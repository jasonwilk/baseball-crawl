Admin access: Admin access is granted via either path:
  • ADMIN_EMAIL env var (bootstrap/fallback): Set ADMIN_EMAIL in your .env file. Any user logging in with that email gains admin access.
  • Database role (primary): Set role = 'admin' on a user row. Use the edit form above, or via SQL: UPDATE users SET role='admin' WHERE email='...';

First admin bootstrap: The first admin must be seeded directly:

docker compose exec app sqlite3 /app/data/app.db \
  "INSERT INTO users (email, role) VALUES ('[your-email]', 'admin');"

Or set ADMIN_EMAIL=[your-email] in .env and log in, then use the edit form to set your role to Admin.
