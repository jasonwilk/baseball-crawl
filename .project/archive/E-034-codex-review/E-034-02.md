# E-034-02: Add CLAUDE.md Commands Entries for Codex Review

## Epic
[E-034: Codex Review Integration](epic.md)

## Status
`DONE`

## Description
After this story is complete, CLAUDE.md's Commands section will include entries documenting both `scripts/codex-review.sh` (code review) and `scripts/codex-spec-review.sh` (spec review), making both Codex review workflows discoverable to all agents and developers who read CLAUDE.md.

## Context
CLAUDE.md has a Commands section that currently lists `./scripts/install-hooks.sh`. Adding both Codex review scripts to this section ensures they are part of the ambient context for all agents and are discoverable without searching. This story depends on both E-034-01 (code review) and E-034-03 (spec review) so that CLAUDE.md describes implemented reality, not planned future state. This is a context-layer file change, so it routes to claude-architect.

## Acceptance Criteria
- [ ] **AC-1**: The Commands section of `/workspaces/baseball-crawl/CLAUDE.md` contains a new bullet documenting `./scripts/codex-review.sh` with a brief description of its purpose (Codex-powered code review of repository changes) and a pointer to the rubric file at `.project/codex-review.md`.
- [ ] **AC-2**: The Commands section of `/workspaces/baseball-crawl/CLAUDE.md` contains a new bullet documenting `./scripts/codex-spec-review.sh` with a brief description of its purpose (Codex-powered spec review of epic/story files) and a pointer to the rubric file at `.project/codex-spec-review.md`.
- [ ] **AC-3**: Each entry is concise -- no more than two lines per command. Entries describe current implemented reality, not future plans.
- [ ] **AC-4**: No other sections of CLAUDE.md are modified.
- [ ] **AC-5**: Neither entry mentions specific model names.

## Technical Approach
1. Read the current CLAUDE.md Commands section.
2. Add bullets for both scripts following the same style as the existing `./scripts/install-hooks.sh` entry.
3. For code review: mention the three modes (uncommitted/base/commit) and the rubric location.
4. For spec review: mention that it targets an epic directory and the rubric location.
5. Do NOT add PM workflow details to CLAUDE.md -- that belongs in the PM agent definition (handled by E-034-04).
