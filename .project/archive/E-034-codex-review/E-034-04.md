# E-034-04: Integrate Optional Codex Spec-Review Step into PM Workflow

## Epic
[E-034: Codex Review Integration](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent definition at `/.claude/agents/product-manager.md` will include an optional Codex spec-review step in its epic refinement workflow. This makes the spec-review capability discoverable to the PM agent and documents the correct invocation pattern (PM requests review through a general-dev agent, not directly).

## Context
E-034-03 creates the spec-review rubric and helper script. This story integrates that capability into the PM's workflow contract so the PM knows about it and uses the correct delegation pattern. The spec-review step is explicitly optional -- it is a refinement aid, not a mandatory gate before READY.

This story modifies a context-layer file (`/.claude/agents/product-manager.md`), so it routes to claude-architect per the dispatch routing rules.

## Acceptance Criteria
- [ ] **AC-1**: The PM agent definition at `/.claude/agents/product-manager.md` contains an optional Codex spec-review step in or near the Quality Checklist section. The step is clearly marked as optional.
- [ ] **AC-2**: The added text explicitly states that the PM requests the review through a `general-dev` agent (not directly), consistent with the PM's no-Bash constraint.
- [ ] **AC-3**: The added text describes the minimal context the PM should provide when requesting a review: (a) epic directory path, (b) optional short note summarizing intent and uncertainties.
- [ ] **AC-4**: The addition does NOT introduce a mandatory gate -- it is advisory language, not a blocking requirement.
- [ ] **AC-5**: The addition references the spec-review helper at `scripts/codex-spec-review.sh` and the rubric at `.project/codex-spec-review.md` by their absolute or repo-relative paths so the PM can direct the implementing agent to the right files.
- [ ] **AC-6**: No other sections of the PM agent definition are materially changed. Minor wording adjustments for flow are acceptable, but the scope is limited to adding the optional spec-review step.
- [ ] **AC-7**: The addition is concise -- no more than 8-10 lines in the agent definition. This is a pointer to the capability, not a reproduction of the rubric.

## Technical Approach
1. Read `/.claude/agents/product-manager.md` in full.
2. Identify the Quality Checklist section (the `## Quality Checklist` heading near the end of the file).
3. Add a brief optional step after the existing checklist items or as a short paragraph below the checklist. Suggested phrasing: the PM may optionally request a Codex spec review before setting READY by dispatching to a `general-dev` agent with the epic directory path and an optional note.
4. Keep the addition minimal and non-disruptive to the existing checklist flow.
