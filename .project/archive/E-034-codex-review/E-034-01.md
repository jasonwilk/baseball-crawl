# E-034-01: Create Code-Review Rubric and Shell Wrapper

## Epic
[E-034: Codex Review Integration](epic.md)

## Status
`DONE`

## Description
After this story is complete, the repo will contain a checked-in Codex code-review rubric at `.project/codex-review.md` and an executable shell wrapper at `scripts/codex-review.sh` that invokes `codex review` with the rubric as review instructions. Developers and agents will be able to run a single command to get a project-aware code review from Codex.

## Context
Codex is already installed in the devcontainer (`npm i -g @openai/codex`). This story creates the two artifacts for the code-review lane (Lane A in the epic): a stable rubric that tells Codex what to look for when reviewing code changes, and a wrapper script that pairs the rubric with the `codex review` CLI. A separate story (E-034-03) handles the spec-review lane.

## Acceptance Criteria
- [ ] **AC-1**: A file exists at `.project/codex-review.md` containing the review rubric. The rubric instructs Codex to: (a) read `CLAUDE.md` first, (b) read story/epic files when the change is story-tied, (c) prioritize bugs, regressions, missing tests, credential risks, schema drift, and planning/implementation mismatch, (d) report findings with file citations, (e) explicitly state "no findings" when the review is clean.
- [ ] **AC-2**: A file exists at `scripts/codex-review.sh` that is executable (`chmod +x`).
- [ ] **AC-3**: Running `scripts/codex-review.sh uncommitted` invokes `codex review` for uncommitted changes, passing the rubric content as review instructions.
- [ ] **AC-4**: Running `scripts/codex-review.sh base <branch>` invokes `codex review` for a branch diff against the specified base branch.
- [ ] **AC-5**: Running `scripts/codex-review.sh commit <sha>` invokes `codex review` for a specific commit.
- [ ] **AC-6**: Running `scripts/codex-review.sh` with no arguments or an invalid mode prints a usage message to stderr and exits non-zero.
- [ ] **AC-7**: The script verifies the rubric file exists before invoking Codex, printing an error and exiting non-zero if missing.
- [ ] **AC-8**: Neither the rubric file nor the script contains any hard-coded model names. Model selection is delegated to the user's Codex configuration.
- [ ] **AC-9**: The script uses `set -euo pipefail` for safe bash execution.
- [ ] **AC-10**: Before writing the script, run `codex review --help` (or equivalent) to verify the actual CLI flags. If the flags differ from `--uncommitted`, `--base`, `--commit`, adapt the script to match the installed CLI. Document any flag discrepancies in a comment at the top of the script.

## Technical Approach
1. Run `codex review --help` to verify the actual CLI syntax for uncommitted, base-branch, and commit review modes. The flags `--uncommitted`, `--base`, `--commit` are from Codex's design input but have not been verified.
2. Create `.project/codex-review.md` with the review rubric content. Keep it concise -- the rubric should fit in a single screen. See the epic Technical Notes (Lane A: Code Review) for content principles.
3. Create `scripts/codex-review.sh` as a thin wrapper. The script should:
   - Accept a mode argument (`uncommitted`, `base`, `commit`)
   - Accept additional arguments per mode (branch name for `base`, SHA for `commit`)
   - Read the rubric file path relative to the repo root
   - Pass the rubric to `codex review` via the mechanism supported by the CLI (stdin pipe, `--instructions` flag, or whatever the CLI supports)
   - Use `set -euo pipefail`
   - Handle the case where `codex` is not installed (command not found)
4. Make the script executable with `chmod +x`.
