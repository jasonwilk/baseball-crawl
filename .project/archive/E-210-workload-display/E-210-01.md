# E-210-01: Expose appearances_7d and update workload format across all surfaces

## Epic
[E-210: Pitching Workload Display Redesign](epic.md)

## Status
`DONE`

## Description
After this story is complete, the pitching workload column on all display surfaces (dashboard opponent detail, dashboard print view, standalone report web, standalone report print) will show the format `Np (Ng)` instead of `N/Nd`. The `get_pitching_workload()` shared query will return `appearances_7d` alongside existing fields, and all three formatting paths (renderer.py, dashboard.py, and templates) will use the new format with updated column headers.

## Context
The `get_pitching_workload()` CTE already computes `appearances_7d` but drops it from the final SELECT. The formatting logic in both `renderer.py` and `dashboard.py` uses `span_days_7d` to produce the `/Nd` suffix. This story threads the new column through the query result and rewrites both formatting paths plus the template headers.

## Acceptance Criteria
- [ ] **AC-1**: Given a pitcher with 72 pitches across 2 games in the 7-day window, the P(7d) cell displays `72p (2g)` on all surfaces (dashboard detail, dashboard print, report web, report print).
- [ ] **AC-2**: Given a pitcher with appearances in the 7-day window but NULL pitch counts for all appearances, the P(7d) cell displays `?p (Ng)` where N is the appearance count.
- [ ] **AC-3**: Given a pitcher with no appearances in the 7-day window, the P(7d) cell displays `—` (em-dash).
- [ ] **AC-4**: Given a pitcher with 0 recorded pitches across 1 game in the 7-day window, the P(7d) cell displays `0p (1g)` (not em-dash).
- [ ] **AC-5**: The column header reads `Pitches (7d)` on all three templates: `scouting_report.html`, `opponent_detail.html`, `opponent_print.html`.
- [ ] **AC-6**: `get_pitching_workload()` returns `appearances_7d` in each player's result dict (integer or NULL).
- [ ] **AC-7**: All existing workload tests updated and passing. No regressions in `tests/test_pitching_workload.py`, `tests/test_report_workload.py`, `tests/test_dashboard_workload.py`, plus any other test files discovered via import-grep of modified modules.

## Technical Approach
The query change in `src/api/db.py` is minimal -- add `sd.appearances_7d` to the final SELECT and to the result dict. The formatting changes in `renderer.py` and `dashboard.py` replace the `span_days_7d`-based branch logic with `appearances_7d`-based logic per the format spec in Technical Notes. The zero-pitches edge case (`pitches_7d == 0`) changes behavior: previously suppressed to em-dash, now displays `0p (Ng)`. Template headers are string replacements.
