# E-030-01: Remove orchestrator from core project infrastructure

## Epic
[E-030: Remove Orchestrator Agent](epic.md)

## Status
`DONE`

## Description
After this story is complete, the orchestrator agent definition is deleted and all core project infrastructure files reflect the PM as the direct entry point for work. CLAUDE.md, dispatch-pattern.md, and workflow-discipline.md no longer reference the orchestrator. The workflow contract is `user -> PM -> implementing agent`.

## Context
The orchestrator was a "smart router" agent that sat between the user and the PM. It added a relay step that increased telephone game risk without providing value the user cannot get by talking directly to the PM. Removing it simplifies the architecture and eliminates a documented source of intent distortion. This story handles the core infrastructure files -- the "spine" of the routing model.
