# E-030-02: Update agent definitions for PM-as-entry-point

## Epic
[E-030: Remove Orchestrator Agent](epic.md)

## Status
`DONE`

## Description
After this story is complete, the product-manager and claude-architect agent definitions no longer reference the orchestrator. The PM definition reflects its role as the direct entry point for all work. The claude-architect definition lists 6 agents in the ecosystem and no longer references orchestrator routing tables or routing sync responsibilities.

## Context
With the orchestrator removed (E-030-01), the two agent definitions that most heavily reference it need updating. The PM definition mentions the orchestrator in its multi-agent-patterns skill trigger ("Receiving a work-initiation request from the orchestrator"). The claude-architect definition lists the orchestrator in its Identity section, Inter-Agent Coordination, and Anti-Patterns. Both need to reflect the simpler `user -> PM -> implementing agent` model.
