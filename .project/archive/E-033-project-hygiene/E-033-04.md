# E-033-04: Fix FastAPI test harness lifecycle and migration comment

## Epic
[E-033: Project Hygiene -- Align Docs and Tests with Current Reality](epic.md)

## Status
`DONE`

## Description
After this story is complete, all FastAPI test files use `TestClient` as a context manager so the ASGI lifespan starts and stops cleanly. A pytest timeout is configured so test hangs fail fast instead of blocking indefinitely. Additionally, the misleading "Schema designed per E-003" comment in the migration file is corrected to accurately describe the file's current contents.

## Context
`tests/test_api_health.py` and `tests/test_dashboard.py` create `TestClient(app)` without using the context manager protocol (`with TestClient(app) as client:`). FastAPI's TestClient (via Starlette) starts the ASGI lifespan on construction. Without the context manager, the lifespan's shutdown may not fire, potentially causing tests to hang. `tests/test_http_session.py` does not use TestClient at all (it uses `respx.mock` + direct `httpx` calls) and completes normally.

Separately, `migrations/001_initial_schema.sql` has a header comment that says "Schema designed per E-003 (Data Model) spec." This is misleading because E-003-01 (the schema rewrite story) is still TODO. The current file contains the pre-E-003 schema. This comment fix is bundled here because it addresses the same "current vs. future state" concern and is too small for its own story.

## Acceptance Criteria

- [ ] **AC-3**: `pytest tests/test_api_health.py tests/test_dashboard.py` completes within 30 seconds total (no hanging). This can be verified by running with `pytest --timeout=30` or equivalent.

- [ ] **AC-7**: No other changes to `migrations/001_initial_schema.sql` beyond the comment edit. The schema SQL itself is untouched.

## Technical Approach

### Test harness fix

**`tests/test_api_health.py`**: Each test method currently does:
```python
client = TestClient(app)
response = client.get("/health")
```

Change to:
```python
with TestClient(app) as client:
    response = client.get("/health")
```

The `patch.dict` context managers that wrap each test can contain the `with TestClient` block.

**`tests/test_dashboard.py`**: The `seeded_client` fixture currently does:
```python
with patch.dict("os.environ", {"DATABASE_PATH": str(db_path)}):
    yield TestClient(app)
```

Change to:
```python
with patch.dict("os.environ", {"DATABASE_PATH": str(db_path)}):
    with TestClient(app) as client:
        yield client
```

The `test_health_endpoint_unaffected` method creates its own TestClient -- apply the same context manager pattern.

### Pytest timeout

Add `pytest-timeout` to requirements:
```
pytest-timeout
```

Add to `pyproject.toml`:
```toml
[tool.pytest.ini_options]
timeout = 30
```

### Migration comment fix

In `migrations/001_initial_schema.sql`, line 4, change:
```sql
-- Schema designed per E-003 (Data Model) spec.
```
to:
```sql
-- Initial schema (will be rewritten by E-003-01).
```
