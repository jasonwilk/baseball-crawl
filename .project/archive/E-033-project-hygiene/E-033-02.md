# E-033-02: Fix hardcoded paths in context-layer files

## Epic
[E-033: Project Hygiene -- Align Docs and Tests with Current Reality](epic.md)

## Status

## Description
After this story is complete, all context-layer files (agent definitions, agent memory, skills) use workspace-relative paths instead of machine-specific absolute paths. The stale "Cloudflare D1" references in the architect's agent-blueprints memory file are also corrected. Agents running in any workspace (local macOS, devcontainer, Codespaces) will find valid file path references.

## Context
The project was originally developed at `/Users/jason/Documents/code/baseball-crawl/`. When the development environment moved to a devcontainer at `/workspaces/baseball-crawl/`, 72 files retained the old absolute path. Context-layer files (`.claude/` directory) are loaded into every agent session as ambient or deferred context. When these files contain paths like `/Users/jason/Documents/code/baseball-crawl/epics/E-NNN-slug/E-NNN-SS.md`, agents in a different workspace cannot resolve them.

The fix is straightforward: replace `/Users/jason/Documents/code/baseball-crawl/` with a project-root-relative path. In context-layer files, paths should be relative to the project root (e.g., `epics/E-NNN-slug/E-NNN-SS.md` or `/epics/E-NNN-slug/E-NNN-SS.md` with a leading slash indicating project root).

Additionally, `.claude/agent-memory/claude-architect/agent-blueprints.md` contains two references to "Cloudflare D1" that should be updated to "SQLite" per the E-009 decision.
