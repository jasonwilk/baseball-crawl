# E-199-02: Render Plays-Derived Stats in Report Template

## Epic
[E-199: Plays-Derived Stats in Standalone Reports](epic.md)

## Status
`DONE`

## Description
After this story is complete, the standalone scouting report template will display FPS% and pitches-per-BF in the pitching table, QAB% and pitches-per-PA in the batting table, and team FPS% and team pitches/PA in the executive summary. When plays data is unavailable, all plays columns show "—" gracefully.

## Context
E-199-01 adds plays data to the generator's data dict. This story consumes that data in the renderer (`src/reports/renderer.py`) and template (`src/api/templates/reports/scouting_report.html`). The coach consultation established that all plays stats must be inline with existing tables (no separate "advanced" section).

## Acceptance Criteria
- [ ] **AC-1**: Given a report with plays data, when rendered, then the pitching table includes "FPS%" and "P/BF" columns after the "Strike%" column showing per-pitcher values, per TN-3.
- [ ] **AC-2**: Given a report with plays data, when rendered, then the batting table includes "QAB%" and "P/PA" columns after the "BB%" column showing per-batter values, per TN-3.
- [ ] **AC-3**: Given a report with no plays data, when rendered, then plays-derived columns show "—" for all values and the report renders without error, per TN-4.
- [ ] **AC-4**: Given a report with plays data, when rendered, then the executive summary strip includes team FPS% (e.g., "62.5% FPS") and team pitches/PA (e.g., "4.2 P/PA"), per TN-3.
- [ ] **AC-5**: FPS% values are formatted as percentages (e.g., "62.5%"). QAB% values are formatted as percentages (e.g., "45.0%"). P/BF and P/PA values are formatted as one decimal (e.g., "3.8").
- [ ] **AC-6**: The FPS%, P/BF, QAB%, and P/PA columns use the `mob-hide` CSS class (hidden on mobile), per TN-3.
- [ ] **AC-7**: Given partial plays coverage (`plays_game_count < game_count`), when rendered, then the executive summary indicates plays coverage (e.g., "4.2 P/PA (8 of 10 games)").
- [ ] **AC-8**: Tests verify the renderer correctly formats plays stats and handles missing data gracefully.

## Technical Approach
The renderer receives plays stats as additional keys in the player dicts and top-level metadata from the generator. The template adds columns to existing tables following the established pattern (see current pitching/batting table markup). The `has_plays_data` boolean controls whether team aggregates appear in the executive summary.
