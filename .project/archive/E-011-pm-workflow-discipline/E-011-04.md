# E-011-04: Update Rules Files to Reference Canonical Standards Document

## Epic
[E-011: PM Workflow Discipline and Status Update Standards](epic.md)

## Status
`ABANDONED`

**Reason**: Superseded by E-016-01 and E-016-06. The rules file updates will happen as part of the PM evolution epic.

## Description
After this story is complete, both `/.claude/rules/project-management.md` and `/.claude/rules/workflow-discipline.md` will contain a reference to `/.claude/rules/pm-workflow-standards.md` as the canonical source for PM status update rules. The single existing line in `project-management.md` that says "Keep the story table in `epic.md` in sync with individual story file statuses" will be replaced with a pointer to the full protocol. The existing rules in `workflow-discipline.md` are not changed -- a reference note is added only. This eliminates the ambiguity of having partial rules in multiple places.

## Context
The PM workflow rules are currently fragmented:
- `project-management.md` has one line: "Keep the story table in `epic.md` in sync with individual story file statuses" -- insufficient, no checklist
- `workflow-discipline.md` covers the READY gate and work authorization but says nothing about when or how PM must update statuses
- MEMORY.md carries ad-hoc summaries that may drift from file truth
- The agent definition (`project-manager.md`) describes Dispatch Mode in detail but the protocol for other action types (creating stories, completing stories, research spikes) is only in the E-011 epic Technical Notes

E-011-01 fixes this by creating the canonical document. This story (E-011-04) completes the fix by pointing the existing partial rules at the canonical document. After this story, anyone reading any of the three rules files will know where to find the full protocol.

This story does NOT rewrite or remove the existing rules in either file -- it adds references. The existing rules remain valid; the canonical document supersedes them for procedural detail.

## Acceptance Criteria

- [ ] **AC-1**: `/.claude/rules/project-management.md` contains a reference to `/.claude/rules/pm-workflow-standards.md` within or immediately after the "Epics and Stories" section.
- [ ] **AC-2**: The reference in `project-management.md` replaces or augments the single existing line "Keep the story table in `epic.md` in sync with individual story file statuses" with text that points to `pm-workflow-standards.md` for the complete protocol. The existing line may be retained or replaced, but the reference must be present and unambiguous.
- [ ] **AC-3**: `/.claude/rules/workflow-discipline.md` contains a reference to `/.claude/rules/pm-workflow-standards.md`. The reference must appear as a visible note (not a comment) that can be read by an agent loading this file.
- [ ] **AC-4**: The reference in `workflow-discipline.md` does not alter any of the existing rules (Epic READY Gate, Work Authorization Gate, Workflow Routing Rule, Direct-Routing Exceptions). It adds information, it does not change it.
- [ ] **AC-5**: Given an agent that loads only `project-management.md`, it can determine that the complete PM status update protocol lives at `/.claude/rules/pm-workflow-standards.md` without reading any other file.
- [ ] **AC-6**: Given an agent that loads only `workflow-discipline.md`, it can determine that the complete PM status update protocol lives at `/.claude/rules/pm-workflow-standards.md` without reading any other file.
- [ ] **AC-7**: Both files remain valid markdown with no broken formatting.

## Technical Approach

### For `project-management.md`

In the "Epics and Stories" bullet list, find the line:
```
- Keep the story table in `epic.md` in sync with individual story file statuses
```

Replace it with two lines:
```
- Keep the story table in `epic.md` in sync with individual story file statuses. The complete atomic status update protocol -- what to update and when for every PM action type -- is in `/.claude/rules/pm-workflow-standards.md`.
```

Or split it as a bullet with a sub-note. Either form is acceptable as long as AC-2 and AC-5 pass.

### For `workflow-discipline.md`

Append a new section at the end of the file:

```markdown
## PM Status Update Protocol

The complete atomic status update protocol for the project-manager agent (what to update and when for every action type: creating stories, completing stories, completing research spikes, and Dispatch Mode pre-dispatch) is documented at:

`/.claude/rules/pm-workflow-standards.md`

The rules in this file (Epic READY Gate, Work Authorization Gate, Workflow Routing Rule) are not superseded -- they remain authoritative for their respective domains. The PM standards document covers the PM-internal update checklist, which is a separate concern.
```
