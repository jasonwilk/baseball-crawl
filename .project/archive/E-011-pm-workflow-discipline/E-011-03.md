# E-011-03: Update PM Agent Definition with Board Audit Step

## Epic
[E-011: PM Workflow Discipline and Status Update Standards](epic.md)

## Status
`ABANDONED`

**Reason**: Superseded by E-016-01 (AC-11). The audit script reference will be added to the rewritten PM agent definition directly.

## Description
After this story is complete, the PM agent definition at `/.claude/agents/project-manager.md` will include a mandatory board audit step in its Dispatch Mode procedure. The current Dispatch Mode procedure has 8 steps; step 3 will be inserted: "Run the board audit script (`scripts/audit_epic_board.py`) before identifying eligible stories. If issues are found, resolve them before proceeding." The story also adds a reference to the PM workflow standards document (`/.claude/rules/pm-workflow-standards.md`) in the agent definition, so the PM agent knows where the canonical protocol lives.

## Context
The PM agent definition currently describes Dispatch Mode in 8 steps (read epic, check READY gate, identify stories, update status, dispatch, verify ACs, update to DONE, repeat). There is no audit step. This means a PM can dispatch stories even when the board has pre-existing inconsistencies -- which is exactly the failure mode E-009 produced.

The fix is simple: add step 3 to the existing numbered list in the Dispatch Mode section. The new step 3 causes the current steps 3-8 to become steps 4-9. No other structural changes are needed.

Additionally, the agent definition references the atomic protocol inline (the Technical Notes of E-011 describe what the PM should do when creating stories, completing stories, etc.) but the canonical version now lives in the rules file created by E-011-01. A brief reference section should be added so a new PM agent knows to look there.

## Acceptance Criteria

- [ ] **AC-1**: The file `/.claude/agents/project-manager.md` contains a step in the Dispatch Mode procedure that instructs the PM to run `scripts/audit_epic_board.py` before identifying eligible stories. The step must use that exact script path.
- [ ] **AC-2**: The audit step in Dispatch Mode is positioned AFTER the READY gate check and BEFORE the "identify eligible stories" step. The rationale: you only run the audit for epics that pass the READY gate, and you run it before changing any status.
- [ ] **AC-3**: The audit step's wording instructs the PM to resolve any issues found before continuing. It must not say "note any issues" or "ignore issues if minor" -- the instruction is to resolve before proceeding.
- [ ] **AC-4**: Given the updated agent definition, a new PM agent reading only the Dispatch Mode section can determine that the audit script must be run before dispatching stories, and knows the script path.
- [ ] **AC-5**: The Dispatch Mode procedure's other steps are unchanged in substance (wording may be adjusted for step renumbering, but no steps are removed or added except the new audit step).
- [ ] **AC-6**: The agent definition contains a reference to `/.claude/rules/pm-workflow-standards.md` as the canonical location for the atomic status update protocol. This reference appears in a logical location (e.g., within the Dispatch Mode section or in a new "PM Standards Reference" callout near the top of the System of Work section).
- [ ] **AC-7**: The existing content of `/.claude/agents/project-manager.md` is otherwise preserved -- no section is removed, no philosophy statement is altered, no other procedures are modified.

## Technical Approach

This is a targeted edit to an existing file. The recommended approach:

1. Read the full `/.claude/agents/project-manager.md` to locate the Dispatch Mode section.
2. Find the numbered list under "Dispatch Procedure" (currently 8 steps).
3. Insert the new step between the current step 2 (READY gate check) and the current step 3 (identify eligible stories). Renumber steps 3-8 to 4-9.
4. New step text (exact wording):
   > **3. Run the board audit script.** Run `scripts/audit_epic_board.py` from the project root. If any issues are reported (mismatches, orphaned files, dangling rows), resolve them before proceeding. Do not dispatch from an inconsistent board state.
5. Add a reference to the PM standards document. Append after the Dispatch Procedure numbered list, before the "Agent Selection Rule" subsection:
   > **PM Workflow Standards**: The complete atomic status update protocol -- what to update and when for every PM action type -- is documented at `/.claude/rules/pm-workflow-standards.md`. That document is the canonical reference; the Dispatch Mode procedure above is a subset of it.
