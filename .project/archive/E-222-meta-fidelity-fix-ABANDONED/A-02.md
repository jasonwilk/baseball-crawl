## Description
After this story is complete, the story template will have a new optional **Named Structures** field that is required when the Technical Approach contains implementer-latitude phrases, and `plan/SKILL.md` will have a new spec-audit criterion in its Phase 3 internal review that checks for the bridging-language → Named Structures pairing. This enforces A9 bridging-language discipline mechanically.

## Context
Per A9, when a spec leaves "implementer determines at execution time" latitude, it MUST include a Named Structures field listing existing structures by name. This prevents bridging-language drift where implementers have to re-derive which structures to modify, which is a known failure mode. A9 is part of the A↔D contract delivered by AD-01; this story installs the enforcement on the authoring side (template + spec audit).

## Acceptance Criteria
- [ ] **AC-1**: A new optional **Named Structures** field is added to `.project/templates/story-template.md`, positioned near the Technical Approach section.
- [ ] **AC-2**: The field is documented as REQUIRED when the Technical Approach contains implementer-latitude phrases ("implementer determines", "implementer decides", or semantically equivalent).
- [ ] **AC-3**: The field format lists existing structures by name (file paths, sections, tables) that the implementer will modify.
- [ ] **AC-4**: A new spec-audit criterion is added to `.claude/skills/plan/SKILL.md` Phase 3 internal review: "Bridging-language check — if any story's Technical Approach contains implementer-latitude phrases, the story MUST include the Named Structures field."
- [ ] **AC-5**: The story template field and the plan skill criterion both cross-reference `.claude/rules/fidelity-protocol.md` A9 section for the authoritative rationale.
- [ ] **AC-6**: Edits to `plan/SKILL.md` do not conflict with B-04 or D-01 anchors. (B-04 touches Phase 0/1/2/5; D-01 touches post-Phase-5 + Phase 3/4 consumer-check hooks; A-02 touches Phase 3 internal review criteria list.)

## Technical Approach

Two edits:

1. `.project/templates/story-template.md` — insert the Named Structures field in the existing structure, positioned near the Technical Approach section. The field uses the same comment-block style as other optional fields.
2. `.claude/skills/plan/SKILL.md` — identify the Phase 3 internal review criteria list and append the Bridging-language check criterion.

**Named Structures for this story**: `.project/templates/story-template.md` (Named Structures field position near Technical Approach) and `.claude/skills/plan/SKILL.md` (Phase 3 Step 1 or internal review criteria list — implementer identifies exact anchor).
