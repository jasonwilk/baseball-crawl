## Description
After this story is complete, `implement/SKILL.md` Phase 3 Step 5 will have a new sub-step where main session writes CR output to `epics/E-NNN-slug/cr-findings.log` before the staging boundary advances. The log is append-only structured markdown, one block per review, written by main session on CR's behalf (because CR does not have Write/Edit tools). This is the third and final Workstream C item.

Findings currently evaporate into the message stream; a persistent log enables post-hoc review and cross-epic analysis without adding monitoring infrastructure.
