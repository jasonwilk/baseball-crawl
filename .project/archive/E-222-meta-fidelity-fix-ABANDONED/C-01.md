## Description
After this story is complete, `code-reviewer.md` will have a new Caller Audit Result field in its Structured Findings Format that is required in every review (even when "None"), and the Caller Audit Bug Pattern Checklist language will be replaced with explicit scope: grep the full codebase, report only callers outside Files Changed AND not part of another story in the current epic. This is a precautionary rubric tightening, not a CR fidelity crisis response (per Framing III).

## Context
Per Framing III, Workstream C is bounded to 3 precautionary items. This story delivers two of them: (1) a mandatory Caller Audit Result field that produces a forensic record in every review, and (2) explicit scope language that prevents CR from either skipping caller audit (because scope was ambiguous) or flagging in-epic callers that another story will handle.

## Acceptance Criteria
- [ ] **AC-1**: A new **Caller Audit Result** sub-section is added to the Structured Findings Format in `code-reviewer.md` with required fields: Scope (function signatures, dataclass fields, behavioral changes audited), Grep patterns run (actual commands), Callers found outside Files Changed (list or "None"), Broken callers (list or "None").
- [ ] **AC-2**: The Caller Audit Result section is documented as REQUIRED in every review, even when the result is "None."
- [ ] **AC-3**: The existing Caller Audit Bug Pattern Checklist language (around line 94 in the current file) is replaced with explicit scope wording: grep the full codebase; report only (a) callers outside Files Changed AND (b) not part of another story in the current epic.
- [ ] **AC-4**: The new language explicitly handles the in-epic case: other-story callers are noted as "deferred to E-NNN-SS review."
- [ ] **AC-5**: Ambiguous findings are documented as reported as MUST FIX.
- [ ] **AC-6**: Edits do not conflict with B-02's additions to `code-reviewer.md` (separate Edit operations on different sections — Structured Findings Format + Bug Pattern Checklist language vs. B-02's Invariant Audit Mode + new checklist items + Review Procedure Step 2).

## Technical Approach

Two separate Edit operations on `.claude/agents/code-reviewer.md`:

1. **Structured Findings Format**: insert a new Caller Audit Result sub-section with the four required fields, positioned alongside existing findings sub-sections.
2. **Bug Pattern Checklist language replacement**: locate the current Caller Audit Bug Pattern Checklist language (approximately line 94) and replace it with the new explicit scope wording.

Implementer reads the current `code-reviewer.md` to identify the exact anchors before editing. This story is parallel-safe with B-02 because the two stories modify different sections; implementer coordinates section boundaries during edit to avoid collision.
