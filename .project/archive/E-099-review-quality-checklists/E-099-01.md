# E-099-01: Add Bug-Pattern Checklists to CR and SE Agent Definitions

## Epic
[E-099: Targeted Review Quality Checklists](epic.md)

## Status
`DONE`

## Description
After this story is complete, both the code-reviewer and software-engineer agent definitions will contain concrete, mechanically executable checklist items targeting the three bug classes that escaped E-097 review: query scope incompleteness, ignored return values, and premature status marking. The CR will check for these patterns during every review; the SE will self-check before reporting completion.

## Context
E-097 shipped four bugs that both SE and CR missed. The common thread: both agents verified presence (code exists, test exists) but not correctness (query filters on all dimensions, return values are consumed, status lifecycle is correct). The current rubric has abstract guidance ("logic errors," "silent failures") but no concrete sub-checks for these specific patterns. Adding targeted checklist items makes detection mechanical rather than relying on general reasoning.

## Acceptance Criteria
- [ ] **AC-1**: `code-reviewer.md` contains a "Bug Pattern Checklist" subsection within the Review Rubric, positioned after Priority 2 (Bugs and Regressions). The checklist contains concrete items for: (a) SQL query scope -- verify every scope parameter (parameters that constrain the query's data range, e.g., `season_id`, `team_id`) appears in the WHERE/JOIN/GROUP BY; flag missing scope parameters as MUST FIX, (b) return value consumption -- verify every call to a fallible operation has its return value captured and failure states affect control flow; flag discarded status returns as MUST FIX, (c) status lifecycle -- when code writes a terminal status to a tracking table, verify the status is written only AFTER all gated downstream work succeeds; flag premature status writes as MUST FIX.
- [ ] **AC-2**: `code-reviewer.md` Priority 3 (Missing or Inadequate Tests) section contains two additional items: (a) multi-scope aggregate tests -- for any aggregate query filtering by multiple dimensions, verify a test includes data for 2+ values of at least the primary filtering dimension (e.g., two seasons for a season-scoped aggregate), (b) error-path tests for orchestration code -- for any new CLI command or pipeline orchestration function that delegates to fallible operations, verify at least one test exercises a failure path and checks exit code/return value + output.
- [ ] **AC-3**: `software-engineer.md` contains a "Pre-Submission Checklist" section that the SE executes before reporting story completion. The checklist contains concrete items for the same three bug classes: (a) SQL parameter check -- every scope parameter in a function signature appears in the SQL query, (b) return value check -- every call to a function returning a status/result type has the return value captured, (c) status lifecycle check -- terminal statuses in tracking tables are written after the full pipeline succeeds. Additionally: (d) multi-scope test obligation -- aggregate queries have tests with data spanning 2+ values of at least the primary filtering dimension, (e) error-path test obligation -- CLI commands and pipeline orchestration functions have tests for the failure path.
- [ ] **AC-4**: All checklist items are concrete and mechanically executable (they describe a specific check to perform, not abstract guidance like "be careful" or "think harder"). Each item follows a pattern of: what to look for, where to look, what to flag.
- [ ] **AC-5**: No other sections of the agent definitions are changed. Existing rubric priorities, review procedure, anti-patterns, and other content are preserved exactly.

## Technical Approach
Read `/.claude/agents/code-reviewer.md` and `/.claude/agents/software-engineer.md`. Add the checklist sections described in the epic's Technical Notes. Place the CR checklist after Priority 2 in the Review Rubric. Place the SE checklist in a new "Pre-Submission Checklist" section. Keep additions minimal -- target the three bug classes only, do not expand scope. Reference the epic Technical Notes for the exact bug class definitions and detection patterns.
