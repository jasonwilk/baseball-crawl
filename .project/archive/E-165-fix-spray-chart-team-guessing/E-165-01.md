# E-165-01: Stop Guessing Team Assignment and Clean Up Bad Data

## Epic
[E-165: Fix Scouting Spray Chart Team Misattribution](epic.md)

## Status
`DONE`

## Description
After this story is complete, the scouting spray chart loader will skip spray events for players who cannot be definitively resolved to a team via roster lookup, instead of guessing by assigning them to the fallback team. Existing misattributed rows will be cleaned up via a documented operator procedure. Per-player WARNING logs will be replaced by a per-game DEBUG summary.

## Context
`_resolve_player_team_id` currently returns `fallback_team_id` when a player isn't in `team_rosters` for either the home or away team. This causes 46.4% of scouting spray chart rows to be assigned to the wrong team. The fix changes the method to return `None` and the caller to skip events for unresolvable players. See epic Technical Notes TN-1 through TN-6 for full context.

## Acceptance Criteria
- [ ] **AC-1**: Given a spray chart JSON where a player UUID is NOT in `team_rosters` for either the home or away team, when the scouting spray loader processes the file, then all spray events for that player are skipped (not inserted) and counted in `LoadResult.skipped`.
- [ ] **AC-2**: Given a spray chart JSON where a player UUID IS in `team_rosters` for one of the game's teams, when the scouting spray loader processes the file, then the spray events are inserted with the correct `team_id` from the roster lookup (existing behavior preserved).
- [ ] **AC-3**: Given the loader processes a game with at least one unresolvable player, when all sections are complete, then exactly one DEBUG log line is emitted summarizing the count of skipped events and unresolvable players for that game, per TN-4. No per-player WARNING is emitted for unresolvable players. No DEBUG summary is emitted for games where all players are resolvable.
- [ ] **AC-4**: The operator cleanup procedure (the roster-join DELETE query from TN-3) is documented as a code comment in the loader module so it ships with the code.
- [ ] **AC-5**: New tests cover: (a) unresolvable player events are skipped with no stub player row created, (b) per-game DEBUG summary is logged only when unresolvable players exist, (c) resolvable players are still inserted correctly, (d) a game with mixed resolvable and unresolvable players produces correct loaded + skipped counts and a single per-game DEBUG summary.
- [ ] **AC-6**: The loader remains idempotent -- re-running on the same raw JSON files produces zero new inserts for already-loaded events (existing `INSERT OR IGNORE` on `event_gc_id` behavior preserved).

## Technical Approach
The fix is in `src/gamechanger/loaders/scouting_spray_loader.py`. The `_resolve_player_team_id` method's return type changes to allow signaling "unknown" instead of returning a fallback. The caller in `_load_game_file` checks for the unknown signal and skips all events for that player, incrementing `skipped`. Per-game skip counts are collected and logged as a single DEBUG message after all sections are processed, only when at least one player was unresolvable. See TN-1 and TN-4 for constraints.

For cleanup, the operator runs the roster-join DELETE query from TN-3 which precisely targets misattributed rows. No reload is required. See TN-3 and TN-5 for constraints.

11 existing tests need changes per TN-6: 7 need roster entries added, 2 need full rewrites (fallback → skip), 2 need roster entries so the correct code path is exercised.

## Notes
### Operator Cleanup Procedure
After deploying the fix, run the following query to remove misattributed rows:
```sql
DELETE FROM spray_charts
WHERE id IN (
    SELECT sc.id FROM spray_charts sc
    JOIN games g ON g.game_id = sc.game_id
    WHERE NOT EXISTS (
        SELECT 1 FROM team_rosters tr
        WHERE tr.player_id = sc.player_id
          AND tr.team_id IN (g.home_team_id, g.away_team_id)
          AND tr.season_id = sc.season_id
    )
);
```

Note: coaching staff may see some opponent spray charts disappear after cleanup. This is expected -- the removed data was incorrectly attributed.
