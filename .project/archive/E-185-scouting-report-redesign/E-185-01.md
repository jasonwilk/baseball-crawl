# E-185-01: Enrich Report Data Pipeline

## Epic
[E-185: Scouting Report Redesign](epic.md)

## Status
`DONE`

## Description
After this story is complete, the standalone scouting report data pipeline will produce all the enriched data needed by the redesigned template: new batting columns (K%, BB%, PA, CS, XBH), heat-map threat levels for each player and stat cell, key players identification, enriched recent form with opponent names and home/away, team spray chart, runs scored/allowed averages, and updated sort orders (pitching by IP desc, batting by PA desc). The renderer will pass this enriched context to the template, but the template itself is unchanged in this story.

## Context
The current generator queries batting and pitching stats but is missing CS (caught stealing) and sorts by AVG/ERA. The renderer computes rate stats for pitching but does not compute K%/BB% for batting, heat-map levels, composite threat scores, key players, or XBH. This story adds all the data-layer and computation-layer changes so Story 02 can focus purely on the template redesign.

## Acceptance Criteria
- [ ] **AC-1**: Given a generated report, the batting data includes `cs`, `_pa`, `_k_pct`, `_bb_pct`, `_xbh`, `_sb_cs` (formatted string e.g. "5-1"), and `_heat` (dict with keys `"avg"`, `"obp"`, `"slg"`, `"thr"`, each mapping to a heat level int 0-4) for each player.
- [ ] **AC-2**: Given a generated report, the pitching data includes `_heat` (dict with keys `"era"`, `"k9"`, `"whip"`, `"thr"`, each mapping to a heat level int 0-4) for each player.
- [ ] **AC-3**: Given a generated report, batting data includes `_thr_score` (float) for each player, computed per TN-1 composite formula. The composite score is percentile-ranked to produce `_heat["thr"]`.
- [ ] **AC-4**: Given a team with ≥ 1 pitcher having ≥ 15 IP and ≥ 1 batter having ≥ 20 PA, the template context includes a `key_players` dict with `top_pitcher` and `top_batter` entries per TN-6.
- [ ] **AC-5**: Given a team with no pitchers meeting the 15 IP threshold, `key_players.top_pitcher` is `None`. Given a team with no batters meeting the 20 PA threshold, `key_players.top_batter` is `None`.
- [ ] **AC-6**: The template context includes both `recent_form` (a list of dicts, each containing `result`, `our_score`, `their_score`, `opponent_name` (str, "Unknown" if NULL), and `is_home` (bool)) AND `recent_form_str` / `has_recent_form` for backward compatibility with the unchanged template. The template continues to use `recent_form_str` in this story.
- [ ] **AC-7**: The template context includes `team_spray_uri` (a base64 data URI string, separate from `spray_data`) when the team has ≥ 20 total spray chart events (per CLAUDE.md "20 BIP for team aggregates" convention), or `None` otherwise.
- [ ] **AC-8**: The template context includes `runs_scored_avg` and `runs_allowed_avg` (formatted strings like `"8.2"`) computed from the games table per TN-8.
- [ ] **AC-9**: Pitching query sorts by `ip_outs DESC` (not ERA ASC). Batting query sorts by PA descending (not AVG DESC).
- [ ] **AC-10**: Small-sample players (< 20 PA batting, < 45 outs pitching) have `_small_sample = True` and all heat levels set to 0.
- [ ] **AC-11**: Heat-map percentiles are computed within the non-small-sample subset of the team roster per TN-1 thresholds.
- [ ] **AC-12**: The template context includes `spray_player_stats` (dict mapping player_id to `{"avg": str, "pa": int, "bip_count": int}`) for every player in `spray_data`. Values sourced from batting data (`_pa`, `h/ab` for AVG) and spray event count.
- [ ] **AC-13**: All existing tests pass. New unit tests cover: heat-level computation (including edge cases: all-same-value, single player, mixed small/large sample), key-players selection, K%/BB%/XBH/SB-CS computation, recent form opponent name resolution, and generator query changes (sort order, runs avg).

## Technical Approach
Changes span two files in `src/reports/`: `generator.py` (SQL queries) and `renderer.py` (computation and context building).

**generator.py changes:**
- `_query_batting()`: Add `COALESCE(psb.cs, 0) AS cs` to the SELECT. Change ORDER BY to sort by PA descending (computed in SQL or as a secondary sort key).
- `_query_pitching()`: Change ORDER BY to `ip_outs DESC, p.last_name ASC`.
- `_query_recent_games()`: Join the `teams` table to resolve opponent name. Add `is_home` flag to each result dict.
- New `_query_runs_avg()`: Query average runs scored and allowed per game from the games table.
- Pass `runs_scored_avg` and `runs_allowed_avg` to the render data dict.

**renderer.py changes:**
- Compute `_pa`, `_k_pct`, `_bb_pct`, `_xbh`, `_sb_cs` for each batter.
- Compute heat-map percentiles and levels for batting (AVG, OBP, SLG) and pitching (ERA inverted, K/9, WHIP inverted). **Important**: Heat-map percentile computation must use raw numeric values computed from counting stats (e.g., `er * 27 / ip_outs` for ERA, `(so * 27) / ip_outs` for K/9), not the formatted string rate stats already on the pitcher dicts. The existing `_compute_pitching_rates()` produces strings; percentile ranking needs floats.
- Compute composite THR score (`_thr_score`) for each player; percentile-rank to `_heat["thr"]`.
- Compute `key_players` dict (top pitcher by IP, top batter by OBP, among non-small-sample players).
- Build team spray chart data URI from aggregated spray events (≥ 20 BIP threshold).
- Build `spray_player_stats` dict: for each player in `spray_data`, compute `{"avg": format_avg(h/ab), "pa": _pa, "bip_count": len(events)}` from batting data and spray event count.
- Pass `recent_form` as structured list with opponent names AND maintain `recent_form_str` / `has_recent_form` for backward compatibility with the unchanged template.
- Add all new context keys to the template render call.
