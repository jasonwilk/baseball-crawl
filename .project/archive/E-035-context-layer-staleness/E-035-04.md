# E-035-04: Trim memory file duplication

## Epic
[E-035: Context Layer Staleness Fixes](epic.md)

## Status
`DONE`

## Description
After this story is complete, the general-dev and api-scout MEMORY.md files will reference CLAUDE.md and rules files instead of duplicating their content. Agent-specific additions (session factory location, respx/responses distinction, security placeholder names) will be preserved. This addresses review findings P3.1, P3.2, R3, R4, and R5.

## Context
The context-layer review (`/.project/research/context-layer-review-2026-03-03.md`, Section 4) found that general-dev MEMORY.md duplicates CLAUDE.md content across three sections: HTTP Request Discipline (~24 lines), Python Style (~6 lines from rules/python-style.md), and Testing Rules (~12 lines from rules/testing.md). The api-scout MEMORY.md has a smaller duplication in its HTTP Request Discipline section (~8 lines). This violates the architect anti-pattern "never duplicate CLAUDE.md content in agent system prompts" -- MEMORY.md files are ambient context just like system prompts.

The fix is to replace duplicated sections with brief references to the source files, keeping only agent-specific additions that are NOT in CLAUDE.md or rules.

3. Replace the section with a one-line reference + the agent-specific additions only.
4. Repeat for `.claude/agent-memory/api-scout/MEMORY.md` (full file, 73 lines).
