# E-213-01: Extract CLAUDE.md Domain Sections to Scoped Rules

## Epic
[E-213: Context Layer Optimization](epic.md)

## Status
`DONE`

## Description
After this story is complete, five domain-specific sections will have been extracted from CLAUDE.md into new scoped rule files, each loading only when agents work on relevant file paths. CLAUDE.md will retain 1-line pointers to each extracted rule. The file will be reduced from 239 lines to ~140-160 lines, near the 150-line target established by the context-layer-guard.

## Context
CLAUDE.md has grown 60% past its 150-line target through organic growth across 200+ epics. Sections like "Key Metrics We Track," "Scouting Data Flows," "Data Model," and "Admin UI" are deep domain reference material that only matters when agents work on specific subsystems. Every agent, every session currently loads all 239 lines. Extracting these sections to scoped rules delivers the highest ROI: ~90 lines removed from universal ambient context.

## Acceptance Criteria
- [ ] **AC-1**: CLAUDE.md line count is between 140 and 160 lines
- [ ] **AC-2**: Five new scoped rule files exist with `paths:` frontmatter per Technical Notes TN-2: `key-metrics.md`, `scouting-data-flows.md`, `data-model.md`, `admin-ui.md`, `architecture-subsystems.md`
- [ ] **AC-3**: Each extracted section's content appears in its target scoped rule (no content loss). Bullet splitting is permitted where a MUST constraint must remain in CLAUDE.md as a 1-line invariant while the surrounding detail moves to the scoped rule.
- [ ] **AC-4**: CLAUDE.md contains a 1-line pointer for each extracted section (e.g., "See `.claude/rules/key-metrics.md` for stat definitions and coaching priorities.")
- [ ] **AC-5**: A grep for key terms (`FPS%`, `QAB`, `appearance_order`, `two-tier enrichment`, `ensure_team_row`, `finalize_opponent_resolution`, `scouting report`, `standalone report`, `spray_charts`) each returns at least one hit in the context layer
- [ ] **AC-6**: Content that remains in CLAUDE.md matches the ambient-only sections listed in Technical Notes TN-3 (no domain-specific detail left behind beyond pointers)

## Technical Approach
The 5 extraction targets and their scoped paths are defined in epic Technical Notes TN-2. What remains in CLAUDE.md is defined in TN-3. The Architecture section requires careful surgery: general principles and 1-line canonical invariants (MUST constraints like canonical team creation, import boundary, repo-root resolution, pipeline caller convention, game ordering convention, season_id derivation, shared query functions) stay in CLAUDE.md; subsystem-specific multi-sentence descriptions (plays pipeline, spray pipeline, reconciliation, LLM, reports, charts, two-tier enrichment) move to `architecture-subsystems.md`. Some bullets contain both a MUST constraint and subsystem detail interleaved — in these cases, split the bullet: the MUST constraint stays as a 1-line invariant in CLAUDE.md, the surrounding implementation detail moves to the scoped rule. Line 142 (worktree guard hook) is entirely redundant with the `worktree-isolation.md` universal rule and can be removed without extraction or pointer — a free line savings.
