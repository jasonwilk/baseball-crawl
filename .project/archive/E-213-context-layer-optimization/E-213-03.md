# E-213-03: Restructure PM MEMORY.md and archived-epics.md

## Epic
[E-213: Context Layer Optimization](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM's MEMORY.md will be under 100 lines (from 146) by replacing enumerated state (Ideas Backlog table, Active Epics table, per-epic status line) with filesystem pointers and curated compact lists. The `archived-epics.md` topic file will be under 40 lines (from 152) in a highlights-only format.

## Context
PM MEMORY.md is at 146 lines -- the densest agent memory file and borderline for the 150-line soft limit. The Ideas Backlog table (62 lines) and Active Epics table (25 lines) consume 87 lines on enumerated state that duplicates filesystem-derivable information (`/.project/ideas/README.md` and `ls /epics/`). The per-epic enumeration on line 5 is a single 1,400+ character line listing every epic with status annotations. The `archived-epics.md` topic file (152 lines) is a flat line-per-epic log that duplicates `ls /.project/archive/`.

## Acceptance Criteria
- [ ] **AC-1**: PM MEMORY.md is under 100 lines
- [ ] **AC-2**: Next-available epic and idea numbers are present and accurate in MEMORY.md
- [ ] **AC-3**: The Active Epics section contains only READY and ACTIVE epics (not COMPLETED, ARCHIVED, DRAFT, or ABANDONED), with a pointer to `/epics/` for full details
- [ ] **AC-4**: The Ideas Backlog section is replaced with a pointer to `/.project/ideas/README.md` plus a curated list of all ideas with "Trigger met" or "Immediately promotable" annotations from the current table
- [ ] **AC-5**: The per-epic enumeration line (line 5) is replaced with a compact entry that preserves only the next-available number and a pointer to the filesystem
- [ ] **AC-6**: Key Architectural Decisions, User Preferences, Key Workflow Contract, and Topic File Index sections are preserved unchanged
- [ ] **AC-7**: `archived-epics.md` is under 40 lines, in a highlights-only format with a header noting `ls /.project/archive/` as the canonical source
- [ ] **AC-8**: The restructured PM MEMORY.md structure matches the target in Technical Notes TN-5

## Technical Approach
The target structure for PM MEMORY.md is defined in epic Technical Notes TN-5. The key judgment call is which ideas to include in the curated promotable list -- select ideas with "Trigger met" or "Immediately promotable" annotations from the current table.
