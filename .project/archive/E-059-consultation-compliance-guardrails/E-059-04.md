# E-059-04: Add implementation-prescriptiveness guardrail to PM agent definition

## Epic
[E-059: Consultation Compliance Guardrails](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent definition will contain guardrails against the PM prescribing specific implementation details in story files. Three changes: (1) a new anti-pattern naming "prescribing implementation details" as a failure mode, (2) strengthened Technical Delegation Boundaries section, and (3) a new Quality Checklist item. The principle: Technical Approach sections describe the problem and constraints, not the code solution.

## Context
During E-058 formation, the PM prescribed specific bash patterns (e.g., `${BASH_SOURCE[0]}` vs `$0`) in story Technical Approach sections. This crosses the Technical Delegation Boundary -- the PM decides what to build and why; the implementing agent decides how. The PM agent definition already states this boundary in prose ("PM delegates: How to build it"), but does not name the violation as an anti-pattern or enforce it through the Quality Checklist.

References the real incident (PM prescribed specific bash patterns in E-058 story Technical Approach).
- [ ] **AC-2**: The Technical Delegation Boundaries section in the same file is strengthened with explicit guidance: "Story Technical Approach sections describe the problem and constraints, not the code solution. The PM specifies what file needs to change and why, not how to code the change."
- [ ] **AC-3**: The Quality Checklist in the same file contains a new item: "Technical Approach sections describe the problem and constraints, not the code solution (no specific function names, variable names, or code patterns)."
- [ ] **AC-4**: No existing anti-patterns, delegation boundaries, or checklist items are removed or have their meaning changed. Only additions and strengthening of existing text.
