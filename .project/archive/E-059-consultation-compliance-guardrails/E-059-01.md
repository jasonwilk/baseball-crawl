# E-059-01: Add consultation compliance anti-pattern and strengthen override rule in PM agent definition

## Epic
[E-059: Consultation Compliance Guardrails](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent definition will contain three reinforcements against skipping user-requested consultations: (1) a new anti-pattern naming the failure mode with a real-incident reference and an explicit escalation path for the spawning constraint, (2) stronger MUST language in the user-directed override paragraph, and (3) a procedural pre-step in "How Work Flows" step 3 (Refinement) that forces scanning for collaboration directives before writing stories.

## Context
The PM violated an explicit "work with SE" directive by writing an epic solo. The root cause: PM was spawned as a subagent (one level deep), could not spawn SE (spawning is one-level-deep -- an architectural constraint), and instead of escalating the constraint to the team lead/user, just skipped the consultation entirely. The existing "User-directed override" paragraph in the Consultation Triggers section was too soft -- it used "honor that request" rather than MUST language, defined no escalation path for when PM cannot spawn, and there was no procedural checkpoint in the planning workflow to catch the directive before stories were written. This follows the E-029 pattern: descriptive rules fail; procedural checkpoints succeed.

## Acceptance Criteria
- [ ] **AC-1**: The Anti-Patterns section in `.claude/agents/product-manager.md` contains a new numbered item that names "skipping user-requested consultation" as a failure mode. The item references the real incident (PM wrote an epic solo after user said "work with SE"), explains the spawning constraint (spawning is one-level-deep), and defines the escalation path (if unable to spawn, escalate to team lead/user with specific questions). Follows the style of existing anti-pattern 4 (which references E-019/E-027).
- [ ] **AC-2**: The "User-directed override" paragraph in the Consultation Triggers section uses "MUST consult the named agent via Task tool before writing any stories -- or if unable to spawn, MUST escalate to the team lead/user with specific questions for the named agent" (not "honor that request"). It includes a concrete negative example: if the user says "work with SE on this," the PM MUST NOT proceed to write stories without first invoking SE or escalating.
- [ ] **AC-3**: The "How Work Flows" step 3 (Refinement) in `.claude/agents/product-manager.md` is expanded to include a consultation pre-step: "Before writing stories, scan the user's request for explicit collaboration directives (imperative verb + agent name). If found, consult the named agent via Task tool first -- or if unable to spawn, escalate to the team lead/user with specific questions for the named agent. Then consult domain experts per the Consultation Triggers table..." The pre-step is in the procedural workflow (How Work Flows), NOT in the Task Types table.
- [ ] **AC-4**: No existing anti-patterns, consultation triggers, or task type definitions are removed or have their meaning changed. Only additions and strengthening of existing text.
- [ ] **AC-5**: The file remains well-formed markdown with consistent formatting relative to surrounding sections.

## Technical Approach
Read `/workspaces/baseball-crawl/.claude/agents/product-manager.md` in full. Make three targeted edits:

1. **Anti-Patterns section** (after the last existing item): Add a new numbered item for "skipping user-requested consultation." Include: the spawning constraint explanation (spawning is one-level-deep, PM cannot spawn peers), the escalation path (message team lead/user with specific questions), and the real incident reference (user said "work with SE to propose a fix," PM wrote the epic solo). Follow the style of anti-pattern 4.

2. **Consultation Triggers > User-directed override paragraph**: Replace "honor that request" with MUST language that includes the escalation path: "the PM MUST consult the named agent via Task tool before writing any stories -- or if unable to spawn, MUST escalate to the team lead/user with specific questions for the named agent." Add a negative example sentence.

3. **How Work Flows > step 3 (Refinement)**: Expand the existing step to include a consultation pre-step before writing stories. The pre-step scans for explicit collaboration directives (imperative verb + agent name) and either invokes the expert via Task tool or escalates to team lead/user. Do NOT put this in the Task Types table -- that table is for classification, not procedure.
