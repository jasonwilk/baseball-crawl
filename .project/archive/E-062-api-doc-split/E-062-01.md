# E-062-01: Migrate Global Reference Sections

## Epic
[E-062: Split API Documentation into Per-Endpoint Files](epic.md)

## Status
`DONE`

## Description
After this story is complete, the shared API reference content (authentication, headers, pagination, content-type conventions, observations, and implementer notes) will live in standalone files under `docs/api/`, ready to be linked from individual endpoint files. The monolith's non-endpoint content will be fully extracted with zero information loss.

## Context
The current monolith contains ~800 lines of global reference material (base URL, authentication, headers, content-type convention, pagination, endpoint priority matrix, key observations, notes for implementers) that is shared across all endpoints. This content must be extracted into standalone reference files before or in parallel with endpoint migration, so endpoint files can link to them rather than duplicating shared information.

## Acceptance Criteria
- [ ] **AC-1**: A `docs/api/` directory exists containing reference files as specified in the format spec from E-062-R-01
- [ ] **AC-2**: All non-endpoint content from `docs/gamechanger-api.md` (sections: Base URL, Authentication, Request Headers, Content-Type Convention, Pagination, Endpoint Priority Matrix, Key Observations, Header Quick Reference, Notes for Implementers) is captured in the reference files under `docs/api/` with no information loss
- [ ] **AC-3**: The reference files follow the structure and conventions established in the format spec from E-062-R-01
- [ ] **AC-4**: No endpoint-specific content is included in the reference files (that belongs in endpoint files)
- [ ] **AC-5**: Internal cross-references within the extracted content are updated to use relative file links

## Technical Approach
The format spec from E-062-R-01 (at `/workspaces/baseball-crawl/.project/research/E-062-format-spec.md`) defines which global sections map to which reference files. Read the spec first, then extract content from `docs/gamechanger-api.md` section by section. The monolith is not deleted in this story -- that happens in E-062-04.
