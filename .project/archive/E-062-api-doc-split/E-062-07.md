# E-062-07: Frontmatter Validation Script + Migration Content Verification

## Epic
[E-062: Split API Documentation into Per-Endpoint Files](epic.md)

## Status
`DONE`

## Description
After this story is complete, the project will have a `scripts/validate_api_docs.py` script that validates every endpoint file in `docs/api/endpoints/` for YAML frontmatter schema compliance, index consistency, and migration completeness. The validation logic will be importable for use in pytest tests, ensuring ongoing correctness as endpoint files are added or modified.

## Context
The E-062 migration creates ~88 endpoint files from the monolith. Without validation, frontmatter errors (wrong field names, invalid status values, missing required fields) and index drift (orphan files, missing entries) will accumulate silently. The validation script serves two audiences: Jason running it from the terminal to check doc health, and agents running it as part of the test suite after modifying endpoint files.

The format spec at `/.project/research/E-062-format-spec.md` defines the canonical YAML frontmatter schema, file naming convention, tag vocabulary, and a complete endpoint inventory (88 endpoints + 1 web-routes reference file). The inventory serves as the migration verification source of truth -- it survives monolith deletion and is what migration stories were built against.

Expert consultation performed: SE (script architecture, test integration, YAML parsing, error reporting), UXD (error output formatting, severity levels).

## Acceptance Criteria
- [ ] **AC-1**: `scripts/validate_api_docs.py` exists and is executable via `python scripts/validate_api_docs.py`. Exit code 0 on success, exit code 1 on any ERROR-level finding.
- [ ] **AC-2**: The script validates these **required frontmatter fields** are present in every endpoint file: `method`, `path`, `status`, `auth`, `profiles`, `response_shape`, `discovered`, `tags`. Missing required fields produce ERROR.
- [ ] **AC-3**: The script validates **allowed values** for enumerated fields:
  - `status`: CONFIRMED | OBSERVED | PARTIAL | UNTESTED | DEPRECATED
  - `auth`: required | none
  - `profiles.{web,mobile}.status`: confirmed | unverified | not_applicable
  - `response_shape`: array | object | string
  - Invalid values produce ERROR.
- [ ] **AC-4**: The script validates **tag vocabulary compliance** -- every tag must be from the controlled vocabulary defined in the format spec (schedule, games, team, player, stats, season, organization, opponent, video, lineup, public, auth, subscription, user, sync, coaching, spray-chart, events, bridge, bulk). Unknown tags produce ERROR.
- [ ] **AC-5**: The script validates **tag count** -- each endpoint should have 2-5 tags. Violations produce WARN (not ERROR).
- [ ] **AC-6**: The script detects **orphan files** -- files in `docs/api/endpoints/` that are not listed in the `docs/api/README.md` index. Orphans produce ERROR.
- [ ] **AC-7**: The script detects **missing files** -- entries in the `docs/api/README.md` index whose referenced file does not exist on disk. Missing files produce ERROR.
- [ ] **AC-8**: The script performs **migration inventory verification** -- for each endpoint in the format spec inventory (`/.project/research/E-062-format-spec.md` Section 7 tables), verify: (a) the expected filename exists on disk, (b) the file's frontmatter `method`, `path`, and `status` values match the inventory. The inventory contains annotated statuses (e.g., "CONFIRMED (empty)", "OBSERVED (HTTP 404)", "CONFIRMED (PARTIAL->CONFIRMED)") -- the script must normalize these to the base keyword before comparison (strip parenthetical annotations). Mismatches produce ERROR.
- [ ] **AC-9**: The script's **scope is `docs/api/endpoints/` only** -- global reference files (`auth.md`, `headers.md`, etc.) are not validated (they have no frontmatter).
- [ ] **AC-10**: Output is **grouped by file**, with ERROR/WARN severity per finding, and a trailing summary line showing error count, warning count, and valid file count. Plain text, no ANSI colors. Example format:
  ```
  docs/api/endpoints/get-teams-team_id-foo.md
    ERROR  missing required field: status
    WARN   tags count below minimum (1 of 2 recommended)

  Summary: 1 error, 1 warning across 1 file. 87 files valid.
  ```
- [ ] **AC-11**: Validation logic is in importable functions (not buried in `__main__`). A separate `tests/test_validate_api_docs.py` test file imports the validation functions and tests them with synthetic endpoint file content (not dependent on the actual `docs/api/endpoints/` directory existing).
- [ ] **AC-12**: `pyyaml` is added to `requirements-dev.in` (dev dependency only) and `requirements-dev.txt` is regenerated via pip-compile.

## Technical Approach
The script parses YAML frontmatter by splitting on `---` delimiters and passing the content to `yaml.safe_load()` (pyyaml). Validation functions are structured for importability -- a `validate_file()` function that returns a list of findings, a `validate_directory()` that orchestrates all files, and a `validate_index_consistency()` for orphan/missing checks. The `main()` function handles CLI output formatting and exit codes.

Migration inventory verification parses the markdown tables in Section 7 of the format spec to extract expected filenames, methods, paths, and statuses. The `web-routes-not-api.md` reference file is included in the inventory check (existence only -- it has no standard frontmatter).

Test file uses inline YAML strings to construct synthetic frontmatter, testing each validation rule independently.
