# E-047-01: User-Directed Consultation Override

## Epic
[E-047: PM Workflow Bugs](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent definition will contain a rule that user-directed collaboration requests during epic formation always override the Consultation Triggers table. When a user says "work with SE on this" or "consult the data engineer before writing stories," the PM will honor that request even if the table says no consultation is needed for the epic's domain.

## Context
The PM currently decides whether to consult experts solely based on the Consultation Triggers table, which maps epic domains to experts. The "Pure process/workflow" row says "None required." When a user explicitly asks the PM to collaborate with a specific agent, the PM has no rule telling it to honor that request -- it follows the table and skips the consultation. This is a bug: the user's explicit directive should always take precedence.

## Acceptance Criteria
- [ ] **AC-1**: The Consultation Triggers section in `.claude/agents/product-manager.md` contains a new rule (above the table) stating that user-directed collaboration requests override the table and must be honored.
- [ ] **AC-2**: The rule explicitly covers the pattern of users naming a specific agent to collaborate with during epic formation (e.g., "work with SE," "consult data-engineer," "define this with api-scout").
- [ ] **AC-3**: The rule includes a signal-word filter: the request must be an explicit directive to collaborate (e.g., imperative verb + agent name), not a passing reference or speculation about what an agent might think. This prevents over-consultation from casual agent mentions.
- [ ] **AC-4**: The existing Consultation Triggers table is unchanged -- the new rule is additive, not a replacement.
- [ ] **AC-5**: The rule is concise (no more than 3 sentences) and positioned where the PM will read it before consulting the table.
