# E-047-03: Fix Spec-Review Skill Orchestration

## Epic
[E-047: PM Workflow Bugs](epic.md)

## Status
`DONE`

## Description
After this story is complete, the spec-review skill will contain clear orchestration guidance that prevents the team lead from running the codex spec-review script in a way that loses output or times out. The skill will specify foreground execution, an appropriate timeout, and expected duration so the team lead can execute Phase 1 reliably.

## Context
The spec-review skill (`/.claude/skills/spec-review/SKILL.md`) tells the team lead to "run the script via Bash" but provides no guidance on:
- **Timeout**: codex processes large prompts (rubric + 4 context files + all epic artifacts) and can take 1-2 minutes for a 7-file epic. The team lead has no expectation of how long this takes and may kill it prematurely or let it hang indefinitely.
- **Foreground vs background**: The skill doesn't specify. The team lead ran it as a background task, which lost the output entirely.
- **Output capture**: No guidance on how to ensure the full output is available for Phase 2.

Observed failure: "spec review epic 45" triggered the skill, but the team lead ran codex as a background command. First run produced no output and timed out at 10 minutes. A second manual run with `timeout 300` worked in 1m 24s.

The script itself (`scripts/codex-spec-review.sh`) is functional -- only the skill's orchestration instructions need to change.

## Acceptance Criteria
- [ ] **AC-1**: Phase 1, Step 1 of the skill specifies that the script MUST be run in the **foreground** (not as a background task). The instruction is explicit -- e.g., "Run in the foreground so output is captured directly."
- [ ] **AC-2**: Phase 1 includes a timeout instruction: wrap the command with `timeout 300` (5 minutes) to prevent indefinite hangs. The exact command template is shown.
- [ ] **AC-3**: Phase 1 includes an expected-duration note: "Codex typically takes 1-2 minutes for a standard epic (3-7 story files). Larger epics may take up to 3 minutes."
- [ ] **AC-4**: Phase 1 includes guidance on distinguishing timeout from script errors: exit code 124 means codex timed out; other non-zero codes mean script errors (codex not installed, invalid directory, missing rubric). On timeout (exit 124), report to the user and ask how to proceed -- do not retry automatically. On script error, report the specific error message.
- [ ] **AC-5**: The existing Phase 1 step structure (Steps 1-4) is preserved. New guidance is integrated into existing steps or added as sub-steps, not as a separate section that could be missed.
- [ ] **AC-6**: No changes to Phase 2, Edge Cases, Anti-Patterns, or any other section of the skill. Changes are scoped to Phase 1 only.

## Technical Approach
Edit `/workspaces/baseball-crawl/.claude/skills/spec-review/SKILL.md`, Phase 1 section only.

Suggested changes to Phase 1, Step 1:
- Update the command template from:
  ```
  ./scripts/codex-spec-review.sh <epic-dir>
  ```
  to:
  ```
  timeout 300 ./scripts/codex-spec-review.sh <epic-dir>
  ```
- Add "Run this command in the foreground" instruction.
- Add expected-duration note after the command template.
- Add timeout-handling guidance distinguishing exit code 124 (timeout) from other non-zero codes (script errors).

The `--note` example in Step 2 should also get the `timeout 300` prefix for consistency.
