## Description
After this story is complete, the five durable lessons from E-064 will be codified in the context layer so they survive epic archival and guide future work. Currently these lessons exist only in the E-064 epic/story files, which will be archived to `/.project/archive/` -- agents do not routinely load archived files.

## Context
E-064 produced five architectural lessons and conventions that are currently ephemeral (exist only in epic files). Without codification, future agents may reintroduce `from scripts.X` imports in `src/`, skip subprocess testing of console script entry points, use cwd-relative paths in `src/` modules, or remove `migrations/__init__.py` without understanding why it exists. This story ensures the lessons are durable.

## Acceptance Criteria

### CLAUDE.md Architecture section
- [ ] **AC-1**: The `scripts/` -> `src/` import boundary is documented: `src/` modules MUST NOT import from `scripts/`. `scripts/` are standalone operator tools that import from `src/`. The reverse direction is not allowed.
- [ ] **AC-2**: The repo-root resolution convention is documented: modules in `src/` use `Path(__file__).resolve().parents[N]` to derive repo-root-relative paths. Never use cwd-relative paths or `sys.path.insert()` in `src/` modules.

### .claude/rules/testing.md
- [ ] **AC-3**: A rule is added requiring subprocess-based smoke tests for console script entry points (e.g., `bb`). The rule explains WHY: in-process test runners (like `typer.testing.CliRunner` or pytest) add the project root to `sys.path`, masking packaging/import errors that only appear when the entry point is invoked as an installed console script.

### .claude/rules/python-style.md
- [ ] **AC-4**: A rule is added: "Never use `sys.path.insert()` or `sys.path.append()` in modules under `src/`. Path manipulation belongs in standalone scripts (`scripts/`) that need to bootstrap `src.*` imports when run directly. `src/` modules are always importable via the editable install."
- [ ] **AC-5**: The `parents[N]` convention is documented as the project standard for repo-root resolution from `src/` modules, with a reference to the existing usage in `src/gamechanger/config.py`.

### SE agent definition
- [ ] **AC-6**: The scripts guidance in `.claude/agents/software-engineer.md` (section "3. Utility and Tooling Scripts", lines 46-53) is updated to describe the import boundary: scripts import from `src/`, CLI modules import from `src/`, but `src/` modules never import from `scripts/`. Mention that reusable logic belongs in `src/` with scripts as thin wrappers.

### Packaging note
- [ ] **AC-7**: `CLAUDE.md` Architecture section documents that `migrations/` is a proper Python package (has `__init__.py`, included in `pyproject.toml`) because `src/db/reset.py` imports from it. This prevents someone from removing the `__init__.py` or the pyproject.toml include without understanding the dependency.
