# E-064-01: Move reusable logic to src/ and fix CLI imports

## Epic
[E-064: Fix bb CLI Import Crash](epic.md)

## Status
`DONE`

## Description
After this story is complete, reusable business logic currently embedded in `scripts/` will live in proper `src/` packages, the `bb` CLI will import from `src/` directly (no `from scripts.X` imports), scripts will become thin CLI wrappers that import from `src/`, `migrations/` will be a proper importable package, and a subprocess smoke test of the actual `bb` entry point will guard against this class of regression.

## Context
The `bb` CLI crashes with `ModuleNotFoundError: No module named 'scripts'` because four CLI modules import from `scripts/`, which is not a Python package and is not on `sys.path` when `bb` runs as a console script entry point. SE consultation recommended Option C (move reusable functions into `src/`, make scripts thin wrappers). Code review follow-up with SE and UXD clarified path rebasing, migrations packaging, subprocess testing, and output formatting. See epic Technical Notes for the full assessment.

## Notes
- When moving functions, do NOT include `logging.basicConfig()` calls or `sys.path.insert()` in the `src/` modules. Those belong in the script wrappers and CLI entry points only. The `src/` modules should use `logging.getLogger(__name__)` and let callers configure logging.
- `scripts/crawl.py` uses module-level name resolution for crawler classes (lazy via `_build_crawlers()`) so tests can patch. Preserve this pattern in the moved version.
- The subprocess smoke test (AC-18) complements, not replaces, the existing CliRunner tests. Both test types serve different purposes: CliRunner tests argument mapping and command behavior in-process; the subprocess test verifies the actual console script entry point works.
- Rich is already a dependency (used by `src/cli/creds.py`, `src/cli/status.py`, `src/cli/db.py`). No new dependency needed for AC-14.
