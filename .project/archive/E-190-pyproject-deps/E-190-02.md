# E-190-02: Update dependency-management rule for dual-source workflow

## Epic
[E-190: Declare Dependencies in pyproject.toml](epic.md)

## Status
`DONE`

## Description
After this story is complete, `/.claude/rules/dependency-management.md` will document the dual-source dependency pattern: `pyproject.toml [project.dependencies]` for editable installs and `requirements.in` for pip-tools lockfiles. Agents adding, removing, or upgrading runtime dependencies will know to update both files.

## Context
The current rule describes the pip-tools workflow (`*.in` -> `*.txt`) but says nothing about `pyproject.toml` declaring dependencies. After E-190-01, runtime deps exist in both places. Without updating the rule, agents will continue adding deps to only `requirements.in`, leaving `pyproject.toml` out of sync and re-introducing the original bug.

## Acceptance Criteria
- [ ] **AC-1**: `dependency-management.md` has a new section explaining the `pyproject.toml [project.dependencies]` declaration and its relationship to `requirements.in`
- [ ] **AC-2**: The "File Layout" table includes a `pyproject.toml [project.dependencies]` row describing its purpose (runtime deps for editable installs, mirrors `requirements.in`)
- [ ] **AC-3**: The "Common Operations" section (Add/Upgrade/Remove) instructs agents to update both `pyproject.toml` and `requirements.in` for runtime dependencies
- [ ] **AC-4**: The rule explains WHY two sources exist (editable installs vs. deterministic lockfiles) so agents understand the constraint rather than blindly following a checklist
- [ ] **AC-5**: The rule notes the version constraint convention difference (`>=` in pyproject.toml vs. `~=` in requirements.in) and why it is intentional
- [ ] **AC-6**: The rule notes that dev dependencies are NOT in pyproject.toml (they remain in `requirements-dev.in` only)
- [ ] **AC-7**: No other context-layer files are modified (devcontainer.md and CLAUDE.md do not need changes per CA consultation)

## Technical Approach
Read the final state of `pyproject.toml` from E-190-01 to confirm section names and dep list format. Update three areas of `dependency-management.md`: add a pyproject.toml section after the File Layout table, expand the File Layout table, and update Common Operations. The changes should be concise -- a few paragraphs, not a rewrite.
