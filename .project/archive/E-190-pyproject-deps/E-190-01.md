# E-190-01: Add runtime dependencies to pyproject.toml

## Epic
[E-190: Declare Dependencies in pyproject.toml](epic.md)

## Status
`DONE`

## Description
After this story is complete, `pyproject.toml` will declare all 14 runtime dependencies in `[project.dependencies]`. The editable install (`pip install -e .`) will pull in runtime deps instead of being hollow.

## Context
`pyproject.toml` currently declares zero deps. The devcontainer's `pip install -e .` registers the package (for the `bb` CLI entry point) but installs nothing. Adding `[project.dependencies]` fixes this by declaring what the package actually needs to run. The `requirements.in` / `requirements.txt` files remain unchanged -- they serve the pip-tools deterministic build pipeline.

## Acceptance Criteria
- [ ] **AC-1**: `pyproject.toml` `[project.dependencies]` lists every package from `requirements.in` with `>=` version constraints (packaging convention, per Technical Notes)
- [ ] **AC-2**: The `uvicorn` entry includes the `[standard]` extra (matching `requirements.in`)
- [ ] **AC-3**: The `typer` entry includes the `[all]` extra (matching `requirements.in`)
- [ ] **AC-4**: `requirements.in` has a comment cross-referencing `pyproject.toml` (e.g., "Keep in sync with [project.dependencies] in pyproject.toml")
- [ ] **AC-5**: `pyproject.toml` has a comment above `[project.dependencies]` cross-referencing `requirements.in`
- [ ] **AC-6**: `requirements.in`, `requirements-dev.in`, `requirements.txt`, `requirements-dev.txt` are otherwise unchanged
- [ ] **AC-7**: Dockerfile is unchanged
- [ ] **AC-8**: `devcontainer.json` is unchanged
- [ ] **AC-9**: `pip install -e .` in the epic worktree resolves all 14 runtime deps (verify with `python -c "import matplotlib; import httpx; import fastapi; import uvicorn; import starlette; import jinja2; import aiofiles; import multipart; import webauthn; import dotenv; import yaml; import typer; import brotlicffi; import numpy"`)
- [ ] **AC-10**: Existing tests pass with no regressions

## Technical Approach
Mirror the 14 packages from `requirements.in` into `pyproject.toml [project.dependencies]`, using `>=` ranges (e.g., `httpx~=0.28` becomes `httpx>=0.28`; `typer[all]>=0.9` stays as-is). Add cross-reference comments in both files. Refer to the epic's Technical Notes for the version constraint convention rationale.
