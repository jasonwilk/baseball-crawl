# E-092-04: Update code-reviewer for worktree-aware review

## Epic
[E-092: Git Worktree Isolation for Agent Teams](epic.md)

## Status
`DONE`

## Description
After this story is complete, the code-reviewer agent definition will document that review assignments may include worktree paths, that the reviewer reads files from the implementer's worktree (not the main checkout), and that the reviewer does not get its own worktree isolation.

## Context
When implementing agents work in worktrees, their changed files live at worktree-absolute paths (e.g., `/tmp/.worktrees/baseball-crawl-abc123/src/crawlers/foo.py`) rather than the main checkout path. The code-reviewer needs to know to use these paths when reading files, running tests, and checking diffs. The reviewer is spawned without `isolation: "worktree"` because it needs to access any implementer's worktree path -- giving it its own worktree would be counterproductive.

## Acceptance Criteria
- [ ] **AC-1**: The code-reviewer agent definition (`.claude/agents/code-reviewer.md`) documents that review assignments include worktree paths when the implementer worked in a worktree
- [ ] **AC-2**: The agent definition documents that the reviewer should use the provided worktree path to read changed files (via Read, Glob, Grep tools) and run `git diff` from the worktree directory. For test execution: the reviewer does NOT run pytest from the worktree -- the editable install's meta path finder hardcodes the main checkout's `src/` path and intercepts all `import src.*` before `sys.path` is consulted, so `PYTHONPATH=src` has no effect. Instead, the implementer runs tests during implementation (reporting results), and the reviewer verifies AC compliance primarily through file inspection
- [ ] **AC-3**: The agent definition documents that the reviewer does NOT get its own worktree -- it reads from the implementer's worktree as directed by the review assignment
- [ ] **AC-4**: The agent definition documents that `git diff` should be run from within the worktree directory to see the implementer's changes

## Technical Approach
Read the current `.claude/agents/code-reviewer.md` and add a "Worktree Review" section (or equivalent) documenting the worktree-aware review workflow. The changes should be additive -- the existing review rubric, structured findings format, and circuit breaker are unchanged. This is about path awareness, not review methodology.
