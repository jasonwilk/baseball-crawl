# E-092-05: Fix dotenv_values() path resolution for worktrees

## Epic
[E-092: Git Worktree Isolation for Agent Teams](epic.md)

## Status
`DONE`

## Description
After this story is complete, three modules that call `dotenv_values()` with no path argument will use `__file__`-relative path resolution to find `.env`, ensuring they work correctly when run from a git worktree (where cwd is not the main checkout root).

## Context
`dotenv_values()` with no arguments searches for `.env` starting from the current working directory, walking up toward the filesystem root. When an agent runs in a worktree at a path like `/tmp/.worktrees/baseball-crawl-abc123/`, the `.env` file is not present (it is gitignored and only exists in the main checkout). Three modules rely on this bare call and will fail to load credentials/config in a worktree. The fix is to use `__file__`-relative path resolution, matching the pattern already established in `src/gamechanger/token_manager.py` (`Path(__file__).resolve().parents[2] / ".env"`).

## Technical Approach
Read each of the three affected files and identify the bare `dotenv_values()` calls.

**`src/gamechanger/client.py`**: This module already has `_DEFAULT_ENV_PATH = Path(__file__).resolve().parents[2] / ".env"` defined at line 57 (used by `TokenManager` at line 175). The fix is a one-line change: replace `dotenv_values()` at line 489 with `dotenv_values(_DEFAULT_ENV_PATH)`. Do NOT create a second path expression -- use the existing constant.

**`src/gamechanger/credentials.py` and `src/http/proxy_check.py`**: These modules do not have an existing env path constant. Define one using the same pattern (`_ENV_PATH = Path(__file__).resolve().parents[N] / ".env"` with the correct depth for each module's location) and use it in the `dotenv_values()` call. Reference `src/gamechanger/token_manager.py` for the established convention.
