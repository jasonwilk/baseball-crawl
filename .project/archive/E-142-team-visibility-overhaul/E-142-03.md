# E-142-03: Dashboard Empty State UI

## Epic
[E-142: Team Visibility Overhaul](epic.md)

## Status
`DONE`

## Description
After this story is complete, when a user selects a team with no stat data on the Batting, Pitching, or Games dashboard tab, they see a clear yellow info card explaining that stats haven't been loaded yet. The Opponents tab follows TN-4's exception logic. Team pills for no-data teams use muted styling to signal pending state. The experience feels "ready and waiting," not broken.

## Context
With E-142-02 making no-data teams visible in the year selector and team pills, the dashboard will attempt to render content for teams with empty stat tables. Without an explicit empty state, the user sees blank tables or confusing zero-stat displays. This story adds the UX layer. See TN-4 for design details from UXD consultation.

## Acceptance Criteria
- [ ] **AC-1**: Given a team with no stat data, when the user selects it on `/dashboard` (Batting tab), then a yellow info card appears in the content area with the message per TN-4, and no empty stat table is rendered.
- [ ] **AC-2**: Same empty state card appears on `/dashboard/pitching` and `/dashboard/games` -- consistent behavior across the Batting, Pitching, and Games tabs. The Opponents tab is handled separately by AC-7.
- [ ] **AC-3**: Given a team with no stat data, when its team pill is rendered in unselected state, then it uses muted styling (gray tones) distinguishable from pills for teams with data, per TN-4.
- [ ] **AC-4**: Given a team with no stat data is the selected/active pill, then it uses normal selected styling (not muted) -- the empty state in the content area communicates status.
- [ ] **AC-5**: The `_team_selector.html` partial receives a `has_stat_data` boolean per team entry to drive pill styling. The `has_stat_data` flag is sourced from `get_teams_with_stat_data()` (from E-142-02), NOT from `team_year_map` membership per TN-4.
- [ ] **AC-5a**: When only one year exists in `available_years`, the year context is still visible to the user as a static year label (no dropdown needed for a single year). The `_team_selector.html` macro is adjusted per TN-4 to render a year label even when `available_years|length == 1`.
- [ ] **AC-6**: Given a team with stat data, when selected, then the dashboard renders exactly as before (no regression in existing team display).
- [ ] **AC-7**: The opponent tab exception per TN-4: if `get_team_opponents()` (as modified by E-142-04) returns a non-empty list for the active team and season, show the opponent list on the Opponents tab (not the generic empty card), even when the team has no stat data.

## Technical Approach
Changes span the dashboard route and templates. The route must call `get_teams_with_stat_data()` (produced by E-142-02) and attach a `has_stat_data` boolean to each team in `team_infos`. Pass this flag to the `_team_selector.html` partial for pill styling and to each tab template for empty-state rendering. Add an empty-state conditional in the Batting, Pitching, and Games templates: if the active team has no stat data, render the yellow info card instead of the stat table. The Opponents tab follows TN-4's exception logic (show opponent list if linked opponents exist, empty card only if truly no opponents). Adjust the `_multi_year` conditional to show year context even with a single year. See TN-4 for full design details.
