# E-163-04: Fix Player Profile Spray Chart Display

## Epic
[E-163: Full-Season Opponent Spray Charts and Dashboard Display](epic.md)

## Status
`DONE`

## Description
After this story is complete, the player profile page will show inline spray charts for any player with ≥1 BIP event. The template context variables (`spray_bip_count`, `spray_season_id`) will be populated by the route handler, and the threshold logic will be simplified to show charts regardless of sample size.

## Context
The `player_profile` route in `src/api/routes/dashboard.py` (around line 1839) does not pass `spray_bip_count` or `spray_season_id` to the template. The template references these variables but they're always undefined, causing the spray chart card to show "Charts will appear after the next sync" for every player. Additionally, the current template thresholds (10+ BIP to show chart, 5-9 for "small sample") conflict with the operator directive to show charts regardless of count.

## Acceptance Criteria
- [ ] **AC-1**: The `player_profile` route passes `spray_bip_count` (integer count of offensive BIP events for the player in the displayed season) and `spray_season_id` (the season_id string) in the template context.
- [ ] **AC-2**: When `spray_bip_count >= 1`, the template renders the spray chart `<img>` tag (with `?season_id={{ spray_season_id }}` in the URL) and a "Based on N balls in play" subtitle, per TN-5.
- [ ] **AC-3**: When `spray_bip_count == 0`, the template shows "No spray chart data available" (exact wording per TN-5; no `<img>` tag).
- [ ] **AC-4**: The old threshold tiers (10+ shows chart, 5-9 "small sample", 1-4 "not enough data") are replaced by the simplified two-tier logic in TN-5.
- [ ] **AC-5**: Spray charts render correctly for member/permitted-team players on the player profile page. (Opponent players are not accessible on the player profile page due to `_check_player_authorization` restricting to `user_team_access` teams. Opponent player spray charts are displayed on the opponent detail and print pages via E-163-05.)

## Technical Approach
1. Add a DB query in the `player_profile` route to count offensive spray events: `SELECT COUNT(*) FROM spray_charts WHERE player_id = ? AND chart_type = 'offensive' AND season_id = ?`
2. Add `spray_bip_count` and `spray_season_id` to the template context dict
3. Simplify the template's spray chart card Jinja2 logic from four tiers to two (≥1 vs 0)

Reference: the current template at `src/api/templates/dashboard/player_profile.html` lines 201-215.
