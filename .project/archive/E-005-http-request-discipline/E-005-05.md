# E-005-05: Write HTTP integration guide

## Epic
[E-005: HTTP Request Discipline](epic.md)

## Status
`DONE`

## Description
After this story is complete, the project will have `docs/http-integration-guide.md` -- a concise reference document for any developer or agent writing new HTTP integrations (API clients, scrapers, etc.). The guide covers how to use `create_session()`, how to inject auth, how to override specific headers, and the rate-limiting contract. It is the authoritative source for "how do I add a new HTTP integration to this project?"

## Context
The session factory and header profile encode the HTTP discipline in code. But an agent picking up a future scraping story, or adding a new API endpoint, needs to know that this infrastructure exists and how to use it. Without documentation, the next integration will likely be written from scratch with bare httpx -- exactly the pattern this epic eliminates.

This story is intentionally last so the guide reflects the final, implemented API rather than speculative design.

## Acceptance Criteria
- [ ] **AC-1**: `docs/http-integration-guide.md` exists and contains a working code example of `create_session()` usage including auth injection.
- [ ] **AC-2**: The guide explains what `BROWSER_HEADERS` is, why it exists, and how to update it when the Chrome version needs to change.
- [ ] **AC-3**: The guide documents the rate limiting contract: default values (1000ms min delay, 500ms jitter), how to override them, and why callers should NOT add their own sleeps.
- [ ] **AC-4**: The guide includes a section "Adding a new integration" with a step-by-step checklist: (1) call `create_session()`, (2) inject credentials, (3) do not add manual delays, (4) do not log headers, (5) mock at transport layer in tests.
- [ ] **AC-5**: The guide references the CLAUDE.md "HTTP Request Discipline" section by name, so readers know where the authoritative policy lives.
- [ ] **AC-6**: All code examples in the guide are syntactically correct Python 3.11+ and use only the actual API of `create_session()` as implemented in E-005-02.

## Technical Approach
Write a markdown file at `docs/http-integration-guide.md`. Structure:

1. **Overview** -- what this is and why it exists (2-3 sentences)
2. **Quick start** -- minimal working example: create session, inject auth, make a GET
3. **The browser header profile** -- what BROWSER_HEADERS contains, how to update the Chrome version, what NOT to add (credentials)
4. **Rate limiting and jitter** -- the contract, defaults, how to override, why callers don't add sleeps
5. **Auth injection pattern** -- how to add auth headers (gc-token for GameChanger), how to add cookies, safe practices
6. **Cookie jar** -- what persists automatically, what does not
7. **Adding a new integration** -- the step-by-step checklist (AC-4)
8. **Testing new integrations** -- mock at transport layer, don't mock at session factory level, caplog for log safety
9. **What not to do** -- anti-patterns: bare httpx.Client(), manual sleeps, logging headers

Keep it under 300 lines. This is a reference, not a tutorial.
