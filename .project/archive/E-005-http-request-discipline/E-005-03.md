# E-005-03: Verify GameChanger client session factory integration

## Epic
[E-005: HTTP Request Discipline](epic.md)

## Status
`DONE`

## Description
Verify that `src/gamechanger/client.py` correctly uses `create_session()` from `src/http/session.py` and injects GameChanger-specific headers (`gc-token`, `gc-device-id`, `gc-app-name`) rather than `Authorization: Bearer`. The client was written after E-005-02 shipped and already uses the session factory -- this story is a verification pass, not a retrofit.

If all acceptance criteria already pass with no code changes, confirm that and report back. If any criteria fail, fix them.

## Context
When E-005 was originally planned, E-001-02 (the GameChanger client) had not yet been written. E-005-03 was scoped as a retrofit story: "replace bare httpx.Client with create_session()." In practice, E-001-02 was implemented after E-005-02 shipped and was written directly against `create_session()`, making the retrofit unnecessary.

Additionally, significant API discovery has occurred since E-005 was authored. The api-scout documented 7 endpoints in `docs/gamechanger-api.md` and confirmed that GameChanger uses:
- `gc-token` (not `Authorization: Bearer`) for authentication
- `gc-device-id` and `gc-app-name: web` as required custom headers
- Vendor-typed `Accept` headers per endpoint (e.g., `application/vnd.gc.com.game_summary:list+json; version=0.1.0`)
- Vendor-typed `Content-Type` for GET requests: `application/vnd.gc.com.none+json; version=undefined`
- `x-pagination: true` request header for paginated endpoints

The client already implements all of these patterns. This story verifies the integration is correct and the test suite covers GameChanger-specific headers.

## Acceptance Criteria
- [ ] **AC-1**: `grep -r "httpx.Client(" src/gamechanger/` returns zero matches -- the client uses `create_session()`, not bare httpx.
- [ ] **AC-2**: Given `GameChangerClient()` instantiated with valid credentials, when a mocked GET request is made, then the request carries all headers from `BROWSER_HEADERS` plus the GameChanger auth headers: `gc-token`, `gc-device-id`, and `gc-app-name`.
- [ ] **AC-3**: Given `GameChangerClient.get()` called with an `accept` parameter, the outgoing request's `Accept` header matches the provided vendor-typed value (e.g., `application/vnd.gc.com.game_summary:list+json; version=0.1.0`). When `accept` is not provided, the session default `Accept` from `BROWSER_HEADERS` is used.
- [ ] **AC-4**: Given `GameChangerClient.get()`, the outgoing request includes `Content-Type: application/vnd.gc.com.none+json; version=undefined`.
- [ ] **AC-5**: `GameChangerClient.__init__` accepts `min_delay_ms` (default 1000) and `jitter_ms` (default 500) and forwards them to `create_session()`. Tests confirm these values are passed through.
- [ ] **AC-6**: All existing tests in `tests/test_client.py` pass with no regressions. The test suite covers: 401/403 -> CredentialExpiredError, 429 -> RateLimitError, 5xx -> retry + GameChangerAPIError, missing creds -> ConfigurationError, auth header injection, Content-Type header, Accept header override.
- [ ] **AC-7**: Given `GameChangerClient()` with mocked credentials and mocked HTTP, when `client.get()` is called, then the outgoing request contains `gc-token`, `gc-device-id`, `gc-app-name` AND all 10 required browser headers from `BROWSER_HEADERS`. This test is added to `tests/test_http_discipline.py` (closing the gap from E-005-04 AC-6 which was skipped).
