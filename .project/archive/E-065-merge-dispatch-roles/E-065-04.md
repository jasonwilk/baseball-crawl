# E-065-04: Update CLAUDE.md, Workflow Discipline, and Multi-Agent Patterns

## Epic
[E-065: Merge Team Lead and PM Roles During Dispatch](epic.md)

## Status
`DONE`

## Description
After this story is complete, three files will consistently reflect the merged dispatch model: CLAUDE.md's Workflow Contract section, the workflow-discipline rule, and the multi-agent-patterns skill. All references to the three-role dispatch model (team lead / PM coordinator / implementers) will be updated to the two-role model (main session coordinator / implementers).

## Context
These three files contain smaller references to the dispatch model that must stay consistent with the rewritten dispatch-pattern.md (E-065-01). None requires a major rewrite -- each has a handful of paragraphs or sections that reference the old model.

**CLAUDE.md** (Workflow Contract section, lines ~380-395, AND Workflows section, line ~176):
- Step 5 describes team lead creating dispatch team and spawning PM. Needs to describe main session spawning implementers directly.
- "Team Lead Boundary" paragraph describes team lead restrictions. Remove or replace with the merged model.
- Workflows section's Implement entry says "The team lead reads the epic for team composition, spawns the PM, and relays the request." Must be updated to describe the main session reading the epic and spawning implementers directly.

**workflow-discipline.md**:
- "Workflow Routing Rule" section describes `user -> team lead -> PM -> implementing agent` pipeline. Needs to describe `user/main session -> implementing agent`.
- "Team lead boundary" paragraph describes team lead restrictions. Remove or replace.
- "Dispatch Authorization Gate" references PM setting READY and presenting to user. PM still does this during planning -- the gate logic stays, but any references to PM chaining into dispatch mode should be reviewed.
- "PM Task Types" section -- verify it does not list dispatch as a PM mode.

**multi-agent-patterns SKILL.md**:
- Routing chain diagram shows `product-manager (plans, dispatches via Agent Teams)`. Remove the "dispatches via Agent Teams" part -- PM plans only, main session dispatches.
- PM Dispatch Checklist references "spawning a teammate." Update to reflect main session dispatching.
