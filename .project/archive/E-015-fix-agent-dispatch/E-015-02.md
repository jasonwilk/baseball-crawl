# E-015-02: Fix Orchestrator Routing for Dispatch Requests

## Epic
[E-015: Fix Agent Dispatch -- Nested Session Error](epic.md)

## Status
`DONE`

## Description
After this story is complete, the orchestrator agent definition will correctly handle "start epic X", "execute story X", and "dispatch stories" requests by telling the user to invoke the PM directly from their main session -- rather than wrapping the PM in a Task call that makes it a subagent. The nested dispatch failure cannot occur when the orchestrator routes this way.

## Context
The orchestrator's current routing example for "Start epic E-001" says:

> Route to **project-manager** with a Task call.

This is the broken pattern. The PM, once invoked as a Task subagent, cannot use the Task tool itself to dispatch stories. The fix is for the orchestrator to recognize dispatch requests as a special case where it must NOT use the Task tool. Instead it tells the user exactly what to do.

The correct dispatch flow:
1. User says "start epic E-011" to the orchestrator
2. Orchestrator recognizes this as a dispatch request
3. Orchestrator responds: "To dispatch stories, invoke the PM directly from your main session. [Instructions]"
4. User invokes PM from the main session (not through orchestrator Task routing)
5. PM's Dispatch Mode runs with Task tool access and dispatches successfully

This is NOT a change to how the orchestrator handles non-dispatch requests (creating epics, reviewing backlog, capturing ideas). Those remain as Task calls and work correctly because they don't require the PM to dispatch further.

## Acceptance Criteria

- [ ] **AC-1**: The orchestrator agent definition at `/.claude/agents/orchestrator.md` no longer contains an example or instruction that routes "start epic X" / "execute story X" / "dispatch stories" requests via a Task call to PM.
- [ ] **AC-2**: The orchestrator definition contains a clearly labeled section or routing rule for dispatch requests that states: the orchestrator must NOT use the Task tool for this case; it must instead tell the user to invoke the PM directly.
- [ ] **AC-3**: The orchestrator's routing instructions for dispatch requests include the verbatim instruction the orchestrator should give to the user. The instruction must: (a) explain why direct invocation is required, (b) tell the user to start a fresh session or switch to the PM agent, and (c) tell the user what to say when they do (e.g., "enter Dispatch Mode for epic E-NNN").
- [ ] **AC-4**: The orchestrator's routing rules for NON-dispatch PM requests (create epic, review backlog, capture idea, refine stories) are unchanged. These still use Task calls to PM.
- [ ] **AC-5**: The routing rules section references `/.claude/rules/dispatch-pattern.md` as the authoritative explanation of why dispatch requests are handled differently.
- [ ] **AC-6**: Given a user says "start epic E-011" to the orchestrator in a fresh session, an agent reading only the updated orchestrator definition would know to respond with user instructions rather than a Task call.

## Technical Approach

Modify `/.claude/agents/orchestrator.md`. Changes are minimal and scoped:

1. Find the "Routing Example: 'Start epic E-001'" section. Replace the example with the corrected routing that produces user instructions instead of a Task call.

2. Add a new routing rule to the Routing Rules list (after rule 1 or as rule 2). Something like:

> **Dispatch requests go directly back to the user.** Requests that ask the PM to dispatch, execute, or start stories ("start epic X", "execute story E-NNN-SS", "dispatch E-003") are a special case. Do NOT route these via the Task tool. The PM cannot dispatch subagents when it is itself a subagent. Instead respond to the user with instructions to invoke the PM directly. See `/.claude/rules/dispatch-pattern.md` for the full constraint explanation.

3. Add the example user instruction the orchestrator should give. For example:

> "PM Dispatch Mode requires direct invocation (not routing through the orchestrator). Please switch to the PM agent directly and say: 'Enter Dispatch Mode for epic E-NNN. Read the epic directory and dispatch all eligible TODO stories.' This ensures the PM runs in the main session where it can use the Task tool."

Read the current orchestrator file before making changes. Preserve all existing routing rules and agent descriptions exactly -- only add the new dispatch routing rule and update the "Routing Example" section.
