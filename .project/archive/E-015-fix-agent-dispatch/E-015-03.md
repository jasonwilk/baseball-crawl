# E-015-03: Update PM Dispatch Mode for Subagent Awareness

## Epic
[E-015: Fix Agent Dispatch -- Nested Session Error](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent definition's Dispatch Mode section will explicitly document the subagent nesting constraint and describe what to do in each operating mode: when running in the main session, dispatch normally via Task tool; when running as a subagent (Task tool unavailable), produce a Dispatch Manifest so the user or main session can execute the dispatches instead. The PM will never again attempt nested Task calls and fail silently or confusingly.

## Context
The PM's Dispatch Mode procedure (in `/.claude/agents/project-manager.md`) currently says at step 5:

> "5. **Dispatch via Task tool.** For each eligible story, invoke the correct implementing agent with the standard context block."

This step works only when the PM is in the main session. When the PM runs as a subagent (invoked via orchestrator Task routing), step 5 fails with "Claude Code cannot be launched inside another Claude Code session."

The fix requires the PM's Dispatch Mode to:
1. Acknowledge the constraint explicitly
2. Define Mode 1 (main session) and Mode 2 (subagent) behavior
3. Specify the Dispatch Manifest format for Mode 2

The Dispatch Manifest format is defined in `/.claude/rules/dispatch-pattern.md` (E-015-01). This story wires that constraint into the PM's own Dispatch Mode procedure.

## Acceptance Criteria

- [ ] **AC-1**: The PM agent definition at `/.claude/agents/project-manager.md` contains a clearly labeled sub-section within Dispatch Mode titled "Operating Mode Check" (or equivalent) that appears as step 1 or step 2 of the Dispatch Procedure.
- [ ] **AC-2**: The Operating Mode Check describes how to determine which mode the PM is in: if the Task tool is available and not nested inside a subagent context, Mode 1 applies; if the PM was invoked via a Task call (subagent context), Mode 2 applies.
- [ ] **AC-3**: The Dispatch Procedure section explicitly states: "Mode 1 (main session): proceed with Task tool dispatch as described below. Mode 2 (subagent context): produce a Dispatch Manifest and stop -- do not attempt Task tool calls."
- [ ] **AC-4**: The Dispatch Manifest format is defined (or referenced) in the PM definition. The format must match the format in `/.claude/rules/dispatch-pattern.md` -- it must include a header, an explanation that the PM is in subagent context, and one numbered Task block per eligible story.
- [ ] **AC-5**: The existing Dispatch Procedure steps (READY gate check, eligible story identification, status updates, AC verification, DONE marking) are preserved unchanged for Mode 1. The Mode 2 path ONLY adds the manifest output step and does NOT remove or alter the Mode 1 procedure.
- [ ] **AC-6**: The PM definition references `/.claude/rules/dispatch-pattern.md` as the authoritative source for the constraint and manifest format.
- [ ] **AC-7**: Given a PM agent reading only the updated definition, it can determine: (a) how to tell whether it is in Mode 1 or Mode 2, (b) what to do in each mode, and (c) the exact format of a Dispatch Manifest -- without reading any other file.

## Technical Approach

Modify `/.claude/agents/project-manager.md`. The changes are confined to the Dispatch Mode section.

Insert a new step at the top of the Dispatch Procedure (before the current step 1 "Read the epic directory"). Label it "Step 0: Operating Mode Check" or renumber the existing steps. The new step reads approximately:

> **Step 0: Operating Mode Check.** Before any other action, determine whether the PM is running in the main session or as a subagent. If the PM was invoked via a Task tool call from another agent (e.g., orchestrator), it is in a subagent context (Mode 2) and CANNOT use the Task tool to dispatch further. In Mode 2: complete steps 1-4 normally (read the epic, check READY gate, identify eligible stories, update story statuses to IN_PROGRESS), then output a Dispatch Manifest (see format in `/.claude/rules/dispatch-pattern.md`) and stop. Do NOT attempt Task tool calls in Mode 2. In Mode 1 (main session, PM invoked directly): complete all steps normally including Task dispatch.

Then add a "Dispatch Manifest" subsection after the "Standard Context Block" subsection, showing the manifest format. Mirror the format from `/.claude/rules/dispatch-pattern.md` exactly.

Read the current PM definition fully before making changes. Preserve the existing structure and all other sections. Only modify the Dispatch Mode section.
