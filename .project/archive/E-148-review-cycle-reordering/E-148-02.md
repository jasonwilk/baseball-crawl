# E-148-02: Reorder Implement Skill Review Phases and Add Scorecard

## Epic
[E-148: Review Cycle Reordering for Plan and Implement Skills](epic.md)

## Status
`DONE`

## Description
After this story is complete, the implement skill's optional review chain (Phase 4) will run the code-reviewer integration review first (Phase 4a) and Codex code review second (Phase 4b), swapped from the current order. Phase 5 (Closure Sequence) will record a review scorecard table in the epic's History section. Per-story code review in Phase 3 Step 5 remains unchanged.

## Context
The implement skill currently runs Phase 4a (Codex code review) before Phase 4b (CR integration review). The same principle from the plan skill applies: fast internal review (CR integration) should run first to catch cross-story interactions, naming inconsistencies, and import conflicts before Codex runs its systematic rubric pass. This story also adds the review scorecard pattern to the closure sequence for convergence visibility.

## Acceptance Criteria
- [ ] **AC-1**: Phase 4a is titled "Code-Reviewer Integration Review" and contains the CR integration review logic currently in Phase 4b (holistic review of the full epic diff for cross-story interactions).
- [ ] **AC-2**: Phase 4b is titled "Codex Code Review" and contains the Codex code review logic currently in Phase 4a (headless codex → prompt-generation fallback).
- [ ] **AC-3**: All existing edge cases in both Phase 4a and 4b are preserved (timeout, codex not installed, clean review, no uncommitted changes, 2 rounds max circuit breaker per TN-4, prompt fallback).
- [ ] **AC-4**: Phase 5 Step 2 includes PM recording a review scorecard table in the epic's History section, per TN-5 format, covering both per-story reviews (Phase 3) and integration/codex reviews (Phase 4).
- [ ] **AC-5**: Phase 5 Step 6 (summary presentation) includes review outcomes reflecting the new order: "CR integration: clean/N fixed/not run; Codex: clean/N fixed/skipped/not run".
- [ ] **AC-6**: Per-story code review in Phase 3 Step 5 is NOT modified.
- [ ] **AC-7**: The Workflow Summary diagram at the bottom of the skill file is updated to reflect the swapped Phase 4a/4b order.
- [ ] **AC-8**: The chaining modifier documentation ("and review" / "and codex review") remains accurate after the reordering.

## Technical Approach
This is primarily a swap of the existing Phase 4a and 4b content, plus adding the review scorecard to Phase 5. The current Phase 4b (CR integration review) becomes Phase 4a, and the current Phase 4a (Codex code review) becomes Phase 4b. The internal logic, circuit breakers, remediation spawn mechanics, and edge case handling within each sub-phase remain unchanged -- only the execution order changes. The scorecard addition to Phase 5 is a new step that PM executes as part of epic completion. Reference the architect's analysis at `/.claude/agent-memory/claude-architect/review-cycle-reordering.md` for the approved direction.
