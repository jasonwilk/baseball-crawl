# E-074-01: Code Review Prompt Skill

## Epic
[E-074: Codex Review Prompt Generator Skill](./epic.md)

## Status
`DONE`

## Description
After this story is complete, a new skill file exists at `.claude/skills/codex-prompt-code/SKILL.md` that enables Claude to generate a self-contained code review prompt for copy-paste into Codex. The skill instructs Claude to gather the diff via Bash git commands (including full untracked file contents), read the code review rubric, verify the static agent roster, assemble everything into a formatted prompt, and present it to the user in a fenced code block.

## Context
The user sometimes prefers to interact with Codex directly rather than piping through the automated `codex-review.sh` script. This skill gives Claude the instructions to assemble a complete code review prompt that the user can copy-paste. The skill follows the same structural patterns as existing skills (`review-epic`, `spec-review`, `ingest-endpoint`) -- activation triggers, prerequisites, step-by-step workflow, edge cases, and anti-patterns.

This is a separate skill file from the spec review prompt skill (E-074-02), following the same split pattern as the existing `review-epic` / `spec-review` pair. Different inputs (diff vs. spec files), rubrics, triggers, and output structures warrant separation.

Expert consultation confirmed: separate skill file, rubric read at execution time, static agent roster table (not dynamic agent def reads), git commands replicated from codex-review.sh, untracked file contents included (closing a gap in the existing script), three-tier size thresholds computed before assembly.

## Technical Approach
The skill follows the same structural pattern as existing skills at `.claude/skills/review-epic/SKILL.md` and `.claude/skills/ingest-endpoint/SKILL.md`. The implementing agent should read those files for format reference.

Key design constraints from expert consultation:
- Separate skill file (not combined with spec review) -- different inputs, rubrics, and workflow structure
- Rubric content read at execution time via Read tool, not embedded
- Static agent roster table embedded in the skill (not dynamic reads of `.claude/agents/*.md`) -- Claude verifies against CLAUDE.md Agent Ecosystem before assembly
- Diff gathered via the same git commands used in `scripts/codex-review.sh` (the agent should read that script for the exact commands)
- Untracked file contents included via Read tool (closing a gap in the existing script)
- Size thresholds computed before assembly to avoid building a huge prompt only to discard it
