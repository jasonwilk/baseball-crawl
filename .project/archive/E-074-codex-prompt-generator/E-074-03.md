# E-074-03: CLAUDE.md Workflows Entries

## Epic
[E-074: Codex Review Prompt Generator Skill](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the CLAUDE.md Workflows section includes two new entries that register the codex prompt generator skills' trigger phrases, so Claude knows to load the correct skill when the user requests a code review prompt or a spec review prompt.

## Context
Every user-triggered workflow skill in the project has a corresponding entry in the CLAUDE.md Workflows section that maps trigger phrases to the skill file path. (Agent-loaded skills like `filesystem-context`, `multi-agent-patterns`, and `context-fundamentals` do not have Workflows entries -- they are loaded by specific agents, not triggered by user phrases.) This story adds two entries -- one for the code review prompt skill (E-074-01) and one for the spec review prompt skill (E-074-02). This follows the existing pattern where `review-epic` and `spec-review` each have their own Workflows entry.

## Acceptance Criteria
- [ ] **AC-1**: The CLAUDE.md `## Workflows` section contains a new entry for the code review prompt skill, referencing `.claude/skills/codex-prompt-code/SKILL.md`
- [ ] **AC-2**: The CLAUDE.md `## Workflows` section contains a new entry for the spec review prompt skill, referencing `.claude/skills/codex-prompt-spec/SKILL.md`
- [ ] **AC-3**: Each entry includes a primary trigger phrase and representative alternatives from the corresponding skill's Activation Triggers section, using the "or similar" pattern consistent with existing entries
- [ ] **AC-4**: Each entry's format matches the existing Workflows entries: a markdown bullet starting with bold skill name, followed by trigger phrases in parentheses after "When the user says", the skill file path, and a one-sentence description of the workflow. Reference the existing "Implement", "Spec review", and "Review epic" entries as the concrete format template
- [ ] **AC-5**: Existing Workflows entries are unchanged

## Technical Approach
Read the current CLAUDE.md Workflows section to understand the existing entry format. Add two new entries following the same pattern. Each entry maps trigger phrases to a single skill file path.

Reference: the trigger phrases come from the Activation Triggers sections of the skill files created in E-074-01 and E-074-02.
