# E-074-02: Spec Review Prompt Skill

## Epic
[E-074: Codex Review Prompt Generator Skill](./epic.md)

## Status
`DONE`

## Description
After this story is complete, a new skill file exists at `.claude/skills/codex-prompt-spec/SKILL.md` that enables Claude to generate a self-contained spec review prompt for copy-paste into Codex. The skill instructs Claude to gather epic/story markdown files via Glob + Read, read the spec review rubric, verify the static agent roster, assemble everything into a formatted prompt, and present it to the user in a fenced code block.

## Context
The user sometimes prefers to interact with Codex directly rather than piping through the automated `codex-spec-review.sh` script. This skill gives Claude the instructions to assemble a complete spec review prompt that the user can copy-paste. The skill follows the same structural patterns as existing skills (`review-epic`, `spec-review`, `ingest-endpoint`) -- activation triggers, prerequisites, step-by-step workflow, edge cases, and anti-patterns.

This is a separate skill file from the code review prompt skill (E-074-01), following the same split pattern as the existing `review-epic` / `spec-review` pair.

Expert consultation confirmed: separate skill file, rubric read at execution time, static agent roster table, spec files gathered via Glob + Read. Edge cases: epic dirs may contain scratch/draft files; stories referencing external docs will not be auto-included.

## Technical Approach
The skill follows the same structural pattern as existing skills at `.claude/skills/spec-review/SKILL.md` and `.claude/skills/ingest-endpoint/SKILL.md`. The implementing agent should read those files for format reference.

Key design constraints from expert consultation:
- Separate skill file (not combined with code review) -- different inputs, rubrics, and workflow structure
- Rubric content read at execution time via Read tool, not embedded
- Static agent roster table embedded in the skill (not dynamic reads of `.claude/agents/*.md`) -- Claude verifies against CLAUDE.md Agent Ecosystem before assembly
- Spec files gathered via Glob + Read (more natural for Claude than shell find+cat)
- Epic directory may contain scratch/draft files -- skill should note this as a caveat
