# E-010-03: Write `context-fundamentals` Skill

## Epic
[E-010: Intent/Context Layer Implementation](epic.md)

## Status
`DONE`

## Description
After this story is complete, a `context-fundamentals` skill file exists at `.claude/skills/context-fundamentals/SKILL.md`. The skill provides foundational grounding on what a context window is, what goes into it for baseball-crawl sessions, how context fills up during complex multi-file work, and how to reason about when to compact vs. start fresh. The file includes a baseball-crawl-specific section on the project's context budget -- what loads automatically per session versus what agents load deliberately per task.

## Context
The muratcankoylan Agent Skills for Context Engineering repository defines a `context-fundamentals` skill in its Foundational category. It covers: what context is, its components (system prompt, conversation history, tool outputs, loaded files), why token efficiency matters, and the failure modes that arise when context management is poor (lost-in-the-middle, context poisoning, attention scarcity).

Baseball-crawl's context budget is specific and worth documenting explicitly. Every session automatically loads:
- The project CLAUDE.md (232 lines)
- All files in `.claude/rules/` (6 files, ~212 lines total)
- The agent's own definition from `.claude/agents/<agent>.md` (~100-200 lines)
- The agent's MEMORY.md from `.claude/agent-memory/<agent>/MEMORY.md` (varies)

That is approximately 600-700 lines of ambient context before any task-specific content is loaded. Task-specific content added on demand:
- Story file (50-150 lines per story)
- Epic Technical Notes (50-200 lines)
- Research artifacts from `.project/research/` (variable, 100-400 lines each)
- Raw API responses or log excerpts (variable, can be large)

Understanding this budget helps agents make intentional choices: when to load a research artifact vs. trusting MEMORY.md, when a session is getting too full to handle another epic's Technical Notes, when to use `/clear` and start fresh.

**Source material**: The muratcankoylan SKILL.md format is documented in `/.project/research/E-008-R-02-agent-skills-summary.md`. The four-section format is: activation triggers, key concepts, practical implementation, references.

**Adaptation requirement**: The skill must include concrete numbers for baseball-crawl's context budget (what loads per session vs. per task). An agent reading this skill should be able to estimate whether a given session has room for another large research artifact without running into the context limit.

## Technical Approach

Write the skill from scratch using the muratcankoylan four-section format, drawing on:
1. The format description in `/.project/research/E-008-R-02-agent-skills-summary.md`
2. Baseball-crawl's CLAUDE.md (the "Workflow" section mentions `/clear` between unrelated tasks -- this is context management advice already embedded in the project)
3. The statusline hook (`.claude/hooks/statusline.sh`) which shows context window percentage -- agents can observe their own context usage
4. The implementation sketch in `/.project/research/E-008-intent-context-layer-recommendation.md` (Phase 1, `context-fundamentals` adaptation notes)
5. General knowledge of LLM context window mechanics

Do NOT fetch the muratcankoylan repository directly. The research summary and listed project files contain sufficient material.

The skill should read as an orientation document that any agent encounters before undertaking complex multi-file work. It is foundational -- not deeply specific to any one task type. The baseball-crawl-specific section (AC-5 through AC-7) is what distinguishes this from a generic context tutorial.
