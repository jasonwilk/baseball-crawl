## Description
After this story is complete, a `filesystem-context` skill file exists at `.claude/skills/filesystem-context/SKILL.md`. The skill teaches agents how baseball-crawl uses the filesystem as a dynamic context store -- story files, epic files, research artifacts, and agent memory files are all context that agents load on demand, not pre-loaded into every session. The file is adapted from the muratcankoylan pattern with project-specific examples replacing generic ones.

## Context
The muratcankoylan Agent Skills for Context Engineering repository (researched in E-008-R-02) defines a `filesystem-context` skill in its Architectural category. The skill's core concept is that context can live in files on the filesystem and be loaded by agents on demand rather than being baked into static prompts -- a technique for managing context windows efficiently.

Baseball-crawl already uses this pattern extensively but has never named or codified it:
- Implementing agents load their story file (full content) at task start
- The PM loads epic.md + all story files before dispatch
- claude-architect loads design docs from `.project/research/` when making architectural decisions
- All agents access their `MEMORY.md` file at session start for persistent context

The skill should make this pattern explicit, so agents understand why they are loading files (not just that they should) and can apply the pattern to new situations.

**Source material**: The muratcankoylan SKILL.md format is documented in `/.project/research/E-008-R-02-agent-skills-summary.md` (sections: "Key Tensions", "Key Q&A" #4, and the SKILL.md structure summary). The four-section format is: activation triggers, key concepts, practical implementation, references.

**Adaptation requirement**: The skill must contain at least one named baseball-crawl example per major concept. Generic descriptions of the filesystem-context pattern are necessary but not sufficient. An agent reading this skill should be able to name three specific files in this project that implement the pattern.

## Technical Approach

Write the skill from scratch using the muratcankoylan four-section format, drawing on:
1. The format description in `/.project/research/E-008-R-02-agent-skills-summary.md`
2. Baseball-crawl's existing file-based context patterns (observable in CLAUDE.md, story files, epic files, and agent memory files)
3. The implementation sketch in `/.project/research/E-008-intent-context-layer-recommendation.md` (Phase 1 section, `filesystem-context` adaptation notes)

Do NOT fetch the muratcankoylan repository directly. The research summary contains sufficient source material. The adaptation requirement means project-specific content is more valuable than faithful reproduction of the upstream source.

The skill file should read as if it were written specifically for baseball-crawl, with the muratcankoylan concepts as the backbone. An agent encountering this skill for the first time should come away knowing: (a) what the filesystem-context pattern is, (b) why baseball-crawl uses it, and (c) exactly which files in this project are the pattern's primary instances.
