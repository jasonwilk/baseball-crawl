# E-050-04: Operator bootstrap guide

## Epic
[E-050: Credential Validation and Crawl Bootstrap](epic.md)

## Status
`DONE`

## Description
After this story is complete, a clear operator guide at `docs/admin/bootstrap-guide.md` documents the end-to-end workflow from "I want to start getting data" to "data is in the database and visible on the dashboard." It covers both credential capture paths (proxy and curl), the bootstrap command, and common troubleshooting scenarios.

## Context
The credential capture mechanisms (proxy and curl) and the bootstrap pipeline (E-050-03) work independently, but there is no documentation tying them together into a coherent operator workflow. A first-time operator (or the current operator returning after a break) needs a single guide that answers: "How do I get this system running?"

## Acceptance Criteria
- [ ] **AC-1**: `docs/admin/bootstrap-guide.md` exists
- [ ] **AC-2**: The guide has a "Quick Start" section at the top with the minimal steps to go from zero to data:
  1. Capture credentials (proxy or curl -- one paragraph each, linking to detailed sections)
  2. Run `python scripts/bootstrap.py`
  3. Check the dashboard
- [ ] **AC-3**: The guide has a "Credential Capture: Proxy" section explaining the proxy path:
  - How to start the proxy (`cd proxy && ./start.sh`)
  - How to configure iPhone to use the proxy (reference the start.sh output)
  - How to verify credentials were captured (check `.env` for `GAMECHANGER_AUTH_TOKEN` key presence -- never show values)
  - Link to `docs/admin/mitmproxy-guide.md` for detailed proxy setup
- [ ] **AC-4**: The guide has a "Credential Capture: Curl" section explaining the curl path:
  - How to copy a curl command from browser dev tools
  - How to save it to `secrets/gamechanger-curl.txt`
  - How to run `python scripts/refresh_credentials.py`
- [ ] **AC-5**: The guide has a "Bootstrap Command" section documenting all `scripts/bootstrap.py` flags:
  - Default behavior (validate + crawl + load)
  - `--check-only` (validate only)
  - `--profile web|mobile`
  - `--dry-run`
- [ ] **AC-6**: The guide has a "Credential Lifecycle" section explaining:
  - Token lifetime (14 days from JWT `exp - iat`)
  - When to refresh (every ~2 weeks, or when the health check reports expired)
  - What happens when credentials expire mid-crawl (crawlers report `CredentialExpiredError`, partial data may be saved)
- [ ] **AC-7**: The guide has a "Troubleshooting" section with common scenarios:
  - "Credentials missing" (run one of the capture paths)
  - "Credentials expired" (recapture)
  - "No teams configured" (use admin UI or edit teams.yaml)
  - "Crawl failed but load succeeded" (partial crawl data was loadable)
  - "Cannot reach GameChanger API" (network issue, check connectivity)
- [ ] **AC-8**: The guide does NOT contain any credential values, API tokens, or example tokens
- [ ] **AC-9**: The guide cross-references existing docs: `docs/admin/mitmproxy-guide.md` for proxy details, `docs/admin/operations.md` for general operations, `docs/admin/getting-started.md` for initial setup

## Technical Approach
1. Create `docs/admin/bootstrap-guide.md`
2. Follow the existing docs style in `docs/admin/` (plain Markdown, clear section headers, code blocks for commands)
3. Keep it operator-focused: what to do, not how the code works
4. Reference the actual script flags (read `scripts/bootstrap.py` and `scripts/check_credentials.py` for current flag names)

## Notes
