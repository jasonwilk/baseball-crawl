## Description
After this story is complete, the project will have a `/ephemeral/` directory tree that is safe by default: it is added to the root `.gitignore`, organized by epic ID for structured exploration data, and documented with a README that explains its purpose and warns against committing its contents. Any developer or agent can immediately drop raw API responses, scratch JSON, or intermediate data files into the correct subdirectory with no risk of accidentally committing that data.

## Context
During API exploration and ETL development, agents and developers will regularly pull raw GameChanger API responses to disk for inspection. Without a designated landing zone, these files end up scattered across the project root, `data/`, or `src/`, where they are at risk of being accidentally staged.

The `/ephemeral/` directory is intentionally NOT the same as `data/raw/`, which is the production pipeline's output directory and may be committed (without PII, after redaction). Ephemeral means: never committed, never permanent, no expectations of longevity.

This story has no blocking dependencies and can execute immediately. It shares no files with any other story in this epic.

## Acceptance Criteria
- [ ] **AC-1**: Given the repository root, a directory `/ephemeral/` exists with subdirectories `E-001/`, `E-002/`, `E-003/`, `E-004/`, `E-005/`, `E-006/`, and `scratch/`
- [ ] **AC-2**: Given the root `.gitignore`, when the file is read, then `/ephemeral/` appears as an entry, causing Git to ignore the entire directory tree
- [ ] **AC-3**: Given a file written to `/ephemeral/E-001/test.json`, when `git status` is run, then that file does not appear as an untracked file
- [ ] **AC-4**: Given `/ephemeral/README.md`, when it is read, then it explains: (a) what the directory is for, (b) that contents are never committed, (c) how to choose the right subdirectory, and (d) that the directory itself (minus README) is gitignored
- [ ] **AC-5**: Given each epic subdirectory (`E-001/` through `E-006/` and `scratch/`), a `.gitkeep` file exists so the directory structure is preserved in the repository (the `.gitkeep` itself is committed, not gitignored)

## Technical Approach
1. Create `/ephemeral/` with the subdirectory structure
2. Add a `.gitkeep` file to each subdirectory (so empty directories are trackable)
3. Write `/ephemeral/README.md` -- see content guidance in Notes below
4. Add `/ephemeral/` to the root `.gitignore` (add a new section at the bottom; do not restructure existing `.gitignore` content)
5. Verify with `git status` and `git check-ignore -v ephemeral/E-001/test.json` that the ignore rule works as expected

Note: The `.gitkeep` files in subdirectories WILL be committed because Git evaluates gitignore rules from the inside out -- a `.gitkeep` inside `/ephemeral/E-001/` is covered by the `/ephemeral/` ignore rule. The way to commit the skeleton is to add a `.gitignore` inside `/ephemeral/` that says `*` + `!.gitkeep` + `!README.md`, and then the parent `.gitignore` does NOT ignore `/ephemeral/` itself -- only its other contents.

Revised approach (more correct):
- Root `.gitignore`: add `ephemeral/**` and `!ephemeral/.gitignore` and `!ephemeral/README.md` and `!ephemeral/**/.gitkeep`
- OR: use `/ephemeral/**` with exceptions for `.gitkeep` and `README.md`
- Whichever approach is chosen must satisfy AC-3 (test.json is invisible to git) and AC-5 (.gitkeep files are tracked)

The implementing agent should choose the cleanest gitignore pattern that satisfies both criteria and document which pattern was used in the commit message.

## Notes

### README Content Guidance
The `/ephemeral/README.md` should include:

**Purpose**: This directory is the safe landing zone for any data file generated during development that contains or might contain real data from the GameChanger API. It is gitignored and NEVER committed.

**What belongs here**: Raw API responses, intermediate JSON/CSV files pulled during exploration, any data file from a real API call.

**What does NOT belong here**: Test fixtures with synthetic data (those go in `tests/fixtures/`), processed output intended for the D1 database.

**How to choose a subdirectory**: Use the epic you are currently working on (e.g., `/ephemeral/E-005/` if you are working on E-005 stories). Use `/ephemeral/scratch/` for one-off experiments not tied to an epic.

**Warning**: Files in this directory are never backed up or versioned. If you need to share an exploration artifact with a teammate, describe the API call that produced it, not the file itself.
