# E-149-03: Testing Rule and Codex Rubric Alignment

## Epic
[E-149: Review Methodology Retro and Gap-Filling](epic.md)

## Status
`DONE`

## Description
After this story is complete, the testing rule will include a test-validates-spec clause that prevents tests from vacuously passing by mirroring buggy implementation data. The Codex rubric will include inline summaries of the CR rubric's expanded checklist items -- keeping the CR agent definition as the single source of truth while ensuring Codex can apply the checks without file access.

## Context
Retro finding 2 showed that a test mocked the wrong API response shape (matching the buggy implementation), so the test passed despite the code being wrong. The testing rule has no clause about verifying mock data against authoritative specs. Meanwhile, the Codex rubric (`.project/codex-review.md`) is a separate, simpler document that doesn't benefit from the detailed checklist items added to the CR rubric in E-149-01. Aligning them ensures all review layers share the same vocabulary. See epic TN-5 and TN-6.

## Acceptance Criteria
- [ ] **AC-1**: `.claude/rules/testing.md` contains a **Test-validates-spec** section per TN-5. The section instructs implementers to verify mock data (API responses, DB query results) matches the authoritative spec (`docs/api/endpoints/`, `migrations/*.sql`, docstrings) -- not the implementation under test.
- [ ] **AC-2**: The test-validates-spec section includes a concrete example based on retro finding #2 (the `team_season.year` vs `season_year` field path divergence from E-147), showing the wrong pattern (mock mirrors buggy implementation's field path) and the right pattern (mock mirrors the API endpoint doc's field path).
- [ ] **AC-3**: `.project/codex-review.md` includes a new priority item with **inline summaries** (one sentence each) of the 7 Bug Pattern Checklist items (6 new + Status lifecycle extension) from the CR agent definition, per TN-6. The summaries are sufficient for a reviewer without file access to apply each check.
- [ ] **AC-4**: The Codex rubric item notes that `.claude/agents/code-reviewer.md` is the authoritative source and that summaries should be updated when the CR checklist changes. The item does NOT use a file cross-reference that requires the model to read another file (the codex-review skill embeds rubric content in the prompt; file references are dead text).
- [ ] **AC-5**: Existing testing rule sections (Test Scope Discovery, Error-Path Testing, subprocess smoke tests) are unchanged.
- [ ] **AC-6**: Existing Codex rubric priorities (1-6) are unchanged; the new item is additive.

## Technical Approach
Read `.claude/rules/testing.md` and `.project/codex-review.md`. For the testing rule, add a new section after "Error-Path Testing" with the test-validates-spec clause and example. For the Codex rubric, add a priority 7 with inline summaries of each CR checklist item (one sentence each, per TN-6). The summaries must be self-contained -- the codex-review skill embeds rubric content directly in the prompt, so Codex cannot follow file references to read the full checklist items.
