# E-053-04: Update .env.example and documentation

## Epic
[E-053: Profile-Scoped Credentials](epic.md)

## Status
`DONE`

## Description
After this story is complete, `.env.example` documents the new profile-scoped credential keys, and `CLAUDE.md` Commands section includes the `--profile` flag for `check_credentials.py`. The operator can see at a glance what keys are available and how to use them.

## Context
The credential section in `.env.example` currently shows only flat keys with a comment saying they are managed by `scripts/refresh_credentials.py`. After E-053-01 and E-053-02, the proxy writes profile-scoped keys and the client reads them. The example file and command docs need to reflect this.

## Acceptance Criteria
- [ ] **AC-1**: `.env.example` has a "GameChanger API credentials" section that documents profile-scoped keys (`GAMECHANGER_AUTH_TOKEN_WEB`, `_MOBILE`, `GAMECHANGER_DEVICE_ID_WEB`, `_MOBILE`, etc.) with comments explaining the naming convention. No flat (unsuffixed) credential keys remain.
- [ ] **AC-2**: The old flat key entries (`GAMECHANGER_AUTH_TOKEN`, `GAMECHANGER_DEVICE_ID`, etc.) are removed from `.env.example`. A comment notes that credentials are written automatically by the mitmproxy credential extractor.
- [ ] **AC-3**: `CLAUDE.md` Commands section includes `python scripts/check_credentials.py [--profile web|mobile]` and `python scripts/bootstrap.py [--profile web|mobile] [--check-only] [--dry-run]`.

## Technical Approach
- Replace the old flat-key credential section in `.env.example` with profile-scoped keys. Group by profile (web, mobile) under a clear heading.
- Add the `check_credentials.py` and `bootstrap.py` command entries to `CLAUDE.md` Commands section (follow-up from E-050).
