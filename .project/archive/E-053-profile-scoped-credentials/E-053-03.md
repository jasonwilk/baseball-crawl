# E-053-03: Profile-aware check_credentials and bootstrap

## Epic
[E-053: Profile-Scoped Credentials](epic.md)

## Status
`DONE`

## Description
After this story is complete, `check_credentials.py` will accept a `--profile` flag to validate credentials for a specific profile, and will default to reporting the status of all available credential sets. `bootstrap.py` already passes `--profile` to the crawl stage; this story makes the credential check stage profile-aware so the bootstrap pipeline validates the right credentials before crawling.

## Context
`check_credentials.py` currently instantiates `GameChangerClient()` with no profile argument (defaults to `"web"`). After E-053-02 lands, the client reads profile-scoped keys -- but `check_credentials.py` needs to know *which* profile to check. `bootstrap.py` already accepts `--profile` and passes it to the crawl stage, but its credential check stage calls `check_credentials()` without a profile, creating a mismatch.

## Acceptance Criteria
- [ ] **AC-1**: `python scripts/check_credentials.py --profile web` validates web-profile credentials (`GAMECHANGER_AUTH_TOKEN_WEB`, `GAMECHANGER_DEVICE_ID_WEB`) and reports the result.
- [ ] **AC-2**: `python scripts/check_credentials.py --profile mobile` validates mobile-profile credentials (`_MOBILE` keys).
- [ ] **AC-3**: `python scripts/check_credentials.py` (no flag) checks both web and mobile credential sets and reports each one's status (e.g., "web: valid -- logged in as Jason Smith", "mobile: missing -- GAMECHANGER_AUTH_TOKEN_MOBILE not set").
- [ ] **AC-4**: When `--profile` is given, exit code is 0 if that profile is valid, non-zero otherwise. When no `--profile` is given, exit code is 0 only when at least one profile is valid (operator may legitimately have only one profile configured).
- [ ] **AC-5**: `bootstrap.py` passes its `--profile` value to `check_credentials(profile=args.profile)` so the bootstrap pipeline validates the correct profile before crawling.
- [ ] **AC-6**: The `_REQUIRED_KEYS` tuple in `check_credentials.py` is removed or updated to use profile-scoped key names (no flat keys).
- [ ] **AC-7**: Tests cover: single profile check (valid, expired, missing), multi-profile summary, bootstrap integration with profile.

## Technical Approach
- Add a `profile` parameter to `check_credentials()`: `def check_credentials(profile: str | None = None)`.
- When `profile` is provided, instantiate `GameChangerClient(profile=profile)` and check that single profile.
- When `profile` is `None`, check both `_WEB` and `_MOBILE` credential sets. Report per-profile status. Missing keys for a profile are reported but do not fail if the other profile is valid.
- In `bootstrap.py`, pass the `profile` argument from CLI args to `check_credentials(profile=args.profile)`.
- Update the CLI argument parser in `check_credentials.py` to accept `--profile`.
- Note: these scripts will eventually be wrapped by E-055's unified `bb` CLI (separate epic, no dependency).
