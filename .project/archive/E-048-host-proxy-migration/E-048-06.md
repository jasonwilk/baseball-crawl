# E-048-06: Update CLAUDE.md and documentation

## Epic
[E-048: Migrate mitmproxy to Host-Based Standalone Proxy](epic.md)

## Status
`DONE`

## Description
After this story is complete, CLAUDE.md and `docs/admin/mitmproxy-guide.md` will reflect the host-based proxy architecture. CLAUDE.md Commands section will reference the new proxy scripts. The mitmproxy guide will be rewritten for the host-based workflow.

## Context
CLAUDE.md currently references `./scripts/proxy.sh {start|stop|status}` and data paths under `data/mitmproxy/`. The mitmproxy guide describes the Docker-in-Docker setup with VS Code port forwarding. Both need updating to describe the Mac host approach: `cd proxy && ./start.sh`, data at `proxy/data/`, certs at `proxy/certs/`, devcontainer reaches proxy via `host.docker.internal:8080`.
