# E-048-05: Clean up devcontainer.json

## Epic
[E-048: Migrate mitmproxy to Host-Based Standalone Proxy](epic.md)

## Status
`DONE`

## Description
After this story is complete, `.devcontainer/devcontainer.json` will have no mitmproxy-related port forwarding or attributes. Ports 8080 and 8081 will be removed from `forwardPorts`. The `portsAttributes` for 8080 and 8081 will be removed. The inert `remote.localPortHost` and `remote.restoreForwardedPorts` settings that were added to debug mitmproxy networking can also be removed if they serve no other purpose.

## Context
With mitmproxy running on the Mac host instead of inside Docker-in-Docker, VS Code no longer needs to forward ports 8080 or 8081. The `portsAttributes` entry for 8080 (`allInterfaces`) was an attempt to solve iPhone access through VS Code forwarding -- that entire approach is now obsolete. The `remote.localPortHost` and `remote.restoreForwardedPorts` settings were added during E-045 troubleshooting and are not needed for app/traefik ports.

- [ ] **AC-3**: The `remote.localPortHost` setting is removed (it was added for mitmproxy; the remaining ports work fine with VS Code defaults).
