# E-048-07: Delete prototype and obsolete files

## Epic
[E-048: Migrate mitmproxy to Host-Based Standalone Proxy](epic.md)

## Status
`DONE`

## Description
After this story is complete, the `proxy-host/` prototype directory and `MITM-TROUBLESHOOTING.md` will be deleted. The old `scripts/proxy.sh` control script will also be deleted since it is replaced by the scripts in `proxy/`.

## Context
`proxy-host/` was a quick prototype to validate the host-based approach. Its functionality is now consolidated into `proxy/` (E-048-01). `MITM-TROUBLESHOOTING.md` documents the Docker-in-Docker networking problems that no longer apply with the host-based approach. `scripts/proxy.sh` managed the old `docker compose --profile proxy` workflow that no longer exists.
