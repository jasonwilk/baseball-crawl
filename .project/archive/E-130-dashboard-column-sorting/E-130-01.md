# E-130-01: Sortable Batting and Pitching Stats Tables

## Epic
[E-130: Dashboard Column Sorting](epic.md)

## Status
`DONE`

## Description
After this story is complete, coaches can click any column header on the team batting or pitching stats tables to sort by that column. Clicking the same header again reverses the sort direction. The active sort column shows a directional indicator (▲/▼). Sort state is preserved in page links.

## Context
The dashboard batting table (`/dashboard`) and pitching table (`/dashboard/pitching`) display stats in a fixed order (batting by AVG DESC, pitching by ERA ASC). Coaches cannot re-sort to answer questions like "who strikes out the most?" or "who has the best OBP?". Both tables follow the same Jinja2 `{% for %}` + `<thead>`/`<tbody>` pattern and share the same route structure in `src/api/routes/dashboard.py`.

## Acceptance Criteria
- [ ] **AC-1**: The `/dashboard` route accepts optional `sort` and `dir` query parameters. When `sort` is a recognized batting column key (per TN-3), the player list is sorted by that column. When `dir` is `asc` or `desc`, the sort direction is applied. Unrecognized `sort` values fall back to the default sort (AVG descending).
- [ ] **AC-2**: The `/dashboard/pitching` route accepts the same `sort` and `dir` query parameters, with pitching column keys per TN-3. Default sort remains ERA ascending.
- [ ] **AC-3**: All column headers in `team_stats.html` and `team_pitching.html` are rendered as clickable `<a>` links. Each link targets the current page with `team_id`, `season_id` (if present), `sort`, and `dir` parameters set appropriately.
- [ ] **AC-4**: When a column is already the active sort column, clicking it toggles the direction (asc becomes desc and vice versa). When a column is not the active sort, clicking it applies the default direction for that column per TN-4.
- [ ] **AC-5**: The active sort column's header displays a `▲` (ascending) or `▼` (descending) indicator adjacent to the column label. Non-active columns show no indicator.
- [ ] **AC-6**: Players/pitchers with zero denominators (0 AB for batting, 0 IP for pitching) sort to the bottom regardless of sort direction, per TN-2.
- [ ] **AC-7**: The `sort` and `dir` values are passed to the template context so the team selector macro can preserve them in its links (same-page team switch preserves sort state).
- [ ] **AC-8**: Tests verify: (a) default sort order when no params provided matches current behavior, (b) recognized sort params produce correctly ordered results, (c) unrecognized sort params fall back to default, (d) direction toggle works, (e) zero-denominator rows sort to bottom.

## Technical Approach
Read `src/api/routes/dashboard.py` for the `team_stats()` and `team_pitching()` route handlers. Both currently fetch data from `src/api/db.py` and pass it to templates. The sorting logic should be added in the route handlers after data fetch (and after `_compute_pitching_rates()` for pitching).

For rate stat sort keys, compute the numeric value from raw columns using the formulas in TN-2. Do not change how templates render formatted values -- the sort operates on the list order, not on display formatting.

For the template changes, study `src/api/templates/dashboard/team_stats.html` and `team_pitching.html` -- the `<th>` elements in `<thead>` become `<a>` links. The route handler passes `current_sort`, `current_dir`, `active_team_id`, and `season_id` to the template so headers can construct their URLs. Consider a Jinja2 macro for the sortable header pattern to avoid repetition across both templates.

The `_team_selector.html` macro may need an update to accept and forward sort params when generating same-page links.
