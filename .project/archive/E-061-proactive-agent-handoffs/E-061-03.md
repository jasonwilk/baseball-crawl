# E-061-03: Update PM Dispatch Procedure for Proactive Handoffs

## Epic
[E-061: Proactive Agent Handoffs](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent definition's Dispatch Procedure will incorporate the proactive handoff patterns established by E-061-01 and E-061-02: reading Agent Hint fields for routing, sending a spawn plan to the team lead at dispatch start, and including Handoff Context declarations in context blocks sent to implementing agents.

## Context
E-061-01 added Agent Hint and Handoff Context fields to the story template. E-061-02 added the Agent Hint paragraph, pre-spawn communication pattern, and Context Packaging update to dispatch-pattern.md. The PM agent definition (`product-manager.md`) contains the PM's own Dispatch Procedure -- a numbered step sequence PM follows during dispatch. This story updates that procedure so PM actually uses the new patterns. Without this story, the patterns exist in templates and rules but PM's operational procedure does not reference them.

## Acceptance Criteria
- [ ] **AC-1**: The Dispatch Procedure in `/.claude/agents/product-manager.md` includes a new sub-step after step 5 (identify available teammates) where PM sends the team lead a message listing all agent types needed across all dependency waves -- not just wave 1 -- per the pre-spawn communication pattern in dispatch-pattern.md. This lets the team lead prepare to spawn later-wave agents on PM's signal rather than waiting for reactive requests.
- [ ] **AC-2**: Step 6 (context-layer routing check) is updated to note that PM reads the Agent Hint field first and falls back to file-path inference only when the hint is absent. Context-layer routing precedence still overrides the hint.
- [ ] **AC-3**: Step 7 (assign stories to teammates) is updated to note that when assigning a story whose upstream dependencies have Handoff Context declarations, PM includes the declared artifact paths and descriptions in the context block alongside the full story file and Technical Notes.
- [ ] **AC-4**: The Context Block Format section is updated to include a "Handoff context from completed dependencies" block that lists any Handoff Context declarations from upstream stories, showing the artifact paths and descriptions declared by the producing story.
- [ ] **AC-5**: No existing dispatch steps are removed or reordered -- the changes are purely additive refinements to existing steps.

## Technical Approach
The PM agent definition is a large file. The implementing agent should read the Dispatch Procedure section (steps 1-11) and the Context Block Format section to understand the existing structure before making surgical additions. The changes are small: a new sub-step, updates to two existing steps, and an addition to the context block format. The implementing agent should reference `/.claude/rules/dispatch-pattern.md` (updated by E-061-02) for the pre-spawn communication pattern terminology and `/.project/templates/story-template.md` (updated by E-061-01) for the Agent Hint and Handoff Context field names.
