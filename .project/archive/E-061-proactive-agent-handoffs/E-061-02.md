# E-061-02: Add Agent Hint and Pre-Spawn Protocol to Dispatch Pattern

## Epic
[E-061: Proactive Agent Handoffs](epic.md)

## Status
`DONE`

## Description
After this story is complete, dispatch-pattern.md will document the Agent Hint routing mechanism, include a pre-spawn communication pattern for multi-wave epics, and update Context Packaging to reference Handoff Context declarations from upstream stories.

## Context
dispatch-pattern.md is the authoritative reference for how dispatch works. It currently describes reactive patterns: PM infers agent types from file paths, spawns are requested as stories become eligible, and context packaging mentions only the story file and Technical Notes. This story adds three refinements: (1) a paragraph in Agent Selection acknowledging that stories may carry an Agent Hint, (2) a pre-spawn communication pattern so PM communicates all needed agent types upfront rather than one-at-a-time, and (3) a Context Packaging note about including Handoff Context from upstream stories.

## Acceptance Criteria
- [ ] **AC-1**: The Agent Selection section in `/.claude/rules/dispatch-pattern.md` includes a paragraph (after the existing Dispatch Team metadata paragraph) explaining that stories may carry an optional Agent Hint field, PM should prefer the hint over file-path inference when present, and PM retains final routing authority (the hint is advisory).
- [ ] **AC-2**: The Routing Precedence paragraph is updated to note that context-layer routing precedence still applies even when an Agent Hint is present.
- [ ] **AC-3**: The Spawning Scenarios section includes a new "Pre-spawn communication" subsection (after the existing "Mid-dispatch (cascading)" paragraph). Content: At dispatch start, PM reviews the full dependency graph and messages the team lead with all agent types needed across all waves, not just the first. The team lead spawns wave-1 agents immediately; PM signals when to spawn later-wave agents. For single-wave epics, the existing "At team creation" batch spawn is unchanged (all agents spawned at once). This subsection explicitly positions itself as extending the existing batch-spawn pattern to multi-wave cases, not replacing it.
- [ ] **AC-4**: The existing "Mid-dispatch (cascading)" paragraph is updated to note that cascading remains as a fallback for truly unexpected agent needs discovered mid-dispatch, with pre-spawn communication as the preferred approach.
- [ ] **AC-5**: The Context Packaging section is updated to note that when assigning a story, PM should check completed upstream stories for Handoff Context declarations and include the declared artifacts in the context block alongside the full story file and Technical Notes.
- [ ] **AC-6**: No existing behavior is broken -- all changes are additive refinements to existing sections.

## Technical Approach
Three targeted additions to existing sections in dispatch-pattern.md. The implementing agent should read the full file to understand current section structure and placement before making changes. The pre-spawn communication should be concise (a few sentences describing a communication pattern, not a multi-step algorithm).
