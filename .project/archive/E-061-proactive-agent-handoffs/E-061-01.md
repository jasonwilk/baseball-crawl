# E-061-01: Add Dispatch Metadata to Story and Epic Templates

## Epic
[E-061: Proactive Agent Handoffs](epic.md)

## Status
`DONE`

## Description
After this story is complete, the story template will include two new optional dispatch metadata fields (Agent Hint and Handoff Context) and the epic template's Dispatch Team section will include guidance for deriving the team composition from story Agent Hints.

## Context
Today, stories declare "Files to Create or Modify" but carry no explicit routing or handoff metadata. PM infers agent types by cross-referencing files against the routing table, and assembles cross-story context ad-hoc after each story completes. Adding optional dispatch metadata to the story template lets PM skip the inference step (Agent Hint) and pre-stage handoff context at planning time (Handoff Context). Updating the epic template's Dispatch Team guidance connects the two: PM derives the team from the union of story hints.

## Acceptance Criteria
- [ ] **AC-1**: The story template at `/.project/templates/story-template.md` contains an "Agent Hint" field between "Files to Create or Modify" and "Definition of Done." The field is a single optional line with a comment explaining: its purpose (routing hint for PM), that it is optional, that valid values are agent names as they appear in `.claude/agents/` filenames (`software-engineer`, `data-engineer`, `claude-architect`, `docs-writer`, `ux-designer`, `api-scout`), and that if the hint disagrees with the routing table, the routing table wins.
- [ ] **AC-2**: The story template contains a "Handoff Context" section after Agent Hint and before "Definition of Done." The section includes a comment explaining its purpose (declaring what artifacts this story produces for downstream stories) and shows the format: a bulleted list where each bullet names a downstream story ID and describes what it needs.
- [ ] **AC-3**: Both fields are clearly marked as optional. Stories with no routing preference omit Agent Hint; stories with no downstream consumers omit Handoff Context.
- [ ] **AC-4**: The Dispatch Team section comment in `/.project/templates/epic-template.md` includes additional guidance that PM should derive the team list from the union of Agent Hint values across all stories when hints are present, use file-path inference for stories lacking a hint, and that the section remains advisory.
- [ ] **AC-5**: Existing template content is preserved -- all changes are additive.

## Technical Approach
Two template files need small additions. The implementing agent should read both templates in full (`/.project/templates/story-template.md` and `/.project/templates/epic-template.md`) before making changes, to understand current section ordering and comment style. The Agent Hint and Handoff Context fields should be grouped visually (they are both dispatch metadata), placed after "Files to Create or Modify" and before "Definition of Done."
