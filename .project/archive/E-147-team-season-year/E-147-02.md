# E-147-02: Admin Add-Team: Thread season_year into INSERT

## Epic
[E-147: Team Season Year and Cohort-Based Dashboard Navigation](epic.md)

## Status
`DONE`

## Description
After this story is complete, the admin add-team flow persists `season_year` from the GameChanger API response into the `teams` table at INSERT time. This is the "golden path" -- the primary way new teams enter the system -- so it must populate `season_year` from day one.

## Context
`TeamProfile` dataclass in `src/gamechanger/team_resolver.py` already parses `year` from the API. The admin add-team flow is three steps: (1) `add_team_phase1()` (POST) resolves `TeamProfile`, builds a redirect params dict, and redirects to the confirm page; (2) the confirm page (GET) renders a form with hidden fields from the redirect params; (3) `confirm_team_submit()` (POST) reads hidden form fields and calls `_insert_team_new()`. The `TeamProfile` object is NOT in scope during steps 2 or 3 -- only values in the redirect params dict (step 1) and hidden form fields (step 2→3) survive. Currently `year` is not included in the redirect params. This story threads `year` through all three steps. Per TN-4 and TN-5 (item 1) in the epic Technical Notes.

## Acceptance Criteria
- [ ] **AC-1**: `add_team_phase1()` includes `season_year` (from `profile.year`) in the redirect params dict. The confirm page template (`confirm_team.html`) includes a hidden form field for `season_year`. The POST handler (`confirm_team_submit()`) reads this field and passes it to `_insert_team_new()`.
- [ ] **AC-2**: `_insert_team_new()` accepts a `season_year` parameter and includes it in the INSERT column list. If the value is None or empty, the INSERT uses NULL rather than failing.
- [ ] **AC-3**: A test verifies that adding a team via the admin flow results in `teams.season_year` being populated with the expected year value.

## Technical Approach
The admin add-team flow is three steps. Step 1: `add_team_phase1()` (POST) resolves `TeamProfile` and builds a redirect params dict (~line 1240 in `admin.py`). Add `season_year` to this dict only when `profile.year is not None` — omitting it entirely when None avoids the `str(None)` = `"None"` bug (a truthy string that would pass through to the INSERT). Step 2: the confirm page (GET) renders `confirm_team.html` with hidden fields. Add a hidden `<input name="season_year">` field (value may be empty string if not in params). Step 3: `confirm_team_submit()` (POST) reads form data. Read the `season_year` field, treat empty string as None, add a `season_year` parameter to `_insert_team_new()`, and include it in the INSERT statement.
