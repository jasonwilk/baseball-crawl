# E-140-06: PM Agent Definition Updates

## Epic
[E-140: Planning Skill](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent definition (`.claude/agents/product-manager.md`) references the new plan skill, and its System of Work section reflects the skill-driven planning workflow. The PM's existing planning-related content (quality checklist, consultation triggers, post-incorporation consistency sweep) remains in the agent definition as capability documentation. The plan skill references these sections; the agent definition does not duplicate skill content.

## Context
The PM agent definition currently describes the planning workflow procedurally in its "How Work Flows" section and "System of Work" section. With the plan skill now codifying the process, the agent definition needs to reference the skill as the authoritative source for planning workflow while retaining the capability content (what PM can do) that the skill orchestrates.

This is the final story in the epic -- it ensures the PM agent definition and the plan skill are consistent and non-duplicative.

## Acceptance Criteria
- [ ] **AC-1**: The PM agent definition's Skills section includes a `plan` entry with the file path (`.claude/skills/plan/SKILL.md`) and a "Load when" description matching the activation triggers from TN-4.
- [ ] **AC-2**: The PM agent definition's "How Work Flows" section (step 3, Refinement) references the plan skill as the authoritative source for the planning workflow, rather than describing the procedure inline.
- [ ] **AC-3**: The PM agent definition retains all existing capability content: quality checklist, consultation triggers table, post-incorporation consistency sweep, task types, anti-patterns. None of this content is removed or moved to the skill file.
- [ ] **AC-4**: The PM agent definition does not duplicate content from the plan skill. Where the agent definition previously described planning procedures that are now in the skill, those descriptions are replaced with references to the skill.
- [ ] **AC-5**: The PM agent definition's "Optional: Codex Spec Review" section is reconciled with the plan skill's automatic spec review phase. The section is updated to clarify that spec review is automatic when the plan skill is loaded, and remains optional only for ad-hoc reviews outside the plan skill workflow.
- [ ] **AC-6**: CLAUDE.md's Workflows section includes a "Plan" entry describing the planning workflow trigger phrases and referencing the plan skill, following the same format as the existing Implement, Ingest endpoint, Spec review, Code review, and Curate the vision entries.

## Technical Approach
Edit the PM agent definition to add the skill reference and update procedural sections to point to the skill. Edit CLAUDE.md to add the Plan workflow entry. The boundary between skill (process) and agent definition (capabilities) is defined in TN-9.
