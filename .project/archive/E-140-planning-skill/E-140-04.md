# E-140-04: Consistency Sweep Automation

## Epic
[E-140: Planning Skill](epic.md)

## Status
`DONE`

## Description
After this story is complete, Phase 4 (Refinement) of the plan skill contains a mandatory post-incorporation consistency sweep that PM executes after incorporating spec review findings. The sweep catches cascade drift where a fix in one story introduces an inconsistency in another.

## Context
The PM agent definition already contains a "Post-Incorporation Consistency Sweep" in its quality checklist. In practice, this sweep is manual and frequently skipped. The plan skill makes it a required gate: PM cannot proceed to Phase 5 (READY) or loop to Phase 3 (re-review) until the sweep passes.

The sweep is a procedural step performed by PM, not a script. PM lists changed values, greps the epic directory, and verifies consistency. The skill codifies the steps and makes them non-skippable within the planning workflow.

## Acceptance Criteria
- [ ] **AC-1**: Phase 4 includes a mandatory consistency sweep step between "PM incorporates findings" and "user decides whether to re-run spec review or proceed to READY."
- [ ] **AC-2**: The consistency sweep step specifies the same procedure as TN-5: (a) list every value changed, (b) grep epic directory for each changed value, (c) verify consistency across epic file and all story files, (d) apply cascading updates before proceeding.
- [ ] **AC-3**: The skill marks the consistency sweep as a gate -- PM cannot proceed past Phase 4 without confirming the sweep is complete. The skill text uses language like "MUST" or "required gate" to make this non-optional.
- [ ] **AC-4**: If the consistency sweep finds drift, PM fixes it inline (within Phase 4) before the user is offered the choice to re-review or proceed.

## Technical Approach
Edit the plan skill file created by E-140-01. Expand the consistency sweep portion of Phase 4 from a stub to full procedural detail. The sweep procedure is defined in TN-5 -- translate it into skill-level steps with gate language.
