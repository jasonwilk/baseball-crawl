# E-056-04: Update Review-Epic Skill for Team Lead Spawning

## Epic
[E-056: Fix Dispatch Pattern -- Team Lead as Spawner](epic.md)

## Status
`DONE`

## Description
After this story is complete, the review-epic skill (`/.claude/skills/review-epic/SKILL.md`) will have the team lead spawn PM AND implementing agents for the review team -- matching the new dispatch pattern where the team lead is the spawner.

## Context
The review-epic skill's Phase 2 currently instructs PM to "spawn implementing agents as teammates." This is impossible since PM is a teammate. The fix: the team lead spawns PM + implementers (determined from the epic's Dispatch Team section or routing table), then PM coordinates the review via messaging.

## Acceptance Criteria
- [ ] **AC-1**: Phase 2's context block instructions are updated. Instead of telling PM to "spawn implementing agents," the instructions tell PM that "the team lead has spawned implementing agents alongside you" and list who is on the team.
- [ ] **AC-2**: Between Phase 1 and Phase 2, the team lead reads the epic's Dispatch Team section (or determines agents from the routing table) and spawns PM + all required implementers. The team lead provides each implementer with: the codex findings, the epic ID/directory, and "wait for PM to assign your review tasks."
- [ ] **AC-3**: The Workflow Summary diagram is updated: Phase 2 shows "Team lead spawns PM + implementing agents" instead of "PM spawns implementing agents."
- [ ] **AC-4**: Phase 1 (team lead runs codex-review.sh) is unchanged.
- [ ] **AC-5**: The review coordination logic (PM tracks dispositions, implementers apply fixes) is unchanged in substance -- only the spawning mechanism changes.
- [ ] **AC-6**: Edge cases are preserved. No new edge cases needed (the implement skill handles spawn failures; this skill can reference the same pattern).

## Technical Approach
Read the current review-epic skill at `/workspaces/baseball-crawl/.claude/skills/review-epic/SKILL.md`. Edit Phase 2 to have the team lead spawn all agents. Update the context block template and Workflow Summary. Use the rewritten `/.claude/rules/dispatch-pattern.md` (E-056-01) as the authoritative reference.
