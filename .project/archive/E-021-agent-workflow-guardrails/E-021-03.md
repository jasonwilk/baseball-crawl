# E-021-03: Add Dispatch Failure Protocol to Workflow Discipline Rule

## Epic
[E-021: Agent Workflow Guardrails](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the `workflow-discipline.md` rule file will contain a "Dispatch Failure Protocol" section that codifies the escalation path for all agents when dispatch fails. This ensures the protocol is visible to every agent in the ecosystem (since rule files are loaded based on path matching), not just the orchestrator.

## Context
The `workflow-discipline.md` rule file is the project-wide contract for workflow routing. It currently defines the Epic READY Gate, Work Authorization Gate, Workflow Routing Rule, PM Task Types, and Direct-Routing Exceptions. It does not address what happens when the normal dispatch path fails. Adding the protocol here makes it ambient context for all agents (the file has `paths: ["**"]`), providing a backstop even if the orchestrator's prompt is not loaded.
