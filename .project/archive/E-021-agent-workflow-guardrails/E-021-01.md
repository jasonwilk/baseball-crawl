# E-021-01: Harden PM Agent -- Tool Restrictions, Anti-Patterns, and Assumption Verification

## Epic
[E-021: Agent Workflow Guardrails](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the product-manager agent will be structurally prevented from executing code or running tests via frontmatter tool restrictions, will have an explicit Anti-Patterns section prohibiting implementation work, and will include an assumption verification checkpoint in the research spike completion workflow. These three changes close Violations 1 and 2 from the parent epic.

## Context
The PM agent is the only agent in the ecosystem without an explicit `tools` field in its YAML frontmatter. This means it receives ALL tools by default -- including Bash and WebFetch -- which enables it to execute code and run tests despite prose-level prohibitions. Every other agent (orchestrator, claude-architect, general-dev, data-engineer, api-scout, baseball-coach) has explicit tool scoping. Additionally, the PM has no Anti-Patterns section (all other implementation-adjacent agents have one) and no checkpoint for verifying user-facing assumptions from research spikes before baking them into epic documentation.
