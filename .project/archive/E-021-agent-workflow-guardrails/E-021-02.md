# E-021-02: Add Dispatch Failure Protocol to Orchestrator

## Epic
[E-021: Agent Workflow Guardrails](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the orchestrator agent will contain a "Dispatch Failure Protocol" section that instructs it to escalate to the user when PM dispatch fails, rather than improvising workarounds like dispatching directly or asking the PM to implement. This closes Violation 3 from the parent epic.

## Context
When the orchestrator routes a dispatch request to the PM and the PM reports that dispatch cannot proceed (Agent tool unavailable, team creation fails, no eligible stories), the orchestrator currently has no defined fallback. Its Anti-Patterns section says "Never route implementation work directly to general-dev or data-engineer" but provides no guidance for what to do when the normal path fails. In practice, this has led to the orchestrator improvising: dispatching implementing agents directly (skipping PM) or instructing the PM to do the implementation itself. Both are workflow violations. The fix is a defined escalation path: report the failure to the user and let the user decide.
