# E-016-03: Update settings.json -- Make Orchestrator Default Agent

## Epic
[E-016: Evolve Project Manager to Product Manager](./epic.md)

## Status
`DONE`

## Description
After this story is complete, `/.claude/settings.json` will contain `"agent": "orchestrator"`, making the orchestrator the default agent for the project. This is identity-level enforcement: instead of relying on context or user memory to invoke the orchestrator first, the system will route to it by default.

## Context
Currently, settings.json has no `agent` key, so Claude Code uses its default behavior (no specific agent). The user must manually invoke the orchestrator or rely on the session-level agent selection. Adding `"agent": "orchestrator"` to settings.json ensures every new session starts with the orchestrator, which is the intended entry point for all requests.

This story depends on E-016-02 (orchestrator rewrite) because the orchestrator must be upgraded before it becomes the default. Making a haiku keyword-matcher the default agent would be worse than no default.

## Acceptance Criteria

- [ ] **AC-1**: The file `/.claude/settings.json` contains the key `"agent"` with value `"orchestrator"` at the top level of the JSON object.
- [ ] **AC-2**: All existing keys in settings.json are preserved: `env`, `statusLine`, and any others present at the time of modification.
- [ ] **AC-3**: The resulting file is valid JSON (parseable by `jq .`).

## Technical Approach

1. Read the current `/.claude/settings.json`.
2. Add `"agent": "orchestrator"` to the top-level JSON object.
3. Preserve all existing keys and values.
4. Write the file with consistent formatting (2-space indentation).
5. Verify the file is valid JSON.
