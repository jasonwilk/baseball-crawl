# E-016-06: Update workflow-discipline.md and multi-agent-patterns Skill

## Epic
[E-016: Evolve Project Manager to Product Manager](./epic.md)

## Status
`DONE`

## Description
After this story is complete, `/.claude/rules/workflow-discipline.md` will reference the PM's new task types and the simplified dispatch model. The `/.claude/skills/multi-agent-patterns/SKILL.md` will include a "When to Shorten the Chain" section and update PM references to reflect the new Product Manager identity. Both files align with the PM rewrite from E-016-01.

## Context
`workflow-discipline.md` is a rules file loaded into all agent contexts (via `paths: ["**"]`). It enforces the epic READY gate, work authorization gate, workflow routing rule, and direct-routing exceptions. The current content is accurate but does not reference the PM's new task types or the simplified dispatch approach.

`multi-agent-patterns/SKILL.md` is an architectural skill that documents the telephone game problem, the routing chain, and verbatim relay mitigations. Its PM references describe the old "spec-formatting" PM. It should be updated to reflect the new strategic PM identity and should gain a "When to Shorten the Chain" section that helps agents decide when the full orchestrator -> PM -> implementing agent chain is warranted vs. when direct routing is appropriate.

- [ ] **AC-1**: `/.claude/rules/workflow-discipline.md` retains the frontmatter `paths: ["**"]`.
- [ ] **AC-2**: The Workflow Routing Rule in `workflow-discipline.md` references the PM's dispatch role but does NOT contain the string "Dispatch Manifest".
- [ ] **AC-3**: `workflow-discipline.md` includes a mention of PM task types by name: discover, plan, clarify, triage, and close. All five must appear explicitly (not just "task types exist").

This section should list at least 3 concrete examples of when to shorten.
- [ ] **AC-5**: The multi-agent-patterns SKILL.md describes the PM as a "Product Manager" (not "Project Manager" or "spec-formatting" role) in its routing chain diagram and PM references.
- [ ] **AC-6**: The multi-agent-patterns SKILL.md does NOT contain the string "Dispatch Manifest".
- [ ] **AC-7**: Both files remain syntactically valid markdown.

## Notes
- The "When to Shorten the Chain" concept comes from the reference project analysis. The idea: not every request needs the full routing chain. Direct-routing exceptions already exist in the system, but they are described as "exceptions" rather than as a deliberate optimization. Framing them as "shortening the chain" makes the pattern more intentional and easier for agents to apply.
- Keep the workflow-discipline.md changes minimal. It is a rules file -- every line is loaded into every agent's context. Do not turn it into documentation.
- The multi-agent-patterns skill is loaded on-demand (via activation triggers), so it can be more verbose without impacting every session's token budget.
