# E-016-05: Update CLAUDE.md Agent Ecosystem Section

## Epic
[E-016: Evolve Project Manager to Product Manager](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the Agent Ecosystem table and Workflow Contract section in `CLAUDE.md` (lines ~186-222) will reflect the new PM identity (Product Manager, opus model), the upgraded orchestrator (sonnet model, can read files), and the simplified dispatch pattern (no Dispatch Manifest references). The rest of CLAUDE.md remains untouched.

## Context
CLAUDE.md is the project's root instruction file -- it is loaded into every agent's context. The Agent Ecosystem section describes each agent's role and the Workflow Contract describes how they interact. Both sections must be updated to match the changes made by E-016-01 (PM rewrite) and E-016-02 (orchestrator rewrite). This story is deliberately sequenced after both rewrites so the CLAUDE.md update matches the actual agent definitions.

## Acceptance Criteria

- [ ] **AC-1**: The Agent Ecosystem table in CLAUDE.md describes the project-manager's role using "Product Manager" language (not "Writes epics, stories, captures ideas, manages backlog"). The description should reflect the new identity: strategic product decisions, task types, delegation to specialists.
- [ ] **AC-2**: The Agent Ecosystem table includes the orchestrator's updated description: smart router with file-reading capability (not just "Routes all requests to the correct agent").
- [ ] **AC-3**: The Workflow Contract section does NOT contain the string "Dispatch Manifest" anywhere.
- [ ] **AC-4**: The Workflow Contract step 5 ("Start epic X dispatches through PM") reflects the simplified dispatch rule: PM runs in main session for dispatch, reference dispatch-pattern.md for the constraint.
- [ ] **AC-5**: No sections of CLAUDE.md outside of "Agent Ecosystem" (lines ~186-222) and the immediately following "Workflow Contract" subsection are modified. The rest of the file (Core Principle, Project Purpose, Tech Stack, Code Style, Security Rules, HTTP Request Discipline, Testing, Git Conventions, Statusline, etc.) remains identical.
- [ ] **AC-6**: The `data-engineer` row in the Agent Ecosystem table says "SQLite" instead of "Cloudflare D1" (this was a stale reference from before E-009's tech stack redesign).

## Technical Approach

3. Read the completed E-016-02 story output (new orchestrator definition) to extract the new orchestrator description.

   - Update the `project-manager` row description.
