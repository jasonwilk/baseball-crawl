# E-016-04: Simplify dispatch-pattern.md Rule

## Epic
[E-016: Evolve Project Manager to Product Manager](./epic.md)

## Status
`DONE`

## Description
After this story is complete, `/.claude/rules/dispatch-pattern.md` will be a concise rules file that documents the subagent nesting constraint and states one simple rule: the PM must run in the main session for dispatch. The Dispatch Manifest format section will be completely removed. In its place, the Mode 2 behavior is a single error message.

## Context
The current `dispatch-pattern.md` (65 lines including frontmatter) was created by E-015 as a tactical fix. It documents two PM operating modes and includes a verbose Dispatch Manifest format (~20 lines of template). E-016 supersedes this: instead of accommodating the PM running as a subagent with a complex manifest handoff, we simplify to "PM must be in main session for dispatch; if not, emit error and stop."

The nesting constraint documentation (CLAUDECODE=1 environment variable) is still valuable and should be preserved -- it explains WHY the rule exists. Only the Dispatch Manifest section is removed.

## Acceptance Criteria

- [ ] **AC-1**: The file `/.claude/rules/dispatch-pattern.md` exists and retains the frontmatter `paths: ["**"]`.
- [ ] **AC-2**: The file documents the CLAUDECODE=1 nesting constraint (main session -> subagent allowed, subagent -> sub-subagent blocked).
- [ ] **AC-3**: The file does NOT contain the string "Dispatch Manifest" anywhere.
- [ ] **AC-4**: The file defines Mode 2 behavior as: the PM emits a one-line error message (exact text: "ERROR: Dispatch requires the PM to run in the main session. Invoke the project-manager agent directly and retry.") and stops. No manifest, no context blocks, no Task templates.
- [ ] **AC-5**: The file preserves the orchestrator routing rule: dispatch-intent requests must NOT be wrapped in a Task tool call to the PM.
- [ ] **AC-6**: The file is shorter than 40 lines (including frontmatter). The current 65 lines should be reducible to ~30-35.

## Technical Approach

1. Read the current `/.claude/rules/dispatch-pattern.md` (65 lines).
2. Rewrite with this structure:
   - **Frontmatter**: Preserve `paths: ["**"]`.
   - **The Nesting Constraint**: Keep the explanation of CLAUDECODE=1 and the depth limit. Tighten prose.
   - **PM Dispatch Rule**: Mode 1 (main session): dispatch normally. Mode 2 (subagent): emit error message and stop. Two sentences per mode.
   - **Orchestrator Routing Rule**: Keep the rule that dispatch requests must not be wrapped in Task calls. Simplify the example.
3. Remove the Dispatch Manifest Format section entirely.
4. Verify line count is under 40.
