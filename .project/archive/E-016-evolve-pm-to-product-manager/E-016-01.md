# E-016-01: Rewrite PM Agent Definition

## Epic
[E-016: Evolve Project Manager to Product Manager](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the project-manager agent definition at `/.claude/agents/project-manager.md` will be a lean (~200-250 line) strategic Product Manager definition. It will contain explicit task types (discover, plan, clarify, triage, close), technical delegation boundaries, consultation triggers, structured response formats, and a simplified dispatch mode. All embedded templates (Epic Template, Story Template) will be removed -- they already exist in `/.project/templates/` and do not need to be duplicated in the agent prompt. The model will be set to `opus`.

## Context
The current PM definition is 523 lines. Roughly 200 of those are embedded copies of the epic and story templates that already live in `/.project/templates/`. The agent knows HOW to format stories but has no guidance on WHAT to build or WHY. It has no task types, no delegation boundaries, and no structured response formats. The Dispatch Manifest (Mode 2) section alone is ~25 lines describing a pattern that should be replaced by a one-line error message.

This is the highest-impact story in E-016: it transforms the PM from a spec-formatting robot into a strategic product thinker.

## Technical Approach

1. Read the current `project-manager.md` fully (523 lines).
2. Read `/.project/templates/epic-template.md` and `/.project/templates/story-template.md` to confirm they exist and are complete.
3. Rewrite the file from scratch, using the current file as source material but restructuring around the new identity:
   - **Identity**: Strategic Product Manager, not spec-formatting robot.
   - **Core Principle**: Keep "Simple first. Complexity as needed."
   - **Task Types**: New section. Five types with descriptions and output formats.
   - **Technical Delegation Boundaries**: New section.
   - **Consultation Triggers**: Adapt from current "When to Consult Experts" table.
   - **Numbering Scheme**: Keep as-is (compact).
   - **File Organization**: Keep as-is but make it a brief reference (3-4 lines, not a code block).
   - **Templates**: Remove embedded templates. Add a line: "Templates are at `/.project/templates/`. Read them when creating epics/stories."
   - **System of Work**: Keep the workflow statuses and the How Work Flows list, but tighten.
   - **Refinement Mode**: Keep but tighten. Remove the "Consultation Procedure" 4-step list (the triggers table is enough).
   - **Decision Gates**: Keep but tighten. Remove the verbose format examples.
   - **Dispatch Mode**: Simplify drastically. Keep Mode 1 (main session dispatch). Replace Mode 2 with one line: emit error. Remove Dispatch Manifest section entirely. Reference `dispatch-pattern.md` for the nesting constraint.
   - **Ideas Workflow**: Keep but tighten.
   - **Quality Checklist**: Keep.
   - **Memory Instructions**: Keep.
   - **Structured Response Formats**: New section. For each task type, specify the expected response structure.
4. Verify line count is under 300.
5. Verify all acceptance criteria are met.
