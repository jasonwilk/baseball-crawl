# E-004-02: Enhanced Team Batting Stats

## Epic
[E-004: Coaching Dashboard](epic.md)

## Status
`DONE`

## Description
After this story is complete, the `/dashboard` page will display a full batting stats table with all stored columns and computed rates (AVG, OBP, SLG), replacing the current minimal table (name, AB, H, BB, SO). Player names will be clickable links to their profile pages. The table will be sorted by batting average descending by default.

## Context
The current `/dashboard` route returns only name, ab, h, bb, so from `player_season_batting`. The schema stores much richer data: doubles, triples, hr, rbi, sb, plus split columns. This story expands the query and template to show all standard counting stats plus computed rate stats that coaches use for lineup decisions.

## Acceptance Criteria
- [ ] **AC-1**: The batting stats table displays columns in this order: Player (linked to `/dashboard/players/{player_id}`), AVG, OBP, GP, BB, SO, SLG, H, AB, 2B, 3B, HR, SB, RBI. Rate stats (AVG, OBP) and key development indicators (GP, BB, SO) are placed immediately after Player so they are visible at 375px without horizontal scrolling. SLG, power stats, and RBI follow and may require scroll on narrow screens. Rationale: coaches use walk rate and strikeout rate as primary development indicators; RBI is lineup-dependent and least actionable.
- [ ] **AC-2**: AVG, OBP, SLG are computed from stored data using the formulas in epic Technical Notes. Division by zero displays "-".
- [ ] **AC-3**: Rate stats (AVG, OBP, SLG) display in ".000" format (three decimal places, leading dot, e.g., ".333" not "0.333").
- [ ] **AC-4**: The table is sorted by AVG descending by default. Players with no AB appear at the bottom.
- [ ] **AC-5**: Player names are clickable `<a>` tags linking to `/dashboard/players/{player_id}`. The player_id is included in the query result.
- [ ] **AC-6**: The team selector (from E-004-01 macro) appears at the top. Changing teams reloads the page with the selected team's batting stats.
- [ ] **AC-7**: Jersey number is displayed next to the player name if available (from `team_rosters.jersey_number`).
- [ ] **AC-8**: The table is wrapped in `overflow-x-auto` for horizontal scroll on narrow screens. At 375px width, the Player + AVG + OBP + GP columns are visible without scrolling. The `<thead>` uses `sticky top-0` so column headers remain visible when scrolling vertically through long rosters.
- [ ] **AC-9**: `db.get_team_batting_stats()` is updated to return all needed fields (player_id, jersey_number, games, ab, h, doubles, triples, hr, rbi, bb, so, sb). The function signature accepts `team_id` and `season_id`.
- [ ] **AC-10**: Tests verify the route returns 200 with seeded data and that the response HTML contains: (a) at least one seeded player's last name, (b) a correctly computed AVG value for a seeded player (e.g., if player has 3 hits in 6 AB, the HTML contains ".500"), (c) the column headers "AVG", "OBP", "SLG".
- [ ] **AC-11**: Tests verify that AVG/OBP/SLG computation handles zero AB correctly (displays "-").

## Technical Approach
1. Update `db.get_team_batting_stats()` to join `player_season_batting` with `players` and `team_rosters` (for jersey_number). Return all columns needed for the template.
2. Compute AVG, OBP, SLG either in the SQL query (using CASE for zero division) or in Python after fetching rows. Python is simpler and more testable.
3. Update `src/api/templates/dashboard/team_stats.html` to render the full table with all columns.
4. Use the `format_avg` helper from E-004-01 for rate stat formatting.
5. Reference: `src/api/db.py` for current query, `/workspaces/baseball-crawl/migrations/001_initial_schema.sql` for schema.
