# E-004-03: Team Pitching Stats Page

## Epic
[E-004: Coaching Dashboard](epic.md)

## Status
`DONE`

## Description
After this story is complete, `GET /dashboard/pitching` will render a team pitching stats page showing all pitchers on the selected team with their season stats and computed rates (ERA, K/9, BB/9, WHIP). Player names link to their profile pages. The page uses the shared team selector and nav shell from E-004-01.

## Context
The `player_season_pitching` table stores: games, ip_outs, h, er, bb, so, hr, pitches, strikes, plus home/away and L/R batter splits. This story creates the pitching counterpart to the batting stats page, giving coaches a quick view of their pitching staff's season performance.

## Acceptance Criteria
- [ ] **AC-1**: `GET /dashboard/pitching` returns 200 and renders a pitching stats table for the active team in the current season. Accepts optional `?season_id=` query parameter (same pattern as E-004-01 AC-9). Returns 403 if `?team_id=` specifies a team not in `request.state.permitted_teams`.
- [ ] **AC-2**: Table columns in order: Player (linked to profile), ERA, K/9, BB/9, WHIP, GP, IP, H, ER, BB, SO, HR. Rate stats (ERA, K/9, BB/9, WHIP) are placed immediately after Player so they are visible at 375px without horizontal scrolling. Counting stats follow and may require scroll on narrow screens. Player names link to `/dashboard/players/{player_id}`.
- [ ] **AC-3**: IP displays in standard notation using the `ip_display` filter from E-004-01 (e.g., 20 outs -> "6.2").
- [ ] **AC-4**: ERA, K/9, BB/9, WHIP are computed using formulas from epic Technical Notes: ERA = `(er * 27) / ip_outs`, K/9 = `(so * 27) / ip_outs`, BB/9 = `(bb * 27) / ip_outs`, WHIP = `(bb + h) * 3 / ip_outs`. Division by zero (ip_outs = 0) displays "-".
- [ ] **AC-5**: ERA displays to two decimal places (e.g., "3.45"). K/9, BB/9 display to one decimal place (e.g., "8.1"). WHIP displays to two decimal places (e.g., "1.23").
- [ ] **AC-6**: Table is sorted by ERA ascending by default. Pitchers with 0 ip_outs appear at the bottom.
- [ ] **AC-7**: Team selector appears at top, using the shared macro from E-004-01 with `base_url="/dashboard/pitching"`.
- [ ] **AC-8**: Jersey number displayed next to player name if available.
- [ ] **AC-9**: The active nav item is "Pitching" (highlighted in the nav bar).
- [ ] **AC-10**: `db.get_team_pitching_stats(team_id, season_id)` is added to `src/api/db.py`. Returns all needed fields including player_id and jersey_number.
- [ ] **AC-11**: Route respects `request.state.permitted_teams` -- returns 403 for unauthorized team_id.
- [ ] **AC-12**: Tests verify: 200 with seeded data, ERA/K9/WHIP computation, 403 for unauthorized team_id, empty state renders "No pitching stats available," and `?season_id=` override returns stats for the specified season.
- [ ] **AC-13**: The `<thead>` uses `sticky top-0` so column headers remain visible when scrolling vertically.

## Technical Approach
1. Add `get_team_pitching_stats()` to `src/api/db.py` joining `player_season_pitching` with `players` and `team_rosters`.
2. Add `GET /dashboard/pitching` route to `src/api/routes/dashboard.py`. Follow the same pattern as the existing `team_stats` route for auth, team selection, and season default.
3. Create `src/api/templates/dashboard/team_pitching.html` extending `base.html`.
4. Compute ERA/K9/BB9/WHIP in Python after fetching rows.
5. Reference: `src/api/routes/dashboard.py` for existing route pattern, `/workspaces/baseball-crawl/migrations/001_initial_schema.sql` for `player_season_pitching` schema.
