# E-004-01: Navigation Shell and Base Layout

## Epic
[E-004: Coaching Dashboard](epic.md)

## Status
`DONE`

## Description
After this story is complete, the base layout and navigation will support all planned dashboard views. The nav bar will contain links to the four main sections (Batting, Pitching, Games, Opponents). A shared Jinja2 macro will render the team selector and season default, so subsequent stories can include them without duplicating logic. An `ip_display` Jinja2 filter will be available for converting ip_outs integers to standard IP notation (e.g., 20 -> "6.2").

## Context
The existing `base.html` has a minimal nav bar with only "Team Stats". The team selector logic is embedded in `dashboard.py` and `team_stats.html`. This story extracts shared patterns (team selector, season default, ip_outs formatting) into reusable components so that stories 02-06 can be implemented independently without file conflicts on shared infrastructure.

## Acceptance Criteria
- [ ] **AC-1**: `base.html` has a **bottom fixed navigation bar** (`fixed bottom-0`) with links to: `/dashboard` (Batting), `/dashboard/pitching` (Pitching), `/dashboard/games` (Games), `/dashboard/opponents` (Opponents). Each link shows a text label and highlights when active (uses a CSS class to indicate the current section). The bottom nav has a `z-10` or higher z-index and a top border for visual separation. The `<main>` content area has `pb-16` (or equivalent) bottom padding so content is not hidden behind the fixed nav.
- [ ] **AC-1a**: The **top bar** retains "LSB Baseball" branding and moves user info (display name, admin link, logout) to the right side. The top bar does NOT contain section navigation links -- those live in the bottom nav only.
- [ ] **AC-1b**: All bottom nav links are at minimum 44px tap height (use `py-3` or equivalent). Links are evenly distributed across the full viewport width.
- [ ] **AC-2**: A Jinja2 macro or include file at `src/api/templates/dashboard/_team_selector.html` renders the team selector pills. It accepts `permitted_team_infos`, `active_team_id`, and `base_url` parameters so any page can use it with its own URL prefix. Team selector pills use `py-2` for 44px touch targets.
- [ ] **AC-3**: A Jinja2 filter `ip_display` is registered on the Jinja2 environment that converts an ip_outs integer to standard IP notation: `ip_display(20)` returns `"6.2"`, `ip_display(9)` returns `"3.0"`, `ip_display(0)` returns `"0.0"`, `ip_display(None)` returns `"-"`.
- [ ] **AC-4**: A Python helper function `format_avg(numerator, denominator)` is added to `src/api/db.py` (or a new `src/api/helpers.py`) that returns a formatted batting average string (".333", ".000", "-" for zero denominator). This function is importable for use in db query functions or as a Jinja2 filter.
- [ ] **AC-5**: The existing `/dashboard` route continues to return the same batting stats content when called without a `?season_id=` parameter (no regression on default behavior). All existing tests pass. The only new behavior is the `?season_id=` override added in AC-9.
- [ ] **AC-6**: The bottom nav bar is usable at 375px width -- all 4 links are visible without wrapping or overlapping, evenly distributed.
- [ ] **AC-7**: Tests verify the `ip_display` filter produces correct output for: 0, 1, 3, 9, 20, None.
- [ ] **AC-8**: Tests verify the `format_avg` helper for: (1, 3) -> ".333", (0, 0) -> "-", (3, 3) -> "1.000", (0, 4) -> ".000".
- [ ] **AC-9**: The existing `/dashboard` route accepts an optional `?season_id=` query parameter. If provided and non-empty, use it as the season filter; otherwise fall back to the computed default (`f"{datetime.date.today().year}-spring-hs"`). This establishes the season selection pattern for all subsequent stories.
- [ ] **AC-10**: The test fixture schema in `tests/test_dashboard.py` is updated to match the real schema in `migrations/001_initial_schema.sql`. Specifically: (a) add the `seasons` table, (b) change `team_rosters.season` to `season_id TEXT NOT NULL REFERENCES seasons(season_id)` with matching UNIQUE constraint, (c) change `games.season` to `season_id TEXT NOT NULL REFERENCES seasons(season_id)`, (d) seed a `seasons` row for the current season. All existing tests continue to pass after the fixture update.

## Technical Approach
1. Update `src/api/templates/base.html`: (a) Keep the top bar (`bg-blue-900`) for branding and user info only -- remove the "Team Stats" nav link. (b) Add a bottom fixed nav bar after `</main>` with 4 section links using `fixed bottom-0 w-full`. Use `flex justify-around` for even distribution. Each link gets `py-3` for 44px+ tap targets. Use a `{% set active_nav = "batting" %}` pattern (or `active_nav` passed from template) to highlight the current section. (c) Add `pb-16` to `<main>` to prevent content from being hidden behind the fixed bottom nav.
2. Create `src/api/templates/dashboard/_team_selector.html` as a Jinja2 macro. Extract the existing selector HTML from `team_stats.html` and parameterize the URL prefix. Increase pill padding to `py-2` for touch targets.
3. Update `src/api/templates/dashboard/team_stats.html` to use the new macro instead of inline selector HTML. Set `active_nav = "batting"`. Move user info (display name, admin link, logout) to the top bar via a base.html block.
4. Register the `ip_display` filter in `src/api/routes/dashboard.py` (on the `templates.env.filters` dict) or in `src/api/main.py`.
5. Add `format_avg` to a helpers module.
6. Reference: `src/api/templates/base.html`, `src/api/templates/dashboard/team_stats.html` for current layout patterns.
