# E-004-05: Opponent Scouting Report

## Epic
[E-004: Coaching Dashboard](epic.md)

## Status
`DONE`

## Description
After this story is complete, `GET /dashboard/opponents` will show a list of opponents the selected team has played or is scheduled to play in the current season. Clicking an opponent opens `GET /dashboard/opponents/{team_id}` showing a scouting report with the opponent's batting and pitching leaders -- the core game-prep tool for coaches.

## Context
The `games` table links to opponent teams. The `player_season_batting` and `player_season_pitching` tables store stats for opponent players (loaded by the opponent crawler in E-002). This story gives coaches a one-click scouting report for any upcoming opponent, showing their top hitters and pitchers so the coaching staff can prepare defensive alignments and batting strategy.

## Technical Approach
1. `get_team_opponents()`: Query `games` to find all teams that appear as home or away opponent of the active team in the season. Group by opponent to get game count and W/L record. Also compute next scheduled game date (MIN of game_date WHERE game_date >= today AND status != 'completed') or most recent game date (MAX of game_date WHERE status = 'completed'). Use CASE expression for opponent resolution (same pattern as E-004-04).
2. `get_opponent_scouting_report()`: Query `player_season_batting` and `player_season_pitching` for the opponent's team_id and season_id. Reuse the same query patterns as E-004-02/03 but for the opponent team. For the Key Players card, include `pitches` and `games` from `player_season_pitching` to compute avg pitches per game.
3. `get_last_meeting()`: Query `games` for the most recent completed game where one side is team_id and the other is opponent_team_id, in the given season. Return game_id, date, scores, and home/away orientation, or None.
4. Add routes for opponent list and scouting report to `src/api/routes/dashboard.py`. The scouting report route must verify the opponent appears in games for at least one of `request.state.permitted_teams` before rendering; return 403 otherwise. The scouting report route also needs the user's active team_id (from `?team_id=` or first permitted team) for the Last Meeting query.
5. Create templates for both views.
6. Reference: `/workspaces/baseball-crawl/migrations/001_initial_schema.sql` for schema.
