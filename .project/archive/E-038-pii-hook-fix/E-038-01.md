# E-038-01: Fix hooksPath to Use Relative Path and Auto-Setup in Devcontainer

## Epic
[E-038: Fix PII Pre-Commit Hook Silent Failure](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PII pre-commit hook will work reliably in any environment (host macOS, devcontainer, any clone location) because `core.hooksPath` will use a relative path. New devcontainers will automatically have the hook configured via `postCreateCommand`.

## Context
The PII safety net has been silently disabled in the devcontainer. `scripts/install-hooks.sh` stores an absolute host path in `core.hooksPath`, which does not exist inside the container. Git silently skips the hook -- no error, no `[pii-scan]` confirmation. This is a critical safety gap: commits with PII could pass through undetected.

## Acceptance Criteria
- [ ] **AC-1**: `scripts/install-hooks.sh` sets `core.hooksPath` to `.githooks` (relative), not an absolute path.
- [ ] **AC-2**: After running the updated `install-hooks.sh`, `git config core.hooksPath` outputs `.githooks` (no leading `/`).
- [ ] **AC-3**: `.devcontainer/devcontainer.json` `postCreateCommand` includes `./scripts/install-hooks.sh` so new devcontainers auto-configure the hook.
- [ ] **AC-4**: The existing `postCreateCommand` entries (claude install, codex install, pip install) are preserved -- only appended to, not replaced.
- [ ] **AC-5**: `.githooks/pre-commit` is not modified.
- [ ] **AC-6**: After running the updated script, a test commit (`git commit --allow-empty -m 'test hook'`) produces `[pii-hook]` output confirming the hook ran.

## Technical Approach
1. Read `/workspaces/baseball-crawl/scripts/install-hooks.sh`.
2. Change `git config core.hooksPath "$REPO_ROOT/.githooks"` to `git config core.hooksPath .githooks`.
3. Keep the `chmod +x` line as-is (absolute path for runtime chmod is fine).
4. Read `/workspaces/baseball-crawl/.devcontainer/devcontainer.json`.
5. Append `&& ./scripts/install-hooks.sh` to the existing `postCreateCommand` value.
6. Run the updated `install-hooks.sh` to apply the fix in the current environment.
7. Verify `git config core.hooksPath` returns `.githooks`.
8. Run a test commit to confirm hook execution.
