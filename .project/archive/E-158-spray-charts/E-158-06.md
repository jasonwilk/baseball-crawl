# E-158-06: Opponent Scouting Spray Chart

## Epic
[E-158: Spray Chart Pipeline and Dashboard Integration](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the opponent scouting report page (`/dashboard/opponents/{opponent_team_id}`) will display a team-aggregate offensive spray chart and per-player spray chart links in the batting leaders table. The team aggregate answers "where does this opponent's lineup tend to hit?" for overall defensive positioning. The per-player links let coaches drill into individual hitters ("where do I shade my outfield against their 3-hole hitter?") — this is the coach's #1 priority per consultation.

## Context
The opponent detail page already shows pitching and batting stats. Per UXD consultation, the team spray chart belongs after the "Team Batting" summary card and before the individual leader tables. Per-player spray charts are surfaced via "View spray →" links in the batting leaders table (not inline thumbnails — loading 13 chart images on one page is slow on mobile). Links point directly to the spray chart image endpoint (`/dashboard/charts/spray/player/{player_id}.png`) which opens the chart PNG in a new tab. This avoids the `permitted_teams` auth check on the full player profile page, which would 403 for opponent players on tracked teams.

The spray_charts table already contains opponent player events from games against our teams (the API returns both teams' data in one call, and the loader in E-158-03 stores both). This story queries those rows filtered by `team_id = opponent_team_id`.

## Acceptance Criteria
- [ ] **AC-1**: A new route `GET /dashboard/charts/spray/team/{team_id}.png` exists that returns a PNG image response (`content-type: image/png`) showing an aggregate spray chart for all players on the specified team. This endpoint requires only an authenticated session (Cloudflare Access) — it does NOT check `permitted_teams`, for the same reason as the player spray endpoint (E-158-05 AC-1).
- [ ] **AC-2**: The route accepts an optional `?season_id=` query parameter (text, defaults to current season). It queries `spray_charts` rows where `team_id = {team_id}`, `chart_type = 'offensive'`, and `season_id` matches the parameter, aggregates across all players and games, calls the rendering function, and returns the PNG bytes. The parameter allows the opponent detail page to pass the active season when viewing historical seasons.
- [ ] **AC-3**: If the team has fewer than 20 total offensive spray chart events, the route returns a 204 No Content response (team aggregate uses 20 BIP threshold per TN-7 team tier).
- [ ] **AC-4**: The opponent detail template (`src/api/templates/dashboard/opponent_detail.html`) includes a "Team Spray Chart" card positioned after the team batting summary card and before the individual leader tables. The card must be inserted inside the State C conditional block (the `{% else %}` branch that renders when full stats are available — see existing template structure).
- [ ] **AC-5**: The card displays the chart image as a full-width `<img>` element with alt text and a subtitle: "Based on N balls in play against our teams" (explicitly noting the data scope per TN-7).
- [ ] **AC-6**: The card heading shows "Team Spray Chart" with the BIP count subtitle from AC-5.
- [ ] **AC-7**: The template implements all four TN-7 team aggregate tiers: (a) >= 20 BIP shows the chart image, (b) 10-19 BIP shows "Small sample — N balls in play against our teams" without a chart, (c) < 10 BIP shows "Not enough data yet — N balls in play recorded," (d) 0 BIP shows "Charts will appear after the next sync."
- [ ] **AC-8**: The batting leaders table includes a "View spray →" link for each opponent player who has >= 10 offensive BIP. The link appears inside the Player Name cell as a secondary action beneath the player name (not as a new table column — the name cell is always visible on mobile without horizontal scrolling). The link points to `/dashboard/charts/spray/player/{player_id}.png?season_id={season_id}` and opens in a new tab (`target="_blank"`). This links directly to the spray chart image endpoint (E-158-05 AC-1), which does NOT require team membership — it only needs spray_charts data to exist. Do NOT link to the player profile page (`/dashboard/players/{player_id}`) because opponent players are on tracked teams and would 403 against the `permitted_teams` auth check. Players with < 10 BIP have no spray link. A note above the table (before the scrollable wrapper) reads: "Only showing spray charts for players with 10+ balls in play."
- [ ] **AC-9**: The opponent detail route passes per-player BIP counts, total team BIP count, and the active `season_id` to the template context so the template can conditionally show the "View spray →" links (per-player counts), implement the team aggregate tiers in AC-7 (total count), and append `?season_id=` to both the team and per-player spray chart image URLs. The `season_id` is the route's already-resolved `season_id` variable (which the route computes server-side before rendering, even when no `?season_id=` query param is present). This ensures spray data always matches the season the rest of the page is rendering. BIP count queries filter by this same `season_id`.
- [ ] **AC-10**: The chart renders correctly at full width on a 375px mobile viewport.
- [ ] **AC-11**: Tests verify: team aggregate image route returns PNG for a team with >= 20 events, 204 for < 20 events, correct aggregation across multiple players, per-player BIP count query returns expected counts, and `?season_id=` parameter filters to the specified season (events from other seasons excluded).

## Technical Approach
Two additions to the opponent detail page: (1) a team aggregate spray chart card (same image-route pattern from E-158-05, but querying by `team_id` instead of `player_id`), and (2) "View spray →" links in the batting leaders table for players with sufficient data.

The opponent detail route needs to query per-player BIP counts from `spray_charts` (grouped by `player_id` where `team_id = opponent_team_id` and `chart_type = 'offensive'`) and pass them to the template context. The template uses these counts to conditionally render the "View spray →" link next to each player name in the batting table.

## Notes
- This story only displays spray data from games against our teams (data already in the DB from the loader). Full opponent season spray data (games against other teams) is a Non-Goal for this epic.
- The team chart image route (`/dashboard/charts/spray/team/{team_id}.png`) is generic — it works for both opponent teams and our own teams, though this story focuses on the opponent scouting use case.
- The "View spray →" links open the raw PNG image in a new tab. This is simple but not the richest UX — a dedicated spray detail page with context (player name, BIP count, threshold messages) is a follow-on improvement.
