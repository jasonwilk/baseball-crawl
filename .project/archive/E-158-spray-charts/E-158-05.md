# E-158-05: Player Profile Spray Chart

## Epic
[E-158: Spray Chart Pipeline and Dashboard Integration](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the player profile page (`/dashboard/players/{player_id}`) will display an inline spray chart image showing the player's season offensive ball-in-play distribution. A new image endpoint serves the rendered PNG. Players with fewer than 10 BIP see a threshold-appropriate message instead (per TN-7: "small sample" for 5-9, "not enough data" for <5).

## Context
The player profile page already shows career stats across seasons and teams. The spray chart adds visual context — "where does this player put the ball?" — as a card below the existing stats. Per UXD consultation: full-width inline card, no side-by-side on mobile, no collapsible/JS required. The chart image is served by a new route that queries the DB and calls the rendering module (E-158-04).

## Acceptance Criteria
- [ ] **AC-1**: A new route `GET /dashboard/charts/spray/player/{player_id}.png` exists that returns a PNG image response (`content-type: image/png`). This endpoint requires only an authenticated session (Cloudflare Access) — it does NOT check `permitted_teams` or team membership, because E-158-06 links opponent players (on tracked teams) directly to this endpoint.
- [ ] **AC-2**: The route accepts an optional `?season_id=` query parameter (text, defaults to current season). It queries `spray_charts` rows for the player where `chart_type = 'offensive'` and `season_id` matches the parameter, calls the rendering function from `src/charts/spray.py`, and returns the PNG bytes. The parameter allows E-158-06's opponent spray links to specify the season being viewed.
- [ ] **AC-3**: If the player has fewer than 10 offensive spray chart events, the route returns a 204 No Content response (no image body).
- [ ] **AC-4**: The player profile template (`src/api/templates/dashboard/player_profile.html`) includes a "Spray Chart" card section below the existing stats cards.
- [ ] **AC-5**: When the player has >= 10 BIP, the card displays the chart image as a full-width `<img>` element with alt text and a "Based on N balls in play" subtitle per TN-7.
- [ ] **AC-6**: The card heading reads "Spray Chart" with the BIP count subtitle (e.g., "Based on 47 balls in play").
- [ ] **AC-7**: The template implements all four threshold tiers per TN-7: (a) >= 10 BIP shows the chart image, (b) 5-9 BIP shows "Small sample — N balls in play" without a chart, (c) < 5 BIP shows "Not enough data yet — N balls in play recorded", (d) 0 BIP shows "Charts will appear after the next sync." The BIP count is passed as a template variable from the player profile route.
- [ ] **AC-8**: The chart renders correctly at full width on a 375px mobile viewport — no horizontal overflow, readable dot clusters.
- [ ] **AC-9**: Tests verify: image route returns PNG bytes for a player with >= 10 events, 204 response for a player with < 10 events, and correct content-type header.

## Technical Approach
Add two pieces: (1) a new image route in `src/api/routes/dashboard.py` that queries spray_charts and calls the renderer, (2) a template update to show the chart card. The existing player profile route already queries the DB for player stats — add a BIP count query there and pass it to the template so the template can decide between showing the image or the empty state.

The image route runs the DB query and renderer in a threadpool (consistent with existing dashboard patterns using `run_in_threadpool`).

## Notes
- Dashboard routes pass `title=None` to the renderer — the HTML card heading handles the label. The PNG title is for CLI/standalone export use only.
- Consider caching: for MVP, on-the-fly generation is acceptable. The data volume per player (~30 games × ~3-4 BIP per game = ~100 events max) renders quickly with matplotlib.
- The empty state detection (BIP count) happens in the profile route, not the image route — this avoids an extra round trip to determine whether to show the `<img>` tag. AC-7 (template tiers) is the primary empty-state path. AC-3 (204 response) is a defensive fallback for direct URL access — do not rely on it as the primary mechanism.
- The spray chart image endpoint (`/dashboard/charts/spray/player/{player_id}.png`) does NOT perform team membership checks. It requires only an authenticated session (Cloudflare Access) and that `spray_charts` rows exist for the player. This is intentional — E-158-06 links opponent players directly to this image endpoint, and opponent players are on tracked teams that would fail the `permitted_teams` check on the full player profile page.
