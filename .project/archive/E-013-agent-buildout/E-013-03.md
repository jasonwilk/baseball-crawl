# E-013-03: Seed Memory Directories for `api-scout` and `baseball-coach`

## Epic
[E-013: Agent Buildout -- Complete All Stubbed Agents](../E-013-agent-buildout/epic.md)

## Status
`DONE`

## Description
After this story is complete, `api-scout` and `baseball-coach` both have seeded memory directories with `MEMORY.md` files containing the institutional knowledge these agents would have accumulated if memory had been active since project inception. Both agents already have strong system prompts -- this story only creates the missing memory infrastructure, it does not modify the agent definition files.

## Context
Both `api-scout.md` and `baseball-coach.md` declare `memory: project` in their frontmatter, indicating they are expected to maintain persistent memory. However, neither has a memory directory at `.claude/agent-memory/<agent-name>/`. When these agents start a session, they have no memory to draw on -- every session starts cold.

This is particularly significant for `api-scout`: the agent is supposed to accumulate knowledge about the GameChanger API across sessions (auth patterns, rate limit observations, endpoint relationships, exploration status). Without a seeded MEMORY.md, the agent cannot even know where exploration left off.

For `baseball-coach`: coaching priorities and decisions made in prior sessions are supposed to carry forward. Without memory, the coach re-discovers the same priorities every session.

The `memory: project` frontmatter field tells Claude Code to load the agent's MEMORY.md into the system prompt at session start. This only works if the file exists. Neither agent's memory directory exists.

The content for these seed files is derived from:
- The PM's MEMORY.md (key architectural decisions and project context)
- Claude-architect's MEMORY.md (project scope and team context)
- The CLAUDE.md (scope, metric priorities, security rules)

This story does NOT modify `api-scout.md` or `baseball-coach.md` -- those files stay unchanged. Skill references will be added to the agent definition files in E-013-04.
