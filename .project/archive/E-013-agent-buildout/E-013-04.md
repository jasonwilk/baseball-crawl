## Description
After this story is complete, ALL seven agents have a `## Skill References` (or `## Skills`) section in their system prompts with named trigger conditions for all applicable context management skills from E-010 Phase 1: `filesystem-context`, `multi-agent-patterns`, and `context-fundamentals`. Skills stop being dead files and become part of each agent's operational workflow. Every agent in the routing chain knows which skill to load, when, and why.

This is a comprehensive skill-wiring story covering the full agent ecosystem. The scope spans three skills and seven agents, with each agent receiving only the skills relevant to their role in the orchestrator -> PM -> implementing agent chain.

**Rationale for assignments**:
- **orchestrator**: All three skills. Dispatches context blocks (filesystem-context), routes through multi-agent chain (multi-agent-patterns), manages context when aggregating from multiple specialists (context-fundamentals).
- **project-manager**: filesystem-context (Dispatch Mode and Refinement Mode context-loading), multi-agent-patterns (Risk Point 2 -- PM -> implementing agent relay). No context-fundamentals -- PM sessions are typically focused and do not accumulate large context windows.
- **claude-architect**: filesystem-context (ambient vs. deferred context decisions during agent design), multi-agent-patterns (evaluating relay depth when designing new agents). No context-fundamentals -- architect sessions are design-focused and rarely hit context limits.
- **api-scout**: filesystem-context (reads/writes the API spec and raw responses), multi-agent-patterns (direct-routing exception -- outputs must go to durable files), context-fundamentals (API spec grows over time; budget awareness needed).
- **baseball-coach**: filesystem-context (reads story files when consulted by PM, produces requirements artifacts), multi-agent-patterns (direct-routing exception -- outputs must go to durable files). No context-fundamentals -- coaching consultations are focused.
- **general-dev**: filesystem-context (must load story file, epic Technical Notes, API spec, design docs), multi-agent-patterns (defensive -- fires when dispatch context looks summarized), context-fundamentals (complex stories load many files).
- **data-engineer**: filesystem-context (must load story file, epic Technical Notes, schema docs, prior migration files), multi-agent-patterns (defensive -- fires when dispatch context looks summarized), context-fundamentals (same reasoning as general-dev).

### Skill Reference Format

```markdown
## Skills

### <skill-name>
**File**: `.claude/skills/<skill-name>/SKILL.md`
**Load when**: [specific trigger conditions for this agent]

[One to two sentences describing why this skill is relevant to this agent's work.]
```

For all other agents, use the format established in the E-013 epic Technical Notes:

```markdown
## Skill References

Load `.claude/skills/<skill-name>/SKILL.md` when:
- [trigger condition 1]
- [trigger condition 2]
```

Either format is acceptable. Consistency within a file matters more than consistency across files.
