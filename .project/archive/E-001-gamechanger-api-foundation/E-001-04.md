# E-001-04: Smoke test -- verify client against live API

## Epic
[E-001: GameChanger API Foundation](epic.md)

## Status
`DONE`

## Description
After this story is complete, the project will have a runnable smoke test script (`scripts/smoke_test.py`) that uses the `GameChangerClient` to make real API calls and verify that the full credential-to-response pipeline works end-to-end. The smoke test prints a human-readable summary of what it found so the user can confirm data is flowing correctly.

## Context
This is a validation story -- it doesn't add new production functionality, but it proves that E-001-01 (credential parser), E-001-02 (HTTP client), and E-001-03 (API spec) all compose correctly against the live GameChanger API. It also serves as an ongoing diagnostic tool: any time credentials are refreshed or the API behaves unexpectedly, the smoke test can be run to quickly confirm everything is working.

The smoke test is explicitly NOT part of the automated test suite -- it makes real network calls and requires live credentials, so it lives in `scripts/` not `tests/`.

## Acceptance Criteria
- [ ] **AC-1**: Given valid credentials in `.env` and the real GameChanger API, when `python scripts/smoke_test.py` is run, then the script prints a summary table showing: endpoint called, HTTP status, HTTP round-trip time (excluding rate-limit delays), and a 1-line description of what was returned (e.g., "game-summaries: 34 games").
- [ ] **AC-2**: The smoke test calls at minimum: (a) `GET /me/teams` to discover team UUIDs, (b) `GET /teams/{team_id}/game-summaries` to verify game data access, and (c) `GET /teams/{team_id}/players` to verify roster access. The `team_id` is discovered from the `/me/teams` response unless overridden by `--team-id`.
- [ ] **AC-3**: Given expired credentials, when the smoke test runs, then it catches `CredentialExpiredError` and prints: `Credentials expired. Refresh by running: python scripts/refresh_credentials.py` (not a raw Python traceback).
- [ ] **AC-4**: The script exits with status code 0 if all tested endpoints returned 2xx, and status code 1 if any endpoint failed, so it can be used in a simple check.
- [ ] **AC-5**: The smoke test prints the GameChanger team name(s) and UUID(s) discovered from `/me/teams`, to give the user a human-readable confirmation they're hitting the right team data.
- [ ] **AC-6**: When no `--team-id` is provided, the smoke test discovers teams by calling `/me/teams`, selects the first team, and uses its UUID for subsequent endpoint calls.
- [ ] **AC-7**: The smoke test accepts a `--team-id UUID` argument. When provided, it skips `/me/teams` discovery and uses the given UUID directly for team-scoped endpoints.

## Technical Approach
Keep the smoke test simple and linear. It is a script, not a framework. Example structure:

```python
#!/usr/bin/env python3
"""
Smoke test: verifies GameChanger API access end-to-end.
Run with: python scripts/smoke_test.py [--team-id UUID]

Requires valid credentials in .env (run scripts/refresh_credentials.py first).
"""
import argparse
import sys
import logging
from src.gamechanger.client import GameChangerClient, CredentialExpiredError

logging.basicConfig(level=logging.WARNING)  # suppress debug noise during smoke test

def run_smoke_test(team_id: str | None = None) -> bool:
    all_passed = True
    client = GameChangerClient()
    # 1. Discover teams via /me/teams (unless --team-id provided)
    # 2. Call /teams/{team_id}/game-summaries
    # 3. Call /teams/{team_id}/players
    # ... print summary table, return bool
    return all_passed
```

For each endpoint tested, wrap the call in a try/except, measure HTTP round-trip time with `time.perf_counter()` around the client call, and print a row of output. Use a simple f-string table, not a library.

Each endpoint call must use the correct vendor-typed `Accept` header as documented in `docs/gamechanger-api.md` (Accept Headers by Endpoint table). Use the `GameChangerClient.get()` method's `accept` parameter for this.

Per-endpoint summary extraction:
- `/me/teams`: "N teams found" + team names
- `/teams/{id}/game-summaries`: "N games found"
- `/teams/{id}/players`: "N players on roster"

The smoke test relies on the `GameChangerClient` for rate limiting between requests -- do not add manual `time.sleep()` calls.
