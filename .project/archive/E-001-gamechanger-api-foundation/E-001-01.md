# E-001-01: Extract credentials from curl and write credential parser

## Epic
[E-001: GameChanger API Foundation](epic.md)

## Status
`DONE`

## Description
After this story is complete, the system will have a script (`scripts/refresh_credentials.py`) that accepts a raw curl command string (pasted by the user), parses out all authentication-relevant headers and cookies, and writes them to a `.env` file in the project root. The script will be the single mechanism for updating credentials whenever they rotate.

## Context
GameChanger uses short-lived credentials that the user extracts from browser/app network traffic as curl commands. The curl command contains everything needed to authenticate: headers like `Authorization: Bearer <token>`, cookies, and potentially custom headers. Rather than having the user manually edit config files every time credentials rotate, this script automates the extraction step.

This story has no dependencies -- it can be started immediately, even before we know the full API shape. The user will provide at least one real curl command to test against. If no curl command is available yet, the script should be written against a realistic mock and tested with that.

## Acceptance Criteria
- [x] **AC-1**: Given a curl command string containing an `Authorization: Bearer <token>` header, when `python scripts/refresh_credentials.py --curl "<curl string>"`, then a `.env` file is written (or updated) containing `GAMECHANGER_AUTH_TOKEN=<token>` and any other extracted values, and the script prints a confirmation message showing which keys were written.
- [x] **AC-2**: Given a `.env` file that already exists with old credentials, when the script runs with new credentials, then the old credential values are replaced and non-credential values in the file are preserved.
- [x] **AC-3**: Given a curl command with cookies in addition to headers, when the script runs, then cookies are extracted and stored as `GAMECHANGER_COOKIE_<NAME>=<value>` entries.
- [x] **AC-4**: Given a malformed curl command (missing auth headers, or not a curl command at all), when the script runs, then it exits with a non-zero status code and a human-readable error message explaining what was missing.
- [x] **AC-5**: Given the script is run with `--help`, then it prints a usage description explaining how to copy a curl command from browser dev tools and pass it to the script.
- [x] **AC-6**: The `.env` file is listed in `.gitignore`. The script must not create or modify any file that could be committed to version control.

## Technical Approach
Parse the curl command with a purpose-built parser (do not shell out to `curl`). A curl command looks like:

```
curl 'https://api.example.com/endpoint' \
  -H 'Authorization: Bearer eyJhb...' \
  -H 'Cookie: session_id=abc123; other=xyz' \
  -H 'X-Custom-Header: value'
```

Use Python's `shlex.split()` to tokenize the curl string, then walk the token list looking for `-H` and `--header` flags. Extract the header name and value for each. Special handling for:
- `Authorization: Bearer <token>` -> `GAMECHANGER_AUTH_TOKEN`
- `Cookie: <name>=<value>; ...` -> individual `GAMECHANGER_COOKIE_<NAME>` entries
- Any other headers -> `GAMECHANGER_HEADER_<NAME>` (uppercased, hyphens to underscores)

Also extract the URL itself and store as `GAMECHANGER_BASE_URL` (scheme + host only, strip path).

For `.env` file updating: read existing `.env`, parse into key-value dict, merge new values, write back. Use a simple line-by-line parser -- do not use a library that might silently corrupt the file.
