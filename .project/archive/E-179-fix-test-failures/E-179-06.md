## Description
After this story is complete, 2 failing tests in `tests/test_pii_hook_integration.py`, 1 failing test in `tests/test_scouting_spray_loader.py`, 1 failing test in `tests/test_admin_teams.py`, and 1 failing test in `tests/test_admin_routes.py` will pass. These are grouped together because they are small (1-2 failures each) and span four separate files.

## Context
These failures come from three different root cause categories:

**PII hook integration (TN-1 category 6 -- test infrastructure):** `_init_repo()` creates a temp git repo that symlinks only `src/safety/` but does not create `src/__init__.py`. The pre-commit hook runs `python3 -m src.safety.pii_scanner` which requires `src` to be a Python package. Without `src/__init__.py` in the temp repo, the module import fails. This is NOT an E-173 change.

**Scouting spray loader (TN-1 category 6 -- test infrastructure):** `test_resolvable_unknown_player_gets_stub_row` fails with `sqlite3.IntegrityError: FOREIGN KEY constraint failed`. The test inserts into `team_rosters` without a matching `players` row, but the `db` fixture enables FK enforcement (`PRAGMA foreign_keys=ON`). The FK on `team_rosters.player_id` references `players(player_id)`, so the INSERT fails. This is NOT an E-173 change.

**Admin teams (TN-1 category 7 -- behavioral drift):** `TestDeleteTeam::test_delete_cleans_up_junction_rows` fails with `sqlite3.IntegrityError: CHECK constraint failed: our_team_id != opponent_team_id`. The test fixture inserts a `team_opponents` row where `our_team_id == opponent_team_id`, violating a CHECK constraint.

**Admin routes (TN-1 category 7 -- behavioral drift):** `TestBottomNavSuppression::test_bottom_nav_present_on_dashboard` asserts `">Games<"` in dashboard HTML -- likely a nav label rename.
