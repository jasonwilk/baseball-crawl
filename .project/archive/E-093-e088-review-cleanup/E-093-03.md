# E-093-03: Fix test_listing_requires_admin and Remove Dead Code

## Epic
[E-093: E-088 Code Review Cleanup](epic.md)

## Status
`DONE`

## Description
After this story is complete, `test_listing_requires_admin` in `tests/test_admin_opponents.py` will pass regardless of whether `DEV_USER_EMAIL` is set in the environment, and the dead `is_duplicate_opponent_public_id` function will be removed from `src/api/db.py`.

## Context
Code review finding #5: `test_listing_requires_admin` uses `patch.dict("os.environ", {"DATABASE_PATH": str(opp_db)})` without clearing `DEV_USER_EMAIL`. When the dev environment has `DEV_USER_EMAIL` set, the session middleware auto-authenticates, producing a 200 instead of the expected 302. The sister test file `tests/test_admin_teams.py` handles this correctly by including `"DEV_USER_EMAIL": ""` in every relevant `patch.dict` call. Additionally, `is_duplicate_opponent_public_id` at `src/api/db.py:976` has zero callers -- it was superseded by `get_duplicate_opponent_name`.

## Acceptance Criteria
- [ ] **AC-1**: The `patch.dict` call in `test_listing_requires_admin` (line 309) includes `"DEV_USER_EMAIL": ""` to clear the dev bypass.
- [ ] **AC-2**: `test_listing_requires_admin` passes both with and without `DEV_USER_EMAIL` set in the outer environment.
- [ ] **AC-3**: The `is_duplicate_opponent_public_id` function is removed from `src/api/db.py`.
- [ ] **AC-4**: No other code references `is_duplicate_opponent_public_id` (grep confirms zero callers).
- [ ] **AC-5**: All existing tests pass (`pytest`).

## Technical Approach
For the test fix: add `"DEV_USER_EMAIL": ""` to the `patch.dict` dict at line 309, matching the pattern used in `test_admin_teams.py`. For the dead code removal: delete the entire `is_duplicate_opponent_public_id` function (lines 976-1001 of `src/api/db.py`). Grep the codebase to confirm no other references exist.
