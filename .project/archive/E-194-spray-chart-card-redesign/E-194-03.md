# E-194-03: Opponent Print Template Card Redesign

## Epic
[E-194: Spray Chart Card Redesign](epic.md)

## Status
`DONE`

## Description
After this story is complete, the opponent print view (`opponent_print.html`) will display spray chart cards matching the same 3-column enriched layout as the standalone scouting report: jersey number + name, AVG/OBP/SLG slash line, PA badge, spray chart image, Left/Center/Right zone counts, and contact type counts. The route handler will fetch and classify spray event data to supply these stats.

## Context
The opponent print template already uses a 3-column grid (`.tendencies-grid`) but cards contain only a name and spray chart image -- no slash line, zone breakdown, or contact type stats. The route handler passes `player_spray_bip_counts` but not the raw spray events needed for zone/contact classification. This story enriches the route handler data and redesigns the card template to match the TN-6 card structure using `.tendency-card-*` class names per TN-7.

## Acceptance Criteria
- [ ] **AC-1**: The opponent print route passes a per-player stats dict to the template containing: `avg` (str), `obp` (str), `slg` (str), `pa` (int), `bip_count` (int), `jersey_number` (str|None), `zones` (dict with `left`/`center`/`right` int counts), and `contacts` (dict with `gb`/`ld`/`fb`/`pu`/`bu` int counts).
- [ ] **AC-2**: OBP and SLG are recomputed from raw batting fields per TN-3 (do NOT rely on `_obp_raw`/`_slg_raw` -- the opponent print route pops these during heat computation). Formatted using baseball convention (`.342` for < 1.0, `1.000` for >= 1.0).
- [ ] **AC-3**: Zone and contact classification uses the shared `classify_field_zone()` and `contact_type_label()` functions from `src/charts/spray.py` (produced by E-194-02).
- [ ] **AC-4**: Each tendency card renders the structure per TN-6: identity row (jersey + name), slash line (AVG/OBP/SLG), PA badge, chart image, direction row (`Left 7  Ctr 3  Right 5` format -- space-separated, no slashes), contact row (`6GB  1LD  4FB  1PU  2BU` format).
- [ ] **AC-5**: CSS classes follow the `.tendency-card-*` naming per TN-7, including identity row truncation, card border/padding specs, and `.tendency-card-empty` for placeholder text.
- [ ] **AC-6**: The tendencies grid uses flexbox layout (`display: flex; flex-wrap: wrap`) for both screen and print per TN-8. Cards use `flex: 0 0 calc(33.333% - 6px)` (was `calc(25% - 5px)` in print).
- [ ] **AC-7**: Players with 0 BIP (no spray events) still display their card with name, slash line, and PA badge, plus "No spray chart data available" in place of the image/direction/contact sections per TN-10. Uses `.tendency-card-empty` class (not inline styles). The existing threshold for showing spray images (`>=1 BIP` in opponent print) is preserved per Non-Goals.
- [ ] **AC-8**: Existing tests continue to pass.

## Technical Approach
Two areas of change:

1. **Route handler** (`src/api/routes/dashboard.py`, `opponent_print` function): Add a new query function in `src/api/db.py` to fetch raw spray events per player (x, y, play_type, play_result) -- the existing `get_player_spray_bip_counts()` returns counts only. Recompute OBP/SLG from raw batting fields per TN-3. Apply zone and contact classification using shared helpers from `src/charts/spray.py` (produced by E-194-02). Build the enriched stats dict and pass to template context.

2. **Template** (`src/api/templates/dashboard/opponent_print.html`): Replace the current minimal `.tendency-card` structure with the full TN-6 card layout. Add new `.tendency-card-*` CSS classes per TN-7. Switch to flexbox layout for both screen and print per TN-8.
