# E-194-02: Data Enrichment + Scouting Report Template Redesign

## Epic
[E-194: Spray Chart Card Redesign](epic.md)

## Status
`DONE`

## Description
After this story is complete, the standalone scouting report (`scouting_report.html`) will display spray chart cards in a 3-column layout with enriched per-player stats: jersey number, name, AVG/OBP/SLG slash line, PA badge, spray chart image (without baked-in title), Left/Center/Right zone counts, and contact type counts (GB/LD/FB/PU/BU). All card stats will be visible in print output.

## Context
The current `_build_spray_player_stats()` in `renderer.py` returns only AVG, PA, and BIP count -- it never reads OBP or SLG even though the data is available. The template uses a 2-column grid with minimal stats and hides `.spray-card-stats` in print CSS. This story fixes the data enrichment bug and implements the UXD-validated card design per TN-6 through TN-8 in the epic.

## Acceptance Criteria
- [ ] **AC-1**: `_build_spray_player_stats()` returns per-player dicts containing: `avg` (str), `obp` (str), `slg` (str), `pa` (int), `bip_count` (int), `jersey_number` (str|None), `zones` (dict with keys `left`, `center`, `right`, each int), and `contacts` (dict with keys `gb`, `ld`, `fb`, `pu`, `bu`, each int).
- [ ] **AC-2**: OBP and SLG are formatted using baseball convention: `.342` for values < 1.0, `1.000` for values >= 1.0. AVG follows the same convention.
- [ ] **AC-3**: Field zone classification uses SVG-space coordinates (raw coords transformed via `_raw_to_svg()` before angle calculation) and angle-based equal thirds from home plate, per TN-4. Events with `x=None` or `y=None` are excluded from zone counts.
- [ ] **AC-4**: Contact type classification uses the five-category mapping per TN-5. Events with `play_type=None` or unmapped play_type values are excluded from contact counts.
- [ ] **AC-5**: Zone classification (`classify_field_zone`) and contact type mapping (`contact_type_label`) are implemented as public functions in `src/charts/spray.py`, co-located with the coordinate transform constants they depend on. Both functions are importable by other modules.
- [ ] **AC-6**: Individual player spray charts are rendered with `title=None` (no baked-in title). Team spray chart retains its title per TN-2.
- [ ] **AC-7**: The `.spray-grid` uses flexbox layout (`display: flex; flex-wrap: wrap`) for both screen and print per TN-8. Cards use `flex: 0 0 calc(33.333% - 6px)`.
- [ ] **AC-8**: The old `.spray-card-stats { display: none }` rule in the print CSS is removed -- all card stats are visible in PDF output.
- [ ] **AC-9**: Each spray card renders the structure per TN-6: identity row (jersey + name), slash line (AVG/OBP/SLG), PA badge, chart image, direction row (`Left 7  Ctr 3  Right 5` format -- space-separated, no slashes), contact row (`6GB  1LD  4FB  1PU  2BU` format).
- [ ] **AC-10**: CSS classes follow the `.spray-card-*` naming per TN-7, including identity row truncation and card border/padding specs.
- [ ] **AC-11**: Existing tests continue to pass; new tests cover the enriched stat computation (OBP/SLG, zones, contacts) and the shared helper functions.

## Technical Approach
Three files change: `src/charts/spray.py` (shared helpers), `src/reports/renderer.py` (data enrichment), and `src/api/templates/reports/scouting_report.html` (card layout and CSS).

In `src/charts/spray.py`: add public `classify_field_zone(x, y)` and `contact_type_label(play_type)` functions, co-located with the coordinate transform constants they depend on.

In `renderer.py`: enrich `_build_spray_player_stats()` to recompute OBP/SLG from raw batting fields (per TN-3 -- do NOT rely on `_obp_raw`/`_slg_raw` from heat computation), classify spray events into L/C/R zones using the shared helpers, and count events by the five contact type categories. Change the `_encode_spray_chart` call for individual players to pass `title=None`.

In `scouting_report.html`: replace the old `.spray-card-name` and `.spray-card-stats` CSS with the new TN-7 classes. Replace the old 2-column grid with 3-column. Replace old card HTML with the TN-6 structure. Update print CSS per TN-8.
