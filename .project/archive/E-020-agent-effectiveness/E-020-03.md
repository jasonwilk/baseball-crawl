## Description
After this story is complete, the baseball-coach agent will have a system prompt that follows the canonical section skeleton, includes Anti-Patterns and Error Handling sections, and has a consolidated memory section. The frontmatter (already clean from E-018) is preserved unchanged.

## Context
The baseball-coach is a domain expert agent that translates coaching needs into technical requirements, validates schemas, and designs scouting report formats. It is a consultative agent (direct-routing exception) that does not require a story reference to begin work.

The baseball-coach's current system prompt (127 lines) has good domain content -- the Key Baseball Analytics Knowledge section and Output Standards section are strong. The main gaps compared to the reference agents are:
1. No Anti-Patterns section -- no guidance on what NOT to do (e.g., do not write code, do not design schemas, do not make implementation decisions)
2. No Error Handling section -- no guidance for when asked about stats outside its knowledge, when sample sizes are too small to give advice, or when coaching requirements conflict
3. The "How You Interact With Other Agents' Work" section is brief and passive ("you may be asked to review")
4. The "Your Context" section duplicates project scope from CLAUDE.md (team names, roster sizes, season length)
5. The memory section is minimal
