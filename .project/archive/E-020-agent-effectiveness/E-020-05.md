## Description
After this story is complete, the data-engineer agent will have a system prompt that follows the canonical section skeleton, removes CLAUDE.md content duplication (database conventions already specified in CLAUDE.md), includes Anti-Patterns and Error Handling sections, and has a single consolidated memory section instead of the current duplicated pair.

## Context
The data-engineer designs database schemas, writes SQL migrations, architects ETL pipelines, and optimizes query patterns. It is an implementing agent that requires a story reference before beginning work.

The data-engineer's current system prompt (185 lines) has moderate CLAUDE.md duplication. The "Database Standards" section partially overlaps with CLAUDE.md's architecture and data conventions. The database conventions (ip_outs, soft referential integrity, timestamps, IDs) are specified in BOTH CLAUDE.md and the data-engineer prompt. Unlike general-dev, the data-engineer's domain-specific standards (migration management, ETL pipeline design, query/index design) are genuinely agent-specific and should be preserved.

The memory section has the same duplication pattern as general-dev: "Memory Instructions" + "Persistent Agent Memory" with overlapping save/don't-save lists.

Other gaps: no Anti-Patterns section, no Error Handling section.
