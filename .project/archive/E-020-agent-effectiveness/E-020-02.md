## Description
After this story is complete, the api-scout agent will have a system prompt that follows the canonical section skeleton, includes Anti-Patterns and Error Handling sections, and has a consolidated memory section. No CLAUDE.md content is duplicated. The frontmatter (already clean from E-018) is preserved unchanged.

## Context
The api-scout explores the GameChanger API, documents endpoints in `docs/gamechanger-api.md`, and manages credential patterns. It is a consultative agent (direct-routing exception) that does not require a story reference to begin work.

The api-scout's current system prompt (166 lines) is already reasonably well-structured with good Identity, Core Responsibilities, Security Rules, and Discovery Methodology sections. The main gaps compared to the reference agents are:
1. No Anti-Patterns section -- no guidance on what NOT to do
2. No Error Handling section -- no guidance for stale specs, expired creds, unexpected API responses
3. The "How You Interact With Other Agents" section is generic ("You produce the knowledge. They consume it.") rather than concrete
4. The memory section at the bottom is minimal and could be more specific about what api-scout experiences are worth preserving

## Acceptance Criteria
- [x] **AC-1**: The YAML frontmatter is UNCHANGED from its current state (E-018 already standardized it)
- [x] **AC-2**: An "Anti-Patterns" section exists with 3-5 concrete items specific to API exploration work. Must include at minimum: (a) never store actual credentials in any file, (b) never assume an endpoint's behavior without re-verification, (c) never modify the API spec based on a single observation without noting sample size
- [x] **AC-3**: An "Error Handling" section exists with 3-5 items covering: expired credentials, unexpected API response shapes, rate limiting/throttling, endpoint that previously worked now returning errors, conflicting information between API spec and actual behavior
- [x] **AC-4**: The "Inter-Agent Coordination" section (currently "How You Interact With Other Agents") has 1-2 concrete sentences per agent describing the specific interaction pattern, not generic statements
- [x] **AC-5**: The memory section is consolidated into a single section with no duplication. Lists specific api-scout knowledge worth preserving (endpoint quirks, auth lifecycle observations, rate limit behavior, etc.)
- [x] **AC-6**: The section order follows the canonical skeleton from the epic Technical Notes
- [x] **AC-7**: No content from CLAUDE.md is duplicated in the system prompt (the api-scout does not currently have this problem, so this AC verifies it is not introduced)
- [x] **AC-8**: All existing content that remains accurate is preserved -- this is a refinement of structure and completeness, not a rewrite
