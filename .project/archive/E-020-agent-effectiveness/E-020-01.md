# E-020-01: Refine claude-architect Agent

## Epic
[E-020: Agent Effectiveness Audit & Refinement](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the claude-architect agent will have clean frontmatter matching the E-018 standard, an updated system prompt that references the project's actual agent file format (markdown + YAML frontmatter, not JSON), project-specific identity instead of generic boilerplate, Anti-Patterns and Error Handling sections, and a consolidated memory section.

## Context
The claude-architect is the meta-agent -- it designs and maintains all other agents. But its own configuration file has not been updated since it was first created. It still has the bloated multi-paragraph description with routing examples in its frontmatter (the same problem E-018 fixed for the other four agents). Its system prompt references a JSON output format (`identifier`, `whenToUse`, `systemPrompt`) that this project does not use -- all agents are markdown files with YAML frontmatter in `.claude/agents/`. Its identity section is generic ("elite-tier expert in the Claude Code ecosystem") rather than specific to this project's agent ecosystem.

This is the highest-impact story in the epic because the architect is the agent that will execute the other four stories. Getting its own house in order first improves the quality of everything that follows. However, all five stories can run in parallel since this story only modifies the architect's own file.
