# E-076-02: Cross-Reference from Workflow-Discipline and CLAUDE.md

## Epic
[E-076: Main Session Compliance Guardrails](epic.md)

## Status
`DONE`

## Description
After this story is complete, the Consultation Compliance Gate in `workflow-discipline.md` will cross-reference the new agent-team-compliance rules file, and CLAUDE.md will have a "Main Session Compliance" subsection under Agent Ecosystem that points to the rules file with one-line pattern summaries. This creates defense-in-depth: the specific procedural checkpoints live in the dedicated rules file (E-076-01), while the broader gate and the project's central documentation both reinforce the requirements.

## Context
The existing Consultation Compliance Gate in `workflow-discipline.md` is scoped to PM behavior. CLAUDE.md's Agent Ecosystem section describes agent roles and collaboration but does not reference main session compliance obligations for ad-hoc team/consultation requests. Adding cross-references from both files ensures the rules are discoverable from multiple entry points and reinforces them through repetition.

## Acceptance Criteria
- [ ] **AC-1**: The Consultation Compliance Gate in `.claude/rules/workflow-discipline.md` includes a cross-reference paragraph that: (a) names `.claude/rules/agent-team-compliance.md`, (b) explicitly states the scope relationship between the two gates -- the Consultation Compliance Gate governs PM behavior during epic formation, while the agent-team-compliance rules govern main session and general agent compliance with ad-hoc team and consultation requests, and (c) states that the agent-team-compliance rules extend the consultation compliance concept to all agents (not just PM). The existing PM-specific gate content is preserved unchanged.
- [ ] **AC-2**: CLAUDE.md contains a new "Main Session Compliance" subsection under the Agent Ecosystem section, placed after "How Agents Collaborate" and before "Workflow Contract" (this is an ecosystem concern, not a workflow concern). The subsection: (a) names the rules file (`.claude/rules/agent-team-compliance.md`), (b) summarizes the three patterns in one line each, (c) uses directive language ("MUST"), (d) is concise (no more than 10 lines total).
- [ ] **AC-3**: No existing compliance rules are weakened or removed by these changes. The PM-specific anti-patterns in the PM agent definition remain as-is. The existing Consultation Compliance Gate text remains intact.
- [ ] **AC-4**: The CLAUDE.md subsection does NOT duplicate the full procedural content from the rules file -- it is a pointer with brief summaries, not a second copy.
