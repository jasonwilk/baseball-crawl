# E-090-01: Add tmux Window Rename to Implement Skill

## Epic
[E-090: Auto-Rename tmux Window on Epic Dispatch](epic.md)

## Status
`DONE`

## Description
After this story is complete, the implement skill's dispatch flow will automatically rename the tmux window to the epic ID (e.g., "E-088") at the start of dispatch, if the session is running inside a tmux session. If not in tmux, the step is silently skipped. Failure to rename does not block dispatch.

## Context
The implement skill (`.claude/skills/implement/SKILL.md`) orchestrates all epic dispatch. The `TMUX` environment variable is set when inside a tmux session. The operator currently performs `tmux rename-window "E-NNN"` manually during Heavy mode dispatch. This story automates that step.

The implement skill is a context-layer file (`.claude/skills/`). Per routing precedence, this story routes to claude-architect. SE/CA consultation confirmed this routing (2026-03-10).

## Acceptance Criteria
- [ ] **AC-1**: Given a tmux session is active (`$TMUX` is set), when the implement skill begins dispatch of epic E-NNN, then the tmux window is renamed to "E-NNN" before team creation begins.
- [ ] **AC-2**: Given no tmux session is active (`$TMUX` is not set or empty), when the implement skill begins dispatch, then the guard short-circuits before invoking `tmux`, no error is produced, and dispatch proceeds normally.
- [ ] **AC-3**: Given a tmux session is active but `tmux rename-window` fails at runtime (e.g., stale socket, permission error), when the implement skill begins dispatch, then stderr is suppressed, exit code is 0 (via `|| true`), and dispatch proceeds without blocking.
- [ ] **AC-4**: The epic ID used for the rename is extracted from the user's trigger phrase (the same E-NNN that the implement skill already parses in its Prerequisites phase).

## Technical Approach
The implement skill already parses the epic ID in its Prerequisites phase. Add a new step between Prerequisites and Phase 1 (Team Composition) that instructs the main session to run a guarded one-liner via the Bash tool. The skill text must explicitly instruct the main session to substitute the actual parsed epic ID into the command (not the literal placeholder "E-NNN"). The command uses three guards (env var check, binary check, stderr suppression) plus `|| true` to fail completely silently with exit code 0 when not applicable. Also update the Workflow Summary diagram in the same file to reflect the new step. See epic Technical Notes for the full command and rationale.
