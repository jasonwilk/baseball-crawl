# E-049-07: Endpoint coaching-value priority matrix

## Epic
[E-049: API Endpoint Documentation and Dual-Header System](epic.md)

## Status
`DONE`

## Description
After this story is complete, a coaching-value priority matrix will exist in the API spec that ranks every discovered endpoint by its value for coaching analytics. This matrix guides which endpoints to build crawlers and loaders for next, connecting API capabilities to coaching use cases like scouting, lineup optimization, player development tracking, and game preparation.

## Context
With 50+ endpoint patterns documented, the team needs a clear roadmap for which ones to integrate first. Some endpoints are critical for coaching (standings, opponent rosters, player stats across teams) while others are infrastructure (sync, subscriptions, widgets). This story produces the prioritized integration roadmap.

## Acceptance Criteria
- [ ] **AC-1**: `docs/gamechanger-api.md` contains a new section "Endpoint Priority Matrix" (positioned after the proxy-discovered endpoints section and before "Key Observations") with a table listing every documented endpoint, its coaching value (Critical / High / Medium / Low / None), its current status (Confirmed / Observed / HTTP 500), and a one-line coaching use case
- [ ] **AC-2**: The matrix groups endpoints into tiers:
  - **Tier 1 (Critical)**: Endpoints that directly enable scouting reports, lineup decisions, or game preparation. These should be the next crawlers built.
  - **Tier 2 (High)**: Endpoints that enhance analysis but are not blocking core workflows
  - **Tier 3 (Medium)**: Nice-to-have data for secondary analysis
  - **Tier 4 (Low/None)**: Infrastructure, config, or video endpoints with no coaching analytics value
- [ ] **AC-3**: The matrix identifies the **top 5 endpoints to integrate next** (after the existing E-002 pipeline) with a brief rationale for each, based on coaching value and implementation feasibility
- [ ] **AC-4**: The matrix notes which high-value endpoints currently return HTTP 500 with web headers (status: unresolved -- see IDEA-011), and flags them as requiring investigation before crawlers can be built
- [ ] **AC-5**: The matrix cross-references related IDEA backlog items: IDEA-008 (plays and line scores) and IDEA-009 (per-player game stats and spray charts) should be linked to the relevant endpoints
- [ ] **AC-6**: Each tier includes a brief narrative (2-3 sentences) explaining what coaching capability that tier unlocks

## Technical Approach
1. Read the completed schema documentation from E-049-01, E-049-02, E-049-03
2. Read `/.project/ideas/IDEA-008-plays-and-line-scores.md` and `/.project/ideas/IDEA-009-per-player-game-stats-spray-charts.md` for context on planned work

This is the primary proxy for coaching priorities since live baseball-coach consultation was not available during planning. The key metrics section covers: OBP, K rate, splits (home/away, L/R), per-game lines, box scores, spray charts, pitch-by-pitch, player identification, opponent tendencies, and longitudinal tracking.
4. Build the tiered matrix with coaching rationale
5. Identify the top 5 integration candidates
6. **Note**: Ideally this story would include baseball-coach consultation for tier validation. In its absence, the implementer should prioritize endpoints that directly serve the "Key Metrics We Track" list and flag any tier assignments that feel uncertain for PM review.
