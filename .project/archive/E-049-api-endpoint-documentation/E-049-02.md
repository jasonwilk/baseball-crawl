# E-049-02: Document /organizations/* endpoint schemas from bulk payloads

## Epic
[E-049: API Endpoint Documentation and Dual-Header System](epic.md)

## Status
`DONE`

## Description
After this story is complete, the `docs/gamechanger-api.md` API spec will contain full response schema documentation for all `/organizations/*` endpoints whose raw payloads were captured in the bulk collection. The organization endpoint family is entirely new -- none were previously documented beyond proxy-log observations. Two of these endpoints returned HTTP 500 with web headers and need special documentation.

## Context
The `/organizations/*` family is a major discovery from the proxy capture. An organization UUID (`8881846c-7a9c-4230-ac17-09627aac7f59`) scopes all requests. If LSB has an org UUID, these endpoints could return all LSB teams, standings, and opponents in single calls rather than iterating per-team. Two endpoints (`org-teams`, `org-opponent-players`) returned HTTP 500 with web headers -- the root cause is unknown (see IDEA-011).

## Technical Approach
1. Read each raw payload file for the `/organizations/*` family
2. Cross-reference with `proxy/data/endpoint-log.jsonl` for request metadata
3. Update the existing "Organizations" subsection under "Proxy-Discovered Endpoints" in `docs/gamechanger-api.md` -- promote each endpoint status from OBSERVED to CONFIRMED or HTTP 500
4. Add full response schema tables with field types and nullability
5. Redact all PII in examples
6. For `org-team-records.json`, document whatever structure is present
