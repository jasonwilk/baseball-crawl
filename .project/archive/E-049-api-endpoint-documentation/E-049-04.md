# E-049-04: Document mobile vs web header differences and endpoint compatibility

## Reason
User feedback (2026-03-06): The "mobile-only" categorization was a hypothesis, not a confirmed fact. The three HTTP 500 endpoints might work with different web headers, different auth, or different parameters -- we only know they failed with the specific web headers we tried. Testing and experimentation to resolve those failures belongs in a separate future epic (see IDEA-011). The factual documentation of the 500 errors is already handled by E-049-01 AC-4 and E-049-02 AC-3/AC-4. The useful header-profile documentation aspect (documenting both profiles side by side) was absorbed into E-049-05 AC-10.
