# E-143-06a: Crawl Jobs Schema Migration

## Epic
[E-143: Admin Management Overhaul](epic.md)

## Status
`DONE`

## Description
After this story is complete, a `crawl_jobs` tracking table exists for recording per-team crawl execution status, timestamps, and outcomes for all UI-triggered syncs (both member and tracked teams). This migration is the prerequisite for the crawl trigger UI (E-143-06).

## Context
The scouting pipeline already has a `scouting_runs` table for tracking scouting status. This story creates a `crawl_jobs` table for all UI-triggered syncs — both member-team crawls and tracked-team scouting syncs triggered from the admin teams page. The table records each crawl invocation's status (running/completed/failed), timing, and outcome so the admin UI can display "Last synced" timestamps and success/failure indicators.
