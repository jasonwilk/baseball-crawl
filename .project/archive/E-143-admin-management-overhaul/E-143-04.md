# E-143-04: Opponent Mapping UX Polish

## Epic
[E-143: Admin Management Overhaul](epic.md)

## Status
`DONE`

## Description
After this story is complete, unresolved opponents are immediately obvious on the Opponents page and the manual Connect flow is discoverable without prior knowledge. The backend for opponent discovery and manual mapping is already fully implemented (see TN-5) -- this story improves the front-end presentation so operators naturally see which opponents need action and how to act on them.

## Context
The baseball-coach rated opponent mapping as the #1 coaching value feature because it unlocks scouting data. SE confirmed all 6 backend routes exist (discover, list, connect, confirm, save, disconnect). The UX designer identified that the "Connect" text link is small and easy to miss, and recommended a summary stat line plus visual hierarchy improvements. The ~14% of opponents that fail auto-resolution need the manual Connect flow -- but the operator must be able to find it.

## Acceptance Criteria
- [ ] **AC-1**: The Opponents page displays a summary stat line above the table showing: total opponent count, resolved count, and unresolved count. Example: "14 opponents -- 10 resolved, 4 need mapping."
- [ ] **AC-2**: Unresolved opponent rows (where `public_id IS NULL` per TN-5) display an orange `Unresolved` badge in a Status column. Resolved rows display a green `Resolved` badge.
- [ ] **AC-3**: The Connect button for unresolved rows uses the primary button style (blue/filled), identical to other primary actions in the admin UI. Manually resolved rows (where `resolution_method = 'manual'`) show a gray Disconnect button (POST form, matching the existing pattern). Auto-resolved rows show no action link (existing behavior preserved per AC-5).
- [ ] **AC-4**: A "Run Discovery" link is prominently displayed on the Opponents page that navigates to `/admin/teams` where the existing per-team Discover buttons are available. No new backend route is added; the existing per-team discover flow is used.
- [ ] **AC-5**: Existing test functions in `tests/test_admin_opponents.py` pass without modification. New tests for the summary stat line, badge styling, and Connect button style are added to the same file. Opponent access control is not changed -- all `/admin/opponents/*` routes remain admin-only per TN-4.

## Technical Approach
This is primarily a template change. The `counts` variable already passed to the opponents template includes the `public_id IS NULL` breakdown (as the "scoresheet only" count) — use this existing context variable for the summary stat line. No route changes are needed. For AC-2/AC-3, modify row styling and the Connect link markup. For AC-4, add a prominent link to the Teams page where existing per-team Discover buttons live.
