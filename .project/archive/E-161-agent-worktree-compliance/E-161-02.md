# E-161-02: Harden Spawn Context and Update Context-Layer References

## Epic
[E-161: Agent Worktree Compliance Audit and Hardening](epic.md)

## Status
`DONE`

## Description
After this story is complete, the implement skill's spawn context will include an explicit negative instruction prohibiting Write/Edit to `/workspaces/baseball-crawl/` paths, and the worktree-isolation rule and CLAUDE.md hook enforcement description will accurately document the expanded hook behavior (allowlist during dispatch, always-on denylist otherwise). Any hardcoded example paths in agent definitions that reference main-checkout non-memory locations will be fixed.

## Context
The E-158 incident showed that advisory "use absolute paths under this directory" language is insufficient -- agents can resolve paths from training data or their own definitions. The hook fix (E-161-01) provides deterministic enforcement, but the advisory layer should also be explicit about what NOT to do. This story adds the negative instruction ("do NOT write to `/workspaces/baseball-crawl/`") to spawn contexts and updates documentation to reflect the new hook scope.

CA consultation (2026-03-26) confirmed: agent definitions should NOT be overhauled with worktree constraints (those are dispatch-time context, not agent identity). Only hardcoded example paths to non-memory main-checkout locations need fixing. The spawn context and rules are the right places for the negative instruction.

## Acceptance Criteria
- [ ] **AC-1**: The universal implementer spawn context (implement skill Phase 2 Step 3) includes an explicit prohibition: agents must NOT use Write/Edit on paths starting with `/workspaces/baseball-crawl/` during dispatch -- that is the main checkout, not their worktree.
- [ ] **AC-2**: The PM and code-reviewer spawn contexts (implement skill Phase 2) include the same prohibition.
- [ ] **AC-3**: `.claude/rules/worktree-isolation.md` Hook Enforcement section documents both modes: (a) always-on denylist for `src/`, `tests/`, `migrations/`, `scripts/`, and (b) dispatch-active allowlist that blocks all main-checkout Write/Edit except `.claude/agent-memory/`.
- [ ] **AC-4**: CLAUDE.md's reference to the worktree-guard hook (Architecture section or wherever hook enforcement is described) says "all paths during dispatch" instead of "implementation paths" only.
- [ ] **AC-5**: No agent definition contains a hardcoded example path referencing `/workspaces/baseball-crawl/` for non-memory output locations. (Agent memory paths like `/workspaces/baseball-crawl/.claude/agent-memory/<agent>/` are correct and must NOT be changed.)

## Technical Approach
This is a multi-file text update across context-layer files. Read the implement skill for the spawn context sections, read the worktree-isolation rule, read CLAUDE.md's hook reference, and scan agent definitions for hardcoded non-memory example paths. Update each with the new language. The changes are advisory -- they describe the hook behavior from E-161-01 and add the negative instruction.
