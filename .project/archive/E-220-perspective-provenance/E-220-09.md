# E-220-09: Clean-Slate Rebuild Procedure

## Epic
[E-220: Perspective-Aware Data Architecture](epic.md)

## Status
`DONE`

## Description
After this story is complete, a documented rebuild procedure exists for wiping the database and `data/raw/` scouting cache, then rebuilding all data from scratch under the new perspective-aware architecture. The procedure includes verification SQL queries that the operator runs post-rebuild to confirm correctness.

## Context
The user authorized a clean-slate wipe. After all loader and schema changes are complete, the existing database and filesystem cache must be rebuilt. This story produces the documentation and verification queries. The operator (user) executes the actual rebuild since it requires Docker, credentials, and network access unavailable in the epic worktree.

## Acceptance Criteria
- [ ] **AC-1**: A documented rebuild procedure exists in `docs/admin/` covering: (a) DB wipe, (b) scouting cache wipe (`data/raw/{season}/scouting/`), (c) migration apply, (d) member team sync, (e) scouting sync for all tracked opponents, (f) report regeneration for any existing reports
- [ ] **AC-2**: Verification SQL queries are included that check: (a) no NULL `perspective_team_id` in any stat table, (b) `game_perspectives` has at least one row per game, (c) no scouting files in `data/raw/{season}/scouting/`, (d) season aggregates are consistent (no double-counting)
- [ ] **AC-3**: The procedure documents expected behavior for own-team data (`data/raw/{season}/teams/` is NOT wiped)
- [ ] **AC-4**: The procedure references the health check (`curl http://localhost:8001/health`) as a post-rebuild validation step

## Technical Approach
Write a rebuild procedure document in `docs/admin/`. Include step-by-step instructions and verification SQL templates. The operator runs the procedure manually. No automation script is required (per user preference against one-off CLI commands).
