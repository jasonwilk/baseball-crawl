# E-221-01: Shared real-schema fixture helper

## Epic

`DONE`

## Description
After this story is complete, `tests/conftest.py` will export a `load_real_schema(conn)` helper that loads `migrations/001_initial_schema.sql` into a SQLite connection with foreign key enforcement enabled. This replaces the ad-hoc pattern used in 22+ test files (each rolling its own `executescript` call) and unblocks the inline-fixture migration in Story 02.

## Context
Round 7 of E-220 discovered that 7 test files define `games`, `teams`, and other tables inline with columns that do not enforce production FK constraints. The missing FK enforcement was the enabling condition for bugs like round 8 R8-P1-1 slipping through test coverage — fixtures said "FK constraint failed" was impossible, production said otherwise.

22+ other test files already load the real schema via `executescript`, but no shared helper exists, so each rolls its own path resolution and forgets to enable foreign keys (a subtle `executescript` pitfall — see `.claude/rules/migrations.md`).

This story establishes the helper. Story 02 migrates the 7 inline-fixture files to use it. Story 03 adds a guardrail preventing future drift.

## Acceptance Criteria
- [ ] **AC-1**: Given `tests/conftest.py` exists and imports `sqlite3`, when a test calls `load_real_schema(conn)` on a fresh in-memory connection, then the connection has all tables from `migrations/001_initial_schema.sql` with their FK constraints enforced.
- [ ] **AC-2**: Given the helper is called, when the calling test attempts an INSERT that violates a foreign key constraint, then SQLite raises `sqlite3.IntegrityError` with a foreign key message (verifying `PRAGMA foreign_keys=ON` took effect, per Technical Notes).
- [ ] **AC-3**: Given the helper resolves its path to `001_initial_schema.sql`, when the test is run from any subdirectory, then the path resolution uses `Path(__file__).resolve().parents[N]` per the repo-root-resolution rule in CLAUDE.md — not cwd-relative or `sys.path` manipulation.
- [ ] **AC-4**: A new test `tests/test_conftest_helper.py` (or equivalent) exercises AC-1 and AC-2 directly. This test MUST pass.
- [ ] **AC-5**: The helper is under 30 lines, has a docstring referencing the `executescript` pragma-prepend requirement, and does no environment-variable reading, pytest fixture wrapping, or parameterization. Simplicity gate.

## Technical Approach
Per TN-1 in the epic. The critical detail is the pragma prepend: calling `conn.execute("PRAGMA foreign_keys=ON")` before `conn.executescript(sql)` does NOT enable FKs for the executescript run — `executescript` implicitly commits and resets connection state. The SQL string passed to `executescript` MUST start with `PRAGMA foreign_keys=ON;\n` for the enforcement to take effect.

Read `.claude/rules/migrations.md` (the "executescript() and PRAGMAs" section) before implementing.
