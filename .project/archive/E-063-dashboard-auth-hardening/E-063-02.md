# E-063-02: Root Route Redirect

## Epic
[E-063: Dashboard Auth Hardening](./epic.md)

## Status
`DONE`

## Description
After this story is complete, `GET /` will return a 302 redirect to `/dashboard`. This ensures visitors who navigate to the root URL land on the dashboard (or get redirected to login if unauthenticated), instead of seeing a 404.

## Context
Currently there is no route registered for `/`. The app serves routes at `/health`, `/auth/*`, `/dashboard/*`, and `/admin/*`. Visitors hitting the root get a FastAPI 404 JSON response. UXD recommended `/` -> `/dashboard` (rather than `/` -> `/auth/login`) because coaches bookmark the app and expect to land on the dashboard. The auth middleware handles the unauthenticated case by redirecting to `/auth/login`.

## Acceptance Criteria
- [ ] **AC-1**: Given an unauthenticated user, when they request `GET /`, then they receive a 302 redirect to `/dashboard`, and the subsequent middleware redirect sends them to `/auth/login`.
- [ ] **AC-2**: Given an authenticated user, when they request `GET /`, then they receive a 302 redirect to `/dashboard` and see the dashboard page.
- [ ] **AC-3**: A test verifies `GET /` returns 302 with `Location: /dashboard`.

## Technical Approach
Add a simple route in `src/api/main.py` (or a small routes file) that returns `RedirectResponse(url="/dashboard", status_code=302)` for `GET /`. The route should be registered before the dashboard router so it does not conflict.
