# E-063-03: Auth Page Template and Error Pages

## Epic
[E-063: Dashboard Auth Hardening](./epic.md)

## Status
`DONE`

## Description
After this story is complete, all auth-flow pages and error pages will use a minimal base template (`base_auth.html`) that shows the top bar but no bottom navigation. The existing 403 page will be reworked to fix its broken link and layout, and new 404 and 500 error pages will be added.

## Context
The current `base.html` always renders the fixed bottom nav with "Batting / Pitching / Games / Opponents" tabs. On auth pages (login, check email, etc.) and error pages, these links are meaningless -- the user is either unauthenticated or in an error state. UXD recommended creating `base_auth.html` with only the `bg-blue-900` top bar (wordmark, no nav links) and no bottom nav. All auth templates and error pages should extend this instead of `base.html`.

The existing `errors/forbidden.html` has two bugs: (1) it links to `/dashboard` which loops if the user lacks access, and (2) it uses inconsistent layout (`mt-8 text-center`) compared to the auth page centered-card pattern.

## Acceptance Criteria
- [ ] **AC-1**: `base_auth.html` exists with the `bg-blue-900` top bar (showing "LSB Baseball" wordmark) and NO bottom navigation.
- [ ] **AC-2**: All 6 auth templates (`login.html`, `check_email.html`, `verify_error.html`, `passkey_prompt.html`, `passkey_register.html`, `passkey_error.html`) extend `base_auth.html` instead of `base.html`.
- [ ] **AC-3**: `errors/forbidden.html` extends `base_auth.html`, uses the centered-card layout consistent with auth pages, and its primary CTA links to `/auth/login` (not `/dashboard`).
- [ ] **AC-4**: `errors/404.html` exists, extends `base_auth.html`, displays "Page Not Found" heading, and links to `/dashboard`.
- [ ] **AC-5**: `errors/500.html` exists, extends `base_auth.html`, displays "Something Went Wrong" heading, and links to `/dashboard`.
- [ ] **AC-6**: FastAPI exception handlers are registered for 404 and 500 status codes that render the new templates.
- [ ] **AC-7**: Tests verify that `GET /auth/login` response body does not contain the bottom nav markup (e.g., the "Batting" / "Pitching" tab links).

## Technical Approach
Create `base_auth.html` as a sibling to `base.html` in `src/api/templates/`. It should contain the same `<head>` setup (meta tags, Tailwind CDN, title block) and the top bar nav, but omit the bottom fixed nav. Update the 6 auth templates to change their `{% extends %}` line. Rework `errors/forbidden.html` and create `errors/404.html` and `errors/500.html`. Register custom exception handlers in `src/api/main.py` using FastAPI's `@app.exception_handler` for `404` and `500`.
