# E-144-01: Add Placeholder Email Allowlist and Tests

## Epic
[E-144: PII Scanner Placeholder Email Allowlist](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PII scanner will automatically skip common placeholder email addresses (like `your@email.com` and `user@domain.com`) without requiring manual `pii-ok` markers. A new exact-match allowlist in `pii_patterns.py` and a companion check function in `pii_scanner.py` will handle this, with full test coverage.

## Context
The PII scanner's RFC 2606 domain allowlist handles reserved domains like `example.com` and `.test`, but placeholder emails using real registerable domains (`email.com`, `domain.com`, `yourcompany.com`) are flagged as violations. This forces documentation authors to add `pii-ok` markers to every placeholder email, which is easy to forget and clutters the file. SE recommended an exact full-email allowlist as the conservative approach -- domain-level allowlisting would risk suppressing real leaks since these are real domains.

## Acceptance Criteria
- [ ] **AC-1**: `PLACEHOLDER_EMAILS` frozenset exists in `src/safety/pii_patterns.py` containing exactly the seed entries per Technical Notes TN-1 (no additional entries beyond TN-1's seed list)
- [ ] **AC-2**: `is_placeholder_email()` function exists in `src/safety/pii_scanner.py`, is imported from `pii_patterns` for the constant, and performs case-insensitive exact matching
- [ ] **AC-3**: `scan_file()` skips emails that match either `is_rfc2606_email()` or `is_placeholder_email()` -- the existing RFC 2606 check is unchanged
- [ ] **AC-4**: A `TestPlaceholderEmailAllowlist` test class exists in `tests/test_pii_scanner.py` per Technical Notes TN-3, covering: (a) each seeded email produces no violations, (b) a similar-but-not-listed email still gets flagged, (c) case-insensitive matching works
- [ ] **AC-5**: All existing PII scanner tests pass without modification

## Technical Approach
Three files change. `pii_patterns.py` gets a new `PLACEHOLDER_EMAILS` frozenset with documented entries. `pii_scanner.py` gets a new `is_placeholder_email()` function and an updated condition in `scan_file()`. `test_pii_scanner.py` gets a new test class mirroring the existing RFC 2606 test structure. See epic Technical Notes TN-1 through TN-4 for design constraints.
