# E-144-02: Update .env.example Auth Comments for Post-E-143 Behavior

## Epic
[E-144: PII Scanner Placeholder Email Allowlist](epic.md)

## Status
`DONE`

## Description
After this story is complete, `.env.example` will accurately describe the post-E-143 auth model for `ADMIN_EMAIL` and `DEV_USER_EMAIL`. The current comments are stale: `ADMIN_EMAIL` is described as "Informational only -- not read by the app" but `_require_admin()` now checks it as a bootstrap/fallback for admin access. `DEV_USER_EMAIL` describes login bypass but does not clarify whether the dev user gets admin privileges.

## Context
E-143-02b changed the admin access model: `_require_admin()` now grants admin access via EITHER `ADMIN_EMAIL` env var match OR `users.role = 'admin'` in the database. The `.env.example` comments were not updated to reflect this. The `ADMIN_EMAIL` comment is actively misleading (says the app does not read it). The `DEV_USER_EMAIL` comment is incomplete (does not document the admin access interaction). The implementer must read the auth code to verify exact behavior before writing the updated comments.

## Acceptance Criteria
- [ ] **AC-1**: The `ADMIN_EMAIL` comment block (lines 173-176 of `.env.example`) accurately describes that `_require_admin()` checks this value as a bootstrap/fallback for admin access, alongside the DB `role='admin'` check
- [ ] **AC-2**: The `DEV_USER_EMAIL` comment block (lines 153-156 of `.env.example`) accurately describes whether the dev bypass user receives admin access and under what conditions (e.g., only if the email also matches `ADMIN_EMAIL` or has `role='admin'`)
- [ ] **AC-3**: No functional code changes -- only `.env.example` comment text is modified

## Technical Approach
Read `src/api/routes/admin.py` (`_require_admin()`) and `src/api/auth.py` (`AuthMiddleware`, `_handle_dev_bypass()`) to verify the exact behavior of both env vars. Then update the `.env.example` comment blocks to match observed behavior. The implementer determines the correct wording based on what the code actually does.
