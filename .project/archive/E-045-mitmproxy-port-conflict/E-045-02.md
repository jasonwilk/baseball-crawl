# E-045-02: Update operator docs and scripts for port clarity

## Epic
[E-045: Resolve mitmproxy Port Conflict with Devcontainer](epic.md)

## Status
`DONE`

## Description
After this story is complete, the mitmproxy operator guide and proxy script include a troubleshooting entry for the devcontainer port-forwarding conflict, and the guide's port references are verified accurate. This ensures the operator understands that VS Code devcontainer forwarding must not duplicate Docker Compose LAN-bound ports, and knows how to diagnose the issue if it recurs.

## Context
The port conflict between VS Code devcontainer forwarding and Docker Compose was not obvious -- it required `lsof` investigation to discover. The mitmproxy guide's troubleshooting section should document this failure mode so future operators (or the same operator after a devcontainer config change) can diagnose it quickly. The `proxy.sh` script's status subcommand should also help by checking for port conflicts.

## Acceptance Criteria
- [x] **AC-1**: `docs/admin/mitmproxy-guide.md` troubleshooting section includes a new subsection titled "Traefik dashboard appears instead of mitm.it" (or equivalent) that explains the devcontainer port-forwarding conflict, its symptoms, diagnosis (`lsof -i :8080`), and fix (remove port from `forwardPorts`).
- [x] **AC-2**: `docs/admin/mitmproxy-guide.md` "Proxy unreachable from iPhone" troubleshooting entry adds a bullet about checking for devcontainer port-forwarding conflicts.
- [x] **AC-3**: `scripts/proxy.sh` `status` subcommand, in addition to `docker compose ps mitmproxy`, runs `lsof -i :8080 2>/dev/null || true` (or equivalent) and prints the output so the operator can see if multiple processes are bound to the proxy port.
