# E-002-R-01: Research: Discover game stats and player stats endpoints

## Epic
[E-002: GameChanger Data Ingestion Pipeline](epic.md)

## Status
`DONE`

## Objective
Determine whether the GameChanger API exposes endpoints for (1) per-game box score data (per-player batting and pitching lines for a specific game) and (2) per-player or per-team season aggregate statistics. The current API spec (`docs/gamechanger-api.md`) documents five endpoints, none of which return player-level stats. Without these endpoints, stories E-002-03 and E-002-04 cannot be implemented as written.

## Time Box
One focused session. If endpoints cannot be discovered from browser traffic capture within that session, document the gap and recommend alternative approaches.

## Research Questions
1. Does GameChanger expose a per-game stats endpoint (e.g., `/games/{game_id}/stats`, `/teams/{team_id}/games/{game_id}/box-score`, or similar) that returns per-player batting and pitching lines for a completed game?
2. Does GameChanger expose a season stats endpoint (e.g., `/teams/{team_id}/stats`, `/players/{player_id}/stats`, or similar) that returns aggregate batting/pitching stats for a season?
3. If neither exists as a dedicated endpoint, does the `game-summaries` response contain player-level stats that were not captured in the initial schema documentation? (The current schema shows only game-level scores and metadata.)
4. Are there any undiscovered endpoints visible in browser network traffic when viewing a game's box score page or a team's stats page on `web.gc.com`?

## Deliverables
- Updated `docs/gamechanger-api.md` with any newly discovered endpoints (full request/response documentation per existing format)
- If endpoints are found: unblock E-002-03 and E-002-04 with specific endpoint details
- If endpoints are NOT found: a written recommendation for alternative approaches (e.g., web scraping, deriving stats from play-by-play data, or accepting the data gap)

## Approach
1. The user navigates to a completed game's box score page on `web.gc.com` and captures the network traffic (browser DevTools > Network tab > Copy as cURL for all XHR requests)
2. The user navigates to a team's season stats page and captures network traffic similarly
3. api-scout analyzes the captured curl commands to identify new endpoints
4. Any discovered endpoints are documented in `docs/gamechanger-api.md` following the existing format
5. If the user cannot provide captures (e.g., the pages do not exist or do not make API calls), document that finding

## Dependencies
- **Blocked by**: None (requires user to provide curl captures from browser traffic)
- **Blocks**: E-002-03 (game stats crawler), E-002-04 (player season stats crawler), E-002-07a (game loader), E-002-07b (player stats loader)

## Findings

### Research Question 1: Per-game stats endpoint (box score)

**NOT FOUND.** The browser traffic capture from the team stats page did not reveal any endpoint returning per-player batting or pitching lines for a specific game. The `GET /teams/{team_id}/season-stats` endpoint confirmed below returns season-aggregate stats only -- not per-game breakdowns. The `game_stream.game_id` and `event_id` fields present in game-summaries responses may serve as path parameters in an undiscovered box-score endpoint, but no such endpoint was captured. E-002-03 remains BLOCKED.

### Research Question 2: Season stats endpoint

**CONFIRMED.** `GET /teams/{team_id}/season-stats` (documented in `docs/gamechanger-api.md`) returns comprehensive per-player season aggregate statistics for all players on a team. The response is a single JSON object (not paginated) containing:

- `stats_data.players`: per-player stats keyed by player UUID, each with `offense`, `defense`, and `general` sub-objects
- `stats_data.streaks`: hot/cold streak data for players currently on a streak (only `streak_H` observed; `streak_C` inferred but not confirmed)
- `stats_data.stats`: team aggregate totals in the same structure

**Batting stats confirmed available:** AVG, OBP, SLG, OPS, AB, PA, H, 1B, 2B, 3B, HR, TB, XBH, BB, SO, HBP, SB, CS, SB%, R, RBI, LOB, BABIP, BA/RISP, QAB, QAB%, BB/K, PS/PA, GB%, FLB%, LND%, SW%, SM%, HARD, WEAK, and many more (full list in API spec).

**Pitching/fielding stats confirmed available:** ERA, IP, WHIP, FIP, K/9 (as K/G), BB/9 (as BB/INN), K/BB, K/BF, BF, GS, SVO, BAA, BABIP, GB%, FLB%, FPS%, plus full fielding lines (PO, A, E, FPCT, outs by position, innings by position, catcher-specific stats).

**Important caveat -- player names absent:** The response keys players by UUID only. Names must be resolved by cross-referencing `GET /teams/{team_id}/players`.

**Important caveat -- defense merges pitching and fielding:** A player who both pitches and plays field positions will have all their pitching stats (ERA, IP, etc.) and fielding stats (PO, A, E, `IP:POS`) combined in one `defense` object. Use `GP:P` (games pitched) and `GP:F` (games played in field) to determine which role context applies.

**Accept header:** `application/vnd.gc.com.team_season_stats+json; version=0.2.0`

**gc-user-action value:** `data_loading:team_stats`

### Research Question 3: Player stats in game-summaries

**Not applicable.** The season-stats endpoint provides the player-level data needed for E-002-04. The game-summaries endpoint was not re-examined in this session -- it is confirmed to return only game-level scores and metadata, not per-player lines.

### Research Question 4: Undiscovered endpoints in browser traffic

The season-stats endpoint (`GET /teams/{team_id}/season-stats`) was the key discovery from this capture. No per-game box-score endpoint was visible in the team stats page traffic. To discover a box-score endpoint, the user must capture traffic from a **specific completed game's detail page** on `web.gc.com`, not the team stats summary page.

## Recommendation

### E-002-04 is UNBLOCKED

The `GET /teams/{team_id}/season-stats` endpoint provides everything needed for story E-002-04 (Crawl player season stats and write raw JSON). The endpoint returns full-season batting and pitching aggregates per player. The coaching-relevant stats are all present: AVG, OBP, SLG, OPS, K-rate, BB-rate, BABIP, BA/RISP, stolen base rate, ERA, WHIP, FIP, K/9, BB/9. **Update E-002-04 status from BLOCKED to TODO immediately.**

The raw file target for this endpoint (`data/raw/{season}/teams/{team_id}/stats.json`) maps cleanly to a single API call per team.

### E-002-03 remains BLOCKED

No per-game box-score endpoint was discovered. The `game_id` and `event_id` fields in game-summaries suggest a box-score endpoint likely exists, but it was not visible in the team stats page capture. **Recommended next step:** Ask the user to open a specific completed game on `web.gc.com`, open DevTools > Network, and copy as cURL any XHR requests that load while on the game's box score or stats tab. That capture should reveal the per-game endpoint if it exists.

### Stat abbreviations of coaching value

The following fields are directly relevant to scouting and game prep:

- **Batting (opponents):** AVG, OBP, SLG, OPS, BA/RISP, SB/CS (stolen base tendencies), K-rate (SO/PA), BB-rate (BB/PA), BABIP, hard/weak contact rates, lineup depth via PA counts
- **Pitching (opponents):** ERA, WHIP, K/9, BB/9, FIP, first-pitch strike rate (FPS%), BAA, BA/RISP allowed
- **Fielding:** Fielding percentage (FPCT), errors (E) by position via `outs-POS` fields
- **Catcher intelligence:** Stolen base success against (`SB:C%`), passed balls (`PB:C`) -- crucial for running game planning
