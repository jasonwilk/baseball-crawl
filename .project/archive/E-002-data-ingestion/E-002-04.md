# E-002-04: Crawl player season stats and write raw JSON

## Epic
[E-002: GameChanger Data Ingestion Pipeline](epic.md)

## Status
`DONE`

## Description
After this story is complete, the system will have a `PlayerStatsCrawler` that fetches season-aggregate stats for each configured team via `GET /teams/{team_id}/season-stats`, writing raw JSON to `data/raw/{season}/teams/{team_id}/stats.json`. One request per team; the response contains all players. This provides the season-level view: batting average, OBP, SLG, strikeout rate, and many more aggregate stats.

## Context
Game-by-game stats (E-002-03) tell you what happened in a specific game. Season stats tell you who a player *is*. Coaches need both: game stats for recent form, season stats for scouting and lineup decisions. This crawler targets the season aggregate endpoint.

The season-stats API response (`docs/gamechanger-api.md`, Schema: season-stats) does NOT include home/away or left/right splits directly. Those split columns in the E-003 schema will remain NULL from this crawl; splits must be computed from per-game data (boxscore or per-player stats endpoint) in a future story.

## Acceptance Criteria
- [ ] **AC-1**: Given a `config/teams.yaml` with one or more team entries and valid credentials, when `PlayerStatsCrawler(client, config).crawl_all()`, then for each team a single `GET /teams/{team_id}/season-stats` call is made and the response is written to `data/raw/{season}/teams/{team_id}/stats.json`.
- [ ] **AC-2**: Idempotency: stats files less than 24 hours old are skipped. Season stats change as games are played, so they are not treated as immutable like game files.
- [ ] **AC-3**: The full API response is written regardless of how many players it contains (including zero). If the response contains players who do not appear in the team's roster, the crawler does not filter or error -- it writes the raw response as-is. Player-to-roster reconciliation is the loader's concern (E-002-07b).
- [ ] **AC-4**: Given an API error for a team's season-stats call, the error is logged with the team ID and the crawl continues to the next team.
- [ ] **AC-5**: The endpoint is team-level: `GET /teams/{team_id}/season-stats` (confirmed by E-002-R-01). One request per team. The response contains all players for that team. Write the full response to `data/raw/{season}/teams/{team_id}/stats.json`.

## Technical Approach
`src/gamechanger/crawlers/player_stats.py`. Makes one request per team to `GET /teams/{team_id}/season-stats`.

**Endpoint details** (confirmed 2026-03-04, E-002-R-01):
- `GET /teams/{team_id}/season-stats` -- returns full per-player season batting, pitching, and fielding aggregates.
- Accept header: `application/vnd.gc.com.team_season_stats+json; version=0.2.0`
- `gc-user-action: data_loading:team_stats`
- Response is a single JSON object (not paginated). Players keyed by UUID under `stats_data.players`.
- Full schema in `docs/gamechanger-api.md` (Schema: season-stats).

`crawl_all()` returns a `CrawlResult` (defined in E-002-01, `src/gamechanger/crawlers/__init__.py`).

Reference files:
- `/workspaces/baseball-crawl/docs/gamechanger-api.md` -- season-stats endpoint spec and schema
- `/workspaces/baseball-crawl/docs/gamechanger-stat-glossary.md` -- stat abbreviation reference

## Notes
**Splits (home/away, L/R)**: The season-stats API response does NOT include home/away or left/right splits. The E-003 schema includes split columns in `player_season_batting` and `player_season_pitching`, but these will remain NULL from the season-stats endpoint data. Splits must be computed from per-game data in a future story (either from boxscore data via E-002-03/07a, or from the per-player stats endpoint which has per-game breakdowns).

**Player name resolution**: The season-stats response keys players by UUID only (no names). The loader (E-002-07b) must cross-reference with roster data (`GET /teams/{team_id}/players`) to resolve names. The crawler just stores the raw response.

**Note**: A separate per-player per-game stats endpoint also exists (`GET /teams/{team_id}/players/{player_id}/stats`) that provides game-by-game breakdowns, spray charts, and cumulative rolling stats. That endpoint is NOT used by this story (it requires one call per player). It is captured as IDEA-009 for future consideration.
