# E-008-R-02: Research: Agent Skills for Context Engineering

## Epic
[E-008: Intent/Context Layer Design](epic.md)

`DONE`

## Objective
Produce a structured research summary of the Agent Skills for Context Engineering repository at https://github.com/muratcankoylan/Agent-Skills-for-Context-Engineering. The summary must be specific enough that a reader can understand what skills exist, what they do, and what it would mean to install them into a project like baseball-crawl -- without visiting the repository.

## Time Box
One focused session. Do not spend more than two hours of elapsed research time. If some skills are thin on detail, note that as a finding. Prioritize breadth over depth -- understand what each category offers before going deep on any single skill.

## Research Questions
1. What are the five skill categories (Foundational, Architectural, Operational, Development, Cognitive) and what does each category address?
2. What is the "progressive disclosure" mechanism? How does it work and what does it require from the agent runtime?
3. What is BDI (Belief-Desire-Intention) mental state modeling? How is it applied in the Cognitive skill category and what would it change about how an agent approaches a task?
4. What does "installing" a skill look like in practice? Is there a standard file format, a placement convention, a manifest file?
5. What agent platforms or runtimes does the project support? Is it Claude Code-compatible?
6. How does this system interact with, or compare to, directory-level context files (like intent nodes or CLAUDE.md files)?
7. What are the three or four most immediately applicable skills to a project like baseball-crawl, and why?
8. What maintenance burden does adopting this system introduce? Who updates the skills, and how?

## Findings
- Repository is publicly accessible (12.8k stars, MIT license, cited in academic research) -- no paywall
- 13 skill modules across 5 categories; each is a SKILL.md file in skills/<name>/ with optional references/ and scripts/ subdirectories
- Claude Code-compatible via plugin marketplace: `/plugin install context-engineering-fundamentals@context-engineering-marketplace`
- The `.claude/skills/` directory in baseball-crawl is **currently empty** -- no existing content to reconcile
- Most immediately applicable skills: filesystem-context (validates baseball-crawl's file-based context approach), multi-agent-patterns (warns about orchestrator bottleneck / telephone game problem), context-fundamentals (foundational grounding for complex multi-file work)
- Progressive disclosure mechanism: skill names/descriptions in static context; full SKILL.md loaded on demand -- requires agent discipline to not pre-load all files
- BDI mental-states and advanced-evaluation skills are academically sophisticated and not applicable at baseball-crawl's scale
- Full research document: /.project/research/E-008-R-02-agent-skills-summary.md

## Recommendation
Partial fit. Two or three individual skills (filesystem-context, multi-agent-patterns, context-fundamentals) describe patterns baseball-crawl already uses or should formalize, and adopting them would be genuinely useful. However, wholesale plugin installation brings along academically sophisticated skills (BDI, advanced evaluation) with no practical application at this scale. Recommended adoption path: manually copy 2-3 specific SKILL.md files into .claude/skills/, adapt to baseball-crawl conventions, skip the plugin install.
