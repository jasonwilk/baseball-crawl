# E-008-01: Compare and Evaluate Architectural Options

## Epic
[E-008: Intent/Context Layer Design](epic.md)

## Status
`DONE`

## Description
After this story is complete, a structured comparison document will exist at `/.project/research/E-008-01-options-comparison.md` that evaluates all five architectural options for baseball-crawl's intent/context layer against a defined set of criteria. The document is the direct input to E-008-02 (the recommendation). It must be opinionated, not neutral -- each option should receive a clear verdict, not a hedge.

## Context
This story synthesizes the two research spikes (E-008-R-01 and E-008-R-02) into a structured evaluation. The five options to compare are defined in the epic's Background & Context section:

1. Status quo with refinements (improve existing CLAUDE.md and rules files)
2. Intent Node hierarchy (intent-systems.com approach)
3. Agent Skills adoption (muratcankoylan approach, installing skills into `.claude/skills/`)
4. Dedicated context-manager agent (new partner agent role)
5. Hybrid (Intent Node hierarchy for structure + skills for agent behaviors)

The evaluation must be grounded in baseball-crawl's actual current state, not in abstract theory. The evaluator (claude-architect) should audit the actual files in `.claude/`, `epics/`, and `CLAUDE.md` before writing the comparison, to ensure the "status quo" description is accurate.

## Acceptance Criteria
- [ ] **AC-1**: Given the two research summaries exist in `/.project/research/`, when this story executes, then the agent reads both documents in full before beginning the comparison.
- [ ] **AC-2**: Given the five architectural options listed in the epic, when the comparison is written, then all five options appear in the document with individual assessments.
- [ ] **AC-3**: Given the six evaluation criteria defined in the epic's Technical Notes (token efficiency, agent onboarding clarity, maintenance burden, compatibility with existing workflow contract, implementation complexity, reversibility), when the comparison is written, then all six criteria are applied to all five options in a consistent format.
- [ ] **AC-4**: Given baseball-crawl's scale constraint (small project, one operator, 7 agents, ~30-game season, no CI/CD pipeline), when options are assessed, then each assessment explicitly states whether the option is appropriately scaled or over-engineered for the project.
- [ ] **AC-5**: Given the comparison document, when E-008-02 reads it, then the document contains a clear ranking or shortlist of the top one or two options with explicit reasoning -- not a neutral "it depends" conclusion.
- [ ] **AC-6**: Given the existing `.claude/` directory structure, when the "status quo" option is assessed, then the assessment is based on the actual files that exist, not on assumed or idealized state.

## Technical Approach
1. Read `/.project/research/E-008-R-01-intent-systems-summary.md` and `/.project/research/E-008-R-02-agent-skills-summary.md` in full
2. Audit the actual current state of baseball-crawl's context layer: read `CLAUDE.md`, `.claude/rules/`, `.claude/agents/`, `.claude/skills/`, `.claude/agent-memory/` directory listings
3. For each of the five options, write a section containing: description, how it would be implemented in baseball-crawl, and an evaluation scorecard against the six criteria
4. Scorecard format: each criterion rated as Strong / Adequate / Weak with a one-sentence justification
5. Write a "Verdict" paragraph for each option: overall fit, who it is right for, why baseball-crawl is or is not that case
6. Close the document with a Shortlist section naming the top one or two options and why

## Notes
- This is a research and design story. No implementation code is written.
- The six evaluation criteria are defined in the epic's Technical Notes section -- do not invent different criteria.
- "Reversibility" is a first-class criterion because baseball-crawl is still early. An approach that is hard to undo carries meaningful cost.
- The target reader of this document is the team reviewing E-008-02's recommendation. Write it so a reader who has NOT read the research spikes can follow the reasoning.
