# E-044-01: Epic template: Dispatch Team section

## Epic
[E-044: Workflow Trigger Phrases](epic.md)

## Status
`DONE`

## Description
After this story is complete, the epic template includes a Dispatch Team section that allows epic authors to declare which agents should be on the dispatch team. This metadata enables the implement workflow skill to read team composition directly from the epic file rather than requiring the PM to infer it from story domains each time.

## Context
The "implement E-NNN" workflow needs to know which agents to spawn. Today, the PM infers this from the Agent Selection table in dispatch-pattern.md by examining each story's domain. Adding a Dispatch Team section to the epic template makes this explicit at planning time. This story must land before E-044-04 (implement skill) because that skill reads the section.

## Acceptance Criteria
- [ ] **AC-1**: The file `/.project/templates/epic-template.md` contains a `## Dispatch Team` section placed between the `## Stories` table and `## Technical Notes`.
- [ ] **AC-2**: The Dispatch Team section includes a comment explaining its purpose: which agents should be on the dispatch team, PM is always included automatically, and the section is advisory (PM still makes final routing decisions).
- [ ] **AC-3**: The Dispatch Team section shows an example format as a bulleted list of agent names (e.g., `- software-engineer`, `- data-engineer`).
- [ ] **AC-4**: The file `/.claude/rules/dispatch-pattern.md` "Agent Selection for Dispatch" section includes a paragraph stating that epics may include a Dispatch Team section, PM should prefer it when present, and fall back to the routing table when it is absent or empty.

## Technical Approach
1. Read `/.project/templates/epic-template.md`.
2. Add a `## Dispatch Team` section after the Stories table and before Technical Notes.
3. Read `/.claude/rules/dispatch-pattern.md`.
4. Add a paragraph to the "Agent Selection for Dispatch" section referencing the Dispatch Team metadata.
