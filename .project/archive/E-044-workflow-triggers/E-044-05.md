# E-044-05: CLAUDE.md Workflows entries

## Epic
[E-044: Workflow Trigger Phrases](epic.md)

## Status
`DONE`

## Description
After this story is complete, the CLAUDE.md Workflows section contains entries for all three new trigger phrases (spec review, review epic, implement), and the team lead can detect and route each phrase to the correct skill file. This is the integration story that wires the skill files (created in E-044-02, E-044-03, E-044-04) into the team lead's detection layer.

## Context
The ingest-endpoint workflow has a single entry in CLAUDE.md Workflows that tells the team lead when to load the skill file. Each of the three new workflows needs the same treatment. This story is last because it depends on all three skill files existing.

## Acceptance Criteria
- [ ] **AC-1**: The `## Workflows` section in `/workspaces/baseball-crawl/CLAUDE.md` contains a bullet for **Spec review** with trigger phrases and a reference to `.claude/skills/spec-review/SKILL.md`.
- [ ] **AC-2**: The `## Workflows` section contains a bullet for **Review epic** with trigger phrases and a reference to `.claude/skills/review-epic/SKILL.md`.
- [ ] **AC-3**: The `## Workflows` section contains a bullet for **Implement** with trigger phrases (including the "and review" modifier) and a reference to `.claude/skills/implement/SKILL.md`.
- [ ] **AC-4**: Each Workflows entry follows the same format as the existing "Ingest endpoint" entry: bold workflow name, trigger phrase examples in the description, skill file path, and a brief explanation of what the workflow does.
- [ ] **AC-5**: The three new entries are ordered logically after the existing "Ingest endpoint" entry: Implement, Spec review, Review epic (or alphabetical -- consistent with the existing style).

## Technical Approach
1. Read `/workspaces/baseball-crawl/CLAUDE.md` and locate the `## Workflows` section.
2. Add three entries following the "Ingest endpoint" format.
3. Each entry should be concise (1-3 sentences) -- the skill file has the full procedure.
