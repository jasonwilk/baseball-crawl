# E-044-02: Spec review workflow skill

## Epic
[E-044: Workflow Trigger Phrases](epic.md)

## Status
`DONE`

## Description
After this story is complete, a skill file at `.claude/skills/spec-review/SKILL.md` defines the full workflow for running a codex spec review on an epic and having domain experts review the findings with PM. The team lead loads this skill when the user says "spec review" and follows its procedure.

## Context
Today spec review is an optional step the PM can request manually (documented in the PM agent definition). This skill formalizes it as a trigger-phrase workflow that the team lead recognizes and executes. The workflow runs the existing `./scripts/codex-spec-review.sh` script, then spawns a review team of PM + relevant domain experts to triage the findings.

## Acceptance Criteria
- [ ] **AC-1**: The file `.claude/skills/spec-review/SKILL.md` exists and follows the ingest-endpoint structural pattern: Activation Triggers, Purpose, Prerequisites, Phases, Workflow Summary (ASCII diagram), Edge Cases, Anti-Patterns.
- [ ] **AC-2**: Activation Triggers section lists: "spec review", "spec review E-NNN", "review the spec for E-NNN", "run spec review on E-NNN", and similar natural variations.
- [ ] **AC-3**: Prerequisites section requires: (a) the target epic directory exists and contains `epic.md`, (b) if no epic ID is specified, ask the user which epic to review.
- [ ] **AC-4**: Phase 1 instructs the team lead to run `./scripts/codex-spec-review.sh <epic-dir>` via Bash. The team lead captures the codex output.
- [ ] **AC-5**: Phase 2 instructs the team lead to spawn PM as a teammate, providing the codex output and the epic directory path. PM determines which domain experts are relevant to the epic (by reading the epic's domain and stories), spawns them as teammates, and leads a review of the codex findings. PM decides which findings to refine into the epic, which to dismiss, and which to defer -- then updates the affected epic and story files accordingly.
- [ ] **AC-6**: The skill explicitly states that PM owns all epic file updates (implementing agents and domain experts provide input but do not modify epic/story files directly).
- [ ] **AC-7**: Edge Cases section covers: (a) codex not installed, (b) codex returns no findings, (c) epic is already ACTIVE (review findings may require story updates, not just epic updates).
- [ ] **AC-8**: Anti-Patterns section includes: (a) do not skip Phase 2 -- codex findings need human judgment, (b) do not auto-apply codex suggestions without team review.

## Technical Approach
1. Create `.claude/skills/spec-review/SKILL.md` following the ingest-endpoint structure at `/.claude/skills/ingest-endpoint/SKILL.md`.
2. Phase 1 is the team lead running the script directly (team lead has Bash access).
3. Phase 2 is a team-based review: team lead spawns PM, PM spawns domain experts.
4. Keep the skill focused on the workflow procedure, not on codex internals.
