# E-052-04: Migrate report scripts to session-aware mode

## Epic
[E-052: Proxy Data Lifecycle](epic.md)

## Status
`DONE`

## Description
After this story is complete, `scripts/proxy-endpoints.sh` and `scripts/proxy-report.sh` read from session directories instead of hardcoded flat files. They default to the latest session (via the `current` symlink) and accept flags for specific sessions, all sessions, or only unreviewed sessions.

## Context
The existing report scripts read from `proxy/data/endpoint-log.jsonl` and `proxy/data/header-report.json`. After E-052-02 routes addon output into session directories, these scripts need to find the data in its new location. This is the user-facing payoff: the operator can now ask "show me only what's new."

## Acceptance Criteria
- [ ] **AC-1**: `proxy-endpoints.sh` with no arguments reads from `proxy/data/current/endpoint-log.jsonl` (the `current` symlink). If no current session exists, prints an error and exits.
- [ ] **AC-2**: `proxy-endpoints.sh --session <id>` reads from `proxy/data/sessions/<id>/endpoint-log.jsonl`. Errors if the session or file does not exist.
- [ ] **AC-3**: `proxy-endpoints.sh --all` aggregates `endpoint-log.jsonl` from all sessions (sorted chronologically). Deduplication logic (method+path grouping) operates across all files.
- [ ] **AC-4**: `proxy-endpoints.sh --unreviewed` aggregates `endpoint-log.jsonl` from all sessions where `session.json` has `reviewed: false`. If all sessions are reviewed, prints "All sessions reviewed."
- [ ] **AC-5**: `proxy-report.sh` with no arguments reads from `proxy/data/current/header-report.json`. If no current session exists, prints an error and exits.
- [ ] **AC-6**: `proxy-report.sh --session <id>` reads from `proxy/data/sessions/<id>/header-report.json`. Errors if the session or file does not exist.
- [ ] **AC-7**: `proxy-report.sh --all` reads the header report from the most recent session (by session ID sort order) that contains a `header-report.json` file and has `status: "closed"` in its `session.json`. Header reports are snapshots, not aggregatable -- "all" means "latest closed session with data." If no closed session with a header report exists, prints an error.
- [ ] **AC-8**: Both scripts print usage when called with `--help`.
- [ ] **AC-9**: Both scripts handle the case where a session directory exists but the expected data file does not (e.g., session was started but no GC traffic flowed). Print a clear message rather than a jq/awk error.

## Technical Approach
- Add argument parsing at the top of each script (`getopts` or manual `case` parsing, consistent with existing style).
- For `--unreviewed` in `proxy-endpoints.sh`: iterate `proxy/data/sessions/*/session.json`, filter by `reviewed: false` using `jq`, then `cat` those sessions' `endpoint-log.jsonl` files together before piping into the existing dedup logic.
- For `--all` in `proxy-endpoints.sh`: `cat proxy/data/sessions/*/endpoint-log.jsonl` (glob sorts alphabetically, which is chronological given the timestamp naming).
- For `proxy-report.sh --all`: iterate session dirs in reverse chronological order, find the first one with `status: "closed"` in `session.json` AND a `header-report.json` file present, then display it.
- Keep the existing output formatting logic unchanged -- only the input source selection changes.

## Notes
- The `--unreviewed` flag is the key user-facing feature: "show me what I haven't looked at yet."
- Header reports are not aggregatable (each is a point-in-time snapshot), so `--all` for `proxy-report.sh` just shows the latest. This is a deliberate simplification.
