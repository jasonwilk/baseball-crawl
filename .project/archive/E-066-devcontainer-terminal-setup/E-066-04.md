## Description
After this story is complete, CLAUDE.md documents the dual-shell contract (ZSH for interactive use, Bash for automation and agent tooling), the three terminal operating modes (Solo, Coordinated, Heavy), and the host-terminal workflow for heavy Agent Teams sessions. The context layer accurately reflects the post-E-066 shell environment so agents and future contributors understand the intentional separation.

## Context
Switching the default interactive shell to ZSH while keeping Bash for automation creates a dual-shell contract that must be maintained long-term. All hook scripts (statusline, PII check, epic archive check) are bash and must remain so. Claude Code's Bash tool runs bash regardless of the user's interactive shell. The `post-create-env.sh` dual-injection pattern (into both `.bashrc` and `.zshrc`) is a convention that future changes must preserve. This context-layer awareness prevents future contributors or agents from accidentally breaking the contract (e.g., assuming all shells are bash, or migrating hooks to zsh).

The consensus plan also identified that split-pane Agent Teams does NOT work in VS Code's integrated terminal. The recommended advanced path is iTerm2 + tmux on the Mac host. The three-mode taxonomy (Solo, Coordinated, Heavy) is a useful decision aid from the discussion.

- [ ] **AC-4**: CLAUDE.md contains a "Terminal Modes" section (or similar) describing the three operating modes: Solo (VS Code, ZSH, no teams), Coordinated (VS Code, Agent Teams in-process), Heavy (host terminal + tmux + devcontainer attach)

## Technical Approach
This is a context-layer documentation story. Two new sections in CLAUDE.md: one for the dual-shell contract (near the existing "Code Style" or "Architecture" section), one for terminal modes (near the existing "Proxy Boundary" section since both deal with the host/container split). Keep both concise -- the operator is technical and does not need hand-holding. The mode taxonomy should be a quick-reference table or list, not a narrative.

The current CLAUDE.md has no shell-related documentation beyond the statusline dependencies line (`Dependencies: bash, jq, git`). The existing "Proxy Boundary" section is the natural anchor point for terminal mode documentation.

See epic Technical Notes for the dual-shell contract details, mode definitions, and tmux session convention.
