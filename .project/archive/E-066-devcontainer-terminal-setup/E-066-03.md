# E-066-03: tmux Installation and Config

## Epic
[E-066: Devcontainer Terminal Setup](epic.md)

## Status
`DONE`

## Description
After this story is complete, tmux is installed in the devcontainer and configured with a minimal project-appropriate config file. The config addresses known VS Code integrated terminal compatibility issues (prefix key conflict, TERM variable, resize behavior). tmux is available for optional use but is not required for any workflow.

## Context
Both proposals agreed that tmux should be installed early (low cost) but adopted gradually. SE identified specific gotchas with tmux inside VS Code's integrated terminal: the default `Ctrl+B` prefix conflicts with VS Code, and the TERM variable and resize behavior need explicit configuration. This story installs tmux and provides a working config without making tmux part of any mandatory workflow.

## Acceptance Criteria
- [ ] **AC-1**: `devcontainer.json` apt-packages feature includes `tmux` in its packages list
- [ ] **AC-2**: A tmux config file exists in `.devcontainer/` and is copied to `~/.tmux.conf` via postCreateCommand
- [ ] **AC-3**: The tmux config changes the prefix key from the default `Ctrl+B` to avoid the VS Code keybinding conflict
- [ ] **AC-4**: The tmux config sets `default-terminal` to `tmux-256color` with terminal overrides for true color support
- [ ] **AC-5**: The tmux config enables mouse support
- [ ] **AC-6**: The tmux config uses 1-based indexing for windows and panes
- [ ] **AC-7**: The tmux config binds `|` for horizontal split and `-` for vertical split, both preserving the current working directory
- [ ] **AC-8**: The tmux config enables aggressive resize
- [ ] **AC-9**: Existing tests pass (`pytest` -- no regressions)

## Technical Approach
Two changes needed. First, add `tmux` to the apt-packages feature list in `devcontainer.json`. Second, create a minimal `.devcontainer/tmux.conf` file and add a `cp` command to postCreateCommand to install it as `~/.tmux.conf`.

See epic Technical Notes for the SE-recommended tmux settings (prefix key, TERM variable, aggressive resize, mouse support) and the tmux session convention (`baseball`).
