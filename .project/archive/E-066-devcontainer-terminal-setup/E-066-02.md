# E-066-02: VS Code Terminal Profiles

## Epic
[E-066: Devcontainer Terminal Setup](epic.md)

## Status
`DONE`

## Description
After this story is complete, the VS Code integrated terminal dropdown shows clearly named terminal profiles with distinct icons and colors. ZSH is the default profile. Bash is available as a named fallback. The operator can instantly identify which terminal is which without guessing.

## Context
The consensus plan identified VS Code terminal profiles as the fastest fix for terminal confusion -- lower risk and lower learning cost than tmux. SE confirmed that terminal profiles work correctly in devcontainers when placed under `customizations.vscode.settings` in `devcontainer.json`. This story depends on E-066-01 (ZSH must be the default shell before we set it as the default terminal profile).

## Acceptance Criteria
- [ ] **AC-1**: `devcontainer.json` `customizations.vscode.settings` includes `terminal.integrated.profiles.linux` with at least two profiles: a ZSH profile and a Bash profile
- [ ] **AC-2**: Each profile has `path`, `icon`, and `color` fields set
- [ ] **AC-3**: `terminal.integrated.defaultProfile.linux` is set to the ZSH profile name
- [ ] **AC-4**: The ZSH profile uses `overrideName: true` so terminal tabs show the profile name instead of the process name
- [ ] **AC-5**: The Bash profile uses `overrideName: true` for the same reason
- [ ] **AC-6**: Existing `customizations.vscode.settings` content (port attributes) is preserved

## Technical Approach
This is a JSON configuration change in `devcontainer.json`. The terminal profiles and default profile setting go under the existing `customizations.vscode.settings` object alongside the existing `remote.portsAttributes` configuration. No new files are created. Profile names should be descriptive and short (e.g., "ZSH" and "Bash"). Colors should be visually distinct. Icons should be meaningful (e.g., `terminal` for ZSH, `tools` for Bash).

See the consensus plan in `DISCUSSION-terminal-setup.md` for the naming convention discussion. The Codex proposal suggested names like `main:zsh` and `ops:bash` -- the implementer should choose clear, simple names.
