# E-096-02: Update Dispatch Review Loop in dispatch-pattern and implement skill

## Epic
[E-096: Resolve All Review Findings During Stories](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the dispatch review loop in `dispatch-pattern.md`, `implement/SKILL.md`, and `CLAUDE.md` will implement the resolution-first model: when the code-reviewer returns findings, the main session triages ALL findings (not just MUST FIX) and routes accepted SHOULD FIX items to the implementer alongside MUST FIX items. No findings are deferred to epic History. Every finding reaches a terminal state of FIXED or DISMISSED during the story.

## Context
The current dispatch loop explicitly instructs the main session to ignore SHOULD FIX findings: "SHOULD FIX findings from the reviewer are recorded in the epic's History section during closure -- they are NOT relayed to the implementer" (implement SKILL.md line 214) and anti-pattern 10 in implement SKILL.md says "Do not relay SHOULD FIX findings to implementers." CLAUDE.md's Agent Ecosystem section reinforces this: "SHOULD FIX findings are recorded in epic History during closure, not relayed to implementers." dispatch-pattern.md is silent on SHOULD FIX handling -- it needs a small addition. This story replaces the deferral pattern with an active triage step where the main session decides fix-or-dismiss for each SHOULD FIX item.

This story depends on E-096-01 because the code-reviewer's updated output format (from that story) defines the contract that this story's triage step consumes.

## Technical Approach
Three files need coordinated updates.

**`/.claude/rules/dispatch-pattern.md`** -- Currently silent on SHOULD FIX handling. Needs a small addition to Step 7: after receiving the reviewer's findings, the main session triages ALL findings (not just MUST FIX) before any merge/status decisions. Fold into the existing APPROVED/NOT APPROVED handling -- no new step number needed. Keep this change minimal.

**`/.claude/skills/implement/SKILL.md`** -- Contains the bulk of the deferral language (APPROVED path line 214, NOT APPROVED path line 216, Workflow Summary diagram "SHOULD FIX -> epic History" annotation, anti-pattern 10). All need rewriting to reflect the triage model. This is the primary file for structural changes.

**`CLAUDE.md`** -- Agent Ecosystem code-reviewer description (line 450) says "SHOULD FIX findings are recorded in epic History during closure, not relayed to implementers." Update to reflect the resolution-first model.

The triage heuristics from the epic's Technical Notes section should land in one of the first two files. CA should decide the right home based on where the triage step is most naturally described.

## Notes

- The circuit breaker (2 rounds max) is unchanged.
- The APPROVED/NOT APPROVED verdict from the reviewer is unchanged -- APPROVED still means no MUST FIX items. What changes is that SHOULD FIX items in an APPROVED review are also triaged (not silently recorded in History).
