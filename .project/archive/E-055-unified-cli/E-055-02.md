# E-055-02: Credential Commands (`bb creds`)

## Epic
[E-055: Unified Operator CLI](epic.md)

## Status
`DONE`

## Description
After this story is complete, the operator can manage GameChanger credentials through `bb creds refresh` and `bb creds check`. These commands wrap the existing `scripts/refresh_credentials.py` and `scripts/check_credentials.py` functionality with consistent Typer interfaces.

## Context
Credential management is the most frequent operator task -- credentials expire every 14 days and must be refreshed from a browser curl capture. Having these under a discoverable `bb creds` group means the operator does not need to remember file paths.

## Acceptance Criteria
- [ ] **AC-1**: `src/cli/creds.py` exists and defines a Typer sub-app mounted on the main app as `bb creds`.
- [ ] **AC-2**: `bb creds refresh` with no flags reads from `secrets/gamechanger-curl.txt` (matching current default behavior), extracts credentials, and writes to `.env`.
- [ ] **AC-3**: `bb creds refresh --curl "curl ..."` accepts an inline curl command (matching `--curl` flag of `refresh_credentials.py`).
- [ ] **AC-4**: `bb creds refresh --file /path/to/file` accepts a file path (matching `--file` flag of `refresh_credentials.py`).
- [ ] **AC-5**: `bb creds check` validates credentials by calling GET /me/user and prints the result with appropriate exit code (0=valid, 1=expired, 2=missing). Supports `--profile` flag to validate a specific profile (web or mobile) or all profiles (per E-053's profile-aware interface).
- [ ] **AC-6**: `bb creds --help` lists both `refresh` and `check` with descriptions.
- [ ] **AC-7**: After a successful `bb creds refresh`, the command prints the list of credential key names written (e.g., `GC_TOKEN_WEB`, `GC_DEVICE_ID_WEB` -- never their values) and the total count of keys in `.env`. This confirms the operation succeeded and which profile was updated.
- [ ] **AC-8**: Tests using `CliRunner` verify argument mapping for `refresh` (default, --curl, --file) and `check` (mocked API call). Include a test that `refresh` output contains key names but not values.
- [ ] **AC-9**: All existing tests continue to pass.

## Technical Approach
1. Create `src/cli/creds.py` with a `typer.Typer(help="Credential management.")` sub-app.
2. `refresh` command imports `parse_curl` and `merge_env_file` from `src/gamechanger/credential_parser.py` directly and reimplements the thin orchestration (read source, parse, merge, print summary). **Do NOT call `refresh_credentials.main()`** -- it uses `argparse.parse_args()` internally and cannot be invoked programmatically. The CLI replaces the argparse layer while reusing the library functions.
3. `check` command imports and calls `check_credentials()` from `scripts/check_credentials.py`. This function accepts `profile: str | None = None` and returns `(exit_code, message)`.
4. The sub-app mount in `src/cli/__init__.py` is pre-wired by E-055-01 -- do not modify `__init__.py`.
5. Tests mock the credential parser and API calls; do not re-test business logic.

## Notes
- The `refresh` command's `--curl` and `--file` flags are mutually exclusive in the current script. Typer does not natively support mutually exclusive options, so the implementation should validate this manually and print a clear error if both are provided.
- **`merge_env_file` signature**: `merge_env_file(env_path: str, ...)` takes a `str`, not a `Path`. The CLI must cast with `str()` if passing a `Path` object.
- **Important**: `refresh_credentials.py` has no `run()` function. Its `main()` calls `argparse.parse_args()` and cannot be invoked programmatically. The CLI must reimplement the thin orchestration using the library functions from `credential_parser.py`: read curl command from the appropriate source, call `parse_curl()`, call `merge_env_file()`, print summary. The default curl file path is `secrets/gamechanger-curl.txt` (relative to project root). The .env file path is `.env` (project root).
