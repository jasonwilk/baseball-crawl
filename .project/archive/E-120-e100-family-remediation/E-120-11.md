# E-120-11: Dashboard back-link navigation fixes

## Epic
[E-120: E-100 Family Code Review Remediation](epic.md)

## Status
`DONE`

## Description
After this story is complete, dashboard back links will use explicit URL construction with team context instead of fragile `javascript:history.back()` calls, and game detail back links will preserve the originating team_id so users return to the correct team view.

## Context
Two related navigation bugs in dashboard templates: (1) `player_profile.html:21` uses `javascript:history.back()` which fails silently when a user navigates directly to the page (bookmark, shared link) — the back action goes nowhere or leaves the dashboard entirely. (2) `game_detail.html:21` back links lose team_id context, so users navigating back from a game detail page may land on an ambiguous or wrong team view.

## Acceptance Criteria
- [ ] **AC-1**: `player_profile.html` back link uses an explicit URL (e.g., linking to the team roster or batting stats page for the player's team) instead of `javascript:history.back()`. The link works correctly when the page is accessed directly via bookmark or shared URL.
- [ ] **AC-2**: `game_detail.html` back link includes the originating team_id in the URL so users return to the correct team's schedule or game list.
- [ ] **AC-3**: All existing tests pass

## Technical Approach
Both fixes are template-level: replace `javascript:history.back()` and context-less back links with explicit URL construction using the team_id already available in the template context. Check for any other `history.back()` usages across dashboard templates and fix those too if found.
