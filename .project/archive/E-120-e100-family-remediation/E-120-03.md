# E-120-03: Admin validation and UX hardening

## Epic
[E-120: E-100 Family Code Review Remediation](epic.md)

## Status
`DONE`

## Description
After this story is complete, `membership_type` will be validated against a whitelist before DB writes (preventing data corruption), and the `connect_opponent_confirm` GET handler will reject already-resolved links (preventing a confusing UX flow where the form renders but submission fails).

## Context
Two admin route issues: (1) `membership_type` is accepted as-is from form input and written directly to the database without validation. Classification has `_VALID_CLASSIFICATIONS` but membership_type has no equivalent. (2) The POST handler for manual opponent linking calls `_check_already_resolved()` but the GET handler does not, so a user can load the confirmation form for an already-resolved link, fill it out, and then get a confusing 400 error on submit.

## Acceptance Criteria
- [ ] **AC-1**: Given `src/api/routes/admin.py`, a `_VALID_MEMBERSHIP_TYPES = {"member", "tracked"}` constant exists near `_VALID_CLASSIFICATIONS` (line ~78), and is used to validate `membership_type` before the DB write in both `confirm_team_submit` (line ~1096) and `edit_team_submit` (line ~1193). Invalid values return 400 (not silently coerced to a default).
- [ ] **AC-2**: Given the `connect_opponent_confirm` GET handler at `admin.py:1486`, when the link already has a `public_id`, then the handler returns 400 — matching the POST handler's behavior (line ~1577). The `_check_already_resolved(link)` call is added between the link fetch (line ~1501) and the URL parsing (line ~1505).
- [ ] **AC-3**: A test in `tests/test_admin.py` verifies that submitting an invalid `membership_type` value to `confirm_team_submit` returns a 400 response
- [ ] **AC-4**: A test in `tests/test_admin.py` verifies that GET `/opponents/{link_id}/connect/confirm` for an already-resolved link returns 400
- [ ] **AC-5**: All existing tests pass

## Technical Approach
1. For AC-1: Define `_VALID_MEMBERSHIP_TYPES = {"member", "tracked"}` near `_VALID_CLASSIFICATIONS` at admin.py:78. In `confirm_team_submit`, validate after `_normalize_confirm_inputs` returns. In `edit_team_submit`, validate before the `_update_team()` call. Return a 400 HTMLResponse for invalid values.
2. For AC-2: Add `_check_already_resolved(link)` call in the GET handler at admin.py:1486, between line 1503 (link null check) and line 1505 (URL parsing). The helper already exists at line 1540.
