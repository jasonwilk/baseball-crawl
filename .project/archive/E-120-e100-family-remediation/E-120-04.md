# E-120-04: Opponent count query and db.py consistency fixes

## Epic
[E-120: E-100 Family Code Review Remediation](epic.md)

## Status
`DONE`

## Description
After this story is complete, the team list will show correct opponent counts (excluding hidden links), `save_manual_opponent_link` will have exception handling consistent with other db.py write functions, and its docstring will no longer reference a nonexistent column.

## Context
Three related issues in the database layer: (1) The `opponent_count` subquery in `_get_all_teams_flat` (admin.py:322-326) counts all opponent_links rows including hidden ones, while the opponents page filters hidden links â€” causing a count mismatch. (2) `save_manual_opponent_link` (db.py:1005) has no try/except around its database operation, unlike every other write function in db.py. (3) The same function's docstring says "Callers must set updated_at explicitly" but `opponent_links` has no `updated_at` column.

## Acceptance Criteria
- [ ] **AC-1**: Given `_get_all_teams_flat` in `admin.py:300`, the `opponent_count` subquery (line ~325) includes `AND ol.is_hidden = 0` so hidden links are excluded from the count displayed on the team list page
- [ ] **AC-2**: Given `save_manual_opponent_link` in `db.py:1005`, the `conn.execute` + `conn.commit` block is wrapped in `try: ... except sqlite3.Error: logger.exception(...); raise` â€” matching the pattern used by `disconnect_opponent_link` (db.py:1031) and other write functions in the file
- [ ] **AC-3**: The `save_manual_opponent_link` docstring at db.py:1006 no longer references `updated_at` (the `opponent_links` table has no such column; the line "Callers must set updated_at explicitly" is removed or corrected)
- [ ] **AC-4**: A test in `tests/test_admin.py` verifies that when both hidden and non-hidden opponent_links rows exist for a team, the `opponent_count` in the team list response only counts non-hidden rows
- [ ] **AC-5**: All existing tests pass

## Technical Approach
1. For AC-1: Add `AND ol.is_hidden = 0` to the WHERE clause of the subquery at admin.py:325.
2. For AC-2: Wrap the `conn.execute` + `conn.commit` block in `save_manual_opponent_link` with `try: ... except sqlite3.Error: logger.exception(...); raise`.
3. For AC-3: Remove the sentence "Callers must set updated_at explicitly (no trigger on this table)." from the docstring.
