# E-120-07: Dashboard pitching HR query and template display fixes

## Epic
[E-120: E-100 Family Code Review Remediation](epic.md)

## Status
`DONE`

## Description
After this story is complete, the game detail pitching box score will display the `hr` column correctly (not blank), tied games will display "T" instead of incorrectly labeling both teams "L", and opponent detail score columns will display "-" instead of literal "None" text when scores are null.

## Context
Three dashboard display bugs found in the second code review: (1) The pitching query in `db.py:309-322` does not SELECT the `hr` column, but `game_detail.html:129` and `team_pitching.html:62` render `{{ pitcher.hr }}`. The `player_game_pitching` table has an `hr` column and the `_PlayerPitching` dataclass in `game_loader.py:159` defines it -- the query simply omits it. (2) `_compute_wl` in `dashboard.py:270-289` returns "L" for both teams when scores are tied (e.g., suspended/called games). It should return "T". (3) The opponent detail template renders literal "None-None" when game scores are null, instead of displaying a placeholder.

## Acceptance Criteria
- [ ] **AC-1**: The pitching query in `db.py` (the `pitching_query` variable in the game detail function) includes `COALESCE(pgp.hr, 0) AS hr` in its SELECT list, matching the pattern of the other stat columns
- [ ] **AC-2**: Verify that the season pitching query path serving `team_pitching.html` already includes `hr` (confirmed: `db.py:174` has `COALESCE(psp.hr, 0) AS hr`). No code change needed — note verification in commit message
- [ ] **AC-3**: `_compute_wl` in `dashboard.py` returns `"T"` when `home_score == away_score` (both non-null)
- [ ] **AC-4**: The opponent detail template (`opponent_detail.html:82-91`) displays a placeholder (e.g., `"-"` or blank) instead of literal `"None"` text when `home_score` or `away_score` is null
- [ ] **AC-5**: A test verifies that `_compute_wl` returns `"T"` for a tied game
- [ ] **AC-6**: All existing tests pass

## Technical Approach
1. For AC-1: Add `COALESCE(pgp.hr, 0) AS hr` to the pitching query SELECT list in the game detail function in `db.py`. The column exists in `player_game_pitching` per the schema.
2. For AC-2: Check whether season/team pitching has a separate query path; if so, add `hr` there too.
3. For AC-3: In `_compute_wl`, add an equality check before the W/L comparison. When `home_score == away_score`, return `"T"`.
4. For AC-4: In the opponent detail template, use Jinja2 conditional or default filter to display "-" when scores are None.
5. For AC-5: Add a test case for `_compute_wl` with equal scores.
