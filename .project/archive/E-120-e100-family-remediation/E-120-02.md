# E-120-02: Resolve-opponents and scouting loader test coverage

## Epic
[E-120: E-100 Family Code Review Remediation](epic.md)

## Status
`DONE`

## Description
After this story is complete, the `resolve-opponents` CLI code path will have test coverage verifying that `load_config()` receives `db_path`, and the scouting loader multi-scope aggregate test will include data for multiple teams so it actually validates the `team_id` filter.

## Context
Two test gaps were identified in the code review: (1) Zero CLI test coverage for the `bb data resolve-opponents` command — the pipeline bug in E-120-01 shipped because no test exercised this path. (2) `test_scouting_loader.py:446` multi-scope aggregate test only inserts data for one team, so a missing `team_id` WHERE clause would still produce correct results — making the test vacuous for that class of bug.

## Acceptance Criteria
- [ ] **AC-1**: A test in `tests/test_cli_data.py` exercises the `resolve_opponents()` function in `src/cli/data.py` and verifies that `load_config()` is called with a `db_path` keyword argument (not bare `load_config()`)
- [ ] **AC-2**: The `test_aggregate_isolated_per_season` test in `tests/test_scouting_loader.py:446` (or a new companion test) inserts stat rows for at least two different `team_id` values with different stat values, then asserts that aggregation for one team does not include the other team's data. Verification: removing the `WHERE team_id = ?` clause from the aggregate query would cause the test to fail.
- [ ] **AC-3**: All existing tests pass

## Technical Approach
1. For AC-1: `tests/test_cli_data.py` already exists (covers sync, crawl, load commands). Add a test that patches `load_config` and `_resolve_db_path`, invokes `resolve_opponents()` via the Typer test runner, and asserts `load_config` was called with `db_path=<resolved_path>`.
2. For AC-2: The existing `test_aggregate_isolated_per_season` at `test_scouting_loader.py:446` inserts game data for one team only (`own_pk`). Add stat rows for `opp_pk` (the second team already created in the fixture at line 475-478) with different values. After aggregation, assert that `own_pk`'s season stats do not include `opp_pk`'s numbers. Alternatively, add a new `test_aggregate_isolated_per_team` test with this structure.
