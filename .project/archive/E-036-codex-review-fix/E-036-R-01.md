# E-036-R-01: Research: How to Pass Custom Rubric to codex review

## Epic
[E-036: Fix Codex Code-Review Wrapper](epic.md)

## Status
`DONE`

## Objective
Determine how to deliver the project-specific review rubric (`.project/codex-review.md`) to `codex review` when using `--uncommitted`, `--base`, or `--commit` flags. The `[PROMPT]` argument cannot be combined with these flags (verified failure in v0.107.0). We need an alternative delivery mechanism or a fundamentally different approach.

## Time Box
Single agent session. If no mechanism is found after exhausting all investigation paths, document that and recommend the `codex exec` fallback approach.

## Research Questions
1. Does `codex review` accept any flag for custom instructions, system prompt, or config file? (Check `codex review --help` output.)
2. Does the `codex` CLI have a global config mechanism (e.g., `--config`, `--instructions`, project-level config file) that `codex review` would inherit?
3. Does Codex read any project-level config files automatically (e.g., `codex.json`, `.codex/config`, `codex.toml`, `.codexrc`)? Check the filesystem and `codex --help`.
4. Does Codex respect any environment variable for injecting custom review instructions?
5. If no injection mechanism exists for `codex review`, can `codex exec` be used with a manually-assembled diff + rubric to produce equivalent review output? Prototype and assess.
6. Are there any other `codex` subcommands or flags (discovered via `codex --help`) that could serve this purpose?

  - Full output of `codex review --help` and `codex --help`
  - Answer to each research question with evidence

  - If recommending `codex exec` fallback: a working prototype command line showing how to assemble diff + rubric and pipe to `codex exec`

## Approach
1. Run `codex review --help` and capture the full output. Look for any flags related to instructions, config, system prompt, or context.
2. Run `codex --help` and capture the full output. Look for global flags or config mechanisms.
3. Check for Codex config file conventions: look for files like `codex.json`, `.codex/`, `codex.toml`, `.codexrc` in the repo root and home directory. Check if `codex` documentation mentions project-level config.
4. Check environment variables: look at `codex --help` output for env var references, and check if common patterns like `CODEX_INSTRUCTIONS` or `CODEX_CONFIG` are documented.
5. If no native mechanism exists, prototype the `codex exec` fallback:
   - Generate a diff using `git diff` (for uncommitted), `git diff <branch>...HEAD` (for base), or `git show <sha>` (for commit)
   - Assemble rubric + diff into a single prompt
   - Pipe to `codex exec --ephemeral -`
   - Verify it produces meaningful review output
6. Write findings document with recommendation.

## Findings
See `/workspaces/baseball-crawl/epics/E-036-codex-review-fix/E-036-R-01-findings.md` for full findings.

Summary: `codex review` has NO mechanism for custom instructions alongside diff-scope flags (`--uncommitted`, `--base`, `--commit`). The `[PROMPT]` argument is the only input path and it is mutually exclusive with all three flags. No config files, env vars, or alternative flags exist. The `codex exec --ephemeral -` fallback is the only viable approach.

## Recommendation
Use `codex exec --ephemeral -` with assembled rubric + diff for all three review modes. This approach is proven (spec-review wrapper uses it), delivers the full rubric, includes equivalent diff content, and preserves the user-facing interface unchanged.
