# E-036-01: Fix codex-review.sh to Work with Project Rubric

## Epic
[E-036: Fix Codex Code-Review Wrapper](epic.md)

## Status
`DONE`

## Description
After this story is complete, `scripts/codex-review.sh` will successfully execute all three review modes (`uncommitted`, `base`, `commit`) without CLI argument errors, and will deliver the project-specific rubric (`.project/codex-review.md`) to Codex during each review. The script's header comments will accurately reflect the verified CLI behavior.

## Context
E-036-R-01 researched how to deliver custom instructions to `codex review` and produced a findings document with a recommended approach. This story implements that recommendation. The current script is completely broken -- every mode fails because the `[PROMPT]` argument is incompatible with `--uncommitted`, `--base`, and `--commit` flags.

## Acceptance Criteria
- [ ] **AC-1**: Running `scripts/codex-review.sh uncommitted` does NOT produce a CLI argument error. It either completes a review or reports "no changes to review" (if working tree is clean).
- [ ] **AC-2**: Running `scripts/codex-review.sh base main` does NOT produce a CLI argument error. It either completes a review or reports "no diff" (if branch matches main).
- [ ] **AC-3**: Running `scripts/codex-review.sh commit $(git rev-parse HEAD)` does NOT produce a CLI argument error. It completes a review of the HEAD commit.
- [ ] **AC-4**: The review output demonstrates awareness of the project rubric (e.g., references project-specific review priorities, not just generic code review). If the recommended approach does not support rubric injection, the findings document from E-036-R-01 must have explicitly documented this limitation and the script must include a comment noting it.
- [ ] **AC-5**: The script header comment (lines 4-9 area) is rewritten to accurately describe the verified CLI behavior. No false claims about PROMPT argument compatibility with review flags.
- [ ] **AC-6**: The `usage()` function output is still accurate (modes, syntax, examples).
- [ ] **AC-7**: The script still validates that `codex` is installed and that the rubric file exists (existing guard clauses preserved).
- [ ] **AC-8**: No changes to `.project/codex-review.md` (rubric content is out of scope).

## Technical Approach
1. Read the research findings at `/workspaces/baseball-crawl/epics/E-036-codex-review-fix/E-036-R-01-findings.md`.
2. Implement the recommended approach from the findings document.
3. If the recommendation is to use `codex exec` with assembled diff + rubric: restructure the script to generate the appropriate diff for each mode, assemble it with the rubric, and pipe to `codex exec --ephemeral -`.
4. If the recommendation is to use a native `codex review` config mechanism: apply that mechanism.
5. Rewrite the header comment to document the actual verified behavior.
6. Test all three modes manually to confirm no CLI errors.
