# E-196-05: Standalone report pitching availability columns

## Epic
[E-196: Pitching Availability and Game Chronological Ordering](epic.md)

## Status
`DONE`

## Description
After this story is complete, standalone scouting reports will display Last/Rest and P(7d) columns in the pitching table, with a JS snippet that transforms server-rendered dates into days-rest in web view and falls back gracefully for PDF/print. The pitching section annotation will show the generation date for coach reference.

## Context
Standalone reports are self-contained HTML snapshots that must work both in a browser (JS enabled) and when printed/saved as PDF (JS disabled). This story adds the two pitching availability columns with a dual-rendering strategy: server renders formatted dates and "Last"/"Generated" headers; client-side JS upgrades to days-rest and "Rest"/"Data through" headers. The workload query comes from E-196-03.

## Acceptance Criteria
- [ ] **AC-1**: `src/reports/generator.py` calls the workload query function from E-196-03 with `reference_date` set to the report's generation date and includes the data in the template context.
- [ ] **AC-2**: `scouting_report.html` pitching table includes "Last" and "P (7d)" columns per Technical Notes TN-5.
- [ ] **AC-3**: Each Last column cell has class `last-outing-cell` and a `data-date` attribute containing the ISO date (YYYY-MM-DD) for JS computation.
- [ ] **AC-4**: The Last column header has class `last-outing-header`. The pitching section annotation line has class `pitching-annotation` and reads "Generated [formatted date]" in server-rendered HTML.
- [ ] **AC-5**: An inline JS snippet per Technical Notes TN-6 is present at the bottom of the report body. It replaces date text in `.last-outing-cell` elements with computed `Xd` values, swaps the `.last-outing-header` text from "Last" to "Rest", and swaps the `.pitching-annotation` text from "Generated" to "Data through".
- [ ] **AC-6**: When JS is disabled, the report displays correctly with formatted dates, "Last" header, and "Generated" annotation -- no broken layout or missing data.
- [ ] **AC-7**: The JS snippet uses `var` (not `let`/`const`) and has no external dependencies, per Technical Notes TN-6.
- [ ] **AC-8**: Tests verify renderer includes workload data in template context, and that the HTML output contains the expected classes and data attributes.
- [ ] **AC-9**: The key-player callout ("Ace" box) in the standalone report includes a workload sub-line. Web (JS enabled): `Rest: Xd * {pitches}/{span}d`. PDF/print (JS disabled): `Last: Mar 28 * {pitches}/{span}d`. No appearances: `No recent outings`. NULL pitch counts: `?/{span}d` for the workload portion. The sub-line cell has class `last-outing-cell` with `data-date` attribute for JS upgrade, consistent with the pitching table pattern.

## Technical Approach
The renderer already builds a data dict for the report template. Add a call to the workload query function and merge the result into the pitching data. The template adds two columns to the pitching table with appropriate CSS classes and data attributes. The JS snippet goes in a `<script>` block at the bottom of the template body -- it iterates `.last-outing-cell` elements, computes days difference from `data-date` to the current client date, and replaces the text content. Header and annotation swaps target their respective classes.

Read the current renderer and template to understand the existing data structure before modifying.

## Notes
- The JS days-rest computation uses the client's current date, which may differ from the server's generation date by a day depending on timezone. This is acceptable -- the coach sees "how stale is this report" relative to when they open it.
- The `data-date` attribute uses ISO format (YYYY-MM-DD) for reliable cross-browser parsing.
- The annotation swap ("Generated" -> "Data through") ensures coaches know the report's data freshness even if the header is off-screen or cropped.
- **Shared file**: Both E-196-02 and E-196-05 modify `src/reports/generator.py`. E-196-02 changes ORDER BY clauses; E-196-05 adds workload query calls. Changes target different parts of the file.
- **"Today" + "/1d" consistency**: When Rest shows "Today" and P(7d) shows `/1d`, both are correct -- the pitcher threw today and the 7d window span is 1 day.
- **NULL pitch counts**: When appearances exist but all pitch counts are NULL, display `?/{span}d` in the P(7d) column (not `0/{span}d`, which misleads the coach into thinking the arm is fresh).
- Verify that the `format_date` Jinja filter handles YYYY-MM-DD string input (not just date objects) before using it for the Last column dates.
