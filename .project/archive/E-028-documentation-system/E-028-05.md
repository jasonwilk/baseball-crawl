# E-028-05: CLAUDE.md and workflow integration updates

## Epic
[E-028: Documentation System and Docs-Writer Agent](epic.md)

## Status
`DONE`

## Description
After this story is complete, the docs-writer agent is fully integrated into the project's convention and workflow infrastructure. CLAUDE.md's Agent Ecosystem table includes docs-writer, and -- critically -- the dispatch pattern and workflow discipline rules are updated to enforce documentation assessment as a required step in the epic lifecycle.

This story wires the documentation maintenance rule (created by E-028-02) into the existing workflow infrastructure, making documentation updates a mandatory part of the epic completion flow rather than an optional convention.

## Context
Creating an agent definition (E-028-01) and a documentation rule (E-028-02) are necessary but not sufficient. The rule must be enforced by the existing workflow machinery. Specifically:
- The dispatch pattern (`.claude/rules/dispatch-pattern.md`) must include a documentation assessment step in its "Completing an epic" flow.
- The workflow discipline rule (`.claude/rules/workflow-discipline.md`) must reference documentation assessment as a required close-out step.
- The PM agent definition (`.claude/agents/product-manager.md`) must include documentation assessment in its Atomic Status Update Protocol for epic completion.

Without these integrations, the documentation rule is just a convention file that agents might not follow. With them, the PM is contractually required to assess doc impact and dispatch docs-writer when needed.

## Acceptance Criteria
- [ ] **AC-1**: The Agent Ecosystem table in `/workspaces/baseball-crawl/CLAUDE.md` (under "## Agent Ecosystem") includes a row for `docs-writer` with its role description, consistent with the agent's `description` field.
- [ ] **AC-2**: The dispatch pattern file at `/workspaces/baseball-crawl/.claude/rules/dispatch-pattern.md` is updated to include a **documentation assessment step** between the final step (epic completion / team shutdown) and the preceding step (check for newly unblocked stories). Specifically, add it as a new step after "PM marks the epic COMPLETED" and before "shuts down teammates and deletes the team" -- or as a new numbered step immediately before the current final step. The step must specify: (a) PM reviews epic scope against documentation update triggers defined in `.claude/rules/documentation.md`, (b) if any trigger fires, PM dispatches docs-writer to update affected docs before archiving, (c) if no trigger fires, PM notes "No documentation impact" in epic History.
- [ ] **AC-3**: The workflow discipline file at `/workspaces/baseball-crawl/.claude/rules/workflow-discipline.md` includes a **Documentation Assessment Gate** section that states: epic completion requires documentation impact assessment per `.claude/rules/documentation.md`. The gate sits between "all stories DONE" and "archive the epic."
- [ ] **AC-4**: The PM agent definition at `/workspaces/baseball-crawl/.claude/agents/product-manager.md` has its "Completing an epic" checklist in the Atomic Status Update Protocol updated to include a documentation assessment step. The step should reference `.claude/rules/documentation.md` and fit between "Update epic status to COMPLETED" and "Move the epic directory to archive."
- [ ] **AC-5**: No other agent definitions or rules files are broken by the changes -- existing content in all modified files is preserved.
