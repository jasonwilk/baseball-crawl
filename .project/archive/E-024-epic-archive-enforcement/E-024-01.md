# E-024-01: PM Protocol and Rule: Explicit Archive Checklist

## Epic
[E-024: Epic Archive Enforcement](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent definition will contain an explicit "Completing an epic" checklist in the Atomic Status Update Protocol, and the existing project-management rule will include an archive enforcement reminder. The PM will no longer be able to mark an epic COMPLETED or ABANDONED without a clear, checklistable prompt to archive the epic directory immediately.

## Context
The PM agent definition at `/.claude/agents/product-manager.md` has checklists for completing a story and completing a research spike, but none for completing an epic. The "How Work Flows" section mentions archiving in step 5, and the Dispatch Procedure mentions it in step 10, but neither is structured as a mandatory checklist. This gap allowed E-022 to be marked COMPLETED without being archived. This story adds the missing protocol checklist and strengthens the existing rule.

## Acceptance Criteria
- [ ] **AC-1**: The Atomic Status Update Protocol section in `/.claude/agents/product-manager.md` contains a new "Completing an epic" checklist with at least these steps: (a) update epic status to COMPLETED/ABANDONED, (b) move the epic directory from `/epics/E-NNN-slug/` to `/.project/archive/E-NNN-slug/`, (c) update MEMORY.md (active epics and archived epics sections), (d) review ideas backlog for newly unblocked candidates.
- [ ] **AC-2**: The Dispatch Procedure step 10 ("Close") in `/.claude/agents/product-manager.md` explicitly references the "Completing an epic" checklist from the Atomic Status Update Protocol, not just a vague "mark the epic COMPLETED."
- [ ] **AC-3**: The rule file `/.claude/rules/project-management.md` includes a statement under "Epics and Stories" that COMPLETED or ABANDONED epics MUST be archived (moved to `/.project/archive/`) before the next commit -- not deferred.
- [ ] **AC-4**: No existing content in the PM agent definition or project-management rule is removed or contradicted by these additions.

## Technical Approach
1. Read `/.claude/agents/product-manager.md` and locate the Atomic Status Update Protocol section (around line 116-143).
2. Add a new "Completing an epic" checklist after the existing "Completing a research spike" checklist. The checklist should include:
   - Update epic.md Status to COMPLETED (or ABANDONED with reason)
   - Move the entire epic directory: `/epics/E-NNN-slug/` to `/.project/archive/E-NNN-slug/`
   - Add History entry with completion date
   - Update MEMORY.md: move epic from Active Epics to Archived Epics, update any unblocked work
   - Review `/.project/ideas/README.md` for newly unblocked CANDIDATE ideas
3. Update Dispatch Procedure step 10 to say: "Follow the 'Completing an epic' checklist in the Atomic Status Update Protocol" instead of the current looser language.
4. Read `/.claude/rules/project-management.md` and add an archive enforcement bullet under "Epics and Stories."
