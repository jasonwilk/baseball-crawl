# E-012-03: Extend PM dispatch and quality checklist procedures

## Epic
[E-012: Filesystem-Context Skill Integration for PM and Architect Agents](../E-012-filesystem-context-integration/epic.md)

## Status
`TODO`

## Description
After this story is complete, the PM's Dispatch Mode procedure in `project-manager.md` will explicitly name the filesystem-context skill as the framework for the file-reading sequence, and the PM's quality checklist will include a verifiable item requiring that story Technical Approach sections name deferred context files by absolute path. These two additions operationalize the skill reference added in E-012-01: a skill referenced but not reflected in behavior is decorative; a skill reflected in concrete procedure steps is active.

## Context
E-012-01 adds a `## Skills` section to `project-manager.md` naming the filesystem-context skill and its triggers. That section tells the PM WHEN to load the skill. This story tells the PM HOW to apply it -- by revising the Dispatch Mode step 1 to name the progressive disclosure sequence explicitly, and by adding a quality checklist item that enforces the "name deferred context files by absolute path" convention in story writing.

The quality checklist gap is the more consequential of the two changes. When a story's Technical Approach says "read the relevant design document" without naming it, the implementing agent must guess which file to load. When it says "read `/.project/research/E-006-precommit-design.md`", the implementing agent can load it as deferred context in one step. This has been a recurrent pain point (noted in MEMORY.md lessons learned: "E-006 refinement pattern -- initial stories had vague technical approach sections and speculative file lists").

## Acceptance Criteria

- [ ] **AC-1**: Given `project-manager.md` after this story, when an agent reads the Dispatch Mode procedure (specifically step 1 "Read the epic directory"), then that step now includes an explicit sub-step or parenthetical referencing the filesystem-context skill and naming the two-pass reading sequence: (1) scan the Stories table for IDs, titles, and statuses; (2) open story files only for those with `Status: TODO` and satisfied dependencies.

- [ ] **AC-2**: Given the revised Dispatch Mode step 1, when an agent follows it for an epic with 8 stories where only 2 are TODO and unblocked, then the procedure directs the agent to open 2 story files (not all 8), consistent with progressive disclosure -- avoiding loading the full content of DONE or BLOCKED stories that are not actionable.

- [ ] **AC-3**: Given the quality checklist in `project-manager.md` after this story, when an agent counts the checklist items, then there is a new item: "Story `Technical Approach` sections name all referenced context files by absolute path (e.g., `/.project/research/E-NNN-slug.md`, `docs/gamechanger-api.md`) so implementing agents can load them as deferred context without guessing."

- [ ] **AC-4**: Given the revised quality checklist, when the PM applies it to a story whose Technical Approach says "consult the GameChanger API spec" (without a path), then the checklist item added in AC-3 would mark that story as failing the checklist, prompting the PM to add the absolute path `docs/gamechanger-api.md`.

- [ ] **AC-5**: Given the full `project-manager.md` after this story, when a diff is taken against the post-E-012-01 version, then the only changes are: (a) the revised Dispatch Mode step 1 (one sub-step or parenthetical added), and (b) one new item in the quality checklist. No sections are removed. No other procedure sections are modified.

## Technical Approach

Read `project-manager.md` after E-012-01 has been applied. Identify two locations:

**Location 1 -- Dispatch Mode step 1** (currently reads approximately):
> "1. **Read the epic directory.** Read `/epics/E-NNN-slug/epic.md` and all story files in the directory."

Revise to add the progressive disclosure sub-step:

> "1. **Read the epic directory.** Apply progressive disclosure (see `.claude/skills/filesystem-context/SKILL.md`): first scan the epic's Stories table to identify IDs, titles, and statuses (first pass -- titles only, no file I/O). Then open story files ONLY for stories with `Status: TODO` whose dependencies are satisfied (second pass -- full content). Do not load the full text of DONE, IN_PROGRESS, or BLOCKED story files unless a specific step requires their content."

**Location 2 -- Quality Checklist** (currently ends with approximately):
> "- [ ] Epic status set to `READY` after all stories pass this checklist"

Add one new item before or after that line:

> "- [ ] Story `Technical Approach` sections name all referenced context files by absolute path (e.g., `/.project/research/E-NNN-slug.md`, `docs/gamechanger-api.md`) rather than by vague description (e.g., 'consult the design document'). Implementing agents must be able to load these as deferred context in one step."
