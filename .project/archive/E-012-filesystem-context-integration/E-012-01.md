# E-012-01: Add filesystem-context skill reference to project-manager agent

## Epic
[E-012: Filesystem-Context Skill Integration for PM and Architect Agents](../E-012-filesystem-context-integration/epic.md)

## Status
`TODO`

## Description
After this story is complete, the `project-manager` agent definition will contain a `## Skills` section naming the `filesystem-context` skill with trigger conditions tied to the PM's actual workflow steps -- specifically Dispatch Mode entry and Refinement Mode context-loading. A new agent reading the PM's definition for the first time will know when and why to load the skill.

## Context
The `filesystem-context` skill at `.claude/skills/filesystem-context/SKILL.md` was created by E-010-01 and teaches progressive disclosure: scan file names and titles first, load full content only when needed. The PM is the primary practitioner of this pattern in baseball-crawl -- every dispatch requires reading epic.md plus a varying set of story files, and every refinement requires reading research artifacts and prior epic history. Without a named trigger in the PM's definition, the skill exists on the filesystem but is never loaded.

The PM's system prompt in `.claude/agents/project-manager.md` currently has no `## Skills` section. This story adds one. It does NOT modify the PM's procedure sections (Dispatch Mode, Refinement Mode, quality checklist) -- those extensions are handled by E-012-03, which depends on this story.

## Acceptance Criteria

- [ ] **AC-1**: Given a copy of `.claude/agents/project-manager.md` after this story, when an agent reads it from top to bottom, then the agent finds a `## Skills` section that names `filesystem-context`, provides the exact file path `.claude/skills/filesystem-context/SKILL.md`, and lists at least two trigger conditions tied to named PM workflows (Dispatch Mode, Refinement Mode).

- [ ] **AC-2**: Given the `## Skills` section, when an agent reads the trigger condition for `filesystem-context`, then the trigger condition names the specific PM actions that should fire it -- not a paraphrase of the generic SKILL.md triggers. Acceptable phrasing must reference at least one of: "Dispatch Mode", "Refinement Mode", "reading epic directory", "reading research artifacts".

- [ ] **AC-3**: Given the `## Skills` section, when an agent reads it, then each skill entry follows the format defined in the epic's Technical Notes: skill name as a `###` heading, `**File**:` with the path, `**Load when**:` with the trigger, and one to two sentences of rationale.

- [ ] **AC-4**: Given the full `project-manager.md` file after this story, when a diff is taken against the prior version, then the only changes are the addition of the `## Skills` section. No existing procedure sections (Dispatch Mode, Refinement Mode, Backlog Management, etc.) are modified by this story.

- [ ] **AC-5**: Given the `## Skills` section placement in the file, when an agent reads the file, then the `## Skills` section appears after the existing `## Context Awareness` section and before the `## Ideas Workflow` section (or at the end of the system prompt if those sections do not exist), so it is discoverable without reading the full 400+ line definition.

## Technical Approach

Read the full `project-manager.md` first to understand current structure. Identify the correct insertion point for the `## Skills` section (after `## Context Awareness`, before `## Ideas Workflow`).

Add the following section (exact wording may be adjusted to match the agent's voice, but content must satisfy all ACs):

```markdown
## Skills

### filesystem-context
**File**: `.claude/skills/filesystem-context/SKILL.md`
**Load when**: Entering Dispatch Mode (before reading the epic directory -- `/epics/E-NNN-slug/epic.md` and story files), OR entering Refinement Mode (before reading research artifacts, prior epic files, or dependency story files to scope a new epic).

The PM's core work is loading context files in the right order -- epic.md first, then story files for TODO stories, then dependency artifacts only if needed. This skill provides the progressive disclosure framework for that sequence: scan titles and statuses first (first pass), load full file contents only for stories that are eligible to dispatch (second pass). Prevents over-loading all story files when only a subset are actionable.
```

Note: the `## Skills` section should list only skills that the PM should actually load. Do not add multi-agent-patterns or context-fundamentals to this story -- those are out of scope per the epic Non-Goals.
