# E-012-02: Add filesystem-context skill reference to claude-architect agent

## Status

## Description
After this story is complete, the `claude-architect` agent definition will contain a `## Skills` section naming the `filesystem-context` skill with a trigger condition tied to agent design decisions -- specifically when deciding what context belongs in a system prompt (ambient) vs. a skill file or memory topic file (deferred). This gives claude-architect an explicit framework for the ambient/deferred distinction that is central to its primary job of designing the agent infrastructure.

## Context
Claude-architect creates and modifies agent definitions. Every such decision involves a context architecture question: does this information belong in the system prompt (loaded every session), in a skill file (loaded on demand), in a memory topic file (persistent across sessions but loaded only when the topic is active), or in a CLAUDE.md (directory-scoped and loaded automatically when files in that directory are opened)?

The filesystem-context skill directly informs this decision hierarchy. Claude-architect's agent definition at `.claude/agents/claude-architect.md` currently has a detailed "Prompt Engineering" methodology section (step 5 in the design methodology) but does not reference the ambient/deferred distinction or the skill that teaches it. This story adds that reference.

This story is parallel to E-012-01 -- both can execute simultaneously because they touch different files.

## Technical Approach

Read the full `claude-architect.md` to understand current structure. The file has a "## Your Design Methodology" section with 7 steps, step 5 being "Prompt Engineering". The `## Skills` section should be inserted after the methodology section.

Add the following section (adjust wording to match the agent's voice, but content must satisfy all ACs):

```markdown
## Skills

### filesystem-context
**File**: `.claude/skills/filesystem-context/SKILL.md`
**Load when**: Designing or reviewing an agent definition -- specifically when deciding whether a piece of context should live in the system prompt (ambient, loaded every session) vs. a skill file (deferred, loaded on trigger) vs. a memory topic file (persistent, loaded when the topic is active). Also load when structuring `MEMORY.md` files to determine what stays in the 200-line ambient section vs. what moves to a linked topic file.

Provides the framework for the ambient vs. deferred context distinction that is central to agent definition quality. A system prompt that tries to carry all possible context becomes expensive and diluting -- this skill teaches where to draw the line.
```

Optionally, add a one-sentence cross-reference in step 5 of the design methodology:

> "See the `filesystem-context` skill (`.claude/skills/filesystem-context/SKILL.md`) for guidance on what belongs in the system prompt (ambient) vs. a skill file or memory topic file (deferred)."

This cross-reference is optional but improves discoverability. If it would make the methodology step feel cluttered, omit it -- the `## Skills` section alone satisfies the acceptance criteria.
