# E-040-01: Create UX designer agent definition and update ecosystem

## Epic
[E-040: UX Designer Agent](../E-040-ux-designer-agent/epic.md)

## Status
`DONE`

## Description
After this story is complete, a fully defined UX designer agent will exist at `.claude/agents/ux-designer.md`, the CLAUDE.md Agent Ecosystem table will include the new agent, and the dispatch-pattern.md agent selection table will have a routing entry for UI/UX design work. The PM will be able to dispatch design stories to this agent for any dashboard or UI work.

## Context
The project is approaching dashboard implementation (E-004) but has no agent specializing in interface design. This story creates that agent. The UX designer fills the gap between coaching requirements (baseball-coach) and implementation (software-engineer) by owning the design layer: layouts, information hierarchy, component structure, and user flows.

The agent must be designed for a text-based medium -- it cannot produce images or use Figma. Its design artifacts are structured text: ASCII wireframes, HTML/Tailwind reference mockups, layout specs, component inventories, and user flow descriptions.

## Acceptance Criteria
- [ ] **AC-1**: A file exists at `.claude/agents/ux-designer.md` with valid YAML frontmatter (`name`, `description`, `model`, `color`, `memory`, `tools`) and a markdown body following the canonical section skeleton (`Identity`, `Core Responsibilities`, `Design Artifacts & Formats`, `Stack Constraints`, `Mobile-First Design Principles`, `Anti-Patterns`, `Error Handling`, `Inter-Agent Coordination`, `Skill References`, `Memory`).
- [ ] **AC-2**: The agent's `Identity` section establishes it as a UX/interface designer for the baseball-crawl coaching dashboard, explicitly scoped to server-rendered HTML (FastAPI + Jinja2 + Tailwind CSS CDN, no JS frameworks).
- [ ] **AC-3**: The agent's `Core Responsibilities` section covers at minimum: (a) layout and information architecture design, (b) wireframe/mockup creation in text-based formats, (c) mobile-first responsive design (375px minimum), (d) user flow specification, (e) reviewing existing templates for design coherence.
- [ ] **AC-4**: The agent's `Inter-Agent Coordination` section defines the relationship with baseball-coach (consult for domain requirements), software-engineer (produces design specs that SE implements), docs-writer (no overlap -- UX designs, docs-writer documents), and product-manager (receives design stories via dispatch).
- [ ] **AC-5**: The agent's `Anti-Patterns` section includes at minimum: (a) never write production application code (routes, db queries, migrations, tests), (b) never introduce client-side JS frameworks or CSS build pipelines, (c) never design features that are not in scope of a referenced story.
- [ ] **AC-6**: The agent definition includes a `description` field that is 1-3 concise sentences suitable for PM routing -- no embedded examples, no multi-paragraph blocks.
- [ ] **AC-7**: The CLAUDE.md Agent Ecosystem table (the `| Agent | Alias | Role |` table) includes a row for `ux-designer` with an appropriate role description. The agent count in any prose referencing the number of agents is updated accordingly.
- [ ] **AC-8**: The `.claude/rules/dispatch-pattern.md` Agent Selection for Dispatch table includes a row mapping UI/UX design work to the new agent type.
- [ ] **AC-9**: The agent's `model` is set to `sonnet` (balanced -- design work does not require opus-level reasoning).
- [ ] **AC-10**: The agent has `memory: project` in frontmatter and a Memory section instructing it to save design decisions, established patterns, user preferences for layout, and component conventions.

## Technical Approach
1. Read all existing agent definitions at `.claude/agents/*.md` to understand the canonical format and conventions used in this project.
2. Read the existing dashboard templates (`src/api/templates/base.html`, `src/api/templates/dashboard/team_stats.html`) and E-004 Technical Notes to ground the agent in the current UI reality.
3. Create `.claude/agents/ux-designer.md` following the canonical section skeleton. Use `docs-writer.md` as a structural reference (similar pattern: specialist agent that reads code but produces non-code artifacts).
4. Update the CLAUDE.md Agent Ecosystem table at `/workspaces/baseball-crawl/CLAUDE.md` -- add the `ux-designer` row and update the agent count from 7 to 8 in any prose that mentions it.
5. Update the Agent Selection for Dispatch table in `/workspaces/baseball-crawl/.claude/rules/dispatch-pattern.md` -- add a row for UI/design stories routing to `ux-designer`.
6. Update the claude-architect agent definition at `/workspaces/baseball-crawl/.claude/agents/claude-architect.md` -- add `ux-designer` to the agent list in the Identity section (currently lists 7 agents).
