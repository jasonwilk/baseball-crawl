# E-080-03: Rewrite codex-review.sh rubric embedding to path-based

## Epic
[E-080: Lean Codex Spec Review Prompt](epic.md)

## Status
`DONE`

## Description
After this story is complete, `scripts/codex-review.sh` will pass the rubric file to Codex as a path reference instead of embedding its full contents. The diff content remains embedded (it is generated at runtime and is not a file on disk). All argument parsing, mode handling, diff generation, error handling, and the `codex exec --ephemeral -` invocation are unchanged.

## Context
`scripts/codex-review.sh` was originally excluded from E-080's scope because "diffs must be embedded." That exclusion was correct for the diff, but the rubric (`RUBRIC_FILE`) is a file on disk that Codex can read by path — the same optimization applied to `codex-spec-review.sh` in E-080-02 applies here. The partial fix: rubric becomes a path reference; diff stays embedded.

## Acceptance Criteria
- [ ] **AC-1**: `assemble_review_prompt()` no longer calls `cat "${RUBRIC_FILE}"`. It outputs the rubric file path and instructs Codex to read it.
- [ ] **AC-2**: The diff content (`echo "${diff_content}"`) remains embedded in the prompt unchanged. Diffs are runtime-generated and not a file on disk.
- [ ] **AC-3**: The REVIEW REQUEST section instructs Codex to read the rubric at the given path before reviewing the diff.
- [ ] **AC-4**: The script header comment no longer implies the rubric is embedded into the prompt. Updated to reflect that the rubric is passed as a path.
- [ ] **AC-5**: Error paths remain unchanged: missing/unknown mode prints usage and exits non-zero; missing branch name for `base` mode prints an error and exits non-zero; missing SHA for `commit` mode prints an error and exits non-zero; missing rubric file prints an error and exits non-zero.
- [ ] **AC-6**: The `codex exec --ephemeral -` invocation in all three mode branches (`uncommitted`, `base`, `commit`) remains unchanged.
- [ ] **AC-7**: The assembled prompt (rubric path + diff + instructions) is shorter than the current prompt by the size of the embedded rubric. The rubric path section is under 5 lines.

## Technical Approach
Only `assemble_review_prompt()` (lines 58-82) changes. Replace the `cat "${RUBRIC_FILE}"` call and its surrounding section header with a short block that emits the rubric path and tells Codex to read it. Update the REVIEW REQUEST echo lines to reference reading the rubric at the given path. Update the header comment at lines 7-9 to remove the implication that the rubric is embedded.
