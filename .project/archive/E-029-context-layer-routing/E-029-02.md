# E-029-02: Add Context-Layer Routing Pre-Check to PM Agent Definition

## Epic
[E-029: Context-Layer Routing Enforcement](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent definition (`/.claude/agents/product-manager.md`) includes a mandatory pre-check in the Dispatch Procedure that requires the PM to scan each story's "Files to Create or Modify" section against the context-layer path list before selecting an agent type. If any file matches a context-layer path, the PM must use `subagent_type: claude-architect` for that story.

## Context
The dispatch-pattern routing table (updated by E-029-01) defines which file paths are context-layer. But the PM's Dispatch Procedure (steps 1-10 in the "Dispatch Procedure" section of `product-manager.md`) does not include a step that checks story file lists against the routing table. The PM is expected to "just know" which agent type to use, relying on memory and domain judgment. This has failed twice (E-019, E-027).

This story adds a concrete procedural step -- between identifying eligible stories and spawning implementers -- that forces the PM to scan each story's file list and apply context-layer routing. This turns an implicit judgment call into an explicit checklist item.

## Acceptance Criteria
- [ ] **AC-1**: The Dispatch Procedure in `/.claude/agents/product-manager.md` includes a new step (between the current steps 5 and 6, or integrated into step 6) that reads: "For each eligible story, scan its 'Files to Create or Modify' section. If any file matches a context-layer path (see `/.claude/rules/dispatch-pattern.md` Routing Precedence), use `subagent_type: claude-architect` for that story."
- [ ] **AC-2**: The existing step 6 instruction ("Use `subagent_type: general-purpose` for implementation work, `subagent_type: claude-architect` for infrastructure work") is updated to reference the context-layer routing check rather than relying on the vague "infrastructure work" classification.
- [ ] **AC-3**: The context-layer path list is referenced by pointer to `dispatch-pattern.md` (not duplicated inline in the PM definition). This ensures a single source of truth.
- [ ] **AC-4**: The PM's "Anti-Patterns" section or equivalent includes a note about the E-019/E-027 pattern: "Never dispatch a story that modifies context-layer files to general-purpose. Always apply the context-layer routing check."
- [ ] **AC-5**: No other sections of `product-manager.md` are modified beyond the Dispatch Procedure and Anti-Patterns sections.

## Technical Approach
Read `/.claude/agents/product-manager.md`. Modify the Dispatch Procedure section to insert the context-layer routing check. Update step 6 to reference the check. Add a note to the Anti-Patterns section. Reference the context-layer path list in `dispatch-pattern.md` by file path rather than duplicating it.
