# E-029-01: Expand Dispatch-Pattern Routing Table with Context-Layer File Paths

## Epic
[E-029: Context-Layer Routing Enforcement](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the "Agent Selection for Dispatch" table in `/.claude/rules/dispatch-pattern.md` includes an explicit "Context-layer files" entry that enumerates the specific file paths and path patterns that must always be routed to `claude-architect`. The table also includes a precedence note explaining that context-layer routing overrides domain-based routing.

## Context
The current routing table has four entries. The fourth entry ("Agent config, CLAUDE.md, rules, skills -> claude-architect") is correct in intent but ambiguous in scope. It does not enumerate specific paths, which means the PM must interpret whether a given file is "agent config" or not. This ambiguity has caused mis-routing in E-019 and E-027, where stories touching CLAUDE.md or `.claude/` files were dispatched to general-dev.

This story replaces the ambiguous fourth entry with an explicit, enumerated list of context-layer file paths and adds a routing precedence note.

## Acceptance Criteria
- [ ] **AC-1**: The "Agent Selection for Dispatch" table in `/.claude/rules/dispatch-pattern.md` includes a "Context-layer files" row that lists all of these path patterns: `CLAUDE.md`, `.claude/agents/*.md`, `.claude/rules/*.md`, `.claude/skills/**`, `.claude/hooks/**`, `.claude/settings.json`, `.claude/settings.local.json`, `.claude/agent-memory/**`.
- [ ] **AC-2**: The table entry maps all context-layer file paths to `claude-architect`.
- [ ] **AC-3**: A "Routing Precedence" note appears immediately after the table stating: if a story's "Files to Create or Modify" includes any context-layer path, route to `claude-architect` regardless of the story's primary domain. The only exception is the PM updating its own memory files during normal status-update work.
- [ ] **AC-4**: The old "Agent config, CLAUDE.md, rules, skills" row is replaced (not duplicated) by the new explicit entry.
- [ ] **AC-5**: No other sections of `dispatch-pattern.md` are modified beyond the routing table and the new precedence note.

## Technical Approach
Read `/.claude/rules/dispatch-pattern.md`. Replace the existing fourth row in the "Agent Selection for Dispatch" table with a new row that enumerates context-layer paths. Add a short "Routing Precedence" paragraph immediately after the table. The path list is defined in the parent epic's Technical Notes.
