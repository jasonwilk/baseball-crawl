# E-136-05: Update Code-Reviewer for Staged Diff Command

## Epic
[E-136: Atomic Epic Commits](epic.md)

## Status
`DONE`

## Description
After this story is complete, the Worktree Review section in `code-reviewer.md` will use `git diff --cached main` instead of `git diff main..HEAD` for reviewing implementer changes.

## Context
The code-reviewer agent definition contains a Worktree Review section that instructs the reviewer to use `git diff main..HEAD` to see implementer changes. The atomic commit workflow means implementers no longer commit -- they stage changes. `git diff main..HEAD` shows nothing when there are no commits. The correct command is `git diff --cached main`, which shows all staged changes relative to main.

## Acceptance Criteria
- [ ] **AC-1**: The Worktree Review section uses `git diff --cached main` instead of `git diff main..HEAD`.
- [ ] **AC-2**: Any surrounding explanatory text is consistent with the staged (uncommitted) workflow.
- [ ] **AC-3**: No other sections of the file are altered.

## Technical Approach
The code-reviewer agent definition at `/.claude/agents/code-reviewer.md` contains a Worktree Review section. Locate the `git diff main..HEAD` command and replace with `git diff --cached main`. Verify surrounding text is consistent with the new workflow (e.g., references to "committed changes" should become "staged changes"). Read the full file to ensure no other stale references.
