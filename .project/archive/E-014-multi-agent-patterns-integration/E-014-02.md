## Context
The project-manager's Dispatch Mode procedure is the PM -> implementing agent relay point. The `multi-agent-patterns` skill names this as Risk Point 2 and provides a PM dispatch checklist (section "How to Dispatch Correctly"). If the PM summarizes the story file or omits the epic Technical Notes in the context block, the implementing agent works from incomplete information. This is the most common form of telephone game failure in the baseball-crawl chain.

The PM's definition (`/Users/jason/Documents/code/baseball-crawl/.claude/agents/project-manager.md`) is the longest agent definition in the project. E-012-01 (a dependency of this story) adds a `## Skills` section to `project-manager.md` with the `filesystem-context` skill reference. This story adds the `multi-agent-patterns` entry to that same `## Skills` section.

**Dependency on E-012-01**: E-012-01 creates the `## Skills` section in `project-manager.md`. This story adds to it. If E-012-01 has not been completed, this story would need to create the section itself -- but because we cannot predict the exact text E-012-01 will add, sequencing is mandatory. Read E-012-01 (story file and its completion state) before beginning this story.

**Priority**: Second in this epic. The PM is the workflow gate -- every story dispatch flows through it, making it the highest-volume relay point for distortion risk.

## Notes
The PM's Dispatch Mode procedure already contains the phrase "Never summarize" in its context block mandate (Step 5 of the Dispatch procedure). This story does not change that procedure -- it adds a skill reference that gives the PM an explicit framework (the telephone game diagnostic) for self-checking when something feels wrong at dispatch time.

The `multi-agent-patterns` skill's PM dispatch checklist maps directly to the PM's existing Dispatch Mode step 4 ("Before invoking the Task tool for a story dispatch"). The skill reference creates a connection between those steps and the broader telephone game framing, which helps the PM understand why those steps exist, not just what they are.
