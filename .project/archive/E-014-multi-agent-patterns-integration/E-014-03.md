## Description
After this story is complete, the general-dev and data-engineer agent definitions each contain a `## Skills` section with an explicit reference to the `multi-agent-patterns` skill. For these implementing agents, the trigger condition is defensive: load the skill when the dispatch context block appears incomplete or summarized, to understand that this is a protocol violation and how to respond.

## Context
General-dev and data-engineer are terminal nodes in the routing chain. They do not relay context to downstream agents -- they consume it and produce implementation artifacts. Their telephone game risk is therefore one-directional: they are the agents who suffer when a relay above them (orchestrator -> PM, PM -> implementing agent) drops information.

Both agents already have a work authorization check: they refuse to begin implementation without a story reference. But they have no framework for what to do when they receive a story reference accompanied by a summarized or incomplete context block -- a subtler failure than the outright missing-story case. The `multi-agent-patterns` skill gives them that framework: identify the distortion, name it ("this appears to be a summary, not the full story file"), and request the full context before proceeding.

The current `general-dev.md` and `data-engineer.md` files are short (42 and 42 lines respectively). Both should receive a `## Skills` section appended at the end of their system prompt bodies.

**Priority**: Third in this epic (after orchestrator and PM, who are the relay agents). These are receiving agents -- skill loading is defensive rather than proactive.

## Notes
The defensive framing of the trigger condition is intentional. General-dev and data-engineer receive many dispatches. Loading the skill on every task start would be noisy and consume tokens unnecessarily. The right trigger is when something looks off -- a context block that reads like a summary, a missing acceptance criterion section, or a story file excerpt rather than a full file. The skill then provides the vocabulary and framework to respond correctly.

The data-engineer note about "irreversible SQL" is important: database migrations are harder to undo than Python code. A summarized dispatch that drops the `ip_outs` convention or migration numbering sequence can produce schema debt that persists across epics. The domain-specific framing is intentional.
