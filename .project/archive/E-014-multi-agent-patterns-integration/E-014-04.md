## Description
After this story is complete, the baseball-coach and api-scout agent definitions each contain a `## Skills` section with a reference to the `multi-agent-patterns` skill. For these consultative agents, the trigger condition is output-focused: load the skill when completing a consultation or exploration session to understand that outputs must be written to durable files rather than communicated verbally, so the PM and implementing agents can read them verbatim without relay distortion.

## Context
Baseball-coach and api-scout are direct-routing exceptions in baseball-crawl's hybrid coordination pattern. They are invoked without PM intermediation. The `multi-agent-patterns` skill explains why this is safe (section "Direct-Routing Exceptions: Why They Are Safe"): their outputs go into durable files that downstream agents read directly, so there is no telephone game relay. But this safety guarantee depends on the agents actually following through on writing to files -- if they communicate findings only through conversational output, the PM would have to summarize those findings to route them, which reintroduces the telephone game problem.

Baseball-coach's output channel: domain requirements clarifications, schema validations, scouting report designs. These should be written to `/.project/research/` or similar artifact locations, not held as conversational memory.

Api-scout's output channel: `docs/gamechanger-api.md`. The skill explicitly names this file as the reason api-scout's direct-routing exception is safe -- the PM reads the spec verbatim rather than relying on a relay.

Both agents already document their output disciplines in their definitions (baseball-coach has Memory Instructions; api-scout has "document as you go" in its Discovery Methodology). This story adds a skill reference that frames those disciplines in terms of the telephone game problem and why the file-write requirement is not just process hygiene but a structural necessity in a multi-agent chain.

**Priority**: Fourth in this epic. These agents are already correctly configured for file-based output; the skill reference is reinforcement and framing, not a new behavioral requirement.

## Notes
The framing for these two agents is different from the implementing agents (E-014-03) and the relay agents (E-014-01, E-014-02). For relay agents, the risk is distortion during dispatch. For implementing agents, the risk is receiving a distorted dispatch. For consultative agents, the risk is producing outputs that cannot be read verbatim by downstream agents -- which happens when findings stay in conversation rather than landing in files.

The skill already explains this distinction clearly in the "Direct-Routing Exceptions: Why They Are Safe" section. The trigger conditions in these story entries are designed to reinforce that section's message: the safety guarantee requires file-write discipline.

If api-scout's exploration session is long or produces many findings, it may be appropriate to produce an interim summary document at `/.project/research/` in addition to updating `docs/gamechanger-api.md`. The skill reference will prompt the agent to think about this rather than leaving it implicit.
