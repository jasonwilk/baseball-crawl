## Description
After this story is complete, the orchestrator agent definition will contain a `## Skills` section that explicitly references the `multi-agent-patterns` skill with trigger conditions tied to Task tool dispatch construction and multi-agent output aggregation. The orchestrator is the highest-priority target for this skill because it sits at Risk Point 1 (orchestrator -> project-manager relay) -- the first place where the user's original intent can be distorted through paraphrase.

## Context
The orchestrator is the supervisor in baseball-crawl's hybrid coordination pattern. Every user work-initiation request flows through it before reaching the PM. If the orchestrator paraphrases the user's request rather than relaying it verbatim, the PM writes stories against a distorted requirement -- and the implementing agent builds against a further-distorted story. The telephone game problem is most acute at this first relay point.

The `multi-agent-patterns` skill (`.claude/skills/multi-agent-patterns/SKILL.md`) contains a dedicated orchestrator checklist (section "How to Receive and Relay Correctly") and explicitly names the orchestrator -> PM relay as Risk Point 1. The skill also explains why the orchestrator should ask for clarification before routing ambiguous requests rather than after.

The orchestrator's definition (`/Users/jason/Documents/code/baseball-crawl/.claude/agents/orchestrator.md`) currently lacks any `## Skills` section. The definition is otherwise complete and should not be reorganized -- only the `## Skills` section should be added.

**Priority**: Highest in this epic. The orchestrator is the entry point for all user requests and the agent most likely to introduce distortion.

## Notes
The orchestrator has no filesystem tools -- it cannot read files directly. This means the `## Skills` section in its definition serves as behavioral instruction (a reminder encoded in its system prompt) rather than a literal "load this file" instruction. The trigger conditions should be phrased as behavioral reminders: "before doing X, apply the verbatim relay discipline documented in the skill file." The skill file path is included so that future maintenance (updating the skill) is traceable.

Related: `workflow-discipline.md` at `/Users/jason/Documents/code/baseball-crawl/.claude/rules/workflow-discipline.md` already establishes the verbatim relay obligation in the "Workflow Routing Rule" section. This story complements that rule by giving the orchestrator an explicit self-check mechanism tied to the skill's diagnostic framework (telephone game problem and mitigation checklist).
