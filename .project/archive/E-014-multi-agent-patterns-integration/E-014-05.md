## Description
After this story is complete, the claude-architect agent definition will contain an entry for `multi-agent-patterns` in its `## Skills` section. The trigger condition is design-time: load the skill when creating or modifying agents that add relay steps to the routing chain, to evaluate whether those steps introduce telephone game risk and how to mitigate it.

## Context
Claude-architect is the agent that designs all other agents. Every new agent or routing change it creates has the potential to add a relay step to the orchestrator -> PM -> implementing agent chain. If a new relay step is not designed with verbatim relay discipline, it introduces a new telephone game risk point. The `multi-agent-patterns` skill provides the vocabulary and framework for evaluating this risk during agent design.

The skill is particularly relevant to claude-architect because of the "Ecosystem Thinking" principle in its definition: "Every agent exists in context. Design for the whole, not just the part." Adding a new agent changes the whole chain -- the skill provides the analytical tool for evaluating that change.

The current `claude-architect.md` is 180 lines and ends with a `## MEMORY.md` section (its memory state). E-012-02 (a parallel story in E-012) adds a `## Skills` section to `claude-architect.md` for the `filesystem-context` skill. This story adds the `multi-agent-patterns` entry to that section.

**Sequencing note**: E-014-05 and E-012-02 both modify `claude-architect.md`. Both are assigned to claude-architect. They should be executed sequentially in a single session if possible. If E-012-02 has not been completed before this story is dispatched, this story should add both entries to the `## Skills` section rather than adding just one and leaving the file in a partial state. Check E-012-02 status before beginning.

**Priority**: Fifth (lowest) in this epic. Claude-architect designs agents infrequently relative to how often orchestrator, PM, and implementing agents execute their relay-step workflows. The skill is valuable but the trigger fires less often.

## Notes
The placement guidance (after "Critical Principles", before "Update Your Agent Memory") is a recommendation based on the current file structure. If E-012-02 placed the `## Skills` section at a different location, use that location for consistency -- do not move the section.

The "Ecosystem Thinking" reference in the explanatory sentences is deliberate: it is one of the five Critical Principles in claude-architect's definition. By referencing a principle that claude-architect already holds, the skill entry feels like a natural extension of existing identity rather than an imposed new behavior.

The secondary trigger ("reviewing or modifying routing logic") is important because claude-architect may modify existing agents -- not just create new ones -- in ways that subtly change relay depth. For example, adding a step where the orchestrator queries a new data source before routing to PM would add a relay point even without creating a new agent. The skill's diagnostic questions help evaluate whether that change is safe.
