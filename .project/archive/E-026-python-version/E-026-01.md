# E-026-01: Migrate to Python 3.13 and Establish Version Source of Truth

## Epic
[E-026: Python Version Governance -- Migrate to 3.13](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the project will have a single source of truth for its Python version (`.python-version` = 3.13), a `pyproject.toml` declaring `requires-python >= 3.13`, and all existing version references (Dockerfile, devcontainer.json, CLAUDE.md Tech Stack) updated from 3.12 to 3.13. All dependencies will install and all tests will pass on Python 3.13.

## Context
The project currently uses Python 3.12 as a model training artifact, not a deliberate choice. There is no `.python-version`, no `pyproject.toml`, and no single source of truth. This story creates the governance files and migrates the version in one pass. The companion story E-026-02 handles documenting the version maintenance policy.

## Acceptance Criteria
- [ ] **AC-1**: A `.python-version` file exists at the project root containing exactly `3.13` (single line, no trailing whitespace beyond a newline)
- [ ] **AC-2**: A `pyproject.toml` file exists at the project root containing `[project]` with `name = "baseball-crawl"` and `requires-python = ">=3.13"` -- no other sections (no `[build-system]`, no `[tool.*]`, no dependencies)
- [ ] **AC-3**: `Dockerfile` uses `FROM python:3.13-slim` (was `python:3.12-slim`)
- [ ] **AC-4**: `.devcontainer/devcontainer.json` specifies `"version": "3.13"` for the Python feature (was `"3.12"`)
- [ ] **AC-5**: The CLAUDE.md Tech Stack section references `.python-version` as the source of truth for the Python version instead of hardcoding a specific version number. The reference should make clear that `.python-version` is authoritative and the other locations (Dockerfile, devcontainer.json, pyproject.toml) must stay in sync with it.
- [ ] **AC-6**: All packages in `requirements.txt` install without errors on Python 3.13 (`pip install -r requirements.txt` succeeds)
- [ ] **AC-7**: All existing tests pass on Python 3.13 (`pytest` exits with code 0, no new failures)
- [ ] **AC-8**: No other files are modified beyond the five listed in "Files to Create or Modify"

## Technical Approach
1. Create `.python-version` with content `3.13`.
2. Create `pyproject.toml` with minimal content (see epic Technical Notes for exact shape).
3. Edit `Dockerfile` line 11: change `FROM python:3.12-slim` to `FROM python:3.13-slim`.
4. Edit `.devcontainer/devcontainer.json`: change `"version": "3.12"` to `"version": "3.13"` in the Python feature.
5. Edit CLAUDE.md Tech Stack section: replace the hardcoded Python version reference with a reference to `.python-version`. For example: "Python (version governed by `.python-version`)" instead of "Python 3.12".
6. Run `pip install -r requirements.txt` and `pytest` to verify compatibility.
