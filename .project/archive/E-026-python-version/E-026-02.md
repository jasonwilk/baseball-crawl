# E-026-02: Document Python Version Maintenance Policy in CLAUDE.md

## Epic
[E-026: Python Version Governance -- Migrate to 3.13](./epic.md)

## Status
`DONE`

## Description
After this story is complete, CLAUDE.md will contain a "Python Version Policy" section that documents where the Python version is specified, when to consider upgrading, and how to verify compatibility. This gives future contributors (human or AI) a clear playbook for version maintenance.

## Context
The project is migrating from a version-unaware state (3.12 hardcoded in two files, no policy) to a governed state (3.13 with `.python-version` as source of truth). E-026-01 handles the migration itself. This story documents the ongoing maintenance policy so the project does not drift back into an ungoverned state. The policy section is deliberately brief -- a few bullet points, not a runbook.

## Acceptance Criteria
- [ ] **AC-1**: CLAUDE.md contains a new section titled "## Python Version Policy" placed after the "## Tech Stack" section and before "## Key Metrics We Track"
- [ ] **AC-2**: The section documents where the version is specified: `.python-version` (source of truth), `pyproject.toml` (`requires-python`), `Dockerfile` (`FROM` tag), `.devcontainer/devcontainer.json` (Python feature version). It must be clear that `.python-version` is authoritative and the other three must match.
- [ ] **AC-3**: The section documents when to consider upgrading: (a) annually when a new stable Python release has been out for 6+ months and key deps support it, or (b) when a dependency drops support for the current version
- [ ] **AC-4**: The section documents how to verify before upgrading: check dep compatibility (pyreadiness.org or per-package classifiers), run `pip install -r requirements.txt`, run `pytest`, check for deprecation warnings
- [ ] **AC-5**: The section includes a one-sentence rationale for the current version (3.13 chosen over 3.14 due to httpx/jinja2 lacking official 3.14 support markers; over 3.12 because all deps support 3.13 and it extends the EOL window)
- [ ] **AC-6**: The section is concise -- no more than 30 lines including blank lines and bullet formatting
- [ ] **AC-7**: No other sections of CLAUDE.md are modified (the Tech Stack update is handled by E-026-01)
