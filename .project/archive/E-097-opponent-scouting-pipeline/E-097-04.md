## Description
After this story is complete, the project's context layer will reflect the new opponent scouting capability: CLAUDE.md will document the `bb data scout` command and the scouting data flow, and relevant context-layer files will reference the scouting pipeline as part of the project's data architecture.

## Context
E-097 adds a significant new capability: opponent scouting via the GameChanger API. The context layer needs to be updated so that agents and operators are aware of the scouting pipeline, its CLI entry point, and where the data flows. This is the final story in the epic, ensuring the context layer stays in sync with the implementation.
