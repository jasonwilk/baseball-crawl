# E-007-07: PM Refinement Mode -- Expert Consultation Protocol

## Epic
[E-007: Orchestrator Workflow Discipline](epic.md)

## Status
`DONE`

## Description
After this story is complete, the PM agent prompt contains an explicit "Refinement Mode" section that tells the PM when and how to consult domain experts (baseball-coach, api-scout, claude-architect, data-engineer) during epic formation -- before stories are written, before acceptance criteria are drafted. The section makes consultation a first-class activity in the PM workflow, not an afterthought, and gives the PM a concrete decision rule for when consultation is required versus when the PM can form an epic alone.

## Context
E-007 established the Dispatch Mode workflow: PM gets triggered by "start epic X" and dispatches stories to implementing agents. That covered the execution side of the workflow. What was not addressed is the formation side: when the PM is creating or refining an epic for the first time, what is the protocol for bringing in domain experts?

Currently, `project-manager.md` says:
- "You do NOT make architectural decisions in isolation -- you propose and document them in epics for discussion." (vague -- no action required)
- "How to Interact with Other Agents" section says PM writes specs that agents execute. No mention of consulting agents during spec formation.

The CLAUDE.md Workflow Contract step 2 says "PM consults domain experts as needed" -- but this is framed around dispatch authorization, not epic formation. The PM has no explicit guidance for the refinement phase.

The gap this creates: when a new epic request comes in that has technical complexity (e.g., a new crawler, a new schema, a new API integration), the PM may write stories with speculative or incorrect technical approach sections because it did not consult the right expert. This happened implicitly in E-006 (had to rewrite stories after the architect's design doc existed). Encoding the consultation norm prevents this.

### Who Gets Consulted and When

The decision tree is straightforward:
- **baseball-coach**: Consult when the epic involves data that coaches will use, new statistics, scouting features, or any domain question about what coaches actually need. Ask: "What data matters here? What does a coach actually want to see?"
- **api-scout**: Consult when the epic involves GameChanger API endpoints, data availability, or anything that depends on what the API actually returns. Ask: "Does the API support this? What does the response look like?"
- **claude-architect**: Consult when the epic involves agent infrastructure, new agent designs, CLAUDE.md changes, rules, skills, or significant architectural decisions. Ask: "What is the right architecture here?"
- **data-engineer**: Consult when the epic involves database schema design, D1 migrations, or ETL patterns that affect the data model. Ask: "Does this schema support the queries we need?"
- **No consultation needed**: When the epic is purely process/workflow (like E-007 itself), purely PM-domain work (idea capture, story refinement), or clearly within scope of existing specifications.

## Technical Approach

This is a pure prompt-editing story. No code. No new files. Two files are modified:

1. `/.claude/agents/project-manager.md` -- Add the "Refinement Mode" section. The most natural placement is immediately before "Dispatch Mode", since Refinement Mode is the PM's first mode and Dispatch Mode is the second. The section name should clearly parallel "Dispatch Mode" to make the two-mode model explicit.

2. `/CLAUDE.md` -- Audit the Workflow Contract step 2. The current text is: "PM reviews and consults. PM reads the relevant epic and stories, consults domain experts (baseball-coach, api-scout) as needed, and authorizes implementation." This is framed around dispatch-time review. Add a note or expand the step to clarify that expert consultation happens during both epic formation (Refinement Mode) and pre-dispatch review. Keep the change minimal -- one or two sentences is sufficient if the concept is already implied.

### Suggested Section Structure for `project-manager.md`

```
## Refinement Mode

Refinement Mode is the PM's first mode. It is active whenever the PM is creating or refining an epic -- from the moment an idea is promoted to an epic through the point where all stories have testable acceptance criteria and the epic is marked READY.

### When to Consult Experts

Before writing stories for a new epic, assess whether expert consultation is needed:

| Epic Domain | Consult | Question to Ask |
|------------|---------|-----------------|
| Data coaching needs, stats, scouting | baseball-coach | "What data does a coach actually need here? What does this report/feature need to show?" |
| GameChanger API endpoints, data availability | api-scout | "Does the API support this? What does the response look like? Are there rate limits or auth concerns?" |
| Agent infrastructure, CLAUDE.md, rules, architecture | claude-architect | "What is the right architecture here? How should this agent/system be structured?" |
| Database schema, D1 migrations, ETL | data-engineer | "Does this schema support the queries we need? What is the right migration approach?" |
| Pure process/workflow, PM-domain, well-scoped existing work | None | Note "No consultation required -- [reason]" in Background & Context |

### Consultation Procedure

1. Identify which experts are needed based on the table above.
2. Use the Task tool to invoke each expert agent with a specific question. Be precise: provide the epic title, the proposed scope, and the exact question you need answered.
3. Wait for the response. Incorporate the answer into the epic's Technical Notes or Background & Context before writing any stories.
4. Consultation happens BEFORE stories are written -- not after. Stories written without domain input will have speculative technical approach sections that need revision.

### When Consultation Is Not Required

If the epic is clearly within the scope of existing specifications (e.g., refining an existing story, adding a process improvement like E-007, or promoting an idea that has already been thoroughly discussed), the PM may proceed without formal consultation. In that case, add a one-line note to the epic's Background & Context: "No expert consultation required -- [reason]."
```
