## Description
After this story is complete, a new rules file at `/.claude/rules/workflow-discipline.md` will exist. This file fires for all file paths (global scope) and states the workflow gate rule in plain language: implementing agents must have a story file reference before beginning any task. It also restates the PM-gatekeeper principle for any agent that encounters the rules context. This is a belt-and-suspenders complement to the agent prompt changes in E-007-02 and E-007-03 -- the rules file ensures the contract is visible even in contexts where individual agent prompts might not surface it.

## Context
Claude Code rules files are markdown documents in `/.claude/rules/` with a YAML front matter `paths:` field that controls when they are injected. A rules file with `paths: ["**"]` (or `["**/*"]`) applies globally -- it is included in every agent's context for any file operation. This makes it the right place for project-wide behavioral norms that apply regardless of which agent is running.

Existing rules files in this project:
- `.claude/rules/crawling.md` (paths: src/*client*, etc.)
- `.claude/rules/project-management.md` (paths: epics/**, .project/**)
- `.claude/rules/python-style.md` (paths: **/*.py)
- `.claude/rules/testing.md` (paths: tests/**, **/*test*.py)
- `.claude/rules/ideas-workflow.md` (paths: .project/ideas/**, etc.)

The workflow discipline rule is deliberately global because it governs agent behavior across all file types and directories.
