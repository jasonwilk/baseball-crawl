## Technical Approach
Edit `/Users/jason/Documents/code/baseball-crawl/CLAUDE.md`. Find the "## Agent Ecosystem" section (or its equivalent). After the agent role table, add a new `### Workflow Contract` subsection. Do not rewrite the existing table or surrounding content.

The five-step contract to encode (exact wording may be adjusted for clarity, but substance must be preserved):
1. **Orchestrator routes to PM.** All work-initiation requests go to project-manager first.
2. **PM reviews and consults.** PM reads the relevant epic/stories and consults domain experts (baseball-coach, api-scout) as needed before authorizing implementation.
3. **"Ready for dev" = Status: TODO.** A story file with `Status: TODO` is PM's authorization signal. No story file = no implementation work.
4. **"Start epic X" dispatches through PM.** PM reads the epic, identifies eligible stories, and dispatches each to the correct implementing agent via Task tool.
5. **Implementing agents require a story reference.** general-dev and data-engineer must have a story file path before beginning any task.
