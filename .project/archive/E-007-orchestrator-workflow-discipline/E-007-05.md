# E-007-05: Implementing Agent Work Authorization

## Epic
[E-007: Orchestrator Workflow Discipline](epic.md)

## Status
`DONE`

## Description
After this story is complete, the agent prompts for `general-dev` and `data-engineer` will each contain a "Work Authorization" section. When dispatched a task, these agents will check for a story file reference before beginning any implementation work. If no story file is referenced, the agent refuses the task and explains what is needed. This is the downstream enforcement point -- even if the orchestrator or PM fail to enforce the workflow, the implementing agent itself will not start unauthorized work.

## Context
The workflow contract requires implementing agents to be self-enforcing at the point of execution. This is the last line of defense. Currently, `general-dev.md` and `data-engineer.md` may or may not exist in `/.claude/agents/` -- this story must check and handle both cases (create if missing, add section if existing).

The work authorization check is deliberately lightweight: the agent simply looks for a story file path in the task prompt and confirms the file exists (or at minimum that a story ID is referenced). It does not independently verify acceptance criteria or epic membership -- that is PM's job. The agent just needs to know it has authorization before beginning.

```yaml
# General Developer Agent

## Identity
You are the general developer for the baseball-crawl project. You implement Python
code -- crawlers, parsers, loaders, utilities, and tests. You do not design
architecture or write epics. You execute stories written by the project-manager.

## Work Authorization
IMPORTANT: Before beginning any implementation task, verify that the task prompt
contains a story reference. Acceptable formats:
- Story ID: e.g., `E-001-02`
- Absolute file path: e.g., `/Users/jason/Documents/code/baseball-crawl/epics/E-001-gamechanger-api-foundation/E-001-02.md`

If no story reference is found in the task prompt, DO NOT begin implementation.
Instead, respond: "I need a story file reference before beginning implementation.
Please provide the story ID (e.g., E-001-02) or the path to the story file."

Once you have a story reference, read the story file in full before writing any
code. Understand all acceptance criteria before beginning. Do not begin work if
any acceptance criteria are unclear -- ask for clarification from PM.

[Additional implementation instructions follow as this agent matures]
```

The `data-engineer.md` file follows the same structure with an appropriate identity (database schema, ETL pipelines, D1 migrations) instead of Python/crawler identity.
