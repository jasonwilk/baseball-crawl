# E-025-01: Update devcontainer.json for Python/Docker Stack

## Epic
[E-025: Devcontainer Update -- Align with Python/Docker Stack](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the `.devcontainer/devcontainer.json` will specify the correct
devcontainer features for the current Python/Docker stack: Python 3.12 runtime, Docker-in-Docker
for running `docker compose up`, and Node.js retained for OpenAI Codex CLI. The `postCreateCommand`
will install Python dependencies, Claude Code CLI, and Codex CLI. Port forwarding will be configured
for the FastAPI app.

## Context
The devcontainer was created before E-009 selected Docker Compose + FastAPI (Python) as the tech
stack. It has no Python runtime and no Docker access. Node.js is present and must be kept -- it is
required by the OpenAI Codex CLI (`@openai/codex` is an npm package). This story adds the missing
Python and Docker features alongside the existing Node.js feature. The companion story E-025-02
verifies the updated devcontainer actually builds and works.

- [ ] **AC-1**: The `features` section includes `ghcr.io/devcontainers/features/python:1` with `"version": "3.12"`
- [ ] **AC-2**: The `features` section includes `ghcr.io/devcontainers/features/docker-in-docker:2` (with default settings)
- [ ] **AC-3**: The `features` section retains `ghcr.io/devcontainers/features/node:1` (required for Codex CLI)
- [ ] **AC-4**: The `features` section retains `ghcr.io/rocker-org/devcontainer-features/apt-packages:1` with `"packages": "git,curl,jq"`
