# E-025-02: Verify Devcontainer Builds and Runs the Full Stack

## Epic
[E-025: Devcontainer Update -- Align with Python/Docker Stack](./epic.md)

## Status
`DONE`

## Description
After this story is complete, the updated devcontainer configuration has been verified to build
successfully, provide Python 3.12 and Docker access, install all dependencies, and run
`docker compose up` with the app responding on the expected port. This is a verification and
fix-up story -- if E-025-01's config does not work on first build, this story iterates until it does.

## Context
E-025-01 updates the devcontainer.json based on known-good devcontainer feature specifications.
However, devcontainer builds can fail due to feature version incompatibilities, Docker-in-Docker
startup timing, or missing prerequisites. This story performs the actual build-and-test cycle
and resolves any issues discovered. It is intentionally separate from E-025-01 so the config
change (a file edit) is cleanly separated from the interactive verification (which requires
running the devcontainer).

## Acceptance Criteria
- [ ] **AC-1**: `devcontainer build` completes without errors (or the devcontainer opens successfully in VS Code / Codespaces)
- [ ] **AC-2**: Inside the devcontainer, `python --version` reports Python 3.12.x
- [ ] **AC-3**: Inside the devcontainer, `docker --version` reports a working Docker CLI
- [ ] **AC-4**: Inside the devcontainer, `docker compose version` reports Docker Compose is available
- [ ] **AC-5**: Inside the devcontainer, `pip install -r requirements.txt` succeeds (or was already run by postCreateCommand)
- [ ] **AC-6**: Inside the devcontainer, `pytest` runs and existing tests pass
- [ ] **AC-7**: Inside the devcontainer, `docker compose up -d` starts the app stack and `curl http://localhost:8000/health` returns a 200 response
- [ ] **AC-8**: Inside the devcontainer, the `.claude` bind mount is present at `/home/vscode/.claude`
- [ ] **AC-9**: Inside the devcontainer, `node --version` reports a working Node.js runtime and `npx @openai/codex --version` (or `codex --version`) confirms Codex CLI is installed
- [ ] **AC-10**: If any AC fails, the issue is diagnosed, `devcontainer.json` is updated to fix it, and verification is re-run until all ACs pass

## Technical Approach
This story requires interactive execution -- building the devcontainer and running commands inside it.
The implementing agent should:
1. Rebuild the devcontainer using `devcontainer build` or by reopening in VS Code Remote Containers
2. Open a terminal inside the devcontainer
3. Run each AC's verification command
4. If any fails, diagnose the issue, update `devcontainer.json`, rebuild, and re-verify

The most likely issues:
- Docker-in-Docker feature may need specific version pinning
- `postCreateCommand` may fail if pip is not on PATH before the Python feature initializes
- Port forwarding may need `portsAttributes` configuration for automatic forwarding

## Notes
- This story is inherently interactive -- it cannot be completed by file editing alone. The
  implementing agent must actually build and run the devcontainer.
- If the user's machine does not have the `devcontainer` CLI installed, VS Code's "Reopen in
  Container" command is the alternative. The agent should ask the user to perform the build
  if tool access is limited.
- On macOS, Docker Desktop must be running for Docker-in-Docker to work (it needs a host Docker
  daemon to bootstrap the inner daemon).
