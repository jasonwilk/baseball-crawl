# E-119-01: Document PII Scanner Invocation in CLAUDE.md

## Epic
[E-119: Fix PII Scanner Discoverability](epic.md)

## Status
`DONE`

## Description
After this story is complete, agents working in any part of the codebase can discover the PII scanner's location and manual invocation syntax from CLAUDE.md. The current gap -- where the scanner path is only documented in `pii-safety.md` (scoped to `src/safety/**`, `.githooks/**`, and `.claude/hooks/pii-check.sh`) -- is closed.

## Context
During E-116 dispatch, the main session tried `python scripts/pii_scan.py` because no always-loaded context-layer file told it where the scanner actually lives. The pre-commit hooks work fine, but agents performing manual scans have no way to discover the correct path without grepping. This story adds that discoverability by placing the scanner path in the Security Rules section of CLAUDE.md, per CA's recommendation.

## Acceptance Criteria
- [x] **AC-1**: The PII scanner path (`src/safety/pii_scanner.py`) and at least one invocation example (e.g., `python3 src/safety/pii_scanner.py --staged`) appear in the Security Rules section of `CLAUDE.md`
- [x] **AC-2**: No changes to files in `src/`, `scripts/`, `tests/`, `migrations/`, or hook files (`.githooks/`, `.claude/hooks/`)
- [x] **AC-3**: The added text is at most 3 lines and is placed in the Security Rules section of `CLAUDE.md`

## Technical Approach
Add a bullet to the Security Rules section of `CLAUDE.md` with the scanner path and the `--staged` invocation mode. Place it near the existing credential-safety bullets. The addition should be concise -- a single bullet point is sufficient.

## Notes
The scanner's own docstring documents three invocation modes:
```
python3 src/safety/pii_scanner.py --staged       # scan git staged files
python3 src/safety/pii_scanner.py --stdin         # read file paths from stdin
python3 src/safety/pii_scanner.py file1 file2     # scan specific files
```
