# E-054-01: Header refresh script

## Epic
[E-054: Header Parity Refresh from MITM Captures](epic.md)

## Status
`DONE`

## Description
After this story is complete, a Python script reads the mitmproxy header capture report and rewrites `src/http/headers.py` to match the captured header fingerprints. The operator runs one command after a proxy session and the header dicts are current. Credential headers, per-request headers, and connection-level headers are excluded from the update. Additionally, the `header_capture.py` addon is fixed to diff each traffic source against the correct canonical dict (web vs. BROWSER_HEADERS, ios vs. MOBILE_HEADERS) instead of comparing everything against BROWSER_HEADERS.

## Context
The header capture addon already produces `proxy/data/header-report.json` with the exact headers seen in real browser and iOS app traffic. The parity report script shows drift. This story closes the loop: instead of showing drift, fix it.

## Acceptance Criteria
- [ ] **AC-1**: `python scripts/proxy-refresh-headers.py` (no flags) reads the header report and prints a diff showing what would change in `src/http/headers.py`, without writing any files (dry-run default).
- [ ] **AC-2**: `python scripts/proxy-refresh-headers.py --apply` reads the header report and writes the updated `src/http/headers.py`, then prints a summary of changes.
- [ ] **AC-3**: The script maps `"web"` source headers to `BROWSER_HEADERS` and `"ios"` source headers to `MOBILE_HEADERS`. Headers from `"unknown"` sources are ignored.
- [ ] **AC-4**: The following headers are excluded from the update even if present in the capture: credential headers (`gc-token`, `gc-device-id`, `gc-signature`, `gc-app-name`, `cookie`), per-request headers (`content-type`, `accept`, `gc-user-action-id`, `gc-user-action`, `x-pagination`), and connection-level headers (`host`, `connection`, `content-length`, `transfer-encoding`, `te`, `trailer`, `upgrade`, `proxy-authorization`, `proxy-authenticate`).
- [ ] **AC-5**: The generated `src/http/headers.py` preserves: the module docstring (with updated "Source" date), the `from __future__ import annotations` import, the dict variable names `BROWSER_HEADERS` and `MOBILE_HEADERS`, and valid Python formatting (importable without errors).
- [ ] **AC-6**: If the header report file does not exist, the script prints an actionable error message ("No capture data found. Run mitmproxy and capture GameChanger traffic first.") and exits with code 1.
- [ ] **AC-7**: If the header report contains only one source (e.g., web but no ios), the script updates only the corresponding dict and leaves the other unchanged.
- [ ] **AC-8**: The script tries `proxy/data/current/header-report.json` first (E-052 session-aware path), then falls back to `proxy/data/header-report.json` (flat path).
- [ ] **AC-9**: `header_capture.py`'s `build_report` function is updated to accept both `BROWSER_HEADERS` and `MOBILE_HEADERS` and diff each source against the correct canonical dict: `"web"` -> `BROWSER_HEADERS`, `"ios"` -> `MOBILE_HEADERS`, `"unknown"` -> `BROWSER_HEADERS`.
- [ ] **AC-10**: Existing `header_capture.py` tests are updated to verify per-source canonical dict selection. The `proxy-report.sh` output is now accurate for iOS traffic.
- [ ] **AC-11**: Tests for the refresh script verify: dry-run output, apply mode writing, excluded header filtering, single-source partial update, missing report file error, and round-trip (generated file is importable and contains correct values).

## Technical Approach
- Write `scripts/proxy-refresh-headers.py` as a standalone script (like `check_credentials.py`).
- Read the header report JSON. Extract `captured_headers` for each source.
- Filter out excluded headers (use a set constant, similar to `_CREDENTIAL_HEADERS` and `_SKIP_DIFF_HEADERS` in `header_capture.py`).
- Read the current `src/http/headers.py` to get the existing dicts for sources not present in the capture.
- Generate the new `headers.py` content using string templates or f-strings. The file is small and well-structured -- no need for AST manipulation.
- For the "Source" date line in the docstring, update it to today's date.
- Diff: compare old file content vs. new file content and print the unified diff.
- `--apply`: write the new content. No `--apply`: print diff only.
- For the `header_capture.py` fix: update `build_report` signature to accept a mapping of source -> canonical dict (e.g., `{"web": BROWSER_HEADERS, "ios": MOBILE_HEADERS}`). The `HeaderCapture` class imports both dicts and passes them to `build_report`. Default for unknown sources: `BROWSER_HEADERS`.
- Note: this script will eventually be wrapped by E-055's unified `bb` CLI (separate epic, no dependency).

## Notes
- The generated `headers.py` should use the same formatting conventions as the current file: double-quoted strings, parenthesized multi-line strings for long values (User-Agent), alphabetical key ordering within each dict.
- The script should NOT import from `src.http.headers` at runtime (since it is about to overwrite that file). Read the existing file as text for the "preserve unchanged dict" logic.
