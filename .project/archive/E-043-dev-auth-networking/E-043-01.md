# E-043-01: Fix dev auth defaults and verify Traefik routing

## Epic
[E-043: Dev Environment Auth and Networking Fix](epic.md)

## Status
`DONE`

## Description
After this story is complete, a developer running `docker compose up` in a devcontainer can visit `http://baseball.localhost:8001/auth/login`, submit their email, and click the magic link (logged to stdout) to complete the login flow -- without setting any env vars. The APP_URL, WEBAUTHN_ORIGIN, and WEBAUTHN_RP_ID defaults will match the actual dev access pattern (`baseball.localhost:8001`), and `.env.example` will document the correct defaults.

## Context
The app container listens on port 8000 internally. Docker Compose maps `127.0.0.1:8001:8000` for direct access from the devcontainer. The user accesses the app at `http://baseball.localhost:8001` (confirmed). The current auth code defaults all point to `localhost:8000`, which is wrong on both hostname and port:

- `APP_URL` defaults to `http://localhost:8000` -- magic links go to the wrong URL
- `WEBAUTHN_ORIGIN` defaults to `http://localhost:8000` -- passkey verification fails (origin mismatch)
- `WEBAUTHN_RP_ID` defaults to `localhost` -- passkey registration fails (RP ID mismatch with `baseball.localhost`)

## Acceptance Criteria

- [ ] **AC-3**: Given no `WEBAUTHN_RP_ID` env var is set, when `_get_webauthn_rp_id()` is called, then it returns `baseball.localhost`.

- [ ] **AC-5**: The `.env.example` file documents `WEBAUTHN_ORIGIN` with `http://baseball.localhost:8001` as the dev default.
- [ ] **AC-6**: The `.env.example` file documents `WEBAUTHN_RP_ID` with `baseball.localhost` as the dev default.
- [ ] **AC-7**: Existing tests continue to pass (`pytest` exits 0 with no new failures).

## Technical Approach
1. In `src/api/routes/auth.py`:
   - Change `_APP_URL_DEFAULT = "http://localhost:8000"` to `_APP_URL_DEFAULT = "http://baseball.localhost:8001"`.
   - Change the default in `_get_webauthn_origin()` from `"http://localhost:8000"` to `"http://baseball.localhost:8001"`.
   - Change the default in `_get_webauthn_rp_id()` from `"localhost"` to `"baseball.localhost"`.
2. In `.env.example`:
   - Update the `APP_URL` comment block to show `http://baseball.localhost:8001` as the dev default.
   - Update the `WEBAUTHN_ORIGIN` comment block to show `http://baseball.localhost:8001` as the dev default.
   - Update the `WEBAUTHN_RP_ID` comment block to show `baseball.localhost` as the dev default.
   - Explain the dev access pattern: port 8001 is direct to the app container (bypasses Traefik); port 8000 goes through Traefik (requires `Host: baseball.localhost`).
3. Run `pytest` to confirm no regressions.
4. If the stack is running, rebuild (`docker compose up -d --build app`) and verify:
   - `curl http://localhost:8001/health` returns 200.
   - Check the magic link URL in app logs uses `http://baseball.localhost:8001`.
   - Attempt `curl -H "Host: baseball.localhost" http://localhost:8000/health` to verify Traefik routing still works. Document the result.
