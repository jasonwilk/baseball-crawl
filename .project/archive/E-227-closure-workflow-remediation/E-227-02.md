# E-227-02: Dispatch prerequisites — committed-plan check

## Epic
[E-227: Closure Workflow Structural Remediation](../E-227-closure-workflow-remediation/epic.md)

## Status
`DONE`

## Description
After this story is complete, the implement skill's Prerequisites section refuses to proceed with dispatch when the target epic's directory at `epics/E-NNN-slug/` has uncommitted changes in the main checkout — preventing the epic worktree from branching from a HEAD that lacks the plan files. A handoff exception skips the check when dispatch arrives via the plan skill's compound-trigger path, because plan skill Step 2a already owns the commit invariant on that path.

## Context
Discovered during E-226 dispatch on 2026-04-20. The planning PM wrote epic files and updated memory but never committed them. Dispatch PM was spawned in an epic worktree that branched from HEAD — which didn't contain the plan files. Main session had to improvise: commit the plan on main, destroy + rebuild the worktree. The plan skill's Step 2a is correctly designed (atomic planning commit with user approval, skip behavior, compound-dispatch downgrade) — the gap is that the implement skill's Prerequisites doesn't backstop the invariant. This story adds the backstop. Per Technical Notes section TN-5 (specification) and TN-6 (verification strategy for the narrative `handoff_from_plan` flag).

## Acceptance Criteria

- [ ] **AC-1**: `.claude/skills/implement/SKILL.md` Prerequisites section gains a new clause (added after the existing epic-status check) that runs `cd /workspaces/baseball-crawl && git status --porcelain -- epics/E-NNN-slug/`. If the output is non-empty, dispatch refuses — per Technical Notes section TN-5 (exact specification).

- [ ] **AC-2**: The refusal message lists the specific uncommitted file paths from the `git status --porcelain` output and includes a concrete remediation command (e.g., `git add epics/E-NNN-slug/ && git commit -m "feat(E-NNN): plan <title> (READY)"`) before retrying dispatch.

- [ ] **AC-3**: The check is skipped when the implement skill was loaded via plan skill Phase 5 Step 3c handoff (the compound-trigger "plan and dispatch" path). On that path, plan skill Step 2a already owns the commit invariant. The skip condition is documented in the Prerequisites clause in terms of the trigger pattern ("loaded via plan skill Phase 5 Step 3c handoff" or "compound trigger") rather than an abstract flag — per Technical Notes section TN-6. The check runs on the standalone "implement E-NNN" invocation path.

- [ ] **AC-4**: Prerequisites does NOT auto-commit. The new clause explicitly states that user approval for planning commits is preserved at plan skill Step 2a and that Prerequisites refuses rather than auto-committing. The plan skill's `.claude/skills/plan/SKILL.md` is not modified by this story.

- [ ] **AC-5**: The implement skill's Workflow Summary's Prerequisites block reflects the new check. If the Prerequisites block in the Workflow Summary enumerates prerequisite checks (epic exists, epic status, committed-plan), the new check is listed alongside the existing ones.

## Technical Approach

Add a new numbered Prerequisites clause immediately after the existing epic-status check and before Phase 0. Exact specification language is given in Technical Notes section TN-5; implementer can carry that language into the skill with wording polish as needed but must preserve the `git status --porcelain -- epics/E-NNN-slug/` command, the non-empty-refuses contract, and the handoff-exception framing.

The handoff-path skip is instruction-level (main session tracks `handoff_from_plan` across skill loads by narrative convention — no shell variable or runtime container). Frame the skip condition in the Prerequisites clause in terms of the trigger pattern so the verification is trigger-pattern-based, consistent with how every other `handoff_from_plan` branch in the implement skill is phrased today.

Do not modify plan skill Step 2a or any other plan-skill content. The fix is a backstop at the implement-skill Prerequisites layer, not a change to the planning commit path.
