# E-195-04: CLI Integration and Pipeline Wiring

## Epic
[E-195: Plays Data Ingestion Pipeline -- FPS% and QAB](epic.md)

## Status
`DONE`

## Description
After this story is complete, operators can run `bb data crawl --crawler plays` and `bb data load --loader plays` to execute the plays pipeline. The plays crawler and loader are wired into the existing `bb data` CLI infrastructure alongside existing crawlers (schedule, spray-chart) and loaders (game, spray-chart, etc.).

## Context
The `bb data` CLI already supports `--crawler` and `--loader` flags for targeted pipeline runs (e.g., `bb data crawl --crawler spray-chart`). This story adds `plays` as a new valid value for both flags, wiring `PlaysCrawler` and `PlaysLoader` into the existing dispatch logic in `src/cli/data.py`.

## Acceptance Criteria
- [ ] **AC-1**: `bb data crawl --crawler plays` invokes `PlaysCrawler.crawl_all()` and reports the crawl result (files written, skipped, errors).
- [ ] **AC-2**: `bb data load --loader plays` invokes `PlaysLoader.load_all()` to load all cached plays files and reports the load result (games processed, plays inserted, skipped, errors).
- [ ] **AC-3**: `bb data crawl` (without `--crawler` flag, i.e., the default "all crawlers" path) includes the plays crawler in its execution.
- [ ] **AC-4**: `bb data load` (without `--loader` flag, i.e., the default "all loaders" path) includes the plays loader in its execution.
- [ ] **AC-5**: When `bb data crawl --help` is run, `plays` appears in the list of valid `--crawler` values. Same for `bb data load --help` and `--loader` values.
- [ ] **AC-6**: In the default "all loaders" execution path, the plays loader executes AFTER the game loader (which populates the `games` rows that `plays` references via FK).
- [ ] **AC-7**: Tests verify the CLI integration: at minimum, a subprocess smoke test for `bb data crawl --help` and `bb data load --help` confirming `plays` appears in the output.

## Technical Approach
Modify `src/cli/data.py` to add `plays` to the `--crawler`/`--loader` flag dispatch. Register `PlaysCrawler` in `src/pipeline/crawl.py` (`_CRAWLER_NAMES` list and `_build_crawlers()` factory) and add a plays loader runner function to `src/pipeline/load.py` with correct ordering (after game loader). Follow the existing patterns for how `spray-chart` was registered in both files.
