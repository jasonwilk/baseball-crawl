# E-146-03: Wire resolve_unlinked into CLI

## Epic
[E-146: Crawl Pipeline Guards and Null-Progenitor Auto-Resolution](epic.md)

## Status
`DONE`

## Description
After this story is complete, `bb data resolve-opponents` will call `resolver.resolve_unlinked()` after `resolver.resolve()`, and the CLI output will include the follow-bridge resolution counts. The admin UI's scouting sync trigger is not affected (it already skips unlinked opponents via the `public_id IS NOT NULL` filter in `scout_all()`).

## Context
E-146-02 adds `resolve_unlinked()` to `OpponentResolver` but does not wire it into any entry point. This story connects the new method to the `bb data resolve-opponents` CLI command, so the full pipeline runs: resolve (progenitor chain) → resolve_unlinked (follow-bridge chain) → user sees combined results. Per Technical Notes TN-4 in the epic.

## Acceptance Criteria
- [ ] **AC-1**: `bb data resolve-opponents` calls `resolver.resolve_unlinked()` after `resolver.resolve()` completes.
- [ ] **AC-2**: The CLI output includes follow-bridge resolution counts (`resolved`, `follow_bridge_failed`, `errors`) from the `ResolveResult` returned by `resolve_unlinked()`, in addition to the existing progenitor resolution counts from `resolve()`.
- [ ] **AC-3**: If `resolve()` raises `CredentialExpiredError`, `resolve_unlinked()` is not attempted (the credential is expired; no point continuing).
- [ ] **AC-4**: If `resolve_unlinked()` raises an unexpected exception, it is caught and logged (does not crash the CLI). Exit code is 1 if either `resolve()` or `resolve_unlinked()` has `errors > 0` or raises an exception; 0 only if both complete with zero errors.
- [ ] **AC-5**: (Regression guard) The `--dry-run` flag skips `resolve_unlinked()`. This is existing behavior -- `--dry-run` already exits before `resolve()` is called, so `resolve_unlinked()` is naturally skipped. Verify no regression.
- [ ] **AC-6**: Tests confirm the wiring: `resolve_unlinked()` is called after `resolve()`, and error handling is correct.

## Technical Approach
Modify the `resolve_opponents` function in `src/cli/data.py` to call `resolver.resolve_unlinked()` after the existing `resolver.resolve()` call. Combine the results for display. The `--dry-run` path already returns early before `resolve()` is called, so it naturally skips `resolve_unlinked()` too.
