# E-141-01: Worktree Guard Hook

## Epic
[E-141: Single Epic Worktree Model and Hook Enforcement](epic.md)

## Status
`DONE`

## Description
After this story is complete, a PreToolUse hook will block Write and Edit operations to implementation file paths (`src/`, `tests/`, `migrations/`, `scripts/`) when the target is the main checkout. This provides deterministic enforcement that implementation work must happen in a worktree.

## Context
The two-tier worktree model relied on agents following instructions to work in worktrees and avoid contaminating main. Six leak pathways in the first concurrent dispatch proved this is insufficient. A hook shifts enforcement from "agents should" to "agents cannot" — a Write to `src/foo.py` from the main checkout is blocked before it executes, regardless of what the agent's instructions say. The hook guards implementation paths only; context-layer isolation during dispatch is enforced by the implement skill (all stories get worktrees), not the hook — preserving direct-routing exceptions for CA and docs-writer outside dispatch.

## Acceptance Criteria
- [ ] **AC-1**: A PreToolUse hook on `Write` blocks writes to files under `src/`, `tests/`, `migrations/`, or `scripts/` when the file path starts with `/workspaces/baseball-crawl/` (main checkout). Writes to the same paths under `/tmp/.worktrees/` pass. Per Technical Notes TN-1.
- [ ] **AC-2**: A PreToolUse hook on `Edit` applies the same blocking logic as AC-1. Per Technical Notes TN-1.
- [ ] **AC-3**: Writes to non-protected paths from main are NOT blocked: `.claude/agent-memory/`, `.claude/hooks/`, `epics/`, `.project/`, `docs/`, `.claude/skills/`, `.claude/rules/`, `.claude/agents/`, `CLAUDE.md`, `.claude/settings.json`, `.claude/settings.local.json`. Per Technical Notes TN-1.
- [ ] **AC-4**: The hook uses the standard stdin JSON protocol (same as `pii-check.sh`): read tool input from stdin, output deny JSON on block, exit 0 silently on pass.
- [ ] **AC-5**: Hook entries are registered in `.claude/settings.json` under the `PreToolUse` key with matchers for `Write` and `Edit`.
- [ ] **AC-6**: Blocked operations produce a clear error message that explains WHY the write was blocked and what the agent should do instead. Per Technical Notes TN-1.

## Technical Approach
Create one hook script (`.claude/hooks/worktree-guard.sh`) and register it in `.claude/settings.json` for both Write and Edit matchers. The script follows the same pattern as the existing `pii-check.sh` hook. The detection logic, protected paths, and error message are specified in Technical Notes TN-1.

## Notes
- The Write/Edit hook only protects implementation paths (`src/`, `tests/`, `migrations/`, `scripts/`). Context-layer paths (`.claude/`, `CLAUDE.md`, `docs/`) are intentionally NOT protected because CA and other agents may work on main outside dispatch (direct-routing exceptions). Context-layer isolation during dispatch is enforced by the implement skill, not the hook.
- The `git add -A` contamination problem is resolved structurally by the worktree model (nothing stray writes to main during dispatch), so no `git add` blocker hook is needed. See Technical Notes TN-2.
- Epic worktree uniqueness does not need a hook — `git worktree add` already fails if the path exists.
