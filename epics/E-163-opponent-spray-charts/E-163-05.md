# E-163-05: Opponent Page Spray Charts

## Epic
[E-163: Full-Season Opponent Spray Charts and Dashboard Display](epic.md)

## Status
`TODO`

## Description
After this story is complete, the opponent detail page will show a team-aggregate spray chart card, and the opponent print page will show per-player spray charts instead of "coming soon" placeholders. Both surfaces use the image routes created in E-163-03.

## Context
The opponent detail page (`opponent_detail.html`) has no spray chart content. The opponent print page (`opponent_print.html`) has placeholder divs saying "Spray chart coming soon" in the Batter Tendencies section. Both need real spray chart images now that the data and image routes are available.

## Acceptance Criteria
- [ ] **AC-1**: The opponent detail page shows a "Team Spray Chart" card with an `<img>` tag pointing to `/dashboard/charts/spray/team/{opponent_team_id}.png?season_id={{ season_id }}` when the opponent has ≥1 offensive BIP event, per TN-6. Shows "No spray chart data available" (per TN-5) when 0 BIP.
- [ ] **AC-2**: The opponent detail route passes `team_spray_bip_count` in the template context (count of offensive BIP events for the opponent team in the scouting season).
- [ ] **AC-3**: The opponent print page replaces the `.spray-placeholder` divs (currently showing "Spray chart coming soon") with `<img>` tags pointing to `/dashboard/charts/spray/player/{player_id}.png?season_id={{ season_id }}` for each batter with ≥1 offensive BIP event, per TN-7. Shows "No spray chart data available" (per TN-5) for batters with 0 BIP.
- [ ] **AC-4**: The opponent print route passes per-player spray BIP counts as a dict `{player_id: bip_count}` (or equivalent) in the template context so the template can look up each batter's BIP count by player_id and conditionally render images vs "No spray chart data available".
- [ ] **AC-5**: Team spray chart card shows "Based on N balls in play" subtitle, consistent with player profile display, per TN-5.

## Technical Approach
1. **Opponent detail**: Add a DB query for team-level offensive BIP count. Add `team_spray_bip_count` to the template context. Add a spray chart card to the template (positioned after the batting table, before batter tendencies if present).
2. **Opponent print**: Replace the `.spray-placeholder` divs (CSS class in `opponent_print.html`) with conditional `<img>` tags. The print route needs per-player BIP counts — pass a dict `{player_id: bip_count}` in the template context so each batter row can look up its BIP count by player_id and conditionally render `<img src="/dashboard/charts/spray/player/{player_id}.png?season_id={{ season_id }}">` vs "No spray chart data available".

Reference opponent detail route and template: `src/api/routes/dashboard.py` `opponent_detail` function, `src/api/templates/dashboard/opponent_detail.html`. Reference opponent print: `opponent_print` function, `opponent_print.html`.
