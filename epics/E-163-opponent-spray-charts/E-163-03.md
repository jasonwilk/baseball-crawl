# E-163-03: Spray Chart Image Routes

## Status

## Description
After this story is complete, the dashboard will serve PNG spray chart images at `/dashboard/charts/spray/player/{player_id}.png` and `/dashboard/charts/spray/team/{team_id}.png`. These routes query the `spray_charts` table and render images using the existing `render_spray_chart()` function.

## Context
The `render_spray_chart()` function exists in `src/charts/spray.py` but is not imported or used anywhere in the app. The template (`player_profile.html`) references these routes but they don't exist as route handlers. This story creates the missing routes that connect DB data to the renderer.

## Acceptance Criteria
- [ ] **AC-1**: `GET /dashboard/charts/spray/player/{player_id}.png` returns a PNG image (200, `content-type: image/png`) when the player has ≥1 offensive BIP event in `spray_charts`. The image is rendered by `render_spray_chart()`.
- [ ] **AC-2**: `GET /dashboard/charts/spray/player/{player_id}.png` returns 204 No Content when the player has no offensive BIP events.
- [ ] **AC-3**: `GET /dashboard/charts/spray/team/{team_id}.png` returns a PNG image (200) when the team has ≥1 offensive BIP event. Returns 204 when no events exist.
- [ ] **AC-4**: Both routes accept an optional `season_id` query parameter. When provided, filter BIP events to that season. When omitted, use the most recent season with spray data for that player/team, per TN-3.
- [ ] **AC-5**: Both routes use `run_in_threadpool` for DB queries and renderer calls (blocking I/O must not block the event loop).
- [ ] **AC-6**: Both routes require an authenticated session but deliberately skip `permitted_teams` authorization check (spray chart auth exception, documented in CLAUDE.md Architecture section).
- [ ] **AC-7**: The player route renders with title "{first_name} {last_name}" and the team route renders with title "{team_name}".

## Technical Approach
Add two route handlers to `src/api/routes/dashboard.py`. Each route:
1. Queries `spray_charts` for offensive events (`chart_type = 'offensive'`) matching the player/team and optional season
2. If no events, return `Response(status_code=204)`
3. Calls `render_spray_chart(events, title=...)` from `src/charts/spray.py`
4. Returns `Response(content=png_bytes, media_type="image/png")`

DB queries need a helper in `src/api/db.py` to fetch spray events. The query returns rows with `x`, `y`, `play_result` (the minimum fields `render_spray_chart` needs).

Reference: CLAUDE.md Architecture section documents the spray chart auth exception pattern.
