# E-142-04: Opponent List Fallback to team_opponents

## Epic
[E-142: Team Visibility Overhaul](epic.md)

## Status
`TODO`

## Description
After this story is complete, the opponent list at `/dashboard/opponents` will include opponents from the `team_opponents` junction table even when no game data exists for them. Linked opponents will appear immediately after being linked, with `--` displayed in the GP, W, and L columns instead of numeric values. Opponents with game data continue to display exactly as before.

## Context
`get_team_opponents()` in `src/api/db.py` queries only the `games` table, so opponents only appear after game rows exist. The `team_opponents` table (populated by the scouting pipeline and admin opponent management) contains opponent relationships with `first_seen_year`, but is completely unused on the dashboard. Coaches expect to see their known opponents before any games are played. See TN-3 for the UNION strategy.

## Acceptance Criteria
- [ ] **AC-1**: Given a team with entries in `team_opponents` but no rows in `games` for those opponents, when `/dashboard/opponents` is loaded, then all linked opponents appear in the list.
- [ ] **AC-2**: The data layer returns `games_played=0`, `wins=0`, `losses=0`, and NULL date columns for opponents from `team_opponents` with no game data.
- [ ] **AC-3**: Opponents that appear in BOTH `games` AND `team_opponents` are not duplicated -- the games-based row takes precedence with full stats.
- [ ] **AC-4**: The `first_seen_year` filter is applied per TN-3: only `team_opponents` entries where `first_seen_year` matches the year extracted from the active `season_id` are included.
- [ ] **AC-5**: The opponent list template renders opponents with `games_played=0` using `--` for GP, W, L columns (not "0" or "0-0") per TN-3 layer separation. Date columns render as empty/blank when NULL.
- [ ] **AC-6**: Existing opponent list display for teams with full game data is unchanged (no regression).
- [ ] **AC-7**: New tests cover: (a) opponents from `team_opponents` only, (b) opponents from `games` only, (c) opponents in both sources, (d) `first_seen_year` filtering.

## Technical Approach
Extend `get_team_opponents()` in `src/api/db.py`. After the existing games-based query, add a second query against `team_opponents` JOIN `teams` to get opponent names, filtered by `first_seen_year = CAST(substr(:season_id, 1, 4) AS INTEGER)` and `our_team_id = :team_id`. Exclude opponent_team_ids already in the games result. Merge the two result sets in Python. Update the opponent list template to handle the zero-game display. See TN-3 for the impedance mismatch details.
