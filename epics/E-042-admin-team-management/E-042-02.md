# E-042-02: URL parser and public API team resolver

## Epic
[E-042: Admin Interface and Team Management](epic.md)

## Status
`TODO`

## Description
After this story is complete, two new modules will exist: a URL parser that extracts `public_id` from GameChanger URLs, and a team resolver that fetches team profile data from the public API given a `public_id`. Together they form the "paste a URL, get a team" pipeline used by the admin interface.

## Context
Admins will paste GameChanger team URLs into the admin interface. The system needs to: (1) parse the URL to extract the `public_id` slug, and (2) call the public API to resolve the team name, location, sport, and other metadata. These modules are pure utilities with no database dependency -- they are consumed by the admin routes in E-042-03.

## Acceptance Criteria
### URL Parser (`src/gamechanger/url_parser.py`)
- [ ] **AC-1**: `parse_team_url(input: str) -> str` accepts a full GC URL like `https://web.gc.com/teams/a1GFM9Ku0BbF/some-slug` and returns `"a1GFM9Ku0BbF"`.
- [ ] **AC-2**: The parser also accepts a bare `public_id` string (e.g., `"a1GFM9Ku0BbF"`) and returns it unchanged.
- [ ] **AC-3**: The parser handles URL variations: with/without trailing slash, with query params, with URL fragments, with `http://` vs `https://`.
- [ ] **AC-4**: The parser raises `ValueError` with a descriptive message for unparseable inputs (empty string, URLs without a `/teams/` segment). The parser accepts any hostname with a `/teams/` path (not just `web.gc.com`) to handle mobile share links and future domain changes.
- [ ] **AC-5**: The parser validates that the extracted `public_id` is alphanumeric (letters and digits only, 6-20 chars). Raises `ValueError` if not.
- [ ] **AC-6**: Tests in `tests/test_url_parser.py` cover all of AC-1 through AC-5 with at least 8 test cases (happy paths + error cases).

### Team Resolver (`src/gamechanger/team_resolver.py`)
- [ ] **AC-7**: `resolve_team(public_id: str) -> TeamProfile` calls `GET /public/teams/{public_id}` using the shared HTTP session from `src/http/session.py` with appropriate headers (`gc-app-name: web`, `Accept: application/vnd.gc.com.public_team_profile+json; version=0.1.0`). No auth headers sent. HTTP timeout is 10 seconds.
- [ ] **AC-8**: `TeamProfile` is a dataclass with fields: `public_id: str`, `name: str`, `city: str | None`, `state: str | None`, `sport: str`, `age_group: str | None`, `season: str | None`, `year: int | None`, `record_wins: int | None`, `record_losses: int | None`, `staff: list[str]`.
- [ ] **AC-9**: The resolver raises `TeamNotFoundError` (a custom exception inheriting from `ValueError`) when the API returns 404.
- [ ] **AC-10**: The resolver raises `GameChangerAPIError` for non-200/non-404 HTTP responses.
- [ ] **AC-11**: If the API returns 200 but the response body is missing required fields (`name` or `sport`), the resolver raises `GameChangerAPIError` with a message indicating the unexpected response shape.
- [ ] **AC-12**: Tests in `tests/test_team_resolver.py` mock the HTTP layer and verify AC-7 through AC-11 with at least 5 test cases (success, 404, 500, malformed 200 missing `name`, timeout).

## Technical Approach
- URL parser is a pure function with no I/O. Use `urllib.parse.urlparse` for URL decomposition.
- Team resolver uses `src/http/session.py`'s `create_session()` for HTTP. The `Accept` header for the public team profile endpoint is `application/vnd.gc.com.public_team_profile+json; version=0.1.0` (from `docs/gamechanger-api.md`, GET /public/teams/{public_id} section).
- No auth headers (`gc-token`, `gc-device-id`) should be sent -- this is a public endpoint.
- The base URL is `https://api.team-manager.gc.com` (from `docs/gamechanger-api.md`).
- Use `httpx` response mocking in tests (consistent with existing test patterns).
