## Status
`TODO`

## Description
After this story is complete, admins can edit a team's metadata (name, level, type) and toggle its active/inactive status from the admin interface. Deactivating a team removes it from the crawl pipeline without deleting data.

## Context
E-042-03 delivers team creation and listing. This story adds the edit and deactivate capabilities. The edit form lets admins correct team metadata (e.g., assign a level after initial add). The deactivate toggle is the primary mechanism for controlling which teams are crawled -- safer than deletion since all historical data is preserved.

## Acceptance Criteria
### Edit Team Page (`GET /admin/teams/{team_id}/edit`)
- [ ] **AC-1**: Renders a form pre-filled with the team's current values: Name (text, required), Level (select), Type (radio: Owned/Tracked), Public ID (read-only display).
- [ ] **AC-2**: Shows the team's current status (Active/Inactive) and last_synced timestamp.
- [ ] **AC-3**: Protected by `_require_admin()`.
- [ ] **AC-4**: Returns 404 if `team_id` does not exist.

### Update Team (`POST /admin/teams/{team_id}/edit`)
- [ ] **AC-5**: Updates `name`, `level`, and `is_owned` in the `teams` table.
- [ ] **AC-6**: Redirects to `/admin/teams?msg=Team+updated` on success.
- [ ] **AC-7**: Protected by `_require_admin()`.

### Deactivate/Activate Toggle (`POST /admin/teams/{team_id}/toggle-active`)
- [ ] **AC-8**: Toggles `is_active` between 0 and 1.
- [ ] **AC-9**: Redirects to `/admin/teams?msg=Team+{activated|deactivated}` with the appropriate verb.
- [ ] **AC-10**: Protected by `_require_admin()`.

### Team List Integration
- [ ] **AC-11**: The team list page (from E-042-03) shows an "Activate"/"Deactivate" button per row that POSTs to the toggle endpoint.

### Tests
- [ ] **AC-12**: Tests in `tests/test_admin_teams.py` (extending from E-042-03) verify: GET edit returns 200 with pre-filled data for admin, 404 for nonexistent team. POST edit updates team and redirects. POST toggle-active flips the flag. Auth guards on all three endpoints.

## Technical Approach
- Add routes to `src/api/routes/admin.py` (append after E-042-03's routes).
- Add `get_team_by_id()`, `update_team()`, `toggle_team_active()` to `src/api/db.py`.
- Create `src/api/templates/admin/edit_team.html` following the `edit_user.html` pattern.
- The toggle endpoint uses a simple POST (no form body needed -- just flips the current value).

## Notes
- No team deletion is provided. Deactivation is the safe alternative. If deletion is needed later, it's a separate story with cascade considerations (games, players, stats all reference team_id).
- The `name` field is editable because the API-resolved name might not match the admin's preferred display name (e.g., "Lincoln Rebels 14U" vs "Lincoln JV").
