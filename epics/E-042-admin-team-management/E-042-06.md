# E-042-06: Database-driven crawl configuration

## Epic
[E-042: Admin Interface and Team Management](epic.md)

## Status
`TODO`

## Description
After this story is complete, the crawl pipeline (`scripts/crawl.py`) can read team configuration from the SQLite database instead of `config/teams.yaml`. A `--source db` flag switches to database-driven mode, where active teams with `is_owned=1` are crawled. The YAML path remains the default for backward compatibility.

## Context
With teams now managed via the admin UI (E-042-03/04), the crawl pipeline needs to read from the database. This story bridges the admin-managed team data with the existing crawl orchestration. The YAML config remains as a fallback/bootstrap mechanism.

## Acceptance Criteria
### Database Config Loader
- [ ] **AC-1**: A new function `load_config_from_db(db_path: Path) -> CrawlConfig` in `src/gamechanger/config.py` queries `SELECT team_id, name, level FROM teams WHERE is_active = 1 AND is_owned = 1` and returns a `CrawlConfig` with `owned_teams` populated from the results.
- [ ] **AC-2**: The `season` field is derived from the most recent season in the `seasons` table (`SELECT season_id FROM seasons ORDER BY year DESC LIMIT 1`). If no seasons exist, raises `ValueError("No seasons found in database")`. If multiple seasons exist for the same year, any one is acceptable (the admin can override via a future `--season` flag if needed).
- [ ] **AC-3**: If no active owned teams are found, returns a `CrawlConfig` with an empty `owned_teams` list (not an error -- the pipeline handles empty gracefully).

### Crawl Script Integration
- [ ] **AC-4**: `scripts/crawl.py` accepts a `--source` argument with values `yaml` (default) or `db`.
- [ ] **AC-5**: When `--source db`, the script calls `load_config_from_db()` with the database path from `DATABASE_PATH` env var (default `./data/app.db`).
- [ ] **AC-6**: When `--source yaml` (or no flag), behavior is unchanged from current.
- [ ] **AC-7**: The script logs which source it is using: `"Loading team config from {source}"`.

### Load Script Integration
- [ ] **AC-8**: `scripts/load.py` accepts the same `--source` argument as `scripts/crawl.py`, mirroring AC-4 (`--source yaml|db`), AC-5 (`DATABASE_PATH` env var), AC-6 (yaml default), and AC-7 (source logging).

### Tests
- [ ] **AC-9**: Tests in `tests/test_config.py` verify `load_config_from_db()`: returns correct teams from seeded DB, handles empty teams list, handles missing seasons table, derives season correctly.
- [ ] **AC-10**: Tests verify that the function only includes `is_active=1 AND is_owned=1` teams (inactive and tracked-only teams are excluded).

## Technical Approach
- Add `load_config_from_db()` to `src/gamechanger/config.py` alongside the existing `load_config()`.
- The function opens its own connection (using the same `sqlite3.connect` + WAL + FK pattern from `src/api/db.py`), reads teams, and closes the connection.
- `scripts/crawl.py` already uses `argparse` -- add `--source` as an argument.
- For season derivation: the pipeline currently uses a single season from YAML. The DB approach picks the latest season. If multi-season crawling is needed later, that's a separate enhancement.

## Notes
- This story intentionally does NOT remove `config/teams.yaml` or make `--source db` the default. That's a future decision once the admin team management is battle-tested.
- The `load_config_from_db` function queries `is_owned=1` because the existing crawl pipeline is designed for owned teams. Opponent crawling uses different paths (public endpoints, no auth needed) and would be a separate crawl mode.
- The season derivation is intentionally simple (latest season). Multi-season or season-selection support can be added later.
