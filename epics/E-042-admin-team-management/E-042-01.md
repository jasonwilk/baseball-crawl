# E-042-01: Schema migration: add public_id to teams

## Epic
[E-042: Admin Interface and Team Management](epic.md)

## Status
`TODO`

## Description
After this story is complete, the `teams` table will have a nullable `public_id` column with a unique partial index, enabling teams to be identified by their GameChanger public identifier in addition to the existing `team_id`. This is the foundation for URL-based team onboarding.

## Context
The current `teams` table uses `team_id` (UUID from authenticated API) as the primary key. For public-API-based team onboarding, we need to store the `public_id` slug (e.g., `"a1GFM9Ku0BbF"`) that identifies teams in the public GameChanger web UI and API. Opponents discovered via authenticated endpoints may not have a `public_id` initially, so the column must be nullable.

## Acceptance Criteria
- [ ] **AC-1**: A new migration file `migrations/005_teams_public_id.sql` exists and adds a `public_id TEXT` column to the `teams` table using `ALTER TABLE`.
- [ ] **AC-2**: The migration creates a unique partial index `idx_teams_public_id` on `teams(public_id) WHERE public_id IS NOT NULL`, ensuring uniqueness while allowing multiple NULLs.
- [ ] **AC-3**: The migration file header comment follows the same conventions as existing migrations (purpose, dependencies, see-epic reference).
- [ ] **AC-4**: `python migrations/apply_migrations.py` applies the new migration without error on a fresh database (after 001, 003, 004).
- [ ] **AC-5**: Existing rows in `teams` have `public_id = NULL` after migration (no data loss).
- [ ] **AC-6**: Inserting two teams with the same non-NULL `public_id` raises `IntegrityError`.
- [ ] **AC-7**: Inserting multiple teams with `public_id = NULL` succeeds (partial index allows multiple NULLs).
- [ ] **AC-8**: A test in `tests/test_migrations.py` verifies AC-4 through AC-7 using an in-memory SQLite database.

## Technical Approach
- Create `migrations/005_teams_public_id.sql` with `ALTER TABLE teams ADD COLUMN public_id TEXT` and `CREATE UNIQUE INDEX`.
- SQLite supports `ALTER TABLE ADD COLUMN` without rebuilding the table.
- The partial index syntax `WHERE public_id IS NOT NULL` is supported in SQLite 3.8.0+.
- Follow the migration comment conventions from `migrations/001_initial_schema.sql` and `migrations/004_coaching_assignments.sql`.
