# E-086-04: Context-Layer Mobile Credential Workflow

## Epic
[E-086: Mobile Credential Capture](epic.md)

## Status
`TODO`

## Description
After this story is complete, CLAUDE.md and `docs/admin/mitmproxy-guide.md` will document the complete mobile credential capture workflow: proxy capture, `bb creds capture`, `bb creds import --profile mobile`, and the 12-hour access token window. Agents will know how to guide operators through mobile credential setup and renewal.

## Context
E-086-01 through E-086-03 add mobile credential capture capabilities. The context layer needs to reflect these so agents give operators accurate guidance. Currently, CLAUDE.md's mobile profile note says "programmatic mobile token refresh is not yet possible" with no alternative workflow described. After this epic, it should describe the supported proxy-based capture path.
