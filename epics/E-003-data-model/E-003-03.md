## Description
After this story is complete, the project will have a `scripts/db_migrate.py` script that applies all pending migration files to either a local SQLite database or a Cloudflare D1 instance. Developers can run `python scripts/db_migrate.py --local` to get a fully migrated local database in `data/dev.db` in under 10 seconds.

## Context
Migration tooling is the operational backbone of the data model. Without it, applying schema changes requires manual, error-prone SQL execution. With it, spinning up a fresh environment is one command.

Cloudflare D1 supports migrations natively via `wrangler d1 migrations apply`, but we should have a Python wrapper that works for both local (SQLite) and remote (D1) targets so all tooling is consistent.

## Acceptance Criteria
- [ ] **AC-1**: Running `python scripts/db_migrate.py --local` applies all migration files in `migrations/` (in numeric order) to `data/dev.db` and prints which migrations were applied.
- [ ] **AC-2**: Running the command again on an already-migrated database applies no new migrations and prints "Already up to date."
- [ ] **AC-3**: A `schema_migrations` table in the database tracks which migrations have been applied, using the migration filename as the key.
- [ ] **AC-4**: Running `python scripts/db_migrate.py --status` (local) prints a table showing all migration files and whether each is applied or pending.
- [ ] **AC-5**: Migration files are applied in ascending numeric order regardless of filesystem ordering.
- [ ] **AC-6**: A `wrangler.toml` file exists with a D1 database binding configured (can be a placeholder with `[FILL IN D1 DATABASE ID]` if the real D1 instance hasn't been created yet).

## Technical Approach
`scripts/db_migrate.py` reads all `.sql` files from `migrations/`, sorts by filename, checks the `schema_migrations` table for already-applied migrations, and runs pending ones in order.

```python
def apply_migrations(db_path: Path) -> None:
    conn = sqlite3.connect(db_path)
    conn.execute("""
        CREATE TABLE IF NOT EXISTS schema_migrations (
            filename TEXT PRIMARY KEY,
            applied_at TEXT NOT NULL DEFAULT (datetime('now'))
        )
    """)
    migration_files = sorted(Path("migrations").glob("*.sql"))
    for mf in migration_files:
        already_applied = conn.execute(
            "SELECT 1 FROM schema_migrations WHERE filename = ?", (mf.name,)
        ).fetchone()
        if already_applied:
            continue
        conn.executescript(mf.read_text())
        conn.execute("INSERT INTO schema_migrations (filename) VALUES (?)", (mf.name,))
        conn.commit()
        print(f"Applied: {mf.name}")
```

For Cloudflare D1, wrap `wrangler d1 migrations apply` via `subprocess`. The `--env production` flag selects the D1 target.
