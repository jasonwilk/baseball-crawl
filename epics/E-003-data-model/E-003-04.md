## Epic

## Status

## Description
After this story is complete, the project will have a seed data fixture (`tests/fixtures/seed.sql`) with realistic test data representing two Lincoln teams, several opponents, a partial season's worth of games, and player stats including splits. Query validation tests confirm that the most important coaching queries return correct results against the seed data.

## Context
A schema is not done until you can run real queries against it and get sensible answers. This story closes the loop: it validates that the schema supports the queries coaches will actually run, and it provides a realistic test fixture that all future loader tests can use as a baseline.

The seed data should represent a realistic scenario: two Lincoln teams (Varsity and JV), three opponents, 10 games played, 15 players per team. Numbers should be plausible baseball stats, not all-zeros.

## Acceptance Criteria
- [ ] **AC-1**: `tests/fixtures/seed.sql` exists and inserts test data for: 2 owned teams, 3 opponent teams, 30 players (15 per owned team), 10 games, batting and pitching stats for all players in all games.
- [ ] **AC-2**: A test `test_seed_data_loads.py` confirms that applying the full migration + seed SQL to a fresh SQLite database completes without error.
- [ ] **AC-3**: A query test confirms: "given a player ID, return their season batting average (H/AB), OBP ((H+BB)/(AB+BB)), and K rate (SO/AB) for the current season" returns numerically correct results against the seed data.
- [ ] **AC-4**: A query test confirms: "given a team ID, return the roster with each player's season stats sorted by OBP descending" returns the correct number of players in the correct order.
- [ ] **AC-5**: A query test confirms: "given a team ID, return their record (W-L) for the current season" returns the correct win-loss record based on game scores in the seed data.
- [ ] **AC-6**: A query test confirms: "given a player ID, return their home vs. away split batting averages" -- either from the split columns if populated, or computed from `player_game_batting` joined to `games`.
- [ ] **AC-7**: All three representative queries from the epic's Success Criteria run in under 100ms against the seed database (use `time.perf_counter()` in the test to assert duration).

## Technical Approach
Write `tests/fixtures/seed.sql` as a plain SQL INSERT script. Use fixed, known values (not random) so query results are deterministic and tests can assert exact values.

Write `tests/test_schema_queries.py` with pytest. Each test applies migrations + seed in a fresh in-memory SQLite database, runs a query, and asserts on the result.

```python
@pytest.fixture
def seeded_db():
    conn = sqlite3.connect(":memory:")
    # apply all migrations
    for mf in sorted(Path("migrations").glob("*.sql")):
        conn.executescript(mf.read_text())
    # apply seed
    conn.executescript(Path("tests/fixtures/seed.sql").read_text())
    return conn

def test_player_batting_stats(seeded_db):
    row = seeded_db.execute(
        "SELECT h, ab, bb, so FROM player_season_batting WHERE player_id = ?",
        ("PLAYER_VARSITY_01",)
    ).fetchone()
    h, ab, bb, so = row
    avg = h / ab
    assert abs(avg - 0.333) < 0.001  # seed data for this player: 10 H in 30 AB
```
