## Description
After this story is complete, the context layer codifies two foundational principles: (1) **"Never suppress. Always contextualize."** -- every stat is shown at full weight with data-depth context, never dimmed or hidden; the coach is the analyst. (2) **Season-length calibration** -- all thresholds are designed for 25-game seasons, not 162-game seasons. The baseball-coach agent definition reflects these principles, and stale threshold references are corrected.

## Context
The user's design philosophy emerged during E-187 report evaluation. A 189.0 ERA on 0.1 IP is useful information ("this kid got shelled"), not noise to suppress. The system presents data; the coach decides what matters. The display philosophy applies to all data surfaces -- reports today, dashboard in the future. It must be embedded in the project's DNA so every agent internalizes it when building data-facing features.

The "don't steer when you can define" principle (already codified in `.claude/rules/context-layer-guard.md`) applies: scoped rules define principles for an area without steering implementation into that area.
