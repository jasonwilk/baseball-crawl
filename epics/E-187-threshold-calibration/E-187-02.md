# E-187-02: Lower Display Thresholds for Youth/HS Seasons

## Epic
[E-187: Threshold Calibration for Youth/HS Seasons](epic.md)

## Status
`TODO`

## Description
After this story is complete, the scouting report renderer will use thresholds calibrated for youth/HS season lengths: 5 PA (batting), 6 IP / 18 outs (pitching), with matching key-player callout thresholds. A report for an opponent with 5 games will show heat-map coloring for most starters instead of suppressing nearly all players.

## Context
The standalone scouting report renderer (`src/reports/renderer.py`) uses thresholds (20 PA, 15 IP) that are appropriate for professional or college seasons but too conservative for youth (25-game) and HS (30-game) seasons. A Millard South report (5 games, 18 batters) showed almost no heat-map coloring because only 2 batters exceeded 10 AB. The user's guidance: "Start at the minimum reasonable level and just get better as the data gets deeper."

## Acceptance Criteria
- [ ] **AC-1**: `_MIN_PA_BATTING` is 5 and `_MIN_IP_OUTS_PITCHING` is 18 (6 IP) in `src/reports/renderer.py`.
- [ ] **AC-2**: `_KEY_PITCHER_MIN_OUTS` is 18 and `_KEY_BATTER_MIN_PA` is 5 in `src/reports/renderer.py`.
- [ ] **AC-3**: The scouting report template footnote text reads "fewer than 5 PA" (batting) and "fewer than 6 IP" (pitching), per Technical Notes TN-2.
- [ ] **AC-4**: Given a batter with PA=8 (above new threshold, below old threshold), when `render_report()` is called, then the batter has non-zero heat-map values and is NOT marked `_small_sample`.
- [ ] **AC-5**: Given a pitcher with ip_outs=21 (7 IP -- above new threshold, below old threshold), when `render_report()` is called, then the pitcher has non-zero heat-map values and is NOT marked `_small_sample`.
- [ ] **AC-6**: Given a batter with exactly PA=5 (the boundary value), the batter is NOT marked `_small_sample` and has non-zero heat-map values.
- [ ] **AC-7**: Given a pitcher with exactly ip_outs=18 (the boundary value, 6.0 IP), the pitcher is NOT marked `_small_sample` and has non-zero heat-map values.
- [ ] **AC-8**: All existing renderer tests pass after updating at minimum 12 tests (test data and assertion strings) to reflect the new thresholds, per Technical Notes TN-6.
- [ ] **AC-9**: Spray chart thresholds (`_MIN_BIP_SPRAY=3`, `_MIN_BIP_TEAM_SPRAY=20`) are unchanged.

## Technical Approach
This is a constants-and-tests update. Change 4 threshold constants in `src/reports/renderer.py` (spray constants unchanged per AC-9), update 2 footnote strings in `src/api/templates/reports/scouting_report.html`, and update at minimum 12 tests in `tests/test_report_renderer.py` that hardcode old threshold values. The heat-map algorithm itself is unchanged -- it uses percentile ranking within qualified players, which works correctly at any threshold level.
