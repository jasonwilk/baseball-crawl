# E-187-02: Replace Stat Suppression with Contextual Display

## Epic
[E-187: Threshold Calibration for Youth/HS Seasons](epic.md)

## Status
`DONE`

## Description
After this story is complete, the scouting report never suppresses stats. Every player's numbers display at full visual weight regardless of sample size. The old dimming/asterisk/footnote system is replaced with inline PA/IP badges that give coaches instant context on data depth. Heat-map color richness scales with team data depth via graduated intensity tiers (TN-2a). Qualification thresholds are lowered to 5 PA (batting) and 6 IP / 18 outs (pitching).

## Context
The user's design philosophy: **"Never suppress. Always contextualize."** A 189.0 ERA on 0.1 IP is useful information -- it tells the coach "this kid got shelled." The system presents data; the coach is the analyst. The current renderer dims small-sample rows to gray, adds asterisks, zeroes heat, and adds footnotes -- all forms of suppression that hide data a coach would want to see. Additionally, the thresholds (20 PA, 15 IP) are calibrated for pro seasons, not 25-game youth seasons.

Technical Notes TN-2 and TN-2a provide the implementation reference: suppression artifacts to remove, badge CSS/HTML, and graduated heat intensity tiers.

### Suppression Removal
- [ ] **AC-6**: The `tr.small-sample td { color: #9ca3af; }` CSS rule is removed from the scouting report template. No player row is dimmed regardless of sample size.
- [ ] **AC-7**: No asterisk (` *`) is appended to any player name in the batting or pitching tables.
- [ ] **AC-8**: The small-sample footnote divs ("* Small sample size (fewer than N PA/IP)") are removed from both batting and pitching sections.

### PA/IP Badge
- [ ] **AC-9**: Every batting row displays an inline badge showing the player's PA count (e.g., `[8 PA]`). The badge uses a compact, visually subordinate style (small font, gray background pill) that does not compete with the stat numbers.
- [ ] **AC-10**: Every pitching row displays an inline badge showing the pitcher's IP (e.g., `[2.1 IP]`). Same styling as the batting badge.
- [ ] **AC-11**: The badge displays for ALL players, not just low-sample players. It is neutral metadata, not a warning marker.

### Graduated Heat Intensity
- [ ] **AC-12**: Heat-map color richness scales with the number of qualified players. The maximum heat level is capped based on team depth per the tier breakpoints in TN-2a. Fewer qualified players = lighter greens only; more qualified players = full gradient.
- [ ] **AC-13**: Players below the per-player threshold (< 5 PA or < 18 outs) always get `heat-0` regardless of team depth. The graduated cap applies only to qualified players.

## Technical Approach
Three categories of changes: (1) Lower 4 threshold constants in `src/reports/renderer.py`. (2) Remove suppression artifacts and add PA/IP badge in `src/api/templates/reports/scouting_report.html` -- remove `tr.small-sample` CSS rule, remove asterisk conditionals, remove footnote divs, add `.depth-badge` CSS class, add badge `<span>` in player name cells. (3) Add graduated heat intensity in `_compute_batting_heat()` and `_compute_pitching_heat()` -- after computing heat levels, clamp to a maximum based on the number of qualified players per TN-2a. The renderer already computes `_pa` (plate appearances) via `_compute_pa()` and passes enriched batting rows to the template -- reuse this existing value in the badge. Update at minimum 12 tests per TN-6.
