## Description
After this story is complete, the implement skill's Phase 4b Step 1 will mirror the round counter / circuit breaker logic implemented by the codex-review skill in Story E-233-02. Both consumers read and write the same shared state files (`.codex-review-rounds`, `.codex-dispositions.jsonl`) at the epic worktree root, following the canonical contract in `.claude/rules/codex-review-breaker.md`. This closes the gap that fresh-CA F-FRESH-1 exposed: implement Phase 4b runs codex via `./scripts/codex-review.sh` (a Bash script invocation) and does NOT load the codex-review skill markdown — so the breaker section in the codex-review skill is invisible to Phase 4b without this mirror.

## Context
This story is the second producer half of the breaker. Story E-233-02 installs the breaker section in the codex-review skill markdown (which loads when the user invokes `codex review` directly). This story installs the SAME breaker logic at the implement skill's Phase 4b Step 1, where it executes when the implement skill's "and review" chain is active.

Both consumers reference `.claude/rules/codex-review-breaker.md` as the canonical contract — there is one round counter per epic worktree, shared by both call sites. An "and review" pass increments the counter from 0 to 1; subsequent user-direct codex-review invocations during the same closure increment the same counter; round 3 hits the gate.

Without this story, the breaker only fires on user-direct invocations. That leaves a gap: a user could "and review" (which would have been round 1 with the breaker, but isn't counted because Phase 4b doesn't load the skill), then run `codex review` directly (which the codex-review skill counts as round 1 instead of round 2), and effectively get 3 codex passes before the gate triggers — defeating the cap=2 guarantee.

This story is paired with E-233-02 — they together establish the round counter as a shared invariant. Both stories follow the canonical contract in `.claude/rules/codex-review-breaker.md` verbatim.
