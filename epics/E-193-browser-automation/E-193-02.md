# E-193-02: Browser Automation Context-Layer Rule

## Epic
[E-193: Browser Automation Infrastructure](./epic.md)

## Status
`TODO`

## Description
After this story is complete, a scoped rule at `.claude/rules/browser-automation.md` teaches agents how to use `agent-browser` for UI verification. The rule covers the CLI commands, the CDP connection URL, the app URL convention, and the worktree constraint. Agents working on template or UI files will have this rule loaded automatically.

## Context
The Chrome sidecar (E-193-01) provides the browser. Agents need to know how to use it. CA recommends a scoped rule (not ambient in CLAUDE.md, not a skill) that fires on template/UI file paths. The rule must be self-contained: an agent reading it for the first time should be able to open the app in a browser and take a snapshot without any other documentation.

## Acceptance Criteria
- [ ] **AC-1**: A rule file exists at `.claude/rules/browser-automation.md` with a `paths:` frontmatter scoped to: `src/api/templates/**`, `src/api/routes/**`, `src/reports/renderer.py`.
- [ ] **AC-2**: The rule documents the complete `agent-browser` command reference: binary path (`./node_modules/.bin/agent-browser`), CDP connection flag (`--connect-over-cdp ws://localhost:3000`), and all commands (`open`, `snapshot`, `click`, `type`, `fill`, `press`, `close`) with brief usage descriptions.
- [ ] **AC-3**: The rule states the app URL convention: `http://localhost:8001` from the devcontainer.
- [ ] **AC-4**: The rule states the worktree constraint: browser automation only works in the main checkout, not in epic worktrees (Docker services are not accessible from worktrees), per Technical Notes "Worktree Constraint."
- [ ] **AC-5**: The rule states that `npx agent-browser` does NOT work and the binary must be invoked via `./node_modules/.bin/agent-browser`.

## Technical Approach
Create a scoped rule file. The `paths:` glob patterns target files where browser verification is relevant: HTML templates (`src/api/templates/**`), route files that serve HTML (`src/api/routes/**`), and the report renderer (`src/reports/renderer.py`). The content should be concise and action-oriented. Connection details are documented in Technical Notes "Chrome Sidecar Configuration" and "agent-browser CLI Usage."
