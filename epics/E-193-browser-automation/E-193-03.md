# E-193-03: UXD Agent Definition Browser Capability

## Epic
[E-193: Browser Automation Infrastructure](./epic.md)

## Status
`TODO`

## Description
After this story is complete, the UXD agent definition (`.claude/agents/ux-designer.md`) includes browser automation as a verification capability. UXD agents will know they can use `agent-browser` to verify UI structure via accessibility-tree snapshots after implementation, and will know where to find the detailed usage rule.

## Context
The browser automation rule (E-193-02) provides the detailed how-to. The UXD agent definition needs a lightweight reference to this capability so that UXD agents are aware they can verify their designs programmatically. CA recommends updating the agent definition rather than relying solely on the scoped rule, since UXD is the primary consumer and should know about browser verification as part of its identity.

## Acceptance Criteria
- [ ] **AC-1**: The UXD agent definition at `.claude/agents/ux-designer.md` references `.claude/rules/browser-automation.md` as the detailed usage guide for browser verification (not duplicating the command reference).
- [ ] **AC-2**: The addition notes the worktree constraint (browser verification is main-checkout only, not available during dispatch).
- [ ] **AC-3**: The addition describes the primary use case: taking accessibility-tree snapshots to verify page structure, element presence, and layout after SE implements UI changes.

## Technical Approach
Read the current UXD agent definition, identify the appropriate location for a browser verification section, and add a concise paragraph or section. The content should establish awareness and point to the rule for details -- not duplicate the command reference. Keep it brief and focused on when and why UXD would use this capability.
