# E-220-01: Schema Rewrite with Perspective Provenance

## Epic
[E-220: Perspective-Aware Data Architecture](epic.md)

## Status
`TODO`

## Description
After this story is complete, the database schema includes `perspective_team_id` columns on all four stat tables (`player_game_batting`, `player_game_pitching`, `spray_charts`, `plays`), a `game_perspectives` junction table, and updated UNIQUE constraints that allow the same data from different perspectives to coexist without conflict.

## Context
This is the foundational schema story. All loader stories (E-220-02 through E-220-06) depend on these columns and constraints existing. The user has authorized a clean-slate database wipe, so this rewrites `001_initial_schema.sql` directly (E-100 pattern) rather than adding an ALTER migration. Current migrations 001-015 are archived.

- [ ] **AC-2**: UNIQUE constraints updated per TN-1: `UNIQUE(game_id, player_id, perspective_team_id)` on batting and pitching; `UNIQUE(game_id, play_order, perspective_team_id)` on plays; `UNIQUE(event_gc_id, perspective_team_id)` on spray_charts

## Technical Approach
Rewrite `001_initial_schema.sql` to include the new columns and constraints from the start, following TN-1 and TN-2. Archive existing migration files. The `perspective_team_id` column uses `NOT NULL` without a default to ensure every insert explicitly provides the value -- this makes contamination a hard insertion error.
