# E-220-04: SprayChartLoader Perspective Tagging

## Epic
[E-220: Perspective-Aware Data Architecture](epic.md)

## Status
`TODO`

## Description
After this story is complete, both the member-team SprayChartLoader and the ScoutingSprayChartLoader pass `perspective_team_id` on every `spray_charts` INSERT, and the updated UNIQUE constraint allows the same event from different perspectives to coexist.

## Context
The spray chart loader inserts ball-in-play events for ALL players in both `offense` and `defense` sections. For opponent players, it falls back to `fallback_team_id` and creates stubs. After this story, every spray_charts row carries `perspective_team_id` identifying which team's API call produced the data.

## Acceptance Criteria
- [ ] **AC-1**: Given spray data loaded via `SprayChartLoader.load_dir()`, when spray_charts rows are inserted, then every row has `perspective_team_id` set to the crawling team's integer PK
- [ ] **AC-2**: Given spray data loaded via `ScoutingSprayChartLoader`, when spray_charts rows are inserted, then every row has `perspective_team_id` set to the scouted team's integer PK
- [ ] **AC-3**: Given the same game's spray data loaded from two team perspectives, when both loads complete, then both sets of rows coexist with different `perspective_team_id` values
- [ ] **AC-4**: All existing spray chart loader tests pass (updated where they assert on schema changes)
- [ ] **AC-5**: Given spray data loaded via the member pipeline path (`SprayChartLoader.load_dir()` called from `load.py`), when the load completes, then every spray_charts row has `perspective_team_id` set to the member team's integer PK
- [ ] **AC-6**: All `INSERT OR IGNORE` statements on spray_charts include `perspective_team_id` in the column list and conflict target per TN-13 (NOT NULL column requires explicit value; UNIQUE on `(event_gc_id, perspective_team_id)` requires inclusion in conflict key)

## Technical Approach
The crawling team's integer PK is available in `SprayChartLoader` (via gc_uuid path resolution) and in `ScoutingSprayChartLoader` (via `team_id` parameter). Pass `perspective_team_id` into every INSERT statement. Update the INSERT OR IGNORE key to include `perspective_team_id`.
