## Description
After this story is complete, a `multi-agent-patterns` skill file exists at `.claude/skills/multi-agent-patterns/SKILL.md`. The skill teaches agents the design patterns for multi-agent coordination -- supervisor, peer-to-peer, hierarchical -- with a specific section on baseball-crawl's orchestrator -> PM -> specialist chain and the telephone game risk that chain introduces. The file is adapted from the muratcankoylan pattern with baseball-crawl's routing model as the primary case study.

## Context
The muratcankoylan Agent Skills for Context Engineering repository defines a `multi-agent-patterns` skill in its Architectural category. Its key warning is about the **telephone game problem** in supervisor architectures: when a supervisor agent (orchestrator) receives output from a specialist and then paraphrases it to the next agent downstream, information is lost or distorted at each relay. This is directly applicable to baseball-crawl's current architecture.

Baseball-crawl's routing chain:
```
User -> orchestrator -> product-manager -> implementing agent (general-dev / data-engineer)
```

This chain has two telephone game risk points:
1. **orchestrator -> PM**: The orchestrator routes a work request to the PM. If the orchestrator paraphrases or summarizes the user's request rather than passing it verbatim, the PM may write stories against a distorted understanding of the requirement.
2. **PM -> implementing agent**: The PM dispatches a story to an implementing agent via the Task tool. The dispatch includes a context block (story file + epic Technical Notes). If the PM summarizes rather than includes the full story file text, the implementing agent may miss acceptance criteria.

The workflow-discipline.md rules and the Dispatch Mode protocol (in product-manager.md) are designed to mitigate these risks, but agents do not currently have a named concept for the underlying failure mode. This skill provides that concept.

**Source material**: The muratcankoylan SKILL.md format is documented in `/.project/research/E-008-R-02-agent-skills-summary.md`. The four-section format is: activation triggers, key concepts, practical implementation, references.

**Adaptation requirement**: The skill must name and describe baseball-crawl's specific routing chain. An agent reading this skill should be able to identify baseball-crawl's two telephone game risk points and know the mitigation rules that address them (verbatim dispatch, full story file inclusion, workflow-discipline.md enforcement).

2. Baseball-crawl's agent ecosystem table in `CLAUDE.md` (the routing chain is described there)
3. The workflow-discipline rules in `.claude/rules/workflow-discipline.md` (the verbatim dispatch requirement and direct-routing exceptions)

The skill should be structured so the orchestrator finds it immediately useful when dispatching a complex routing sequence (trigger: "I'm about to route this through multiple agents"). It should read as a reference card for multi-agent coordination decisions, not a textbook.
