## Technical Notes

### TN-1: Skill loading is mandatory
The user's explicit direction is that claude-architect use `.claude/skills/context-fundamentals/SKILL.md` for this work. Dispatch to CA with a prompt that names the skill file by absolute path and instructs CA to load it before making any edits. CA should confirm skill loading in its first reply to the main session.

### TN-2: Per-lesson disposition discipline
The 8 lessons in `.claude/agent-memory/product-manager/lessons-learned.md` are the starting set. Each lesson needs an explicit disposition. "Disposition drift" — where a lesson is read, acknowledged, and then not acted on — is the failure mode this story is designed to prevent. Use a table or per-lesson heading in the commit message to make disposition auditable.

### TN-3: Context-layer-guard compliance
`.claude/rules/context-layer-guard.md` defines where each type of context belongs (CLAUDE.md vs scoped rules vs skills vs agent definitions vs agent memory). claude-architect must respect that placement framework — a lesson about "per-story CR insufficient for cross-cutting invariants" probably lives in the CR agent definition or a code-review rule, not in CLAUDE.md.

### TN-4: CLAUDE.md target
Per `.claude/rules/context-layer-guard.md`, CLAUDE.md targets ~150 lines. Any E-220 lesson promoted into CLAUDE.md must pass the "does every agent need this on every interaction?" test. Most lessons from E-220 will NOT pass that bar — they belong in scoped rules or agent definitions, not CLAUDE.md.
