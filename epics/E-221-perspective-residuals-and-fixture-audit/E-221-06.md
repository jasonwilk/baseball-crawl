## Description
After this story is complete, `_delete_team_cascade` Phase 1b will include a DELETE against `reconciliation_discrepancies` scoped by `perspective_team_id = team_id`. Currently Phase 1b cleans 5 tables this way but omits the sixth (reconciliation_discrepancies, added during E-220 round 7), which causes team deletes to fail with a FK violation when the deleted team owns reconciliation rows.

## Context
E-220 round 7 added `perspective_team_id INTEGER NOT NULL REFERENCES teams(id)` to `reconciliation_discrepancies`. This makes the column part of the FK graph — if you delete a team row, any reconciliation_discrepancies rows pointing to it via `perspective_team_id` block the delete with a FK violation.

`_delete_team_cascade` Phase 1b currently cleans `plays`, `spray_charts`, `player_game_batting`, `player_game_pitching`, and (one other) via `WHERE perspective_team_id = ?`. The sixth table (`reconciliation_discrepancies`) was added AFTER Phase 1b's current implementation and was never backfilled.

Impact: admin delete fails silently (exception caught and logged, same pattern as round 7's P1-1 pre-fix for the reports path) any time the deleted team has reconciliation rows.

This is a one-line fix (one additional DELETE statement) plus a RED test.

## Acceptance Criteria
- [ ] **AC-1**: Given a team has rows in `reconciliation_discrepancies` with `perspective_team_id = T`, when `_delete_team_cascade(conn, T, ...)` is called, then the cascade completes without raising `sqlite3.IntegrityError`, and all the team's reconciliation_discrepancies rows are deleted.
- [ ] **AC-2**: Given a team with no reconciliation_discrepancies rows, when `_delete_team_cascade` runs, its behavior is unchanged. The new DELETE executes as a no-op.
- [ ] **AC-3**: A RED test is added before the fix that demonstrates the FK violation. The test MUST use `load_real_schema(db)` from Story 01 so that FK enforcement is active (without FK enforcement, the bug cannot be reproduced).
- [ ] **AC-4**: The RED test asserts both: (a) the cascade completes without exception, and (b) the reconciliation rows for the deleted team's perspective are gone. The test passes only after the fix lands.
- [ ] **AC-5**: The fix is a single new DELETE statement in Phase 1b, added alongside the existing 5. No refactoring of Phase 1b's structure.

## Technical Approach
Per TN-6 in the epic. The fix is:

```sql
DELETE FROM reconciliation_discrepancies WHERE perspective_team_id = ?
```

Added to Phase 1b in the existing order of DELETEs (alongside the other 5 tables). Bind parameter is the team_id being deleted.

The RED test seeds a team and a minimal game, loads reconciliation_discrepancies rows with `perspective_team_id = team_id`, then calls `_delete_team_cascade`. Pre-fix: IntegrityError. Post-fix: succeeds and rows are gone.
