# E-037-04: Update context-layer references from "soft referential integrity" to stub-player pattern

## Epic
[E-037: Codex Review Remediation](epic.md)

## Status

## Description
After this story is complete, all live context-layer files (agent definitions, agent memory files) will describe the correct orphan-player handling strategy: insert a stub player row before the stat row (maintaining FK integrity), log a WARNING. The stale "soft referential integrity (orphaned player IDs accepted with WARNING, do not reject)" language will be replaced.

## Context
The project's schema uses hard FK constraints (`player_id REFERENCES players(player_id)`) and `PRAGMA foreign_keys=ON`. The original design aspiration was "soft referential integrity" -- accept orphaned player IDs in stat rows. But the delivered schema enforces FKs, making that impossible. E-037-02 fixes the spec. This story fixes the context-layer files that still describe the stale pattern.

Archived files (`/.project/archive/`) are frozen historical records and MUST NOT be modified.
