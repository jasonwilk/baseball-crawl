# E-037-01: Fix dashboard query column name and season_id format

## Epic
[E-037: Codex Review Remediation](epic.md)

## Status
`TODO`

## Description
After this story is complete, the dashboard query in `src/api/db.py` will use the correct `season_id` column name and `src/api/routes/dashboard.py` will pass a season_id slug (e.g., `"2026-spring-hs"`) instead of a bare year string. The dashboard will display player batting stats when seed data is present.

## Context
E-003-01 renamed the `season` column to `season_id` across all tables and introduced the `seasons` table with slug-format IDs (e.g., `"2026-spring-hs"`). The dashboard code was not updated to match. Two bugs:
1. `src/api/db.py` line 85 queries `psb.season` -- the column is now `psb.season_id`.
2. `src/api/routes/dashboard.py` line 85 computes `season = str(datetime.date.today().year)` producing `"2026"`, which never matches the slug `"2026-spring-hs"`.

Both bugs are masked by the `except sqlite3.Error` handler in `get_team_batting_stats()`, which returns an empty list. The dashboard renders with zero players and no visible error.

## Acceptance Criteria
- [ ] **AC-1**: The SQL query in `get_team_batting_stats()` uses `psb.season_id` (not `psb.season`) in the WHERE clause.
- [ ] **AC-2**: The `get_team_batting_stats()` function signature uses `season_id` as the parameter name (not `season`), with a default value of `"2026-spring-hs"`.
- [ ] **AC-3**: `src/api/routes/dashboard.py` computes the default season_id as `f"{datetime.date.today().year}-spring-hs"` (producing e.g., `"2026-spring-hs"`). The variable is named `season_id` (not `season`).
- [ ] **AC-4**: All callers of `get_team_batting_stats()` pass the updated parameter name. Grep the codebase for any other call sites.
- [ ] **AC-5**: After rebuilding the app and loading seed data (`python scripts/reset_dev_db.py`), the dashboard at `/dashboard` shows 5 players with non-zero stats (matching the seed data in `data/seeds/seed_dev.sql`).
- [ ] **AC-6**: Existing tests in `tests/` still pass. If any tests reference the old `season` parameter name, update them.

## Technical Approach
Two files to modify:

**`src/api/db.py`** (function `get_team_batting_stats`):
- Rename parameter `season` to `season_id`
- Change default from `"2026"` to `"2026-spring-hs"`
- Change SQL `WHERE psb.season = ?` to `WHERE psb.season_id = ?`
- Update docstring to reflect `season_id` parameter

**`src/api/routes/dashboard.py`** (function `team_stats`):
- Change line 85 from `season = str(datetime.date.today().year)` to `season_id = f"{datetime.date.today().year}-spring-hs"`
- Update all references from `season` to `season_id` in the function body (the call to `_fetch_dashboard_data` and the logger.debug call)
- Update `_fetch_dashboard_data` signature and body to use `season_id`

Reference files:
- `/workspaces/baseball-crawl/src/api/db.py` -- the query function (lines 57-96)
- `/workspaces/baseball-crawl/src/api/routes/dashboard.py` -- the route handler (lines 36-132)
- `/workspaces/baseball-crawl/data/seeds/seed_dev.sql` -- seed data uses `'2026-spring-hs'` as season_id
- `/workspaces/baseball-crawl/migrations/001_initial_schema.sql` -- column is `season_id` (line 154)
