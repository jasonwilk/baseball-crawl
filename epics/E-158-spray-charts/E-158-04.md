## Description
After this story is complete, a production rendering module will generate spray chart PNG images from a list of ball-in-play events, replicating GameChanger's exact field geometry, coordinate transforms, and binary hit/out color scheme. The module is importable from both dashboard routes and CLI. matplotlib is added as a project dependency.

## Context
The spike at `.project/research/spray-chart-spike/render.py` has a working implementation that replicates GC's rendering. This story ports that spike to a production module with a clean API: accept a list of event dicts (or DB rows), return PNG bytes. The rendering logic (coordinate transform, field geometry paths, hit/out classification, HR zone bubbles) comes directly from the spike.

## Acceptance Criteria
- [ ] **AC-1**: A rendering module exists at `src/charts/spray.py` with a public function `render_spray_chart(events: list[dict], title: str | None = None) -> bytes`. Each event dict must contain at minimum `x` (float | None), `y` (float | None), and `play_result` (str) keys matching DB column names. Returns PNG image bytes (not a file path).
- [ ] **AC-2**: The module applies the coordinate transform per Technical Notes TN-1 (exact constants from the spike).
- [ ] **AC-3**: The module draws the GC field geometry (outfield arc, foul lines, base diamonds) using the exact SVG path data from the spike.
- [ ] **AC-4**: Events are classified as hit or out per Technical Notes TN-3 and rendered with GC's exact colors (#00D682 hit, #B90018 out).
- [ ] **AC-5**: HR zone bubbles render for non-in-the-park home runs per Technical Notes TN-4, with zone (left/center/right) determined from the x-coordinate. When `x` and `y` are `None` and `play_result` is `home_run` (over-the-fence HR — no fielder location recorded), default to center zone per TN-4.
- [ ] **AC-5a**: Events with `x=None` or `y=None` skip the circle-rendering loop entirely (no coordinate transform possible). For `home_run` events with None coordinates, only the HR zone bubble count is incremented (center zone per AC-5). Non-HR events with None coordinates are silently skipped (defensive — should not occur under normal loader behavior per E-158-03 AC-5).
- [ ] **AC-6**: Outs render first, hits render on top (GC render order).
- [ ] **AC-7**: The function accepts an optional `title` parameter (default `None`). When provided, the title renders as a heading above the chart inside the PNG. When `None`, no title is drawn (dashboard routes rely on the HTML card heading instead, avoiding double-labeling).
- [ ] **AC-8**: Output is a 4×6 inch PNG at 150 DPI per Technical Notes TN-8.
- [ ] **AC-9**: `matplotlib` and `numpy` are added to `requirements.in` (with `~=` version ranges) and `requirements.txt` is regenerated via `pip-compile --generate-hashes --output-file=requirements.txt --strip-extras requirements.in`.
- [ ] **AC-10**: `src/charts/__init__.py` exists (package init).
- [ ] **AC-11**: Tests verify: coordinate transform math (known input → known SVG output), hit/out classification for each known play_result value, and that the render function returns non-empty PNG bytes for a sample event list.

## Technical Approach
Port the spike's `render.py` to a clean module. The public API is `render_spray_chart(events: list[dict], title: str | None = None) -> bytes` per AC-1. Event dicts use `x`, `y`, `play_result` keys matching DB column names. Use `io.BytesIO` to capture the PNG output instead of writing to disk. Keep the matplotlib `Agg` backend for headless rendering.

The spike code at `.project/research/spray-chart-spike/render.py` contains all the constants, path data, and rendering logic to port.
