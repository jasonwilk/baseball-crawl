## Status

## Description
After this story is complete, the `spray_charts` table will have the additional columns and indexes needed for idempotent ingestion and efficient querying. This unblocks the loader story (E-158-03) which depends on the `event_gc_id` UNIQUE column for `INSERT OR IGNORE` idempotency.

## Context
The `spray_charts` table exists in `migrations/001_initial_schema.sql` but is missing the event identifier needed for idempotent loading and has no query indexes. Since the table is unpopulated, ALTER TABLE ADD COLUMN is safe with no backfill needed. DE consultation confirmed: no `hr_location` column needed (computed at render time), no `created_at TEXT` load timestamp needed.

## Acceptance Criteria
- [ ] **AC-1**: A new migration file `migrations/006_spray_charts_indexes.sql` exists and is idempotent per migration conventions in `.claude/rules/migrations.md`.
- [ ] **AC-2**: The migration adds an `event_gc_id TEXT` column to `spray_charts` via `ALTER TABLE ADD COLUMN`.
- [ ] **AC-3**: The migration adds a `created_at_ms INTEGER` column to `spray_charts` via `ALTER TABLE ADD COLUMN` (stores the API's `createdAt` Unix millisecond timestamp).
- [ ] **AC-3a**: The migration adds a `season_id TEXT` column to `spray_charts` via `ALTER TABLE ADD COLUMN` (enables per-season filtering per fresh-start philosophy — the loader stores the `{season}` path segment directly, e.g., `2026-spring-hs`).
- [ ] **AC-4**: The migration creates a UNIQUE index `idx_spray_charts_event_gc_id ON spray_charts(event_gc_id)`.
- [ ] **AC-5**: The migration creates index `idx_spray_charts_player ON spray_charts(player_id, team_id, season_id)`.
- [ ] **AC-6**: The migration creates index `idx_spray_charts_game ON spray_charts(game_id)`.
- [ ] **AC-7**: All three `CREATE INDEX` statements use `IF NOT EXISTS` for idempotency.

## Technical Approach
The migration adds three columns via ALTER TABLE and three indexes. Since the `spray_charts` table is unpopulated, this is a pure schema-addition migration with no data backfill. The `event_gc_id` UNIQUE index is the critical piece — it enables the loader's `INSERT OR IGNORE` idempotency pattern per Technical Notes TN-6.

Note: ALTER TABLE ADD COLUMN does not support `IF NOT EXISTS` in SQLite, but per `.claude/rules/migrations.md`, this is safe because `apply_migrations.py` tracks applied migrations and runs each exactly once.
