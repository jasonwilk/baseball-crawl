# E-005-03: Retrofit GameChanger client to use session factory

## Epic
[E-005: HTTP Request Discipline](epic.md)

## Status
`TODO`

## Description
After this story is complete, `src/gamechanger/client.py` (`GameChangerClient`) will use `create_session()` from `src/http/session.py` instead of constructing a bare `httpx.Client`. All existing `GameChangerClient` acceptance criteria (E-001-02) continue to pass -- this story adds no new behavior, it changes the plumbing so that browser headers, cookie jar, and rate limiting are inherited automatically from the session factory.

## Context
E-001-02 wrote the GameChanger client before the HTTP Request Discipline existed. That client builds its own httpx.Client, which means it gets httpx defaults: no browser headers, no cookie jar, no jitter. E-005-02 fixes the infrastructure; this story wires it into the existing client.

This story should be executed AFTER E-001-02 is DONE. If E-001-02 has not yet been started when this story is picked up, the implementing agent should coordinate: either write E-001-02 directly against `create_session()` (making this story a no-op, in which case mark it ABANDONED), or finish E-001-02 as specced then apply this retrofit.

## Acceptance Criteria
- [ ] **AC-1**: Given `src/gamechanger/client.py`, when grepped for `httpx.Client(`, then zero matches are found -- the client uses `create_session()` instead.
- [ ] **AC-2**: Given `GameChangerClient()` instantiated with valid credentials, when a mocked GET request is made, then the request carries all headers from `BROWSER_HEADERS` plus the `Authorization` header injected by the client.
- [ ] **AC-3**: All existing E-001-02 acceptance criteria continue to pass (CredentialExpiredError on 401, RateLimitError on 429, retry logic on 5xx, ConfigurationError on missing credentials, request delay). Run the full E-001-02 test suite to confirm.
- [ ] **AC-4**: The `request_delay_ms` parameter on `GameChangerClient.__init__` now maps to `min_delay_ms` in `create_session()`. The default changes from 500ms to 1000ms to match the session factory default. If the existing E-001-02 tests hardcode the 500ms default, update them.
- [ ] **AC-5**: `GameChangerClient` does not import or use `httpx.Client` directly -- all httpx usage goes through the session from `create_session()`.

## Technical Approach
In `src/gamechanger/client.py`, replace:
```python
import httpx
...
self._client = httpx.Client(headers=self._build_headers(credentials), ...)
```

With:
```python
from src.http.session import create_session
...
self._client = create_session(min_delay_ms=self._request_delay_ms)
self._client.headers["Authorization"] = f"Bearer {credentials['auth_token']}"
# cookies, if any, injected similarly
```

The auth injection after `create_session()` is the pattern documented in the epic's Technical Notes. `create_session()` provides the base headers; the client layers credentials on top.

Note: since `create_session()` already enforces rate limiting via event hooks, remove any manual `time.sleep()` calls in `GameChangerClient.get()` -- the session handles it. The `request_delay_ms` constructor param can be kept for caller control but should now pass through to `create_session(min_delay_ms=...)`.
