# E-005-04: Write header discipline tests

## Epic
[E-005: HTTP Request Discipline](epic.md)

## Status
`DONE`

## Description
After this story is complete, the project will have a focused test module (`tests/test_http_discipline.py`) that verifies the complete HTTP discipline contract: correct headers on every request, auth tokens never appearing in logs at any log level, rate limiting enforcement, and jitter variance. These tests are integration-level -- they test the session factory and the GameChanger client together, with HTTP mocked at the transport layer.

## Context
E-005-01 and E-005-02 each have their own unit tests (test_http_headers.py, test_http_session.py). This story covers the integration seam: does the GameChanger client, using the session factory, actually send the right headers end-to-end? Does auth injection work without leaking to logs? These are the questions that matter most for security and correctness, and they are best answered at a higher level than unit tests.

Note: this story depends only on E-005-02 (the session factory). It does not require E-005-03 (the GameChanger client retrofit) -- the discipline tests can run against the session factory directly. If E-005-03 is done, add a brief integration test for the full client stack.

## Acceptance Criteria
- [ ] **AC-1**: Given `create_session()`, when a mocked GET request is made, then the test asserts all ten required header keys are present in the outgoing request and match their expected values from `BROWSER_HEADERS`.
- [ ] **AC-2**: Given `create_session()` with a mock auth token injected (`session.headers["Authorization"] = "Bearer test-token-abc"`), when a request is made and all log output is captured, then no captured log record at any level (DEBUG through CRITICAL) contains the string `"test-token-abc"`, `"Bearer"`, or `"Cookie"`.
- [ ] **AC-3**: Given `create_session(min_delay_ms=200, jitter_ms=100)`, when ten sequential mocked requests are made, then: (a) no inter-request gap is shorter than 200ms, and (b) not all inter-request gaps are identical (jitter is actually varying).
- [ ] **AC-4**: Given a mock server that sets `Set-Cookie: gc_auth=xyz; Path=/` on request 1, when request 2 is made on the same session, then the outgoing request 2 contains `Cookie: gc_auth=xyz`.
- [ ] **AC-5**: Given `create_session()` used with a mock server that returns 200, when the test checks `response.status_code`, then it is 200 -- confirming the session does not interfere with normal response handling.
- [ ] **AC-6** (if E-005-03 is done): Given `GameChangerClient()` with mocked credentials and mocked HTTP, when `client.get("/v2/teams/abc")` is called, then the outgoing request contains both the `Authorization: Bearer <token>` header AND all required browser headers -- neither category is missing.

## Technical Approach
Use `respx` for mocking httpx at the transport level -- it captures outgoing requests so headers can be inspected:

```python
import respx
import httpx
import time

@respx.mock
def test_browser_headers_sent():
    route = respx.get("https://example.com/test").mock(
        return_value=httpx.Response(200, json={})
    )
    session = create_session()
    session.get("https://example.com/test")
    request = route.calls.last.request
    assert request.headers["user-agent"].startswith("Mozilla/5.0")
    assert "sec-ch-ua" in request.headers
    ...
```

For log capture, use pytest's `caplog` fixture with `caplog.set_level(logging.DEBUG)` to catch everything:

```python
def test_no_token_in_logs(caplog):
    with caplog.at_level(logging.DEBUG):
        session = create_session()
        session.headers["Authorization"] = "Bearer test-token-abc"
        # make mocked request
    for record in caplog.records:
        assert "test-token-abc" not in record.message
        assert "Bearer" not in record.message
```

For timing tests, use `time.perf_counter()` around sequential requests and collect the gaps into a list. Assert `min(gaps) >= 0.200` and `len(set(round(g, 2) for g in gaps)) > 1` (not all identical when rounded to 10ms).
