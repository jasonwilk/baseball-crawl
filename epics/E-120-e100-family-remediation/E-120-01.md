# E-120-01: Fix resolve-opponents pipeline bug

## Epic
[E-120: E-100 Family Code Review Remediation](epic.md)

## Status
`TODO`

## Description
After this story is complete, `bb data resolve-opponents` will correctly resolve opponents by passing `db_path` to `load_config()` and will fail fast when a team's `internal_id` is None instead of silently using 0.

## Context
E-116-01 fixed the `load_config()` missing `db_path` bug in `src/gamechanger/pipeline/load.py`, but the same bug exists in the `resolve-opponents` CLI command. Currently, `load_config()` at `data.py:460` is called without `db_path`, so every `TeamEntry.internal_id` is None. The `or 0` fallback in `opponent_resolver.py:197` then silently converts None to 0, causing every opponent link to be written with `our_team_id=0` — which matches no real team. The command appears to succeed but writes zero useful rows.

## Acceptance Criteria
- [ ] **AC-1**: Given `src/cli/data.py` `resolve_opponents()`, when the function calls `load_config()`, then `db_path` is passed as a keyword argument (i.e., `load_config(db_path=db_path)`) — matching the fix pattern from E-116-01 in `src/pipeline/load.py:167`
- [ ] **AC-2**: Given `src/cli/data.py` `resolve_opponents()`, when `_resolve_db_path()` is called, then it executes before `load_config()` so that `db_path` is available to pass
- [ ] **AC-3**: Given `opponent_resolver.py` `_resolve_team()`, when `team.internal_id` is None, then a `ValueError` is raised with a message identifying the team (not silently converted to 0)
- [ ] **AC-4**: The `or 0` fallback at `opponent_resolver.py:197` is removed entirely
- [ ] **AC-5**: All existing tests pass (including `tests/test_crawlers/test_opponent_resolver.py`)

## Technical Approach
Two files, two changes. The fix pattern is identical to E-116-01 (commit `c72597d`).

1. `src/cli/data.py:460-464`: Currently `load_config()` is called at line 460 without `db_path`, and `_resolve_db_path()` is called at line 464. Reorder so `db_path = _resolve_db_path()` executes first, then call `load_config(db_path=db_path)`.
2. `src/gamechanger/crawlers/opponent_resolver.py:197`: Replace `our_team_id: int = team.internal_id or 0` with a guard that raises `ValueError` when `team.internal_id is None`.
