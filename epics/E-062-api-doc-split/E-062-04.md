# E-062-04: Build Endpoint Index and Remove Monolith

## Epic
[E-062: Split API Documentation into Per-Endpoint Files](epic.md)

## Status

## Description
After this story is complete, `docs/api/README.md` will serve as the navigable index for all API documentation -- listing every endpoint with method, path, status, and auth requirement in a scannable table with links to individual files. The monolithic `docs/gamechanger-api.md` will be deleted. Agents will use the index to find endpoints and load only what they need.

## Context
With all content migrated (reference files in E-062-01, endpoint files in E-062-02, E-062-03, and E-062-06), this story ties everything together with an index and removes the source of truth that is no longer needed. The index is the new entry point for all API documentation -- it replaces the monolith's table of contents and serves as the machine-scannable directory listing.

## Acceptance Criteria
- [ ] **AC-1**: `docs/api/README.md` exists with a table listing every endpoint file: columns for Method, Path, Status, Auth, and a link to the file
- [ ] **AC-2**: The table is sorted by path (alphabetically), with `/public/*` and `/teams/public/*` endpoints grouped together under a "Public Endpoints" subheading
- [ ] **AC-3**: The README includes links to all global reference files (authentication.md, headers.md, pagination.md, content-type.md, observations.md, etc.)
- [ ] **AC-4**: The README includes a brief "How to Use This Documentation" section explaining the structure for both agents and humans
- [ ] **AC-5**: `docs/gamechanger-api.md` is deleted (git rm)
- [ ] **AC-6**: A completeness check confirms: (a) total endpoint files in `docs/api/endpoints/` matches the spike inventory count, (b) total rows in the index table matches the file count, (c) the inventory endpoint count matches both -- three-way match with counts logged in a commit message or story notes
- [ ] **AC-7**: Every endpoint file in `docs/api/endpoints/` is listed in the index (no orphan files)

## Technical Approach
Read all endpoint files from `docs/api/endpoints/` and parse their YAML frontmatter to extract method, path, status, and auth fields. Build the index table from this data. Cross-check against the endpoint inventory from E-062-R-01 (at `/workspaces/baseball-crawl/.project/research/E-062-format-spec.md` or its inventory section) to verify completeness.

After confirming completeness, delete `docs/gamechanger-api.md`.
