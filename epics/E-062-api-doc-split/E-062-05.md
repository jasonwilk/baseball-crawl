# E-062-05: Update Context Layer References

## Epic
[E-062: Split API Documentation into Per-Endpoint Files](epic.md)

## Status
`TODO`

## Description
After this story is complete, all references to `docs/gamechanger-api.md` across the context layer (CLAUDE.md, agent definitions, rules, skills, and agent memory files) will point to the new `docs/api/` structure. Agents will find the API documentation at the correct path without encountering stale references. The ingest-endpoint skill workflow will be updated from "append to monolith" to "create/update endpoint file."

## Context
Multiple context-layer files reference `docs/gamechanger-api.md` as the "single source of truth for API knowledge." With the monolith replaced by the `docs/api/` directory structure, these references must be updated. Claude-architect consultation (2026-03-07) identified 17+ context-layer files containing references -- significantly more than originally scoped. Some updates are simple path swaps; others require semantic rewording (e.g., changing "maintains a single file" to "maintains the `docs/api/` directory structure").

### Path Mapping Rules (from claude-architect consultation)
- References to the monolith as "the API spec" or "source of truth" -> `docs/api/README.md`
- References to specific sections that became reference files (authentication, headers, pagination) -> the specific reference file (e.g., `docs/api/authentication.md`)
- References in a "load this before API work" context -> `docs/api/README.md` (agents start at the index, then load specific files as needed)
- The ingest-endpoint skill's "document in the API spec" instructions -> rewrite to reference creating/updating files in `docs/api/endpoints/`
