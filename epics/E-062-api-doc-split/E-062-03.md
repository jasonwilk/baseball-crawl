## Description
After this story is complete, all endpoints from the "Confirmed Endpoints (2026-03-07)" section of the monolith will exist as individual files under `docs/api/endpoints/`, following the format spec from E-062-R-01. These are endpoints verified via curl with brief documentation (status code, response snippet, notes).

## Context
The "Confirmed Endpoints" section contains endpoints that were verified by direct curl testing but documented more briefly than the main "Endpoints" section. Some of these endpoints also appear in the "Proxy-Discovered" sections with additional observational notes. When an endpoint assigned to this story also has mentions in proxy-discovered sections, all information from all mentions must be consolidated into the single endpoint file.

## Acceptance Criteria
- [ ] **AC-1**: Every endpoint assigned to this story by the spike inventory (E-062-R-01) exists as an individual file in `docs/api/endpoints/`
- [ ] **AC-2**: Each file follows the YAML frontmatter schema and body structure from the format spec at `/workspaces/baseball-crawl/.project/research/E-062-format-spec.md`
- [ ] **AC-3**: File naming follows the convention from the format spec
- [ ] **AC-4**: When an endpoint assigned to this story also appears in other monolith sections (e.g., proxy-discovered), all information from all mentions is consolidated into the single file
- [ ] **AC-5**: The `status` field in frontmatter is set to `CONFIRMED` for all endpoints in this story (they were verified via curl)
- [ ] **AC-6**: All information from the monolith is preserved for each endpoint -- discovery dates, status codes, observational notes, behavioral quirks, and cross-references
- [ ] **AC-7**: Cross-references to global content use relative links to reference files in `docs/api/`
- [ ] **AC-8**: No endpoint file created in this story duplicates one assigned to E-062-02 or E-062-06 per the spike inventory

## Technical Approach
Read the format spec and endpoint inventory from E-062-R-01 (at `/workspaces/baseball-crawl/.project/research/E-062-format-spec.md`). The inventory assigns each endpoint to exactly one story and flags endpoints with mentions in multiple monolith sections. For each endpoint assigned to this story, gather all mentions from across the monolith (use the inventory's cross-reference notes), consolidate into a single file, and write to `docs/api/endpoints/`.

The monolith is not modified or deleted in this story -- that happens in E-062-04.
