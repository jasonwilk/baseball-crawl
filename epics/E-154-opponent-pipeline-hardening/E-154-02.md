# E-154-02: Remove dead resolve_missing_public_ids bridge code

## Epic
[E-154: Opponent Resolution Pipeline Hardening](epic.md)

## Status
`TODO`

## Description
After this story is complete, the `resolve_missing_public_ids` method will be removed from `ScoutingCrawler`, its call site in the CLI scout command will be removed, and the associated test file will be deleted. This dead code path attempted to backfill `teams.public_id` via the bridge endpoint, which returns 403 for non-managed teams and can never succeed for opponent teams.

## Context
`resolve_missing_public_ids` was added as a workaround for the missing `teams.public_id` gap. It calls `GET /teams/{gc_uuid}/public-team-profile-id` for each opponent with `gc_uuid` but no `public_id`. However, the bridge endpoint returns 403 for teams the authenticated user does not manage (confirmed by api-scout 2026-03-09 and 2026-03-11). Since opponent teams are by definition not managed by the operator, this method will never succeed. With E-154-01 fixing the root cause (writing `public_id` to `teams` during resolution), this method is doubly dead.

Remove the method, its imports, its call site, and its tests per TN-5 in the epic Technical Notes. Check each import (`BridgeForbiddenError`, `resolve_uuid_to_public_id`, `ConfigurationError`) for usage elsewhere in `scouting.py` before removing.
