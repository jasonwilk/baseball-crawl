## Description
After this story is complete, a new `proxy-review.sh` script allows the operator to mark sessions as reviewed and to list sessions with their review status. This gives the operator a clear answer to "what proxy captures have I not looked at yet?"

## Context
E-052-01 creates `session.json` with a `reviewed` field (default `false`). This story provides the tooling to flip that flag and to list sessions by status. The report scripts (E-052-04) will use this flag for the `--unreviewed` filter.

## Acceptance Criteria
- [ ] **AC-1**: A new script `scripts/proxy-review.sh` exists with the following subcommands:
  - `list` -- prints a table of all sessions with columns: session_id, profile, status, endpoint_count, reviewed
  - `mark <session-id>` -- sets `reviewed: true` in the specified session's `session.json`
  - `mark --all` -- sets `reviewed: true` in all closed sessions' `session.json` files
- [ ] **AC-2**: `proxy-review.sh list` sorts sessions by session_id (chronological, since IDs are timestamps).
- [ ] **AC-3**: `proxy-review.sh list` shows the `current` (active) session with a `*` marker in the status column (e.g., `active *`).
- [ ] **AC-4**: `proxy-review.sh mark <session-id>` fails with an error message if the session directory does not exist.
- [ ] **AC-5**: `proxy-review.sh mark <session-id>` updates `session.json` in place using `jq`, preserving all other fields.
- [ ] **AC-6**: `proxy-review.sh mark --all` only marks sessions with `status: "closed"` (does not mark active sessions).
- [ ] **AC-7**: Running `proxy-review.sh` with no arguments or with `--help` prints usage.
- [ ] **AC-8**: The script uses `proxy/data/sessions/` as the sessions root directory. If no sessions exist, prints a "No sessions found" message.

## Technical Approach
- Pure bash script using `jq` for JSON manipulation (consistent with existing `proxy-report.sh` and `proxy-endpoints.sh`).
- `list` subcommand: iterate `proxy/data/sessions/*/session.json`, extract fields with `jq`, format with `printf`. Check if `proxy/data/current` symlink target matches for the `*` marker.
- `mark` subcommand: `jq '.reviewed = true' session.json > tmp && mv tmp session.json`.
- Place in `scripts/` alongside the other proxy-related scripts.
