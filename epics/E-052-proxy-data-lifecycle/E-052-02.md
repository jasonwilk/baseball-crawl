# E-052-02: Route addon output to active session

## Epic
[E-052: Proxy Data Lifecycle](epic.md)

## Status
`TODO`

## Description
After this story is complete, the `EndpointLogger` and `HeaderCapture` addons write their output files into the active session directory (determined by `PROXY_SESSION_DIR` env var) instead of the hardcoded `proxy/data/` flat files. When `PROXY_SESSION_DIR` is not set, addons fall back to the old paths for backward compatibility.

## Context
Currently `EndpointLogger` writes to `/app/proxy/data/endpoint-log.jsonl` and `HeaderCapture` writes to `/app/proxy/data/header-report.json` -- both hardcoded. E-052-01 introduces session directories and passes `PROXY_SESSION_DIR` to the container. This story makes the addons session-aware.

## Acceptance Criteria
- [ ] **AC-1**: `EndpointLogger` reads `PROXY_SESSION_DIR` from `os.environ` at initialization. If set, `LOG_PATH` resolves to `$PROXY_SESSION_DIR/endpoint-log.jsonl`. If not set, `LOG_PATH` falls back to `/app/proxy/data/endpoint-log.jsonl` (current behavior).
- [ ] **AC-2**: `HeaderCapture` reads `PROXY_SESSION_DIR` from `os.environ` at initialization. If set, `_REPORT_PATH` resolves to `$PROXY_SESSION_DIR/header-report.json`. If not set, `_REPORT_PATH` falls back to `/app/proxy/data/header-report.json` (current behavior).
- [ ] **AC-3**: Both addons create the output directory (`mkdir -p`) if it does not exist on first write (existing behavior preserved).
- [ ] **AC-4**: Existing unit tests in `tests/test_proxy/test_endpoint_logger.py` continue to pass (they should not set `PROXY_SESSION_DIR`, so the fallback path is used).
- [ ] **AC-5**: Existing unit tests in `tests/test_proxy/test_header_capture.py` continue to pass.
- [ ] **AC-6**: New test: when `PROXY_SESSION_DIR` is set (via `monkeypatch.setenv`), `EndpointLogger` writes to `$PROXY_SESSION_DIR/endpoint-log.jsonl`.
- [ ] **AC-7**: New test: when `PROXY_SESSION_DIR` is set (via `monkeypatch.setenv`), `HeaderCapture` writes to `$PROXY_SESSION_DIR/header-report.json`.
- [ ] **AC-8**: The old flat files `proxy/data/endpoint-log.jsonl` and `proxy/data/header-report.json` are deleted from the repository (they are gitignored data files, but if any tracked references exist, those are updated).

## Technical Approach
- Change `LOG_PATH` and `_REPORT_PATH` from module-level constants to instance attributes set in `__init__()`. Read `os.environ.get("PROXY_SESSION_DIR")` and construct the path accordingly.
- For `EndpointLogger`: move `LOG_PATH` into `__init__` as `self.log_path`. Update `_append_entry` to accept the path or make it an instance method that uses `self.log_path`.
- For `HeaderCapture`: move `_REPORT_PATH` into `__init__` as `self.report_path`. Update `_write_report` to use `self.report_path`.
- In tests, use `monkeypatch.setenv("PROXY_SESSION_DIR", str(tmp_path))` and verify output lands in the right place. Use `monkeypatch.delenv("PROXY_SESSION_DIR", raising=False)` for fallback tests.
- The `loader.py` file instantiates addons at import time. Since `PROXY_SESSION_DIR` is set before the container starts (by `start.sh`), the env var will be available at addon instantiation time.
