# E-143-01: Program List and Create Admin Page

## Epic
[E-143: Admin Management Overhaul](epic.md)

## Status
`TODO`

## Description
After this story is complete, the operator can create and view programs entirely from the admin UI via a new "Programs" sub-tab. Programs are the umbrella entity grouping teams under an organizational program (e.g., `lsb-hs` = Lincoln Standing Bear HS). Currently the `programs` table exists and is seeded but has no admin form -- programs can only be created via SQL.

## Context
The `programs` table already exists in the schema (see TN-6 for column details). The admin UI reads programs for team-assignment dropdowns via `_get_programs()` but has no create/list surface. The UX designer recommended a dedicated Programs sub-tab (cleaner than embedding in the Teams page). This is the simplest story in the epic -- no dependencies, no migrations, pure list-and-create against an existing table.

## Acceptance Criteria
- [ ] **AC-1**: A "Programs" tab appears in the admin sub-navigation per TN-7 and TN-1a. The tab is added to all 7 existing admin templates that contain the inline sub-nav: `teams.html`, `users.html`, `opponents.html`, `edit_user.html`, `edit_team.html`, `confirm_team.html`, `opponent_connect.html`, plus the new `programs.html` template.
- [ ] **AC-2**: `GET /admin/programs` renders a page listing all programs with columns: name, program ID (slug), type (hs/usssa/legion), org name, team count, created date.
- [ ] **AC-3**: An "Add Program" form on the Programs page accepts: program ID (text slug), display name, program type (dropdown: hs/usssa/legion), and org name (optional). On submit (`POST /admin/programs`), the program is created and a success flash confirms it.
- [ ] **AC-4**: Duplicate program ID submission returns a clear validation error flash (not a 500 error).
- [ ] **AC-5**: Newly created programs immediately appear in the program dropdown on the Phase 2 team-add confirm page (`/admin/teams/confirm`) and on the team edit page (`/admin/teams/{id}/edit`) without app restart.

## Technical Approach
Add new routes for program list and create. The `programs` table uses a TEXT PK (`program_id` is the operator-chosen slug). Team count is a query against `teams.program_id`. Template follows existing admin page patterns (table + form on same page). Per TN-1a, the admin sub-nav is inline markup duplicated across 7 existing templates (not a shared partial): `teams.html`, `users.html`, `opponents.html`, `edit_user.html`, `edit_team.html`, `confirm_team.html`, `opponent_connect.html` -- adding the "Programs" tab requires updating all 7 existing templates plus the new programs template.
