## Description
After this story is complete, the standalone scouting report HTML template will feature the redesigned layout with heat-map color coding, executive summary strip, colored recent-form chips, key-players callout, reordered and expanded table columns, sort annotations, 2-per-row spray charts with team aggregate, and an enhanced footer -- all rendered as self-contained HTML that produces print-ready PDFs via browser print dialog.

## Context
Story 01 enriched the data pipeline to provide all the new context keys (heat levels, key players, K%/BB%/XBH/SB-CS, structured recent form, team spray chart, runs averages). This story rewrites the Jinja2 template to consume that enriched context and implement the UXD-designed layout. The template is self-contained (inline CSS, no external dependencies) and must preserve PDF fidelity through `print-color-adjust: exact`.

- [ ] **AC-9**: Column groups are separated by subtle vertical borders per TN-11 (identity | threat | rate | volume | counting groups).

## Technical Approach
Complete rewrite of `src/api/templates/reports/scouting_report.html`. The template consumes enriched context from Story 01's renderer changes.

**CSS additions:**
- Heat-map CSS classes (`.heat-0` through `.heat-4`) with background colors per TN-1
- Print color fidelity: `print-color-adjust: exact` and `-webkit-print-color-adjust: exact` on the `body` selector (not inside `@media print`) to propagate to all children
- Executive summary strip styling
- Recent form chip styling (colored inline badges)
- Key players callout box styling
- Column group separator borders
- Sort annotation styling (small italic text below section headers)
- Updated spray chart grid (2 columns instead of 3)
- Diamond indicator styling

**Template structure changes:**
- Header block: reverse label/name order per TN-10
- New executive summary strip block
- New recent form chips block (iterate `recent_form` list)
- New key players callout block (conditional on `key_players`)
- Pitching table: add THR column, handedness in player name, heat-map classes on stat cells
- Batting table: add THR/K%/BB%/PA/XBH/SB-CS/HBP columns, reorder columns per TN-3, diamond indicator, heat-map classes
- Spray section: team chart first, then 2-per-row individual charts with stat line
- Footer: add expiry date and bbstats.ai

**Jinja2 patterns:**
- Heat-map class assignment: `class="heat-{{ player._heat.obp }}"` (levels are integers 0-4)
- Conditional small-sample styling: `{% if player._small_sample %}class="small-sample"{% endif %}`
- Diamond indicator: `{% if player.player_id in spray_data %}♦{% endif %}`
