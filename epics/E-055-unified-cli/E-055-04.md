## Description
After this story is complete, the operator can view proxy analysis reports and manage proxy data through `bb proxy report|endpoints|refresh-headers|review`. These commands shell out to existing bash and Python scripts. Proxy lifecycle commands (start/stop/status/logs) are excluded because the proxy runs on the Mac host, not inside the devcontainer where the CLI runs.

## Context
The proxy analysis scripts are split across two directories (`scripts/proxy-report.sh`, `scripts/proxy-endpoints.sh`, `scripts/proxy-refresh-headers.py`, `scripts/proxy-review.sh`). The CLI wraps them under a unified `bb proxy` group so the operator does not need to remember which script lives where. These commands work inside the devcontainer because they read files from the mounted `proxy/data/` volume or modify source files -- they do not need Docker access.

After E-052 (Proxy Data Lifecycle), the report and endpoints scripts gain `--session`, `--all`, and `--unreviewed` flags. After E-054 (Header Parity Refresh), `scripts/proxy-refresh-headers.py` exists. This story wraps the post-E-052/E-054 interfaces.

## Technical Approach
1. Create `src/cli/proxy.py` with a `typer.Typer(help="Proxy analysis commands.")` sub-app.
2. `report` command forwards `--session` and `--all` flags (no `--unreviewed`). `endpoints` command forwards `--session`, `--all`, and `--unreviewed` flags.
3. `refresh-headers` invokes `scripts/proxy-refresh-headers.py` via `subprocess.run()`, forwarding `--apply` when specified.
4. `review` invokes `scripts/proxy-review.sh` via `subprocess.run()`.
5. Script paths should be resolved relative to the project root (use `pathlib.Path(__file__).resolve().parent.parent.parent` or similar to find the project root). **All `subprocess.run()` calls MUST pass `cwd=PROJECT_ROOT`** because the bash scripts use relative paths internally (e.g., `proxy-report.sh` reads `proxy/data/header-report.json`). Without `cwd`, these scripts break when the operator runs `bb` from any directory other than the project root.
6. The sub-app mount in `src/cli/__init__.py` is pre-wired by E-055-01 -- do not modify `__init__.py`.

## Notes
- Proxy lifecycle commands (start/stop/status/logs) are intentionally excluded. The proxy runs on the Mac host via Docker Compose; the `bb` CLI runs inside the devcontainer. These commands cannot work across the container boundary. See CLAUDE.md "Proxy Boundary (Host vs. Container)" for details.
- `bb proxy report` defaults to showing the latest session (no flags needed). `--all` aggregates across sessions. `--session` targets a specific session by ID. No `--unreviewed` flag (header reports are point-in-time snapshots).
- `bb proxy endpoints` defaults to showing the latest session. `--all` aggregates across sessions. `--unreviewed` filters to unreviewed sessions only. `--session` targets a specific session by ID.
- `bb proxy refresh-headers` is dry-run by default (shows what would change). The `--apply` flag writes the changes to `src/http/headers.py`.
