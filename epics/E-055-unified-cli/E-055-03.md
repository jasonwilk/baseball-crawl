# E-055-03: Data Pipeline Commands (`bb data`)

## Epic
[E-055: Unified Operator CLI](epic.md)

## Status

## Description
After this story is complete, the operator can run the data pipeline through `bb data sync` (full bootstrap: validate + crawl + load), `bb data crawl` (crawl only), and `bb data load` (load only). These wrap the existing `scripts/bootstrap.py`, `scripts/crawl.py`, and `scripts/load.py` with consistent Typer interfaces.

## Context
The data pipeline is the core operator workflow. `bb data sync` becomes the "one command to refresh everything" that the operator runs most frequently. `crawl` and `load` are available for debugging or partial runs.

## Acceptance Criteria
- [ ] **AC-1**: `src/cli/data.py` exists and defines a Typer sub-app mounted on the main app as `bb data`.
- [ ] **AC-2**: `bb data sync` runs the full bootstrap pipeline (validate creds, check teams, crawl, load) with default flags (profile=web, no dry-run). Exit codes match `bootstrap.py` behavior.
- [ ] **AC-3**: `bb data sync --check-only` runs validation only, no crawl/load.
- [ ] **AC-4**: `bb data sync --profile mobile` passes the mobile profile to the crawl stage.
- [ ] **AC-5**: `bb data sync --dry-run` passes dry-run through to crawl and load stages.
- [ ] **AC-6**: `bb data crawl` runs all crawlers. Supports `--dry-run`, `--crawler NAME`, `--profile`, and `--source db|yaml` (default: yaml). The `--source` flag maps to the `source` parameter added by E-042-06.
- [ ] **AC-7**: `bb data load` runs all loaders. Supports `--dry-run`, `--loader NAME`, and `--source db|yaml` (default: yaml). The `--source` flag maps to the `source` parameter added by E-042-06.
- [ ] **AC-8**: `bb data --help` lists all three commands with descriptions.
- [ ] **AC-9**: Tests using `CliRunner` verify argument mapping for sync, crawl, and load (mock underlying `run()` functions). Tests include `--source db` and `--source yaml` forwarding for crawl and load.
- [ ] **AC-10**: All existing tests continue to pass.

## Technical Approach
1. Create `src/cli/data.py` with a `typer.Typer(help="Data pipeline commands.")` sub-app.
2. `sync` command imports and calls `scripts.bootstrap.run()`.
3. `crawl` command imports and calls `scripts.crawl.run()`.
4. `load` command imports and calls `scripts.load.run()`.
5. The sub-app mount in `src/cli/__init__.py` is pre-wired by E-055-01 -- do not modify `__init__.py`.
6. Tests mock the `run()` functions and verify correct argument forwarding.

Reference files:
- `/workspaces/baseball-crawl/scripts/bootstrap.py` (has `run(check_only: bool, profile: str, dry_run: bool) -> int`)

## Notes
- The `crawl` command's `--crawler` flag accepts choices: roster, schedule, opponent, player-stats, game-stats. Use Typer's `typer.Option` with a callback or enum to validate.
- The `load` command's `--loader` flag accepts choices: roster, game, season-stats.
- The `--source` flag on both `crawl` and `load` accepts `db` or `yaml` (default: `yaml`). When `db`, team config is loaded from SQLite; when `yaml`, from `config/teams.yaml`. This flag is added by E-042-06 and must be forwarded to the underlying `run()` functions.
- Do not expose `data_root` or `db_path` as CLI flags -- those are test-only parameters.
- **`bb data sync` intentionally has NO `--source` flag.** Bootstrap's `run()` has no `source` parameter -- it always uses the default config path. Operators who need DB-sourced config must use `bb data crawl --source db` and `bb data load --source db` separately. Adding `--source` to sync is deferred to IDEA-012 (crawl orchestration).
- **`bb data sync` help text guidance**: The help description for `sync` should mention that it uses YAML team config by default. Include a note like "For database-sourced team config, use `bb data crawl --source db` and `bb data load --source db` separately." This prevents operator confusion when `--source` is not available on `sync`.
