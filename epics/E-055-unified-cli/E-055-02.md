# E-055-02: Credential Commands (`bb creds`)

## Epic
[E-055: Unified Operator CLI](epic.md)

## Status
`TODO`

## Description
After this story is complete, the operator can manage GameChanger credentials through `bb creds refresh` and `bb creds check`. These commands wrap the existing `scripts/refresh_credentials.py` and `scripts/check_credentials.py` functionality with consistent Typer interfaces.

## Context
Credential management is the most frequent operator task -- credentials expire every 14 days and must be refreshed from a browser curl capture. Having these under a discoverable `bb creds` group means the operator does not need to remember file paths.

## Acceptance Criteria
- [ ] **AC-1**: `src/cli/creds.py` exists and defines a Typer sub-app mounted on the main app as `bb creds`.
- [ ] **AC-2**: `bb creds refresh` with no flags reads from `secrets/gamechanger-curl.txt` (matching current default behavior), extracts credentials, and writes to `.env`.
- [ ] **AC-3**: `bb creds refresh --curl "curl ..."` accepts an inline curl command (matching `--curl` flag of `refresh_credentials.py`).
- [ ] **AC-4**: `bb creds refresh --file /path/to/file` accepts a file path (matching `--file` flag of `refresh_credentials.py`).
- [ ] **AC-5**: `bb creds check` validates credentials by calling GET /me/user and prints the result with appropriate exit code (0=valid, 1=expired, 2=missing). Supports `--profile` flag to validate a specific profile (web or mobile) or all profiles (per E-053's profile-aware interface).
- [ ] **AC-6**: `bb creds --help` lists both `refresh` and `check` with descriptions.
- [ ] **AC-7**: Tests using `CliRunner` verify argument mapping for `refresh` (default, --curl, --file) and `check` (mocked API call).
- [ ] **AC-8**: All existing tests continue to pass.

## Technical Approach
1. Create `src/cli/creds.py` with a `typer.Typer(help="Credential management.")` sub-app.
2. `refresh` command imports and calls the parsing/merging logic from `src/gamechanger/credential_parser.py` (same as `refresh_credentials.py` does).
3. `check` command imports and calls `check_credentials()` from `scripts/check_credentials.py`.
4. The sub-app mount in `src/cli/__init__.py` is pre-wired by E-055-01 -- do not modify `__init__.py`.
5. Tests mock the credential parser and API calls; do not re-test business logic.
