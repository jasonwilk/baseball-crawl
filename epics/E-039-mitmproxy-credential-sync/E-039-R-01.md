## Objective
Determine whether existing open-source mitmproxy addons or projects already provide credential extraction, header capture, or API discovery functionality -- so we build only what is missing. Also validate that the `mitmproxy/mitmproxy` Docker image supports the Python environment we need for addon scripts.

## Time Box
One focused session (2-3 hours equivalent). Do not prototype or write production code -- research and document only.

## Research Questions
1. **Existing addons**: Are there published mitmproxy addon scripts (GitHub, PyPI, mitmproxy docs) that extract authentication headers, capture header profiles, or log API endpoints? List any candidates with links, last-updated dates, and relevance assessment.
2. **Addon API surface**: What mitmproxy addon hooks are available for our use cases? Specifically: `request()`, `response()`, `configure()`, flow metadata API, and any built-in filtering mechanisms. Document the minimum API we need.
3. **Docker image environment**: What Python version ships in the `mitmproxy/mitmproxy` Docker image (latest tag)? What packages are pre-installed? Can we `pip install python-dotenv` inside it, or do we need a custom Dockerfile layer?
4. **Configuration options**: How does mitmproxy handle addon loading (`--scripts` vs. config file)? Can multiple addons run concurrently? Is there a recommended pattern for addon-to-addon communication (e.g., shared metadata on flows)?
5. **mitmweb vs. mitmdump vs. mitmproxy**: Which variant is best for our use case (headless Docker service with web UI access)? `mitmweb` provides the web UI; `mitmdump` is headless. Can we run `mitmweb` and still use `--scripts`?
6. **Certificate persistence**: How does mitmproxy handle CA certificate generation across container restarts? Is a volume mount to `~/.mitmproxy/` sufficient for cert persistence?
7. **iOS proxy configuration**: Any known gotchas with iOS 17+ trusting mitmproxy CA certificates? Does the `mitm.it` automatic cert installation still work reliably?

## Deliverables
- A markdown document at `/workspaces/baseball-crawl/epics/E-039-mitmproxy-credential-sync/E-039-R-01-findings.md` summarizing findings for each research question above
- A recommendation section covering: (a) which existing addons to reuse vs. build, (b) which mitmproxy variant to use, (c) whether a custom Docker image is needed, (d) any design changes to the epic Technical Notes based on findings

## Approach
1. Search GitHub for mitmproxy addon repositories related to: credential extraction, header logging, API documentation, traffic recording. Check mitmproxy's official addon examples.
2. Read the mitmproxy addon API documentation (https://docs.mitmproxy.org/) for hooks, flow metadata, and multi-addon patterns.
3. Check the `mitmproxy/mitmproxy` Docker Hub page for image details (Python version, installed packages).
4. Review mitmproxy docs and community resources for iOS HTTPS interception guidance.
5. Summarize findings and make recommendations.
