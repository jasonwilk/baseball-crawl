## Description
After this story is complete, mitmproxy runs as an optional Docker Compose service behind the `proxy` profile. `docker compose --profile proxy up -d` starts the proxy alongside the app stack. The proxy listens on port 8080 (proxy traffic) and 8081 (mitmweb UI), both bound to `0.0.0.0` for LAN access. The Traefik dashboard moves from port 8080 to 8180 to avoid the conflict. CA certificates persist across container restarts via a named volume.

## Context
This is the infrastructure foundation for all mitmproxy addons. No addon scripts are loaded yet -- this story just gets the service running and reachable. The proxy must be reachable from the iPhone on the LAN, so it binds to `0.0.0.0` rather than `127.0.0.1`. The profile approach ensures the proxy does not start by default and has zero impact on users who do not need it.

## Acceptance Criteria
- [ ] **AC-1**: Given the `docker-compose.yml` file, when `docker compose up -d` is run (no profile flag), then only `app`, `traefik`, and `cloudflared` services start -- mitmproxy does NOT start.
- [ ] **AC-2**: Given the `docker-compose.yml` file, when `docker compose --profile proxy up -d` is run, then the `mitmproxy` service starts alongside the existing services and reports healthy.
- [ ] **AC-3**: Given the mitmproxy service is running, when an HTTP request is sent through `<host-ip>:8080` as a proxy, then the request is proxied successfully (verify with `curl -x http://localhost:8080 http://httpbin.org/ip`).
- [ ] **AC-4**: Given the mitmproxy service is running, when `http://localhost:8081` is opened in a browser, then the mitmweb UI loads and shows intercepted traffic.
- [ ] **AC-5**: Given the mitmproxy service is stopped and restarted, when the CA certificate fingerprint is checked, then it matches the previous run (certificates persist via volume).
- [ ] **AC-6**: Given the docker-compose.yml file, the Traefik dashboard is accessible on port 8180 (not 8080). Existing Traefik routing (`baseball.localhost:8000`) continues to work.
- [ ] **AC-7**: Given `devcontainer.json`, ports 8080, 8081, and 8180 are listed in `forwardPorts`.
- [ ] **AC-8**: The `data/mitmproxy/` directory is listed in `.gitignore`.
- [ ] **AC-9**: The `mitmproxy/` top-level directory exists with an `addons/__init__.py` file (empty, placeholder for addon scripts).

## Technical Approach
1. Add a `mitmproxy` service to `docker-compose.yml` using the `mitmproxy/mitmproxy` image (or `mitmweb` variant per R-01 findings). Set `profiles: [proxy]`. Mount a named volume for cert persistence (`mitmproxy-certs:/home/mitmproxy/.mitmproxy`). Mount `./mitmproxy/addons` for addon script loading. Bind ports `0.0.0.0:8080:8080` and `0.0.0.0:8081:8081`.
2. Change the Traefik dashboard port from `8080:8080` to `8180:8080`.
3. Update `devcontainer.json` `forwardPorts` to include 8081 and 8180 (8080 is already there).
4. Add `data/mitmproxy/` to `.gitignore`.
5. Create `mitmproxy/addons/__init__.py` (empty file).
6. If R-01 findings indicate a custom Dockerfile is needed, create `mitmproxy/Dockerfile` with the necessary additions.
