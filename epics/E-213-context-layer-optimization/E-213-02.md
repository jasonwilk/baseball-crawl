## Description
After this story is complete, the Workflow Routing Rule section in `workflow-discipline.md` will be a 2-line pointer to `dispatch-pattern.md` instead of a near-verbatim copy. Any unique content from the workflow-discipline version will have been migrated to `dispatch-pattern.md` first. ## Context
Both `dispatch-pattern.md` (49 lines) and `workflow-discipline.md` (98 lines) are universal rules (`paths: "**"`). The Workflow Routing Rule section in workflow-discipline.md (line 68-70, a single dense paragraph) substantially duplicates dispatch-pattern.md's content: main session role, file operation constraints, PM dispatch role, staging boundary protocol, code-reviewer gate. Together they impose ~600 words of content duplication on every session. The audit (F-05, F-14) identified this as the primary universal rule optimization target.
