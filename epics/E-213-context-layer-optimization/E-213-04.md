# E-213-04: Fix Rule Frontmatter and Compress Universal Rules

## Epic
[E-213: Context Layer Optimization](epic.md)

## Status

## Description
After this story is complete, three rule file issues will be fixed: `gc-uuid-bridge.md` will use `paths:` instead of `globs:` (ensuring correct scoping), `pytest-verbose.md` will have explicit frontmatter declaring its universal scope, and `agent-team-compliance.md`'s "Why These Rules Exist" section will be compressed from 9 lines to ~3 lines while preserving the core enforcement insight.

## Context
The audit identified three rule frontmatter/content issues:
- **F-06**: `gc-uuid-bridge.md` uses `globs:` instead of `paths:` -- if `globs:` is not a recognized alias, this 66-line rule may load universally on every session instead of only on its scoped paths.
- **F-07**: `pytest-verbose.md` has no frontmatter at all (27 lines). Universal loading is arguably intentional (any agent might run pytest), but it should be explicit.
- **F-15**: `agent-team-compliance.md`'s "Why These Rules Exist" section (9 lines of historical context about 6 failed epics) loads on every session but the narrative is background, not enforcement. The core insight ("when a user names specific agents, they are requesting those agents' judgment") and the three patterns are what matter.

## Acceptance Criteria
- [ ] **AC-1**: `gc-uuid-bridge.md` frontmatter uses `paths:` key (not `globs:`) with the same scope values. The `description:` field is removed (inconsistent with all other rule files).
- [ ] **AC-2**: `pytest-verbose.md` has a YAML frontmatter block with `paths: "**"` (explicitly universal)
- [ ] **AC-3**: `agent-team-compliance.md`'s "Why These Rules Exist" section is reduced to 3 lines or fewer while preserving the core insight about user intent when naming agents
- [ ] **AC-4**: The three patterns (Explicit Team Request, Explicit Consultation Directive, Anti-Fabrication Rule) and their trigger/required/prohibited blocks in `agent-team-compliance.md` are unchanged
- [ ] **AC-5**: All three files remain syntactically valid markdown with correct frontmatter format

## Technical Approach
Three independent fixes in three files. For gc-uuid-bridge.md, change the YAML key and remove the description field. For pytest-verbose.md, add a 3-line frontmatter block. For agent-team-compliance.md, compress the "Why" section while keeping the core insight sentence.
