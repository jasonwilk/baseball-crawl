## Description
After this story is complete, Phase 5 of the implement skill produces a single `feat(E-NNN): <title>` commit per epic closure that atomically includes the feat changes patched from the epic worktree, the archive `git mv` of `epics/E-NNN-slug/` to `.project/archive/E-NNN-slug/`, and the PM memory update moving the epic from Active to Archived. Worktree cleanup (`git worktree remove` + `git branch -D epic/E-NNN`) becomes its own first-class numbered step that runs after the closure commit.

## Context
Phase 5 currently has Step 2 flip epic status to `COMPLETED`, then Step 8 patches and commits. `.claude/hooks/epic-archive-check.sh` denies any `git commit` while a `COMPLETED` epic still lives under `/epics/` — so Step 8's commit is structurally blocked until Step 9's archive runs. Additionally, worktree cleanup is buried as a sub-step of Step 8's closure merge sequence, and was dropped when the outer step got improvised during E-226's closure. This story restructures Phase 5 to eliminate the cycle dependency and promote cleanup to a first-class step. Per Technical Notes TN-2, the restructure implements option A (archive-before-commit); option B (epic-aware hook parsing) was rejected.

## Technical Approach

The restructure touches only Phase 5's existing top-level Steps 8-12 and related cross-references. Steps 1-7a are left unchanged. Sub-step ordering inside the new Step 8 is specified in Technical Notes section TN-3.

The archive rename must run before `git add -A` so that staging captures both the rename and the applied patch in a single `git commit`. This is the root-cause fix for F2 (the hook's grep pattern `^\s*\`(COMPLETED|ABANDONED)\`\s*$` at `.claude/hooks/epic-archive-check.sh:34` no longer finds the epic file at `epics/*/epic.md` at commit time because the file has been renamed to `.project/archive/...`).

The approval-gate text currently at Step 8's sub-step 7 is preserved verbatim at its new position. Do not paraphrase the four exact-match words or the non-approval examples.

`.claude/rules/dispatch-pattern.md` is a rules file (frontmatter `paths: "**"`) loaded for every interaction — the archive-commit reference must remain consistent with the skill. Scan lines 8, 12, 40, 41, 42 for stale language; update anything that references a separate archive commit.

The Workflow Summary block at ~lines 568-595 of `.claude/skills/implement/SKILL.md` is a code-fenced ASCII diagram; rewrite it to match Technical Notes section TN-8's target shape.
