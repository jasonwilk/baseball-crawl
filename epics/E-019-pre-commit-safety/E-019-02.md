# E-019-02: Pre-Commit Hook Scaffold (Git + Claude Code)

## Epic
[E-019: Pre-Commit Safety Gates](./epic.md)

## Status
`DONE`

## Description
After this story is complete, both hook layers of the two-layer defense are installed and configured: a Git pre-commit hook at `.githooks/pre-commit` and a Claude Code PreToolUse hook at `.claude/hooks/pii-check.sh`. Both are wired to call `src/safety/pii_scanner.py`, but if the scanner does not yet exist they pass silently. A one-command install script at `scripts/install-hooks.sh` configures the Git hook. The Claude Code hook fires automatically from `.claude/settings.json`. A scoped rule protects the safety code from accidental weakening.

## Context
The architect's design at `/.project/research/E-006-precommit-design.md` specifies all implementation details for both hooks. Read that document in full before starting. Key decisions:

- **Git hook**: `.githooks/` directory with `git config core.hooksPath`, NOT the `pre-commit` Python framework (overkill) and NOT raw `.git/hooks/` (not version-controlled).
- **Claude Code hook**: A `PreToolUse` hook on the `Bash` matcher in `.claude/settings.json`. Intercepts only `git commit` commands -- all other Bash calls pass through with a single grep check.
- **Scanner absent**: At scaffold time the scanner does not exist yet (E-019-03). Both hooks must detect this and exit 0 silently (or with a brief warning for the Git hook).

This story has no dependencies and can execute in parallel with E-019-01. It shares no files with E-019-01.

## Acceptance Criteria

- [ ] **AC-2**: Given the Claude Code project is loaded, when Claude Code attempts a `Bash` tool call containing `git commit`, the PreToolUse hook fires (`.claude/hooks/pii-check.sh` runs). If the scanner does not exist, the tool call is allowed through silently
- [ ] **AC-3**: Given the scanner does not yet exist at `src/safety/pii_scanner.py`, both hooks pass without blocking commits or Bash tool calls (stub behavior until E-019-03 is done)
- [ ] **AC-4**: `./scripts/install-hooks.sh` completes in one command with no manual steps beyond running it; it sets `core.hooksPath` and verifies the hook is executable
- [ ] **AC-5**: `.claude/settings.json` contains the `hooks.PreToolUse` block for `pii-check.sh`, merged with existing settings (`agent`, `env`, `statusLine` are all preserved unchanged)
- [ ] **AC-6**: `.claude/rules/pii-safety.md` exists with `paths` frontmatter scoping it to `src/safety/**`, `.githooks/**`, and `.claude/hooks/pii-check.sh`, containing rules against weakening patterns or adding blanket exclusions
- [ ] **AC-7**: All three shell scripts (`.githooks/pre-commit`, `.claude/hooks/pii-check.sh`, `scripts/install-hooks.sh`) are executable (`chmod +x`)

## Technical Approach
Read `/.project/research/E-006-precommit-design.md` in full. It contains exact pseudocode and file contents. Summary of each deliverable:

### `.githooks/pre-commit`
Shell script. Gets staged files from `git diff --cached --name-only --diff-filter=ACM`. Passes them to the scanner via stdin (`echo "$STAGED_FILES" | python3 "$SCANNER" --stdin`). If the scanner does not exist, prints `[pii-hook] Scanner not installed yet. Skipping PII check.` and exits 0. Must be executable. See design doc for exact script content.

### `.claude/hooks/pii-check.sh`
Shell script. Reads JSON from stdin (`INPUT=$(cat)`). Extracts `tool_input.command` via `jq`. If the command does not match `^git\s+commit`, exits 0 immediately. If it matches, runs `python3 "$CLAUDE_PROJECT_DIR/src/safety/pii_scanner.py" --staged`. If the scanner exits non-zero, outputs a JSON denial object (`permissionDecision: "deny"`) and exits 0 (denial is signaled via JSON output, NOT via exit code). If the scanner does not exist, exits 0. Must be executable. Requires `jq` -- if `jq` is not available, the script should detect this and exit 0 (fail open).

### `scripts/install-hooks.sh`
Shell script. Runs `git config core.hooksPath "$REPO_ROOT/.githooks"` and `chmod +x "$REPO_ROOT/.githooks/pre-commit"`. Prints confirmation. One command to run.

### `.claude/settings.json` update
Add a `hooks` key with a `PreToolUse` array entry. The command is `"$CLAUDE_PROJECT_DIR"/.claude/hooks/pii-check.sh`, timeout 10 seconds, statusMessage `"Checking staged files for PII..."`. Merge with existing keys -- do NOT overwrite or reorder `agent`, `env`, or `statusLine`. Read the file first, then add the new key.

### `.claude/rules/pii-safety.md`
YAML frontmatter with `paths` scoping to `src/safety/**`, `.githooks/**`, `.claude/hooks/pii-check.sh`. Body contains advisory rules: never weaken regex patterns without approval, never add blanket exclusions, run tests before committing scanner changes, scanner must stay under 1 second for 20 files.

### `.claude/hooks/README.md` update
Add a section or row documenting `pii-check.sh` -- its purpose, when it fires, and what it does.

### Verification (manual, part of Definition of Done)
1. Run `./scripts/install-hooks.sh`
2. Run `git commit --allow-empty -m "test: verify hook fires"` -- must see the "Scanner not installed" warning and succeed
3. Confirm `.claude/settings.json` has the hooks block with all other keys intact

## Notes
- The design document at `/.project/research/E-006-precommit-design.md` is the spec. If anything in this story conflicts with the design doc, the design doc wins. Flag genuine contradictions to the PM rather than guessing.
- Do NOT create `src/safety/pii_scanner.py` in this story. That is E-019-03. The hooks should detect scanner absence and pass silently.
- The `pii-check.sh` hook exits 0 even when it issues a denial -- the denial is communicated via JSON output, not exit code. This follows the Claude Code hooks protocol.
- `jq` is required for `pii-check.sh`. If `jq` is not available, the script should detect its absence and exit 0 (fail open, do not block commits).
