## Epic

## Status

## Description
After this story is complete, Phase 0 (Team Formation) of the plan skill contains domain-based team composition suggestions that the main session presents to the user. The main session parses the user's request for domain signals, suggests a planning team (PM + domain experts), and spawns the team after user confirmation. Implementing-type domain experts (SE, DE, docs-writer) are spawned in consultation mode; primary-capacity agents (api-scout, baseball-coach, claude-architect) are spawned without consultation mode per `workflow-discipline.md`.

## Context
Today, Jason manually specifies which agents to include in every planning session. The domain almost always determines the team (schema work → PM + DE, dashboard work → PM + SE + coach, etc.), but there is no guidance for this in any skill. The implement skill has a Dispatch Team section and routing table; planning has nothing equivalent.

This story fills in Phase 0 of the plan skill created by E-140-01 with the team composition suggestion table (TN-3), user-override handling (per `agent-team-compliance.md`), and consultation mode spawning conventions.

## Acceptance Criteria
- [ ] **AC-1**: Phase 0 of the plan skill includes the domain signal → suggested team mapping table per TN-3.
- [ ] **AC-2**: Phase 0 describes how the main session parses the user's request for domain keywords and presents the suggested team composition to the user for confirmation or adjustment.
- [ ] **AC-3**: Phase 0 specifies that when the user explicitly names agents in their request, those names override the suggestion table. Multi-agent naming triggers Pattern 1 (Explicit Team Request -- use TeamCreate); single-agent naming triggers Pattern 2 (Explicit Consultation Directive -- spawn that agent). Cross-reference to `agent-team-compliance.md`.
- [ ] **AC-4**: Phase 0 specifies the consultation mode spawning convention: implementing-type agents (software-engineer, data-engineer, docs-writer) are spawned with the consultation mode phrase from `workflow-discipline.md` ("Consultation mode: do not create or modify implementation files or planning artifacts"). Primary-capacity agents (api-scout, baseball-coach, claude-architect) and PM are spawned WITHOUT consultation mode, per `workflow-discipline.md`'s "When to declare consultation mode" guidance.
- [ ] **AC-5**: Phase 0 specifies the PM spawn context, including: PM operates in discover/plan mode, writes DRAFT epic and stories, runs quality checklist, and does not set READY until Phase 5.

Edit the plan skill file created by E-140-01. Expand the Phase 0 section from a stub to full procedural detail. The team composition table and spawning conventions are defined in the epic's Technical Notes (TN-3, TN-2) -- translate those into skill-level procedural steps.
