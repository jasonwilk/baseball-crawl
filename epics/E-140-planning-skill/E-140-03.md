## Description
After this story is complete, Phase 3 (Spec Review) and the triage portion of Phase 4 (Refinement) of the plan skill contain full procedural detail for automatically chaining the codex spec review, parsing findings, routing triage to PM and domain experts, and managing the spec review → refine loop with a circuit breaker.

## Context
Today, after PM writes a DRAFT epic, Jason manually invokes "spec review E-NNN" as a separate interaction. The plan skill should auto-chain this step. The codex-spec-review skill already handles the mechanics of running the review and parsing output -- the plan skill invokes the underlying script directly (per TN-7) rather than nesting skill loading.

The spec review → triage → refine loop can repeat for complex epics. The circuit breaker (TN-8) prevents infinite loops. Triage follows the same model as dispatch review triage: each finding gets ACCEPT (fix it) or DISMISS (false positive / incorrect finding only).

## Acceptance Criteria
- [ ] **AC-1**: Phase 3 specifies that after PM completes Phase 2 (DRAFT epic with stories), the main session automatically runs the codex spec review script per TN-7 without requiring a separate user command.
- [ ] **AC-2**: Phase 3 handles all three spec review outcomes: (a) clean review (no findings) → proceed to Phase 5, (b) findings present → proceed to triage, (c) error/timeout → present to user and ask how to proceed.
- [ ] **AC-3**: Phase 3 triage specifies that PM and domain experts on the team assess each finding, with disposition options: ACCEPT (PM incorporates the fix) or DISMISS (finding is incorrect -- false positive, misunderstanding, or targets code outside epic scope).
- [ ] **AC-4**: Phase 4 specifies the loop-back condition: after PM incorporates accepted findings and runs the consistency sweep, the user decides whether to re-run spec review (loop to Phase 3) or proceed to Phase 5. The skill presents this choice explicitly.
- [ ] **AC-5**: The circuit breaker per TN-8 is implemented: maximum 3 spec review iterations, with escalation options presented to the user when the breaker fires.
- [ ] **AC-6**: Phase 3 specifies that the codex-spec-review script is run directly (`timeout 600 ./scripts/codex-spec-review.sh <epic-dir>`) rather than loading the codex-spec-review skill, per TN-7.

Reference the codex-spec-review skill for output format expectations. The script invocation, triage model, and circuit breaker are defined in the epic's Technical Notes (TN-7, TN-8).
