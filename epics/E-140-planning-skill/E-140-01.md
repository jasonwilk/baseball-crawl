## Description
After this story is complete, a new skill file exists at `.claude/skills/plan/SKILL.md` containing the full planning workflow: activation triggers, prerequisites, six phases (Team Formation → Discovery → Planning → Spec Review → Refinement → READY), workflow summary flowchart, edge cases, and anti-patterns. The main session can load this skill on planning triggers and follow its phases to produce a READY epic.

## Context
This is the foundational story for E-140. It creates the skeleton of the plan skill with all phases defined. Subsequent stories (E-140-02 through E-140-05) fill in specific subsections with deeper logic. This story establishes the overall structure, phase entry/exit conditions, and the workflow summary that ties everything together.

The skill follows the same structural conventions as the implement skill (`.claude/skills/implement/SKILL.md`) per TN-1.

## Acceptance Criteria
- [ ] **AC-1**: `.claude/skills/plan/SKILL.md` exists and follows the skill structure convention per TN-1 (Activation Triggers, Purpose, Prerequisites, numbered Phases with Steps, Workflow Summary, Edge Cases, Anti-Patterns sections).
- [ ] **AC-2**: Activation Triggers section lists all planning triggers and compound triggers per TN-4, including the "plan and dispatch" compound modifier.
- [ ] **AC-3**: Prerequisites section verifies: (a) the user's request describes a problem or feature to plan, (b) the main session checks `/epics/` for existing DRAFT epics and, if a potentially overlapping epic exists, asks the user whether this is a new epic or a continuation of the existing one before proceeding.
- [ ] **AC-4**: All six phases (0-5) are defined with entry conditions, steps, and exit conditions per TN-2. Phase stubs for team composition (Phase 0), spec review integration (Phase 3), consistency sweep (Phase 4), and implement handoff (Phase 5 compound path) reference the Technical Notes sections where subsequent stories will add detail.
- [ ] **AC-5**: Workflow Summary section contains an ASCII flowchart showing the full phase flow, including the spec review → refine loop and the compound trigger branch to implement skill.
- [ ] **AC-6**: Edge Cases section covers at minimum: user request too vague for an epic (capture as idea instead), codex not installed, spec review timeout, no findings on spec review, circuit breaker per TN-8.
- [ ] **AC-7**: Anti-Patterns section covers at minimum: do not skip expert consultation when the domain warrants it, do not auto-dispatch without user authorization (unless compound trigger), do not duplicate rule content (reference rules by path), do not nest skill loading (run the spec review script directly per TN-7).

## Technical Approach
The skill file is a new markdown document following the established skill conventions. Reference the implement skill for structure and the epic's Technical Notes for phase architecture, activation triggers, and boundaries. The spec review invocation, team composition suggestions, consistency sweep, and handoff mechanics are stubs in this story -- subsequent stories flesh them out.
