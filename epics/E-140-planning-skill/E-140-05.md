## Epic
[E-140: Planning Skill](epic.md)

## Description
After this story is complete, the plan skill's Phase 5 supports the "plan and dispatch" compound trigger by chaining into the implement skill after READY. The implement skill's activation triggers recognize the handoff. The unified team lifecycle is codified: agents from the planning team transition to dispatch roles without a team teardown.

## Context
The user feedback on unified team lifecycle (validated 2026-03-16) says: "Refinement and dispatch should use the same team. Bring agents online for consultation during refinement, then dispatch those same agents to implement. No team teardown between phases."

Today, planning teams are torn down and dispatch teams are created fresh. This story codifies the handoff so that when the user says "plan and dispatch," the planning team transitions into a dispatch team. PM stays, domain-expert-implementers transition from consultation to implementation mode, code-reviewer is spawned fresh, and non-implementer experts can be shut down.

The implement skill already has activation triggers for dispatch, but does not recognize a handoff from the plan skill. This story adds the handoff recognition.

## Acceptance Criteria
- [ ] **AC-1**: Phase 5 of the plan skill specifies that when the compound trigger was used ("plan and dispatch"), the skill signals dispatch authorization and chains into the implement skill per TN-6.
- [ ] **AC-2**: Phase 5 specifies that when the non-compound trigger was used ("plan an epic"), the skill stops at READY and presents the epic summary. No dispatch occurs.
- [ ] **AC-3**: The implement skill is updated in every section that assumes a fresh team (Purpose, Phase 1, Phase 2, and Workflow Summary) to recognize a handoff from the plan skill: when a planning team is already active, the implement skill reuses existing agents per TN-6 team transition mechanics rather than creating a fresh team. Each affected section includes a conditional path for the handoff case.
- [ ] **AC-4**: Team transition mechanics are documented per TN-6: PM stays, domain-expert-implementers transition from consultation mode to implementation mode, non-implementer experts are offered shutdown, code-reviewer is spawned fresh.
- [ ] **AC-5**: The dispatch authorization gate in `workflow-discipline.md` is preserved: the compound trigger phrase serves as the user's explicit dispatch authorization. The plan skill does not invent a new authorization mechanism.

## Technical Approach
Edit the plan skill (Phase 5 compound path) and the implement skill (all sections that assume a fresh team: Purpose, Phase 1, Phase 2, and Workflow Summary). The handoff mechanics are defined in TN-6. The key challenge is the team transition: the implement skill currently assumes it creates a fresh team in multiple places. This story adds a conditional handoff path in each affected section.
