# E-148-01: Restructure Plan Skill Review Phases

## Epic
[E-148: Review Cycle Reordering for Plan and Implement Skills](epic.md)

## Status
`TODO`

## Description
After this story is complete, the plan skill's review phases will run internal reviews (CR spec audit + holistic team review) first in Phase 3, followed by Codex as a final validation pass in Phase 4. The user will explicitly decide when to advance from internal to Codex tiers. Phase 5 (READY gate) will record a review scorecard table in the epic's History section and handle CR already being on the team during compound dispatch handoff. All existing edge cases (timeout, codex not installed, clean review, circuit breaker) will be preserved.

## Context
The plan skill currently runs Codex spec review first (Phase 3) and refinement (Phase 4) as a single review tier. E-147 showed that internal reviews (CR + holistic team) find more practical issues than Codex, but they run after Codex in the current workflow. This story restructures Phase 3-5 so internal reviews clean up mechanical issues before Codex runs its systematic rubric pass.

## Acceptance Criteria
- [ ] **AC-1**: Phase 3 is titled "Internal Review Cycle" and implements the two-sub-pass-per-iteration mechanism per TN-2: CR spec audit (with spec-review criteria in the assignment message, not the code-focused rubric) and holistic team review (domain experts review from their perspective, PM collects and triages combined findings).
- [ ] **AC-2**: Phase 3 has a 3-iteration circuit breaker per TN-4, including "advance to Codex" as an option when the circuit breaker fires.
- [ ] **AC-3**: After each Phase 3 iteration, the user is presented with options per TN-3: (a) another internal iteration, (b) advance to Codex, (c) proceed to READY. A clean iteration (no findings) still presents these options per TN-6.
- [ ] **AC-4**: Phase 3 refinement includes a post-incorporation consistency sweep as a required gate, mirroring the existing consistency sweep mechanism from the current Phase 4 Step 2.
- [ ] **AC-5**: Phase 4 is titled "Codex Validation Pass" and contains the existing Codex spec review logic (run script, handle timeout/error/clean/findings, triage, refine, consistency sweep) relocated from the current Phase 3-4.
- [ ] **AC-6**: Phase 4 has a 2-iteration circuit breaker per TN-4.
- [ ] **AC-7**: Phase 5 (READY Gate) includes PM recording a review scorecard table in the epic's History section per TN-5 format, and the epic summary (Step 2) covers both internal review and Codex tier outcomes.
- [ ] **AC-8**: Phase 5 compound dispatch handoff handles CR already being on the team (reuse CR if spawned during Phase 3; spawn fresh only if Phase 3 was skipped or CR was not spawned). The implement skill's Phase 2 handoff path is updated to reflect that CR may already be on the team from the plan skill's Phase 3 (not just from Phase 5 Step 3a).
- [ ] **AC-9**: All existing edge cases per TN-6 are preserved in the restructured phases (timeout, codex not installed, clean review with tier options, circuit breaker, async paste, prompt fallback).
- [ ] **AC-10**: Elements per TN-7 remain unchanged: Phase 0, Phase 1, Phase 2.
- [ ] **AC-11**: The Workflow Summary diagram and Edge Cases section are updated to reflect the restructured phases, including internal review-specific edge cases (e.g., no planning team experts beyond PM, CR spawn failure during Phase 3).

## Technical Approach
The current Phase 3 (Spec Review) and Phase 4 (Refinement) are replaced by three phases: Phase 3 (Internal Review Cycle), Phase 4 (Codex Validation Pass), and the existing Phase 5 (READY Gate, with scorecard and handoff updates). The internal review cycle introduces CR spec audit and holistic team review as new review mechanisms that do not exist in the current skill. The Codex logic from current Phase 3-4 is relocated to Phase 4 with a reduced circuit breaker (2 iterations instead of 3, since the spec should be cleaner by the time Codex runs). The consistency sweep gate from current Phase 4 Step 2 applies to both Phase 3 and Phase 4 refinement passes. The Phase 5 compound dispatch handoff must account for CR already being on the team from Phase 3. Reference the architect's analysis at `/.claude/agent-memory/claude-architect/review-cycle-reordering.md` for the approved direction and E-147 evidence.
