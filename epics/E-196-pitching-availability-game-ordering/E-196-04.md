# E-196-04: Dashboard pitching availability columns

## Epic
[E-196: Pitching Availability and Game Chronological Ordering](epic.md)

## Status

## Description
After this story is complete, the opponent detail page and opponent print view will display Rest and P(7d) columns in pitching tables, and the "Their Pitchers" card on the opponent detail page will show a workload sub-line under each pitcher's stats.

## Context
The dashboard's opponent views currently show season aggregate pitching stats with no workload context. This story adds the two new columns to the dashboard's pitching surfaces and the summary sub-line to the pitcher card, using the shared query function from E-196-03. Display formats follow Technical Notes TN-5.

## Acceptance Criteria
- [ ] **AC-1**: The opponent detail route (`src/api/routes/dashboard.py`) calls the workload query function from E-196-03 with `reference_date` of today and merges the data into the template context for pitching tables.
- [ ] **AC-2**: `opponent_detail.html` pitching table includes "Rest" and "P (7d)" columns per Technical Notes TN-5.
- [ ] **AC-3**: `opponent_detail.html` "Their Pitchers" card shows a sub-line under each pitcher's stats: `Last: Xd ago * {pitches}/{span}d` (or `No recent outings` when no appearances), per Technical Notes TN-5.
- [ ] **AC-4**: `opponent_print.html` pitching table includes "Last" and "P (7d)" columns per Technical Notes TN-5 (print format: `Mar 28` for Last, same workload format).
- [ ] **AC-5**: The opponent print route enriches pitching data with the same workload query as the detail page.
- [ ] **AC-6**: Tests verify template rendering with workload data present and absent.

## Technical Approach
The dashboard route already queries pitching stats for the opponent detail page. Add a call to the new workload function (E-196-03), merge the workload dict into the pitcher data by player_id, and pass the enriched data to templates. Template changes add two columns to the pitching `<table>` and a sub-line to the pitcher card. The print view uses formatted date instead of days-ago.

Read the current opponent detail/print routes and templates to understand the existing data flow before modifying.

## Notes
- "Today" should be used instead of "0d" when last_outing_days_ago is 0 (Rest column, dashboard only).
- The print view uses static formatted dates since it has no JS context.
- **Enrichment ordering**: Workload data must be merged into the `pitchers` list BEFORE `_get_top_pitchers()` is called, so that the top-pitchers card also has workload sub-lines.
- **"Today" + "/1d" consistency**: When Rest shows "Today" and P(7d) shows `/1d`, both are correct -- the pitcher threw today and the 7d window span is 1 day.
- **NULL pitch counts**: When appearances exist but all pitch counts are NULL, display `?/{span}d` in the P(7d) column (not `0/{span}d`, which misleads the coach into thinking the arm is fresh).
