# E-096-R-01: Research: Interactive Codex Review Conversation

## Epic
[E-096: Resolve All Review Findings During Stories](./epic.md)

## Status
`TODO`

## Objective
Determine whether Codex can be used conversationally during the dispatch review loop -- i.e., the main session poses a question about a specific finding and gets a response, rather than running a batch review. This would enable the main session to consult Codex when triaging ambiguous SHOULD FIX findings.

## Time Box
1 focused session (< 2 hours)

## Research Questions
1. Can `codex exec --ephemeral` (or similar) accept a scoped question with file context and return a useful answer interactively from the command line?
2. Is the response quality sufficient for nuanced review discussions (e.g., "Is this naming convention better than the current one? Here's the project's style guide...")?
3. What are the latency characteristics? Is it fast enough to be useful in a dispatch loop where the main session is triaging 3-5 findings?
4. What are the cost implications compared to the existing batch review approach?
5. Are there any Codex features or modes that enable true back-and-forth conversation (multi-turn), or is it strictly single-prompt?

## Deliverables
- A markdown document in this directory (`E-096-R-01-findings.md`) summarizing:
  - Whether interactive Codex review is feasible
  - The specific command/approach that works (if any)
  - Latency and quality observations
  - Recommendation: integrate, defer, or discard

## Approach
1. Review current Codex documentation and capabilities in `docs/admin/codex-guide.md` and `.codex/config.toml`
2. Test `codex exec --ephemeral -` with a scoped review question piped via stdin
3. Test with a realistic finding scenario: provide a code snippet, a project convention, and ask whether the finding should be fixed or dismissed
4. Assess whether multi-turn conversation is possible or if each question requires a fresh invocation
5. Document findings with concrete examples

- **Blocks**: None (the main stories do not depend on this research -- the resolution-first model works without Codex interactive review)
