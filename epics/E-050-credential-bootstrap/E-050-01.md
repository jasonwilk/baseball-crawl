## Description
After this story is complete, the operator can run `python scripts/check_credentials.py` to verify that GameChanger credentials in `.env` are present and valid. The script makes a single API call (`GET /me/user`) and reports the result with a clear, actionable message. Exit codes enable scripting: 0 = valid, 1 = expired/error, 2 = missing credentials.

## Context
Today, the only way to discover that credentials have expired is to run a crawl and hit a `CredentialExpiredError`. This wastes time and produces confusing error output. The health check provides a fast, explicit validation step that the bootstrap script (E-050-03) will call as its first gate.

## Acceptance Criteria
- [ ] **AC-1**: `scripts/check_credentials.py` exists and is executable via `python scripts/check_credentials.py`
- [ ] **AC-2**: When `.env` is missing or lacks `GAMECHANGER_AUTH_TOKEN`, `GAMECHANGER_DEVICE_ID`, or `GAMECHANGER_BASE_URL`, the script prints a message listing the missing keys and exits with code 2
- [ ] **AC-3**: When all required keys are present, the script makes a single `GET /me/user` request using `GameChangerClient` and reports the result
- [ ] **AC-4**: On a successful response (HTTP 200), the script prints "Credentials valid" along with the user's display name or email (from the `/me/user` response) and exits with code 0
- [ ] **AC-5**: On a 401 response, the script prints "Credentials expired -- refresh via proxy capture or scripts/refresh_credentials.py" and exits with code 1
- [ ] **AC-6**: On a 403 response, the script prints "Access denied -- credentials may be expired or revoked" and exits with code 1
- [ ] **AC-7**: On any other error (network timeout, 5xx, connection refused), the script prints the error details and exits with code 1
- [ ] **AC-8**: The script NEVER prints or logs credential values (`gc-token`, `gc-device-id`). Only key names appear in output.
- [ ] **AC-9**: The core validation logic is in a function `check_credentials() -> tuple[int, str]` (exit_code, message) that can be imported by `scripts/bootstrap.py` without running the CLI
- [ ] **AC-10**: Tests exist in `tests/test_check_credentials.py` covering: missing env vars (exit 2), valid credentials (exit 0), expired credentials 401 (exit 1), forbidden 403 (exit 1), and network error (exit 1). All tests mock the HTTP layer -- no real API calls.

## Technical Approach
1. Create `scripts/check_credentials.py` following the same structure as `scripts/refresh_credentials.py` (project root on `sys.path`, logging setup, `main()` entry point)
2. The `check_credentials()` function:
   - Use `dotenv_values()` to read `.env` (same pattern as `GameChangerClient._load_credentials()`)
   - Check for required keys before making any API call
   - Instantiate `GameChangerClient` and call `client.get("/me/user")`
   - Catch `ConfigurationError` (missing env vars), `CredentialExpiredError` (401), `ForbiddenError` (403), and general exceptions
   - Return `(exit_code, human_readable_message)`
3. The `GET /me/user` endpoint returns a JSON object with at minimum an `email` field -- use that for the success message
4. Reference: `docs/gamechanger-api.md` for `/me/user` response schema; `src/gamechanger/client.py` for error types
