# E-228-03: Pitcher prediction -- convert always-on to opt-in checkbox

## Epic
[E-228: Matchup Strategy Report](./epic.md)

## Status
`TODO`

## Description
After this story is complete, the existing predicted-starter section will be controlled by an "Include pitcher predictions" checkbox on the admin generation form (default ON, preserving today's behavior). The corresponding CLI flag `--no-pitcher-predictions` opts out. The change is operator-facing only -- no engine or rendering changes.

## Context
Per user scope decision (2026-04-27), the admin generation form gains TWO new checkboxes: "Include matchup section" (E-228-01) and "Include pitcher predictions" (this story). Today, the predicted-starter section ships always on every report (gated only by `FEATURE_PREDICTED_STARTER`). The checkbox lets operators turn it off for cleaner reports when the feature isn't relevant (e.g., generating a report for archive purposes, or when the LLM cost is undesirable for a one-off).

This is small and could fold into E-228-01, but separating preserves clean story boundaries and lets the architectural lift land first as a clean commit.
