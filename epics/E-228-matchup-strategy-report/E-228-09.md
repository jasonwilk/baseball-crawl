## Description
After this story is complete, `src/reports/renderer.py` and `src/api/templates/reports/scouting_report.html` will render the "Game Plan" section per UX's v1 layout (delivered 2026-04-27): a single conditional `<section id="game-plan">` block placed immediately after the executive summary strip, BEFORE the Predicted Starter section, containing six sub-sections in fixed order with a deterministic TL;DR line at the top, inline italic gray provenance citations, recent-form overlay with Unicode arrows, and inline "AVOID:" sub-bullets. The section is silently absent when `matchup is None` or `matchup.confidence == "suppress"`.

## Context
Per UX consultation (2026-04-27, relayed by team-lead) and the user's decision-B override (independent checkboxes; Predicted Starter and Game Plan can both render together), the Game Plan section is opt-in via the matchup checkbox AND coexists with Predicted Starter when both are checked. The visual treatment is field-language ("Game Plan", three syllables) — distinct from the data-prediction language of Predicted Starter ("Predicted Starter") to signal prescriptive intent.

Critical UX-derived constraints:
1. **TL;DR is DETERMINISTIC, not LLM-generated.** Three middot-joined directive slots produced by the engine from the same dataclass that drives sub-sections. If LLM Tier 2 fails, TL;DR still renders.
2. **Provenance is inline italic gray parenthetical** — NOT badges, NOT tooltips, NOT a separate column.
3. **Head-to-head is a top-of-section banner**, NOT a sidebar (sidebars don't print well at 9pt with ~2.5in column).
4. **AVOID is an indented sub-bullet**, "AVOID:" prefix in 7pt small caps bold, no color/icon/border (B&W print survives).
5. **Recent form overlay** is `Season X (PA) / Last 7G Y (PA) [arrow]`, Unicode arrow ↑↔↓ at ≥.100 OBP swing threshold; no sparkline, no delta-only.
6. **Empty head-to-head omits the banner entirely** — no empty placeholder.

### Behavior — TL;DR Line (Deterministic)

- [ ] **AC-8**: TL;DR line at the top of Game Plan section, BELOW the section header and ABOVE the head-to-head banner. If `enriched_matchup is None` (LLM failed or unavailable), TL;DR still renders.
- [ ] **AC-9**: TL;DR slot population (engine produces these strings; renderer composes):
  - **Slot 1**: LSB starter recommendation directive. Format: `"Pitch <Name>"` when `pitcher_recommendation` populated; OMIT slot 1 when None (TL;DR has 2 directives, not 3).

  - **Slot 3**: Top hitter to avoid walking from highest BB% threat. Format: `"Avoid walking <Name>"`. Populated when `threat_list` has at least one hitter with `bb_pct >= _AVOID_WALK_BB_PCT_THRESHOLD` (constant in engine, default 0.10).

## Technical Approach
- The Jinja2 conditional follows the existing `scouting_report.html` pattern. Add ONE new conditional `{% if matchup and matchup.confidence != "suppress" %}` block placed AFTER the executive summary and BEFORE Predicted Starter
- The `.exec-summary` class for head-to-head banner is the existing class — reuse, do not redefine
- The `.starter-narrative-style` class for the LLM narrative block is the existing predicted-starter narrative class — reuse for the matchup analysis narrative at the bottom of section 6
- For inline citations, render the LLM prose strings as `{{ section_text|safe }}` ONLY if the LLM output is post-validated for safety (E-228-07's hallucination guardrail filters); otherwise escape and accept the parenthetical pattern survives standard Jinja escape (parens are not escaped by default)
- For TL;DR composition, the engine produces three strings; the renderer joins with `' · '` (middot with surrounding spaces) into one HTML element
- Print CSS extends the existing print block
