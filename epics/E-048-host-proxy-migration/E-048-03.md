# E-048-03: Update report scripts

## Epic
[E-048: Migrate mitmproxy to Host-Based Standalone Proxy](epic.md)

## Status
`DONE`

## Description
After this story is complete, the report scripts (`scripts/proxy-report.sh` and `scripts/proxy-endpoints.sh`) will read from `proxy/data/` instead of `data/mitmproxy/`. The scripts remain in `scripts/` and are run from the project root.

## Context
The report scripts read addon output files to produce human-readable summaries. With addons now writing to `proxy/data/` (E-048-02), the scripts must read from the same location. They are designed to run from the project root inside the devcontainer, so relative paths like `proxy/data/header-report.json` work correctly.

## Acceptance Criteria
- [ ] **AC-1**: `scripts/proxy-report.sh` sets `REPORT_FILE="proxy/data/header-report.json"` and reads from that path.
- [ ] **AC-2**: `scripts/proxy-endpoints.sh` sets `LOG_FILE="proxy/data/endpoint-log.jsonl"` and reads from that path.
- [ ] **AC-3**: The comment at the top of each script references the correct new path.
- [ ] **AC-4**: The "no traffic captured" messages still reference `proxy/start.sh` (or a generic "start the proxy") instead of the old `./scripts/proxy.sh start`.

## Technical Approach
1. In `/workspaces/baseball-crawl/scripts/proxy-report.sh`: change `REPORT_FILE="data/mitmproxy/header-report.json"` to `"proxy/data/header-report.json"`, update comment.
2. In `/workspaces/baseball-crawl/scripts/proxy-endpoints.sh`: change `LOG_FILE="data/mitmproxy/endpoint-log.jsonl"` to `"proxy/data/endpoint-log.jsonl"`, update comment.
3. Update the "no file found" help messages to reference the new start command.

- **Blocks**: E-048-06 (docs reference report commands)
