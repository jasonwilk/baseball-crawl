# E-048-04: Remove mitmproxy from project docker-compose.yml

## Epic
[E-048: Migrate mitmproxy to Host-Based Standalone Proxy](epic.md)

## Status
`IN_PROGRESS`

## Description
After this story is complete, the project's `docker-compose.yml` will have no mitmproxy service, no `mitmproxy-certs` volume, and no `proxy` profile. The file will contain only the app, traefik, and cloudflared services. The Traefik dashboard port comment referencing mitmproxy will also be updated.

## Context
mitmproxy is moving to its own `proxy/docker-compose.yml` on the Mac host (E-048-01). The project compose file should no longer reference it. The mitmproxy service definition is at lines 55-86, the named volume at lines 100-103.

## Acceptance Criteria
- [ ] **AC-1**: The `mitmproxy` service block (lines 55-86) is removed from `docker-compose.yml`.
- [ ] **AC-2**: The `mitmproxy-certs` volume definition (lines 100-103) is removed from `docker-compose.yml`. If no other volumes remain, the `volumes:` top-level key is also removed.
- [ ] **AC-3**: The Traefik dashboard port comment (line 42: "Moved from 8080 to 8180 to free 8080 for mitmproxy.") is simplified to just describe the dashboard port without the mitmproxy migration context.
- [ ] **AC-4**: `docker compose config` (run from the devcontainer) validates the modified file without errors.
- [ ] **AC-5**: `docker compose up -d` starts only app, traefik, and cloudflared (no mitmproxy service listed in `docker compose ps`).

## Technical Approach
1. Open `/workspaces/baseball-crawl/docker-compose.yml`.
2. Remove the entire `mitmproxy:` service block (lines 55-86).
3. Remove the `mitmproxy-certs:` volume and its comment (lines 100-103). Remove the `volumes:` key if empty.
4. Update the Traefik port comment on line 42.
5. Verify with `docker compose config`.
