## Context
This is the highest-impact story in the epic -- roughly 40% of all failures live in this single test file. The failures span most root cause categories from TN-1. The connect flow tests (TestConnectForm, TestConnectConfirm, TestConnectPost, TestConnectSetsTeamsPublicId, TestPublicIdCollisionOnConnect) are the most complex because they need route path changes, mock additions for the auto-scout background task (per TN-4), AND assertion updates for changed POST `/connect` behavior (now calls `finalize_opponent_resolution()`, sets `resolved_team_id`, triggers `run_scouting_sync`). The badge system is now pipeline_status-based (6 states, not simple labels) -- see TN-1 category 1 for the complete mapping.
