## Description
After this story is complete, all cross-source reconciliation signals (pitcher and batter stats) will match at 99%+ accuracy. Five targeted fixes close the remaining gaps: boxscore pitch/strike supplement for high-confidence pitchers, IBB inclusion in BB outcomes, games-table scores for game_runs, and boxscore BF sum for game_pa_count. The two game-level signals are converted to data-availability checks (tautological). Boxscore-supplemented signals include enriched `correction_detail` recording the original plays-derived values for transparency.

## Context
E-198 built the reconciliation engine and BF-boundary correction algorithm. An 8-round investigation identified root causes for every remaining gap. All fixes are in `src/reconciliation/engine.py` -- no parser changes, no schema changes, no new migrations. The research doc at `/.project/research/E-198-accuracy-gaps.md` contains the complete analysis with SQL verification against the live database.

## Acceptance Criteria
- [ ] **AC-1**: Given a pitcher where BF, SO, and BB all match the boxscore (high-confidence gate), when reconciliation runs, then `pitcher_pitches` and `pitcher_total_strikes` discrepancy rows have `plays_value` set to the boxscore `pitches` and `total_strikes` values (supplemented), and `correction_detail` records the original plays-derived values per TN-1.
- [ ] **AC-2**: Given a pitcher where BF, SO, or BB do NOT all match the boxscore, when reconciliation runs, then `pitcher_pitches` and `pitcher_total_strikes` discrepancy rows use the raw plays-derived values (no supplement applied).
- [ ] **AC-3**: Given a play with `outcome = 'Intentional Walk'`, when reconciliation runs, then the play is counted in both `pitcher_bb` and `batter_bb` signals (the `_BB_OUTCOMES` constant includes `"Intentional Walk"`).
- [ ] **AC-4**: Given a game where `games.home_score` and `games.away_score` are non-NULL, when reconciliation runs, then `game_runs` uses `games.home_score`/`away_score` for both boxscore and plays sides per TN-1 Fix 4, and the signal is MATCH. When `games.home_score` or `games.away_score` is NULL, the `game_runs` signal is skipped (no discrepancy row emitted).
- [ ] **AC-5**: Given a game with pitching data and an abandoned final PA (boxscore BF > plays PA count), when reconciliation runs, then `game_pa_count` uses the boxscore BF sum for both sides per TN-1 Fix 5, and the signal is MATCH.
- [ ] **AC-6**: All existing reconciliation tests pass without modification (unless a test explicitly asserted old behavior that the fixes change, in which case the test is updated to reflect the new correct behavior).
- [ ] **AC-7**: New tests cover all 5 fixes including positive cases, negative cases, and edge cases per TN-4 test matrix.

## Technical Approach
The research doc at `/.project/research/E-198-accuracy-gaps.md` has the complete root cause analysis and fix designs (see "Final Fix List for Implementation" near line 810 and Findings 12, 15, 19, 25-27). All changes are in `engine.py` -- the functions `_check_pitcher_signals()`, `_check_game_level_signals()`, the module-level `_BB_OUTCOMES` constant, and the `reconcile_game()` query. The `reconcile_game()` games-table query needs `home_score` and `away_score` added to its SELECT clause, with those values passed through to `_check_game_level_signals()`.
