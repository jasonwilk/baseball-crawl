# E-194-04: Dashboard Spray Chart Enrichments + Global Title Removal

## Epic
[E-194: Spray Chart Card Redesign](epic.md)

## Status
`TODO`

## Description
After this story is complete, the player spray chart image endpoint will no longer bake a player name title into the PNG (since all views now show the name in HTML context). The player profile and opponent detail dashboard views will display L/C/R zone counts and contact type counts alongside their spray chart images, using the shared classification helpers.

## Context
The `/dashboard/charts/spray/player/{id}.png` endpoint currently passes a `"First Last"` title to `render_spray_chart()`, which bakes it into the PNG. After the card redesign (E-194-02, E-194-03), all print/report views show the name in HTML card headers, making the baked-in title redundant. The dashboard views (player_profile.html, opponent_detail.html) also show the name in HTML context (page heading, batting table row). Removing the title globally is safe per TN-11.

The dashboard views currently show spray chart images without direction or contact type breakdowns. Adding these enrichments (using the shared helpers from E-194-02) gives coaches the same scouting data across all views. The dashboard views use Tailwind CSS (not inline styles) and extend `base.html` -- structurally different from the print templates. See TN-12 for the specific layout of each view.

## Technical Approach
Three areas of change:

1. **Chart endpoint** (dashboard.py, `player_spray_chart_image`): Change `render_spray_chart(events, title)` to `render_spray_chart(events, title=None)`. One-line change.

2. **Player profile route + template**: The `player_profile` route currently fetches only `spray_bip_count` (a scalar via `get_player_spray_bip_count()`), NOT raw spray events. To classify events into zones and contact types, the route must call `get_player_spray_events(player_id, season_id)` -- this returns `(events, title)` where events have `x`, `y`, `play_result`, `play_type`. Ignore the title return value (`events, _ = ...`). Classify using shared helpers, pass zone/contact stats to the template. Add direction/contact display below the chart image in `player_profile.html`.

3. **Opponent detail route + template**: The route currently has `team_spray_bip_count` (a scalar) but no raw team-aggregate spray events. Add a new `get_team_spray_events(team_id, season_id)` function to `src/api/db.py` that returns all offensive spray events for a team (equivalent to aggregating all per-player events). Classify and count using shared helpers. Pass to template. Add direction/contact display below the team spray chart card in `opponent_detail.html`.
