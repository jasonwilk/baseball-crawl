# E-089-02: Update Code-Reviewer Review Procedure with Test Scope Verification

## Epic
[E-089: Test Scope Discovery Rule](./epic.md)

## Status

## Description
After this story is complete, the code-reviewer's review procedure will include a step verifying that the implementer's test run covered all test files importing from modified source modules. The current Step 1 guidance ("target changed modules first") will be updated to require broader test discovery verification, closing the blind spot that allowed the E-085 regression.

## Context
The code-reviewer's Step 1 currently says: "For large test suites, target changed modules first: `pytest tests/test_<module>.py`." This guidance actively encourages narrow test scoping. During E-085, the reviewer ran the same narrow scope as the implementer, missing the broken test in `test_check_credentials.py`. The reviewer needs a verification step: after the implementer reports completion, confirm that all test files importing from modified modules were actually tested.

## Acceptance Criteria
- [ ] **AC-1**: The code-reviewer's Review Procedure Step 1 in `/.claude/agents/code-reviewer.md` is updated to include test scope verification. The updated step must: (a) require the reviewer to identify all source modules changed by the implementer (from the Files Changed list); (b) require the reviewer to discover test files importing from those modules; (c) require the reviewer to verify those test files pass (either by running them or confirming the implementer already ran them).
- [ ] **AC-2**: The "target changed modules first" guidance is either removed or revised so it no longer encourages the narrow scoping pattern that caused the E-085 blind spot. The replacement guidance must reference the broader discovery approach.
- [ ] **AC-3**: A test scope gap (test files importing from a modified module that were not run) is classified as a MUST FIX finding, documented inline in the updated Step 1 text (not under Priority 3). Rationale: test scope gaps are procedural (what the reviewer ran), not judgments about test adequacy.
- [ ] **AC-4**: The existing review rubric priorities (1-7) and structured findings format are preserved. Changes are additive or revise existing text -- they do not restructure the rubric.

## Technical Approach
The problem is that the code-reviewer's procedure encourages narrow test scoping and has no verification step for cross-file test coverage. The solution is updating `/.claude/agents/code-reviewer.md` Step 1 to add test scope verification and revising the "target changed modules first" guidance. The implementing agent (claude-architect) should read the full code-reviewer agent definition before making changes, and should ensure the new verification step is actionable (a reviewer following it step-by-step would discover `test_check_credentials.py` when reviewing changes to `credentials.py`).

Key constraint: the updated Step 1 must remain practical for reviews. It should not require the reviewer to run the entire test suite -- just the discovered files plus the story-scoped files.
