# E-119-01: Document PII Scanner Invocation in Context Layer

## Epic
[E-119: Fix PII Scanner Discoverability](epic.md)

## Status
`TODO`

## Description
After this story is complete, agents working in any part of the codebase can discover the PII scanner's location and manual invocation syntax from the always-loaded context layer (CLAUDE.md or a broadly-scoped rule file). The current gap -- where the scanner path is only documented in `pii-safety.md` (scoped to `src/safety/**` and `.githooks/**`) -- is closed.

## Context
During E-116 dispatch, the main session tried `python scripts/pii_scan.py` because no context-layer file told it where the scanner actually lives. The pre-commit hooks work fine, but agents performing manual scans have no way to discover the correct path without grepping. This story adds that discoverability.

## Acceptance Criteria
- [ ] **AC-1**: The PII scanner path (`src/safety/pii_scanner.py`) and at least one invocation example (e.g., `python3 src/safety/pii_scanner.py --staged`) appear in a context-layer file that is loaded for all agent work (either CLAUDE.md or a rule file with `paths: "**"`)
- [ ] **AC-2**: No changes to files in `src/`, `scripts/`, `tests/`, `migrations/`, or hook files (`.githooks/`, `.claude/hooks/`)
- [ ] **AC-3**: The added documentation is concise (no more than 3-4 lines) and placed in a contextually appropriate location (e.g., near the existing `[pii-scan]` confirmation note in CLAUDE.md, or in the Security Rules section)

## Technical Approach
The context layer needs a small addition so agents can find the scanner. Two options exist -- adding to CLAUDE.md (most discoverable, always loaded) or broadening the `pii-safety.md` scope. The implementer should choose the approach that best fits the existing structure. The key constraint is that the result must be visible during all agent work, not only when editing safety-related files.

## Notes
The scanner's own docstring documents three invocation modes:
```
python3 src/safety/pii_scanner.py --staged       # scan git staged files
python3 src/safety/pii_scanner.py --stdin         # read file paths from stdin
python3 src/safety/pii_scanner.py file1 file2     # scan specific files
```
