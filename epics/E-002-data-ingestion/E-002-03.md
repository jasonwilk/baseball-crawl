# E-002-03: Crawl game stats (box score) and write raw JSON

## Epic
[E-002: GameChanger Data Ingestion Pipeline](epic.md)

## Status
`BLOCKED`

## Description
After this story is complete, the system will have a `GameStatsCrawler` that reads game IDs from previously crawled schedule files, fetches box-score-level stats for each completed game, and writes raw JSON to `data/raw/{season}/teams/{team_id}/games/{game_id}.json`. This is the most data-dense crawl in the pipeline and is the primary source for hitting, pitching, and fielding stats.

## Context
Game stats are the core product for coaching intelligence. Box scores contain at minimum: per-batter performance (AB, H, HR, RBI, BB, K), per-pitcher performance (IP, H, ER, BB, K), and game-level context (score by inning, home/away). These are the numbers coaches use to evaluate performance and prepare for opponents.

This crawler reads schedule files written by E-002-02 rather than calling the schedule API directly, which keeps crawlers decoupled.

## Acceptance Criteria
- [ ] **AC-1**: Given schedule files for all configured teams, when `GameStatsCrawler().crawl_all()`, then for each completed game in each schedule, a file is written to `data/raw/{season}/teams/{team_id}/games/{game_id}.json`.
- [ ] **AC-2**: Only games with `game_status: "completed"` (as returned by the game-summaries endpoint, not the schedule endpoint) are crawled. The crawler reads game-summaries data (from `data/raw/{season}/teams/{team_id}/game_summaries.json` written by E-002-02 or fetched inline) to determine which games are completed. Scheduled or cancelled games are skipped with an INFO log.
- [ ] **AC-3**: Idempotency: if a game file already exists (any age -- completed game stats do not change), it is not re-fetched.
- [ ] **AC-4**: The written JSON is the unmodified API response for that game.
- [ ] **AC-5**: Given an API error for a specific game, the error is logged with the game ID and crawl continues.
- [ ] **AC-6**: At the end of a run, the crawler logs a summary: total games found, games already cached, games fetched, games errored.

## Technical Approach
`src/gamechanger/crawlers/game_stats.py`. The crawler reads game-summaries JSON from disk to identify completed games (via `game_status: "completed"`), then fetches per-game box score data for each completed game. Completed game files are treated as immutable (AC-3) -- no freshness check needed since past games don't change.

**BLOCKED**: The exact endpoint for per-game box score data (per-player batting/pitching lines) has not been discovered yet. The game-summaries endpoint returns game-level scores but NOT player-level stats. E-002-R-01 must confirm the endpoint before this story can proceed. Possible endpoints to investigate: `/games/{game_id}/stats`, `/teams/{team_id}/games/{game_id}`, or embedded in a different response shape.

## Notes
The game file is keyed on `{team_id}/{game_id}` because the same game may appear in two teams' schedules (if both teams are in `config/teams.yaml`). Write the file under the team that "owns" the schedule entry to avoid coordination complexity. If both teams are ours, writing it twice (same content) is acceptable.

**Dedup implication for E-002-07a**: When loading game data from raw files, the same game may appear under multiple team directories. The loader must handle this (upsert by game_id ensures idempotency).
