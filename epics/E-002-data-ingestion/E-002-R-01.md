## Objective
Determine whether the GameChanger API exposes endpoints for (1) per-game box score data (per-player batting and pitching lines for a specific game) and (2) per-player or per-team season aggregate statistics. The current API spec (`docs/gamechanger-api.md`) documents five endpoints, none of which return player-level stats. Without these endpoints, stories E-002-03 and E-002-04 cannot be implemented as written.

## Time Box
One focused session. If endpoints cannot be discovered from browser traffic capture within that session, document the gap and recommend alternative approaches.

## Research Questions
1. Does GameChanger expose a per-game stats endpoint (e.g., `/games/{game_id}/stats`, `/teams/{team_id}/games/{game_id}/box-score`, or similar) that returns per-player batting and pitching lines for a completed game?
2. Does GameChanger expose a season stats endpoint (e.g., `/teams/{team_id}/stats`, `/players/{player_id}/stats`, or similar) that returns aggregate batting/pitching stats for a season?
3. If neither exists as a dedicated endpoint, does the `game-summaries` response contain player-level stats that were not captured in the initial schema documentation? (The current schema shows only game-level scores and metadata.)
4. Are there any undiscovered endpoints visible in browser network traffic when viewing a game's box score page or a team's stats page on `web.gc.com`?

## Deliverables
- Updated `docs/gamechanger-api.md` with any newly discovered endpoints (full request/response documentation per existing format)
- If endpoints are found: unblock E-002-03 and E-002-04 with specific endpoint details
- If endpoints are NOT found: a written recommendation for alternative approaches (e.g., web scraping, deriving stats from play-by-play data, or accepting the data gap)

## Approach
1. The user navigates to a completed game's box score page on `web.gc.com` and captures the network traffic (browser DevTools > Network tab > Copy as cURL for all XHR requests)
2. The user navigates to a team's season stats page and captures network traffic similarly
3. api-scout analyzes the captured curl commands to identify new endpoints
4. Any discovered endpoints are documented in `docs/gamechanger-api.md` following the existing format
5. If the user cannot provide captures (e.g., the pages do not exist or do not make API calls), document that finding

## Dependencies
- **Blocked by**: None (requires user to provide curl captures from browser traffic)
- **Blocks**: E-002-03 (game stats crawler), E-002-04 (player season stats crawler), E-002-07a (game loader), E-002-07b (player stats loader)
