## Description
After this story is complete, the system will have a `PlayerStatsCrawler` that reads player IDs from previously crawled roster files and fetches season-aggregate stats for each player, writing raw JSON to `data/raw/{season}/teams/{team_id}/stats.json` (or per-player files if the API returns individual records). This provides the season-level view: batting average, OBP, SLG, strikeout rate, home/away splits, left/right splits.

## Context
Game-by-game stats (E-002-03) tell you what happened in a specific game. Season stats tell you who a player *is*. Coaches need both: game stats for recent form, season stats for scouting and lineup decisions. This crawler targets the season aggregate endpoint.

The home/away splits and left/right pitcher splits mentioned in the project brief are most likely found here (or derived from game-level data). The API spec (E-001-03) will clarify whether these splits are provided directly or must be computed from game-level data.

## Acceptance Criteria
- [ ] **AC-1**: Given roster files for all configured teams, when `PlayerStatsCrawler().crawl_all()`, then season stats are fetched for each player and written to the appropriate path under `data/raw/{season}/teams/{team_id}/`.
- [ ] **AC-2**: Idempotency: stats files less than 24 hours old are skipped. Season stats change as games are played, so they are not treated as immutable like game files.
- [ ] **AC-3**: Given a player that appears in a roster but has no stats (e.g., has not played yet), the API response (even if empty) is written and no error is raised.
- [ ] **AC-4**: Given an API error for a specific player or team stats endpoint, the error is logged and crawl continues.
- [ ] **AC-5**: The endpoint is team-level: `GET /teams/{team_id}/season-stats` (confirmed by E-002-R-01). One request per team. The response contains all players for that team. Write the full response to `data/raw/{season}/teams/{team_id}/stats.json`.

## Technical Approach
`src/gamechanger/crawlers/player_stats.py`. Reads roster JSON from disk. Depending on API shape (team-level vs. player-level endpoint), either makes one request per team or iterates player IDs.

**RESOLVED (2026-03-04)**: E-002-R-01 confirmed `GET /teams/{team_id}/season-stats` returns full per-player season batting, pitching, and fielding aggregates at the team level (one request per team). See `docs/gamechanger-api.md` Schema: season-stats section for the full response shape.
