# E-002-06: Load raw roster JSON into database

## Epic
[E-002: GameChanger Data Ingestion Pipeline](epic.md)

## Status
`TODO`

## Description
After this story is complete, the system will have a `RosterLoader` that reads `roster.json` files from `data/raw/` and upserts player and team records into the SQLite database at `./data/app.db`. Running the loader is idempotent: loading the same data twice does not create duplicates.

## Context
The load layer is responsible for transforming raw API JSON into the normalized relational schema defined in E-003. The loader must never modify the raw JSON files -- it reads from them and writes to the database. If the schema changes, the raw files can be reloaded without re-crawling.

Players move between organizations over time (a player might be on Lincoln Freshman one year and Varsity the next). The schema must track `player_id` as a stable cross-organization identifier.

## Acceptance Criteria
- [ ] **AC-1**: Given a valid `roster.json` file, when `RosterLoader().load_file(path)`, then all players in the file are upserted into the `players` table and all team-membership records are upserted into the `team_rosters` table.
- [ ] **AC-2**: Running `load_file()` twice with the same file produces the same database state as running it once (idempotent via upsert, not insert).
- [ ] **AC-3**: A player that exists in multiple teams' rosters (across different seasons) results in one `players` record and multiple `team_rosters` records.
- [ ] **AC-4**: Given a JSON field that does not map to any schema column, the unknown field is ignored (logged at DEBUG level) and the load continues.
- [ ] **AC-5**: Given a roster JSON with a missing required field (e.g., no `player_id`), the record is skipped, the error is logged with the offending raw data, and the load continues.

## Technical Approach
`src/gamechanger/loaders/roster.py`. The loader maps raw JSON fields to schema fields using an explicit field map (not dynamic reflection). Use Python's `sqlite3` module to connect directly to `./data/app.db`. The schema is managed by `migrations/apply_migrations.py`.

```python
class RosterLoader:
    def __init__(self, db: DatabaseConnection): ...
    def load_file(self, path: Path) -> LoadResult: ...
    def _map_player(self, raw: dict) -> Player: ...  # raises on missing required fields
```

`LoadResult` is a dataclass with: `loaded: int`, `skipped: int`, `errors: int`.
