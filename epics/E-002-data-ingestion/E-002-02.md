# E-002-02: Crawl game schedule and write raw JSON

## Epic
[E-002: GameChanger Data Ingestion Pipeline](epic.md)

## Status
`TODO`

## Description
After this story is complete, the system will have a `ScheduleCrawler` class that fetches the full game schedule for each configured team from the GameChanger API and writes raw JSON to `data/raw/{season}/teams/{team_id}/schedule.json`. The schedule is the index that downstream crawlers use to know which games to pull stats for.

## Context
The schedule is the spine of the ingestion pipeline. Without knowing which games were played (and who the opponents were), we cannot crawl game stats or opponent data. This story must produce a file that contains at minimum: game IDs, dates, home/away status, and opponent team IDs.

## Acceptance Criteria
- [ ] **AC-1**: Given valid credentials and a configured team list, when `ScheduleCrawler().crawl_all()`, then `data/raw/{season}/teams/{team_id}/schedule.json` is written for each team.
- [ ] **AC-2**: The written schedule JSON contains at minimum one record per game with: `game_id`, `date`, `home_team_id`, `away_team_id`, `status` (scheduled/completed/cancelled). (Exact field names from API; these are illustrative.)
- [ ] **AC-3**: Idempotency rule: schedule files less than 1 hour old are skipped; schedule files 1+ hours old are re-fetched. (Shorter window than roster because schedules change -- games get added, postponed, cancelled.)
- [ ] **AC-4**: Given an API error, the error is logged and the crawl continues to the next team.
- [ ] **AC-5**: The `freshness_hours` threshold is configurable, defaulting to 1.

## Technical Approach
Same pattern as `RosterCrawler`. Create `src/gamechanger/crawlers/schedule.py` with a `ScheduleCrawler` class. The exact API endpoint comes from E-001-03. Placeholder: `GET /v2/teams/{team_id}/schedule?season={season}`.
