## Description
After this story is complete, the system will have a top-level orchestration script (`scripts/crawl.py`) that runs all crawlers in the correct order, and a manifest system that records what was crawled, when, and how many records were fetched. Running `python scripts/crawl.py` will be the single command a user needs to refresh all data.

## Context
Individual crawlers are useful for development and debugging, but coaches and operators need a single command that "just works." The orchestration script is that command. It runs crawlers in dependency order, handles errors gracefully, and produces a manifest that makes it easy to answer "when was this data last updated?"

The manifest also serves as an audit log for debugging data quality issues.

## Acceptance Criteria
- [ ] **AC-1**: Running `python scripts/crawl.py` executes all crawlers (roster, schedule, game stats, player stats, opponent) in the correct dependency order.
- [ ] **AC-2**: Running `python scripts/crawl.py --dry-run` prints which crawlers would run and which files would be written, without making any API calls or writing any files.
- [ ] **AC-3**: Running `python scripts/crawl.py --crawler roster` runs only the roster crawler (supports named subsets: `roster`, `schedule`, `game-stats`, `player-stats`, `opponent`).
- [ ] **AC-4**: After a successful run, `data/raw/{season}/manifest.json` is written (or updated) with: crawl timestamp, which crawlers ran, count of files written per crawler, count of files skipped (already fresh), count of errors per crawler.
- [ ] **AC-5**: Given that any single crawler raises an unhandled exception, the orchestrator logs the error and continues to the next crawler rather than aborting the full run.
- [ ] **AC-6**: The script exits with code 0 if all crawlers completed (even with some per-record errors), and code 1 if any crawler raised an unhandled exception.

## Technical Approach
`scripts/crawl.py` uses `argparse` for CLI flags. Import and instantiate each crawler class. Run them in order. Collect `CrawlResult` objects from each and write the manifest.

Manifest structure:
```json
{
  "crawled_at": "2025-04-15T18:30:00Z",
  "season": "2025",
  "crawlers": {
    "roster": {"files_written": 4, "files_skipped": 0, "errors": 0},
    "schedule": {"files_written": 4, "files_skipped": 0, "errors": 0},
    "game_stats": {"files_written": 87, "files_skipped": 12, "errors": 1},
    "player_stats": {"files_written": 4, "files_skipped": 0, "errors": 0},
    "opponent": {"files_written": 23, "files_skipped": 5, "errors": 2}
  }
}
```
