# E-002-01: Crawl team roster and write raw JSON

## Epic
[E-002: GameChanger Data Ingestion Pipeline](epic.md)

## Status
`TODO`

## Description
After this story is complete, the system will have a `RosterCrawler` class that fetches the current roster for one or more configured teams from the GameChanger API and writes the raw API response as JSON to `data/raw/{season}/teams/{team_id}/roster.json`. The crawl is idempotent: if a fresh file already exists, it skips the fetch.

## Context
The roster is the foundational player record. Every other stat (batting, pitching, fielding) references a player by their GameChanger player ID. Getting rosters right first means downstream stories can assume `player_id` is the stable identifier across all stats.

This story fetches roster data for "owned" teams only (Lincoln Standing Bear teams). Opponent rosters, to the extent they are accessible, are handled in E-002-05.

## Acceptance Criteria
- [ ] **AC-1**: Given a `config/teams.yaml` with one or more manually-populated team entries (team IDs obtained from `/me/teams` endpoint or user knowledge) and valid credentials in `.env`, when `crawler = RosterCrawler(); crawler.crawl_all()`, then a `roster.json` file is written to `data/raw/{season}/teams/{team_id}/roster.json` for each team.
- [ ] **AC-2**: Given a `roster.json` that already exists and is less than 24 hours old, when `crawl_all()` is called, then the existing file is not re-fetched and a log message at INFO level notes it was skipped.
- [ ] **AC-3**: Given a `roster.json` that is more than 24 hours old, when `crawl_all()` is called, then it is re-fetched and overwritten.
- [ ] **AC-4**: The written `roster.json` is the unmodified API response -- no fields are removed or transformed.
- [ ] **AC-5**: Given an API error during crawl, when it occurs, then the error is logged with the team ID and the crawl continues to the next team (partial failure does not abort the run).
- [ ] **AC-6**: The `freshness_hours` threshold is configurable, defaulting to 24.

## Technical Approach
Create `src/gamechanger/crawlers/roster.py`. The crawler uses `GameChangerClient` for all HTTP. Example structure:

```python
class RosterCrawler:
    def __init__(self, client: GameChangerClient, config: CrawlConfig, freshness_hours: int = 24): ...
    def crawl_team(self, team_id: str, season: str) -> Path: ...
    def crawl_all(self) -> list[Path]: ...
    def _is_fresh(self, path: Path, freshness_hours: int) -> bool: ...
```

`CrawlConfig` is a dataclass loaded from `config/teams.yaml`. Use `pathlib.Path` for all file operations. Write JSON with `json.dumps(data, indent=2)` for human-readable raw files.

The API endpoint for roster is confirmed: `GET /teams/{team_id}/players` (see `docs/gamechanger-api.md`). Accept header: `application/vnd.gc.com.player:list+json; version=0.1.0`. This endpoint works for both owned teams and opponent teams.
