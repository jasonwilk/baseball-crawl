# E-002-07a: Load raw game JSON into database

## Epic
[E-002: GameChanger Data Ingestion Pipeline](epic.md)

## Status
`BLOCKED`

## Description
After this story is complete, the system will have a `GameLoader` that reads game JSON files from `data/raw/` and upserts game records and per-player batting/pitching lines into the appropriate database tables. Coaches will be able to query game results and individual player game performances from SQL.

## Context
This loader handles two related record types from the same raw files: game-level data (one row per game in `games` table) and per-player game lines (rows in `player_game_batting` and `player_game_pitching` tables). These come from the per-game box score files written by E-002-03.

Home/away context is available from the game-summaries data (`home_away` field). The raw box score data shape depends on what E-002-R-01 discovers.

## Acceptance Criteria
- [ ] **AC-1**: Given `games/{game_id}.json` files, when `GameLoader().load_all()`, then each completed game is upserted into the `games` table with: game ID, date, home team, away team, final score, season_id. (Exact column names per the delivered `migrations/001_initial_schema.sql` from E-003-01.)
- [ ] **AC-2**: Given the same game files, player batting lines are upserted into `player_game_batting` and pitching lines into `player_game_pitching`: one row per (player_id, game_id). Running twice is idempotent.
- [ ] **AC-3**: The loader returns a `LoadResult` (loaded, skipped, errors counts).
- [ ] **AC-4**: Given a game record where a player ID does not exist in the `players` table (orphaned reference), the game stat row is still written with the player ID, and a WARNING is logged. (Referential integrity is soft -- do not reject records because a player was not in a roster file.)
- [ ] **AC-5**: Given the same game appearing under multiple team directories (see E-002-03 Notes on dedup), the loader upserts by game_id so duplicate raw files produce the same database state.

## Technical Approach
`src/gamechanger/loaders/game_loader.py` -- `GameLoader` class. Follows the same pattern as `RosterLoader` (E-002-06). Extract field mapping to a separate `_map_*` method for testability.

The implementing agent must read the delivered `migrations/001_initial_schema.sql` to get exact column names and types. Do not rely on table/column names assumed in this story file.
