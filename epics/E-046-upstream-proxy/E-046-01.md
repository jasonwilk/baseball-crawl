## Description
After this story is complete, `create_session()` will accept an optional `proxy_url` parameter and pass it to `httpx.Client(proxy=...)`. A helper function `get_proxy_config()` will read `PROXY_ENABLED` and `PROXY_URL` from environment variables and return the proxy URL string (or `None`). When `PROXY_ENABLED=true` but `PROXY_URL` is empty, a WARNING is logged and `None` is returned. The proxy URL is never logged.

## Context
All Python HTTP traffic flows through `create_session()` in `src/http/session.py`. Adding proxy support here covers all crawlers and the GameChanger client automatically. The env var reading is separated into a helper so callers can inspect proxy config without creating a session, and so the session factory remains testable without environment manipulation.

## Acceptance Criteria
- [ ] **AC-1**: `create_session(proxy_url="http://proxy:7777")` returns an `httpx.Client` configured with `proxy="http://proxy:7777"`.
- [ ] **AC-2**: `create_session()` (no proxy_url) returns an `httpx.Client` with no proxy configured (current behavior preserved).
- [ ] **AC-3**: A `get_proxy_config()` function in `src/http/session.py` reads `PROXY_ENABLED` and `PROXY_URL` from `os.environ`. Returns the URL string when enabled and set; returns `None` when disabled or unset.
- [ ] **AC-4**: When `PROXY_ENABLED=true` and `PROXY_URL` is empty or unset, `get_proxy_config()` logs a WARNING (message includes "PROXY_ENABLED is true but PROXY_URL is not set") and returns `None`.
- [ ] **AC-5**: The proxy URL value never appears in any log message from `src/http/session.py`. Test this: call `get_proxy_config()` with a known URL, capture logs, assert the URL string is absent.
- [ ] **AC-6**: `.env.example` has a new "Upstream Proxy" section with `PROXY_ENABLED` and `PROXY_URL` entries (commented out), with a brief usage comment explaining both Python crawlers and mitmproxy respect these vars.
- [ ] **AC-7**: All existing tests in `tests/test_http_session.py` continue to pass.
- [ ] **AC-8**: New tests cover AC-1 through AC-5 (proxy kwarg passed, no-proxy default, env var reading, graceful degradation, log safety).

## Technical Approach
1. Add `get_proxy_config() -> str | None` to `src/http/session.py`. It reads `os.environ.get("PROXY_ENABLED", "").lower()` and `os.environ.get("PROXY_URL", "")`. Logic: if enabled != "true", return None. If enabled == "true" and URL is empty, log WARNING, return None. Otherwise return the URL.
2. Add `proxy_url: str | None = None` parameter to `create_session()`. Pass it as `proxy=proxy_url` to `httpx.Client()`.
3. Add the "Upstream Proxy" section to `.env.example` after the WebAuthn section.
4. Write tests using `monkeypatch.setenv()` for env var tests and direct parameter passing for session tests. Use `caplog` to verify log safety and WARNING messages.
