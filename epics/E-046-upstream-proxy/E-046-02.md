## Status

## Description
After this story is complete, the host-based mitmproxy stack (`proxy/`) will start in upstream proxy mode when `PROXY_ENABLED=true` and `PROXY_URL` is set. When proxy is disabled (or `PROXY_URL` is empty despite `PROXY_ENABLED=true`), mitmproxy runs in its current regular intercepting mode. The `proxy/status.sh` script will display whether upstream mode is active.

## Context
E-048 migrated mitmproxy from a Docker Compose profile in the project stack to a standalone `proxy/` directory on the Mac host. The proxy has its own `proxy/docker-compose.yml` with a static `mitmweb` entrypoint and command block. Upstream mode is activated by adding `--mode upstream:URL` to the command. A wrapper entrypoint script replaces the static entrypoint and checks env vars at container startup to build the appropriate command line. This avoids Docker Compose variable interpolation in the `command:` block, which would inline the credential-bearing `PROXY_URL` into `docker compose config` output.

## Acceptance Criteria
- [ ] **AC-1**: When `PROXY_ENABLED=true` and `PROXY_URL=http://proxy:7777` are set, `cd proxy && ./start.sh` starts mitmproxy with `--mode upstream:http://proxy:7777` in its process args. Verify by `docker inspect` on the running container.
- [ ] **AC-2**: When `PROXY_ENABLED` is unset, empty, or `false`, mitmproxy starts without `--mode` (current behavior -- regular intercepting proxy).
- [ ] **AC-3**: When `PROXY_ENABLED=true` but `PROXY_URL` is empty or unset, mitmproxy starts without `--mode` (graceful degradation, same as AC-2). The entrypoint logs a warning to stderr.
- [ ] **AC-4**: The `PROXY_URL` value does not appear in `cd proxy && docker compose config` output. The wrapper entrypoint receives `PROXY_URL` as a runtime env var, not interpolated into the compose `command:` block.
- [ ] **AC-5**: `proxy/status.sh` displays whether upstream proxy mode is active (e.g., "Upstream proxy: enabled" or "Upstream proxy: disabled"). Use `docker inspect` to check for `--mode upstream` in the running container's process args rather than reading `.env` files.
- [ ] **AC-6**: `PROXY_ENABLED` and `PROXY_URL` are passed as environment variables to the mitmproxy service in `proxy/docker-compose.yml`.
- [ ] **AC-7**: `proxy/docker-compose.yml` reads both `../.env` (project root) and `proxy/.env` via `env_file`, so the operator only needs to set `PROXY_ENABLED`/`PROXY_URL` in the root `.env`. `proxy/.env` listed second for override precedence.

## Technical Approach
1. Create `proxy/proxy-entrypoint.sh` -- a bash wrapper that:
   - Checks `PROXY_ENABLED` and `PROXY_URL`
   - If both are set and valid, execs `mitmweb --web-host 0.0.0.0 --scripts /app/proxy/addons/loader.py --set web_password=${MITMWEB_PASSWORD:-} --mode upstream:${PROXY_URL}`
   - If `PROXY_ENABLED=true` but `PROXY_URL` is empty, logs a warning to stderr and execs mitmweb without `--mode`
   - Otherwise, execs mitmweb without `--mode` (current behavior)
2. Update `proxy/docker-compose.yml`:
   - Change `entrypoint` from `["mitmweb"]` to `["/app/proxy/proxy-entrypoint.sh"]`
   - Remove the `command:` block (args move into the entrypoint script)
   - Add `PROXY_ENABLED` and `PROXY_URL` to the `environment` list (as `${PROXY_ENABLED:-}` and `${PROXY_URL:-}` to handle unset gracefully)
   - Add `../.env` to the `env_file` list (before the existing `proxy/.env` entry so proxy/.env can override)
3. Update `proxy/status.sh` to check the running container for `--mode upstream` via `docker inspect` and display upstream proxy status.

## Notes
- The entrypoint wrapper approach is preferred over Docker Compose variable interpolation in `command` because Compose would inline the URL (containing credentials) into the resolved config, violating credential safety.
- No automated tests for this story -- it is Docker infrastructure verified by manual inspection of container config. AC verification uses `docker inspect` on the running container.
- The `proxy/` stack volume-mounts `..:/app`, so the entrypoint script at `proxy/proxy-entrypoint.sh` is accessible inside the container at `/app/proxy/proxy-entrypoint.sh`.
