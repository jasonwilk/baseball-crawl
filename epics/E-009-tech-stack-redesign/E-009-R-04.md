## Research Question
This spike produces the infrastructure head-to-head comparison that E-009-01 (Technology
Decision Record) needs to make a final call. R-01, R-02, and R-03 evaluate specific layers
(database, API, dashboard). This spike evaluates the deployment model as a whole:

- **Option A -- Native Cloudflare**: Workers + D1 + Pages, deployed and managed entirely
  within Cloudflare's platform.
- **Option B -- Docker + Cloudflare Access**: Docker Compose on a Linux VPS, fronted by
  Cloudflare Tunnel and Zero Trust Access (the n8n-wilk-io pattern).

The output is a structured comparison and a recommendation -- not an implementation plan.

## Context

### What Each Option Looks Like End-to-End

#### Option A (Native Cloudflare)
```
Local dev:   wrangler dev (simulates Workers + D1 via Miniflare)
Deploy:      wrangler deploy -> Cloudflare Workers + D1 + Pages
Auth:        Cloudflare Access (on top of Workers)
Database:    Cloudflare D1 (SQLite syntax, managed, edge-distributed)
API:         Cloudflare Workers (TypeScript, or Python Workers if viable)
Dashboard:   Cloudflare Pages (static or Workers-backed SSR)
Cost:        Cloudflare free tier (Workers, Pages, D1 are all free at this scale)
```

#### Option B (Docker + Cloudflare Access -- n8n-wilk-io Pattern)
```
Local dev:   docker compose up (identical to production)
Deploy:      docker compose up on a Linux VPS
Access:      Cloudflare Tunnel (cloudflared) + Zero Trust Access
Database:    SQLite in Docker volume (or Postgres if justified)
API:         FastAPI (Python)
Dashboard:   FastAPI + Jinja2 (server-rendered HTML)
Proxy:       Traefik (internal reverse proxy, routes by Host header)
Cost:        ~$4-10/month for a small VPS (Hetzner CX11 or DigitalOcean Droplet)
```

### Why the User Provided n8n-wilk-io as a Reference
The user operates a production deployment using the Option B pattern. The n8n-wilk-io
deployment has proven in production:
- Docker Compose works identically locally and in production (env vars only differ)
- Cloudflare Tunnel eliminates exposed ports and SSL cert management
- Cloudflare Zero Trust Access handles authentication without application-layer login code
- A VPS at ~$4/month handles the workload easily
- No platform vendor lock-in for compute -- any Linux server with Docker works

This is not a theoretical architecture. It is a working production system the user can
explain, debug, and extend. This reduces Option B's implementation risk significantly.

### Non-Candidates (Explicitly Out of Scope)
The user constraint is: Cloudflare (Option A) OR Docker + Cloudflare Access (Option B).
Do NOT evaluate Fly.io, Render, Railway, Heroku, or AWS. They are out of scope.

The note must answer:

1. **Local development experience**:
   - Option A: How reliable is `wrangler dev` for D1 + Workers + Pages in 2026? Are there
     known cases where local behavior diverges from production? What is the deploy cycle
     time for a dashboard change (edit -> see result in browser)?
   - Option B: `docker compose up` gives an identical-to-production environment. Are there
     any known Docker Compose limitations for this stack? What is the edit-reload cycle
     for a Jinja2 template change (does FastAPI support hot reload in dev mode)?

2. **Language consistency**:
   - Option A (TypeScript Workers): The crawlers are Python; the API would be TypeScript.
     What is the practical cost of this language split for a small team? Is there a viable
     Python Workers runtime for Cloudflare as of 2026?
   - Option B (Python FastAPI): Python end-to-end. No language context switch. What are
     the tradeoffs of running a persistent Python HTTP server vs. a serverless edge function?

3. **Cloudflare Tunnel + Zero Trust Access setup (Option B)**:
   - What is the one-time setup cost for Cloudflare Tunnel and Zero Trust Access?
   - How are service tokens configured for crawler-to-API authentication?
   - How are human access policies configured (WARP for coaching staff, public bypass for
     specific pages)?
   - What does the `cloudflared` Docker Compose service configuration look like?
   - Reference the n8n-wilk-io pattern directly where applicable.

4. **Production reliability and operational burden**:
   - Option A: What is the operational burden of a Cloudflare Workers + D1 + Pages
     deployment? What can go wrong that requires operator attention?
   - Option B: What is the operational burden of a VPS + Docker Compose deployment?
     OS updates, Docker updates, disk space monitoring, VPS provider reliability. Is
     a $4/month Hetzner CX11 adequate? What is the recommended VPS spec?

5. **Dev/prod parity**:
   - Option A: Are there known divergences between `wrangler dev` behavior and production
     Cloudflare Workers behavior? D1 local emulation fidelity? Pages local dev fidelity?
   - Option B: Is the `docker-compose.yml` identical between local and production, with
     environment variables as the only difference? Confirm this is achievable with the
     chosen stack (FastAPI + SQLite + Traefik + cloudflared).

6. **Cost comparison**:
   - Option A: Total cost at this scale (Workers, D1, Pages -- all free tier?). What
     triggers paid tier?
   - Option B: VPS cost (~$4-10/month) + any Cloudflare Zero Trust cost (what does CF
     Zero Trust cost for 5-10 users?). Is the free tier of CF Zero Trust adequate?

7. **Risk factors**:
   - Option A: Cloudflare platform risk (pricing changes, runtime deprecation, D1 limits).
     Vendor lock-in depth.
   - Option B: VPS management overhead. Cloudflare Tunnel uptime dependency. What happens
     if the VPS goes down?

8. **Final recommendation**: A clear recommendation -- Option A or Option B -- with a
   concise paragraph explaining the reasoning. This recommendation feeds directly into
   E-009-01 (Technology Decision Record) and is the PM's primary input for that gate story.

## Notes

The PM's prior, updated after learning about the n8n-wilk-io reference:

**Option B is the stronger prior**. The user already operates this pattern in production.
The implementation risk is low. Dev/prod parity is guaranteed by the Docker Compose model.
Cloudflare Tunnel eliminates the network security complexity that a VPS deployment would
otherwise require. Python end-to-end is a significant advantage for a team whose crawlers
are already Python.

Option A remains viable if: (1) `wrangler dev` provides a reliable enough local dev loop,
AND (2) a Python Workers runtime is production-ready in 2026 (eliminating the language
split). If either condition fails, Option B is the clear choice.

The Cloudflare Zero Trust Access cost for 5-10 users is the one unknown that could tip
the balance. If CF Zero Trust has a free tier that covers this use case, Option A's
"zero cost" advantage disappears (both options use Cloudflare products for auth/access).
Verify this in question 6.
