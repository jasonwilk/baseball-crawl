# E-009-02: Docker Compose Environment -- Database and API

## Epic
[E-009: Tech Stack Redesign -- Portable, Agent-Browsable, Cloudflare-Integrated](epic.md)

## Status
`DONE`

## Description
After this story is complete, the data-engineer will have created a working Docker Compose
environment that runs FastAPI with SQLite as the database and Traefik as the internal
reverse proxy. Running `docker compose up` from the project root starts the full backend
stack without errors. The SQLite database initializes with the project schema on first run.
This is the foundation that every subsequent E-009 story builds on.

## Context
The technology decision (E-009-01) selected Option B: Docker + Cloudflare Access (the
n8n-wilk-io pattern). This story implements the database and API layer of that stack.

The dashboard layer is a separate story (E-009-03), but both services share the same
`docker-compose.yml` file. Coordinate on service names and port assignments before writing
the dashboard service definition. The division of labor is:
- **This story (E-009-02)**: `docker-compose.yml` skeleton, FastAPI service, SQLite setup,
  Traefik service, health checks, migration bootstrap, `.env.example`.
- **E-009-03**: Adds the Jinja2 template layer and dashboard route to the same FastAPI app.

Key architectural decisions from the decision record:
- **SQLite placement**: Host-mounted file (`./data/app.db`) rather than a named Docker
  volume. Keeps the database visible on the filesystem for inspection and migration dry-runs.
- **Migration tooling**: Numbered SQL scripts (`migrations/001_schema.sql`, etc.) with a
  lightweight `apply_migrations.py` bootstrap script. No Alembic.
- **Single monolithic app**: API JSON endpoints and dashboard HTML are both served by the
  same FastAPI process. No microservices.
- **FastAPI + SQLite async pattern**: Use `run_in_threadpool` (or `aiosqlite`) for database
  calls to avoid blocking the async event loop. Document the chosen approach clearly.
- **Traefik**: Internal reverse proxy. Routes by Host header. No exposed ports to the
  internet (that is Cloudflare Tunnel's job in production; locally, ports are exposed only
  for dev convenience).

## Technical Approach

### `docker-compose.yml` Structure

```yaml
services:
  app:
    build: .
    env_file: .env
    volumes:
      - ./data:/app/data      # Host-mounted data dir (SQLite lives here)
      - ./migrations:/app/migrations
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 10s
      timeout: 5s
      retries: 3
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.app.rule=Host(`baseball.localhost`)"
    depends_on:
      traefik:
        condition: service_started

  traefik:
    image: traefik:v3
    command:
      - "--api.insecure=true"        # Dev only
      - "--providers.docker=true"
    ports:
      - "8000:80"    # App traffic
      - "8080:8080"  # Traefik dashboard (dev only)
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
```

### FastAPI App Structure

The app entrypoint is `src/api/main.py`. The `apply_migrations.py` script runs before
uvicorn starts (in the Dockerfile CMD or via a startup event, not as a separate service).

```
src/
  api/
    main.py           # FastAPI app factory, health endpoint
    routes/
      health.py       # GET /health
    db.py             # Database connection helper (get_db dependency)
  gamechanger/        # Existing crawlers (unchanged)
  safety/             # Existing PII module (unchanged)
```

### SQLite + FastAPI Async Pattern

Use `asyncio.get_event_loop().run_in_executor(None, ...)` or the Starlette
`run_in_threadpool` utility for all database calls. This prevents synchronous SQLite
operations from blocking FastAPI's async event loop.

```python
from starlette.concurrency import run_in_threadpool

async def get_health():
    result = await run_in_threadpool(db.check_connection)
    return {"status": "ok", "db": "connected" if result else "error"}
```

### apply_migrations.py

```python
# migrations/apply_migrations.py
# Applies all numbered SQL files in migrations/ that have not yet been applied.
# Tracks applied migrations in a _migrations table in the database itself.
# Idempotent: safe to run multiple times.
```

### `.env.example` Required Variables

```bash
DATABASE_PATH=./data/app.db   # Path to SQLite file (relative to app working dir)
APP_ENV=development            # development | production
LOG_LEVEL=INFO
# Add Cloudflare Tunnel token here (E-009-04 sets this up):
# CLOUDFLARE_TUNNEL_TOKEN=
```
