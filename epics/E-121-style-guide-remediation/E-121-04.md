## Description
After this story is complete, the code-reviewer agent definition will include a proportionality qualifier for convention-violation severity, and both the software-engineer and data-engineer agent definitions will note the consultation-mode exception to the story-reference work authorization requirement.

## Context
The triage audit found that the CR agent def classifies ALL convention violations as MUST FIX without proportionality, and that SE/DE agent defs don't mention the consultation-mode exception defined in workflow-discipline.md. The proportionality issue causes review friction when minor cosmetic violations are classified at the same severity as functional bugs. The missing consultation-mode exception causes agents to refuse advisory consultations because they lack a story reference.

Findings addressed: #6 (CR proportionality), #13 (SE/DE consultation-mode exception).

## Acceptance Criteria
- [ ] **AC-1**: `.claude/agents/code-reviewer.md` Priority 6 section includes a proportionality qualifier. Convention violations remain MUST FIX by default. A convention violation MAY be downgraded to SHOULD FIX only when ALL THREE of: (a) the violation has no functional impact (runtime behavior, correctness, security, or test reliability is unaffected), (b) the violation is in code that follows an established pattern already present in the same file or module (the implementer matched existing style, not invented something new), and (c) the violation is NOT in the categories: security rules, credential handling, SQL scope, or test coverage (those stay MUST FIX always). The qualifier must not weaken the rule for substantive violations.
- [ ] **AC-2**: `.claude/agents/code-reviewer.md` the existing "MUST FIX classification guardrail" text (line 117 area) is consistent with the new proportionality qualifier -- not contradictory.
- [ ] **AC-3**: `.claude/agents/software-engineer.md` Work Authorization section includes a note that consultation-mode spawns (per `/.claude/rules/workflow-discipline.md`) do not require a story reference. The note should reference the exact rule file path.
- [ ] **AC-4**: `.claude/agents/data-engineer.md` Work Authorization section includes the same consultation-mode exception note as AC-3.
- [ ] **AC-5**: The consultation-mode exception notes in SE and DE agent defs are consistent with each other and with the authoritative definition in `/.claude/rules/workflow-discipline.md`.

## Technical Approach
Three files are modified: CR agent def (add proportionality qualifier to Priority 6 and reconcile with guardrail text), SE agent def (add consultation-mode exception to Work Authorization), DE agent def (same). The implementer should read workflow-discipline.md for the authoritative consultation-mode definition, then apply consistent exception notes to both SE and DE agent defs.

## Notes
The proportionality qualifier (finding #6) is the most nuanced change in this story. The wording must thread the needle: permit SHOULD FIX for truly cosmetic violations while preventing agents from using proportionality as an escape hatch for substantive issues.

CR consultation confirmed the three-part test in AC-1 and specifically recommended the "established pattern in the same file or module" clause to prevent the qualifier from excusing sloppy new code. CR also noted that the error handling fallback (line 213: "If you cannot find a documented rule, it is SHOULD FIX") is fine as-is -- the problem was fabricated rules creating phantom MUST FIX findings, not the fallback logic.
