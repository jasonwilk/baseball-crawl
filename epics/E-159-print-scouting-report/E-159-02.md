# E-159-02: Add Print Link to Existing Scouting Report

## Epic
[E-159: Print-Optimized Scouting Report View](epic.md)

## Status
`TODO`

## Description
After this story is complete, the existing opponent scouting report page (`/dashboard/opponents/{id}`) will include a visible "Print / Save as PDF" link near the top of the page that navigates to the print-optimized view, forwarding the current team_id, year, and season_id context.

## Context
E-159-01 creates the print route and template. This story adds the entry point from the existing interactive scouting report so coaches can easily find and use the print view.

## Acceptance Criteria
- [ ] **AC-1**: The opponent detail page (`opponent_detail.html`) includes a "Print / Save as PDF" link visible above the stats sections (after the header div, before the first content card), within the full-stats `{% else %}` block.
- [ ] **AC-2**: The link points to `/dashboard/opponents/{opponent_team_id}/print?team_id={active_team_id}&season_id={season_id}`, mapping the template's `active_team_id` variable to the `team_id` query parameter. When `year` is non-None, `&year={year}` is appended; when `year` is None, the `year` param is omitted entirely (no `&year=None` in the URL).
- [ ] **AC-3**: The link is only rendered in the full-stats state (not in unlinked or linked_unscouted states — no point printing an empty report).
- [ ] **AC-4**: All existing tests pass. At least one test verifies the print link appears in the response for full_stats and is absent for unlinked/linked_unscouted states.

## Technical Approach
Add a small link element to `src/api/templates/dashboard/opponent_detail.html` inside the `{% else %}` block (full stats state), positioned after the header div and before the first content section. Build the URL using the same template variables already available in the template context (`opponent_team_id`, `active_team_id`, `year`, `season_id`).
