## Epic
[E-128: Credential Workflow Redesign](epic.md)

## Status

## Description
After this story is complete, `docs/admin/bootstrap-guide.md` and `docs/admin/credential-refresh.md` reflect the new credential workflow introduced by E-128. The primary web path is `bb creds setup web` (email+password), the device ID is auto-generated, and curl/proxy capture are documented as fallback paths. These docs were identified as stale during the E-128 refinement pass (docs-writer consultation, 2026-03-18).

## Context
E-128 changes the primary web credential path from "capture curl from browser DevTools" to "provide email+password, run `bb creds setup web`." Two admin docs still describe the old flow as primary:
- `docs/admin/bootstrap-guide.md`: Quick Start describes curl/proxy as the two web paths; device ID listed as "browser capture (one-time)"
- `docs/admin/credential-refresh.md`: "What Each Credential Is" table says device ID is "Browser capture (one-time)"; recovery paths don't mention `bb creds setup web`

## Acceptance Criteria
- [ ] **AC-1**: `docs/admin/bootstrap-guide.md` Quick Start section lists `bb creds setup web` as the primary web credential path. Curl capture and proxy capture are listed as fallback alternatives, not primary paths.
- [ ] **AC-2**: `docs/admin/bootstrap-guide.md` credential lifecycle table shows `DEVICE_ID_WEB` as "Auto-generated by `bb creds setup web`" (not "browser capture").
- [ ] **AC-3**: `docs/admin/credential-refresh.md` "What Each Credential Is" table shows `DEVICE_ID_WEB` as auto-generated and `REFRESH_TOKEN_WEB` as optional when email+password are present.
- [ ] **AC-4**: `docs/admin/credential-refresh.md` recovery paths include `bb creds setup web` as the recommended recovery when email+password are in `.env` and refresh fails completely.
- [ ] **AC-5**: No stale references to the old flow as the primary web path remain in either document.

## Technical Approach
Targeted edits to two existing docs. Read each doc, identify sections that describe the old web credential flow, update to reflect E-128's new primary path. The docs are well-structured -- the changes are surgical, not rewrites.

Key files to study: `docs/admin/bootstrap-guide.md`, `docs/admin/credential-refresh.md`.
