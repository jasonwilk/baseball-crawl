# E-204-01: Schema + Loader Forward-Fill

## Epic
[E-204: Starter vs. Relief Appearance Tracking](epic.md)

## Status
`TODO`

## Description
After this story is complete, migration 015 adds the `appearance_order INTEGER` column to `player_game_pitching`, and the game loader populates it for all new loads. Every pitcher in a game gets a 1-based appearance order (1 = starter, 2+ = relievers in order of appearance). The scouting loader and report generator inherit this behavior automatically via their delegation to `GameLoader.load_file()`.

## Context
The boxscore JSON `stats` array is already ordered by appearance (per TN-1), but the loader currently discards the index. This story captures the order by adding `enumerate()` to the existing loop and including the new column in the INSERT/UPSERT. The scouting loader delegates to GameLoader (per TN-2), so it gets this change for free.

## Acceptance Criteria
- [ ] **AC-1**: Migration 015 adds `appearance_order INTEGER` (nullable) to `player_game_pitching`. The app starts successfully after migration.
- [ ] **AC-2**: Given a boxscore JSON with 3 pitchers for a team, when the game loader processes it, then `player_game_pitching` rows have `appearance_order` values 1, 2, 3 matching the order in the JSON `stats` array.
- [ ] **AC-3**: Given an existing `player_game_pitching` row, when the same boxscore is re-loaded (upsert path), then `appearance_order` is updated via the `ON CONFLICT DO UPDATE` clause.
- [ ] **AC-4**: The `_PlayerPitching` dataclass includes `appearance_order: int | None` and it is populated from the enumeration index in `_load_pitching_group()`.

## Technical Approach
The boxscore JSON `stats` array is ordered by appearance (see TN-1 for the confirmed structure). The game loader's `_load_pitching_group()` iterates this array at line 823 -- wrapping it with `enumerate(..., start=1)` captures the index. The `_PlayerPitching` dataclass needs a new field, and `_upsert_pitching()` needs the column in both INSERT and ON CONFLICT DO UPDATE. Migration 015 is a single `ALTER TABLE` statement.
