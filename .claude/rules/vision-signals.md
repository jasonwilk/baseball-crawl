# Vision Signal Capture

## What Is a Vision Signal?

A statement about what this project will become, new capabilities it should have, user scenarios it should support, or strategic direction -- anything not yet captured in `docs/VISION.md`. Examples:

- "Eventually we should let assistant coaches build their own reports"
- "I want pitch sequence data feeding into a pre-game scouting sheet"
- "This should work for Legion teams too, not just high school"

## When You Notice One

Append it to `docs/vision-signals.md` under the `## Signals` heading using the established format:

```
- **YYYY-MM-DD**: Brief description of the signal. *(optional source context)*
```

## Guidelines

- **Err on the side of capturing.** Signals can be discarded during curation; lost signals cannot be recovered.
- **Do not stop the flow.** Append the signal and continue your current work.
- **Do not modify `docs/VISION.md` directly.** That file is updated only during deliberate curation sessions with the user ("curate the vision").
- **Keep entries brief.** One or two sentences capturing the essence of what was said.
