# Code Review Findings: E-100 + E-114 + E-115

Date: 2026-03-16

## Summary
The team-ID migration and the E-114 admin/template fixes look largely intact, but the ETL layer still does not honor the full E-100 schema. The biggest problems are in the loaders: boxscore ingestion still drops multiple now-supported per-game fields, `games.game_stream_id` is never persisted, and season/scouting season-stat upserts only populate a legacy subset of the columns the epic says should be loaded.

## Priority 1: Bugs and Regressions
- `src/gamechanger/loaders/game_loader.py` still behaves as if the old per-game schema were in place, so it silently discards boxscore data the current DDL explicitly supports. The mapping constants mark `R`, `TB`, `HBP`, `CS`, `E`, `WP`, `#P`, `TS`, and `BF` as “not in schema”, and the insert dataclasses/upserts omit `batting_order`, `positions_played`, `is_primary`, `decision`, `r`, `tb`, `hbp`, `shf`, `cs`, `e`, `wp`, `pitches`, `total_strikes`, and `bf`. That means current loads lose valid boxscore data instead of populating the E-100 tables. See [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L69), [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L133), [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L620), [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L695), [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L842), [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L886), [migrations/001_initial_schema.sql](/workspaces/baseball-crawl/migrations/001_initial_schema.sql#L150), and [epics/E-100-team-model-overhaul/epic.md](/workspaces/baseball-crawl/epics/E-100-team-model-overhaul/epic.md#L190).
- `games.game_stream_id` exists specifically to preserve the GC game-stream ID, but `GameLoader._upsert_game()` never writes it even though `GameSummaryEntry` already carries `game_stream_id`. That leaves every loaded game without the identifier E-100 added for public game-detail access. See [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L124), [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L341), [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L784), [migrations/001_initial_schema.sql](/workspaces/baseball-crawl/migrations/001_initial_schema.sql#L126), and [epics/E-100-team-model-overhaul/epic.md](/workspaces/baseball-crawl/epics/E-100-team-model-overhaul/epic.md#L156).
- The season loaders only persist a small legacy stat subset, not the endpoint-bounded season coverage E-100 defined. `SeasonStatsLoader` upserts only `gp/ab/h/2B/3B/hr/rbi/bb/so/sb` for batting and `gp_pitcher/ip_outs/h/er/bb/so/hr/pitches/total_strikes` for pitching; `ScoutingLoader` aggregates an even smaller subset. All of the other E-100 season columns (`pa`, `singles`, `r`, `hbp`, `shb`, `shf`, `gidp`, `roe`, `fc`, `ci`, `pik`, `tb`, `xbh`, `games_tracked` for API loads, plus the expanded pitching counts) remain unpopulated even when the API provides them. See [src/gamechanger/loaders/season_stats_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/season_stats_loader.py#L239), [src/gamechanger/loaders/season_stats_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/season_stats_loader.py#L300), [src/gamechanger/loaders/scouting_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/scouting_loader.py#L340), [src/gamechanger/loaders/scouting_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/scouting_loader.py#L388), and [epics/E-100-team-model-overhaul/epic.md](/workspaces/baseball-crawl/epics/E-100-team-model-overhaul/epic.md#L212).

## Priority 4: Schema Drift
- The current loader SQL no longer matches the active DDL in a meaningful way: the schema defines broader per-game and per-season columns, but the upsert statements still target the older reduced column sets. This is most visible in `GameLoader`, `SeasonStatsLoader`, and `ScoutingLoader`. See [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L842), [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L886), [src/gamechanger/loaders/season_stats_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/season_stats_loader.py#L259), [src/gamechanger/loaders/season_stats_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/season_stats_loader.py#L323), [src/gamechanger/loaders/scouting_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/scouting_loader.py#L366), [src/gamechanger/loaders/scouting_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/scouting_loader.py#L414), and [migrations/001_initial_schema.sql](/workspaces/baseball-crawl/migrations/001_initial_schema.sql#L150).

## Priority 5: Planning/Implementation Mismatch
- E-100’s technical notes define “complete stat coverage per endpoint” for both per-game and season tables, with only split-population, spray-chart ingestion, fielding/catcher/pitch-type tables, and stat blending deferred. The current loaders do not satisfy that contract: they leave many non-computed endpoint fields unhandled even when the schema and epic say they should be stored. See [epics/E-100-team-model-overhaul/epic.md](/workspaces/baseball-crawl/epics/E-100-team-model-overhaul/epic.md#L51), [epics/E-100-team-model-overhaul/epic.md](/workspaces/baseball-crawl/epics/E-100-team-model-overhaul/epic.md#L186), [src/gamechanger/loaders/game_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/game_loader.py#L69), and [src/gamechanger/loaders/season_stats_loader.py](/workspaces/baseball-crawl/src/gamechanger/loaders/season_stats_loader.py#L239).

## Priority 6: Style and Convention Violations
None in the reviewed E-100/E-114/E-115 runtime files beyond the functional issues above.

## Cross-Cutting Concerns
- No active TEXT-team-ID or `is_owned` remnants were found in the reviewed runtime paths for the data layer, admin routes, dashboard routes, and E-115 specs.
- No stale `display_name` or `is_admin` references were found in the reviewed dashboard/admin templates; the E-114 template cleanup appears to hold.
- The reviewed SQL uses parameter binding for values; I did not find SQL string interpolation with user-supplied values in the scoped files.
- I did not find `src/` importing from `scripts/` in the reviewed scope.
- Test isolation looked intact in the reviewed test files, but the loader suites are under-scoped for the expanded E-100 columns.
- The primary cross-cutting failure is schema-code alignment in the ETL layer, as noted in Priorities 1, 4, and 5.
